# 画布系统 V8 重构 SDD

> 版本: v1.0  
> 日期: 2026-06-01  
> 状态: 设计阶段  
> 原则: 知识库 → Skill → 工具 → 媒体，主次分明，节点少而精

---

## 一、问题陈述

### 1.1 现状痛点

| 痛点 | 表现 |
|------|------|
| 节点数量爆炸 | 42 个节点类型，32 个平铺在侧栏，用户无从下手 |
| 产品灵魂缺失 | 知识库、Skill、工具在画布里不可见，媒体节点喧宾夺主 |
| UI 风格分裂 | 文本节点、LLM节点、图片生成节点三种完全不同的视觉风格 |
| 无自由缩放 | 节点大小固定，文本多了溢出，大参数表单挤在固定框里 |
| 执行控制弱 | 没有停止按钮，运行中的节点无法中断 |
| 命名混乱 | prompt/content/input/output/reply 同一概念多种叫法 |
| 多余节点 | 抠图、放大、画板、编辑、videoOutput 等 T8 遗留节点永远隐藏但占据注册表 |

### 1.2 设计目标

1. **节点从 42 → 10 种**，每种功能凝结、视觉统一
2. **每个节点可自由拖拽缩放**，手感流畅
3. **每个节点有 ▶ 执行 / ■ 停止 / ✕ 删除**，状态可见
4. **LLM 节点 = 产品大脑**，内聚知识库 + Skill + 工具 + 模型选择
5. **文本节点 ≥ EditorPanel**，Tiptap 富文本，内容驱动缩放
6. **媒体节点参数精简到 ≤ 3 层**，输入输出端口语义清晰

### 2.2 废弃节点（V7 移除）

以下 32 个 V7 节点类型不再暴露在用户界面，代码保留但标记 `@deprecated`：

| 分类 | 废弃节点 | 废弃原因 |
|------|---------|---------|
| T8 核心 | `runninghub`, `runninghubWallet`, `seedance`, `rhTools`, `rhConfig` | 功能合并到 `imageGen`/`videoGen`/`audioGen` 的模型选择器 |
| 素材 | `upload`, `materialSet`, `output` | 上传合并到结果节点右键菜单；素材集用 Group 替代 |
| 流程 | `pickFromSet`, `framePair` | 极少使用，textSplit 覆盖分镜需求 |
| 图像 | `resize`, `combine`, `removeBg`, `upscale`, `gridCrop`, `imageCompare`, `drawingBoard`, `frameExtractor` | 画布不应是图像编辑器；确有需求走本地工具节点 |
| 工具箱 | `cinematic`, `videoMotion`, `multiAngleVisual` | 概念太窄，LLM 节点的 prompt 自由发挥即可 |
| 辅助 | `idea`, `bp`, `relay`, `edit`, `videoOutput`, `browserNode`, `storyboardGrid` | idea/bp/relay 无实质功能；其他合并或废弃 |
| 其他 | `file`, `tool`, `group` | file 由结果节点右键菜单替代；tool 合并到 LLM 节点工具开关；group 由 VueFlow 原生选择+框选替代 |

## 三、通用节点骨架 (NodeShell)

每种节点都包裹在同一个 `NodeShell` 组件内，保证交互一致性。

### 3.1 结构

```
┌─────────────────────────────────────┐
│ 🏷️ [图标]  节点名称          [⤡]   │  ← 头部 (NodeHeader)
│             状态指示器               │     彩色左边条标记角色
├─────────────────────────────────────┤
│                                     │
│    [各节点自己的表单/编辑器]          │  ← 身体 (slot: default)
│                                     │
│                                     │
├─────────────────────────────────────┤
│ ⬅ 2个输入源    ⬆ 1K tokens   [▶] [■] [✕] │  ← 底部 (NodeFooter)
│                            执行 停止 删除  │
└─────────────────────────────────────┘
     ↑ 右下角拖拽手柄 (3 条斜线)
```

### 3.2 缩放机制

```ts
// 所有节点共享的缩放行为
interface NodeShellProps {
  id: string
  label: string
  icon: string           // Material Symbol 名称
  color: NodeRoleColor   // 左边条颜色
  status: 'idle' | 'running' | 'success' | 'error' | 'stopped'
  minWidth: number       // 默认 260
  minHeight: number      // 默认 120
  defaultWidth: number   // 默认 320
}

// 缩放手柄置于右下角
// 拖拽时使用 pointer-events 捕获，CSS transition 只在松手时生效
// 内部 node.data.width / node.data.height 持久化
```

缩放行为：
- **右下角手柄** 拖拽 → 实时改变宽高（`requestAnimationFrame` 节流）
- **松手** → 写入 `node.data` 持久化
- **双击头部** → 恢复默认尺寸
- **内容溢出** → 节点自动扩展高度（最小高度下限）
- **最小尺寸** → `minWidth: 240px`, `minHeight: 100px`

### 3.3 角色颜色系统

```css
角色-输入:    #3b82f6 (蓝色)  -- text 节点左边条
角色-思考:    #8b5cf6 (紫色)  -- llm 节点左边条
角色-生成:    #10b981 (绿色)  -- imageGen/videoGen/audioGen 左边条
角色-结果:    #6b7280 (灰色)  -- result 节点左边条
角色-编排:    #f59e0b (琥珀)  -- loop/textSplit 左边条
```

### 3.4 端口语义统一

| 位置 | Handle ID | 类型 | 含义 |
|------|-----------|------|------|
| 左侧-上 | `left-prompt` | target | 文本输入 |
| 左侧-下 | `left-image` | target | 图片参考输入 |
| 右侧-上 | `right-text` | source | 文本输出 (仅 llm) |
| 右侧-下 | `right-media` | source | 媒体输出 |

**规则：**
- `text` 节点只有 `right-text` 出口
- `llm` 节点有 `left-prompt` 入口和 `right-text` 出口
- `imageGen`/`videoGen` 有 `left-prompt` + `left-image` 入口，`right-media` 出口
- `audioGen` 只有 `left-prompt` 入口，`right-media` 出口
- 结果节点只有 `left-prompt` + `left-image` 入口，无出口

## 四、节点详细规格

### 4.1 `text` — 文本节点

```
┌──────────────────────────────────┐
│ 📝  文本                        │ ← 蓝色左边条
├──────────────────────────────────┤
│                                  │
│  # 这是一个标题                   │  ← Tiptap 富文本编辑器
│                                  │     支持: 标题/粗体/列表/引用/
│  这是一段正文内容，支持            │           代码块/图片/链接/表格
│  **粗体** *斜体* ~删除线~         │           [[双向链接]]
│                                  │     placeholder: "输入内容..."
│  - 列表项 1                      │
│  - 列表项 2                      │
│                                  │
│  按 Ctrl+Enter 自动 AI 续写       │
│                                  │
├──────────────────────────────────┤
│ 1.2K 字符  ·  ≈450 tokens  [▶] [■] [✕] │  ← 执行=AI续写/润色
└──────────────────────────────────┘
    ↗ 右下角拖拽 = 缩放
    ↕ 内容溢出 = 自动增高
```

**功能清单：**
- Tiptap 完整富文本（标题H1-H4、粗体、斜体、删除线、下划线、高亮、列表、引用、代码块、链接、图片、表格、任务列表）
- `[[双向链接]]` — 链接到知识库其他文档
- 字数/token 实时统计（底部显示）
- 自动增高：内容超过当前高度时 `node.height` 自动增长
- 「▶ 执行」= AI 润色/扩写/续写（调 LLM，结果替换/追加到编辑器）
- 拖拽 `.md` / `.txt` 文件到节点 → 自动导入内容
- 右键菜单：导出为 .md、复制纯文本、清空

**数据模型：**
```ts
interface TextNodeData extends NodeShellData {
  type: 'text'
  content: object          // Tiptap JSON (ProseMirror doc)
  plainText: string        // 纯文本提取（用于连线传递）
  charCount: number
  tokenCount: number       // 近似值
}
```

### 4.2 `llm` — AI 大脑节点

```
┌──────────────────────────────────────────────────┐
│ 🧠  AI 大脑                            运行中 ●  │ ← 紫色左边条
├──────────────────────────────────────────────────┤
│                                                  │
│  模型: [Claude Sonnet 4 ▼]         温度: [0.7]   │
│                                                  │
│  ── System Prompt ─────────────────────────────  │
│  ┌──────────────────────────────────────────────┐│
│  │ 你是一个专业的内容创作助手...                   ││
│  └──────────────────────────────────────────────┘│
│                                                  │
│  知识库: [☐ 无] [☑ 产品文案库 ▼] [+ 添加]        │
│  Skill:  [☐ 无] [☑ 品牌指南 ▼]     [+ 添加]      │
│                                                  │
│  工具:                                         │
│  [☐ 浏览器] [☐ 写代码] [☐ Office] [☐ 知识图谱]   │
│                                                  │
│  ── 上游文本预览 ──────────────────────────────  │
│  ┌──────────────────────────────────────────────┐│
│  │ 来自 "文本节点A"：请根据产品特点写一段营销文案... ││
│  └──────────────────────────────────────────────┘│
│                                                  │
│  [展开高级设置 ▼]  最大Token: 4096  Top-P: 0.9   │
│                                                  │
├──────────────────────────────────────────────────┤
│ ⬅ 1个输入源    ☑ 产品文案库  ☑ 品牌指南  [▶] [■] [✕] │
└──────────────────────────────────────────────────┘
```

**功能清单：**
- 模型选择器（从 `/v1/models` 动态拉取）
- 温度滑块（0-2，步长 0.1）
- System Prompt（可选，textarea，自动增高）
- 知识库选择器（多选，从 `vaultStore` 拉取用户的知识库列表）
- Skill 选择器（多选，从 `agentStore` 拉取用户的 Skill 列表）
- 工具开关（浏览器/写代码/Office/知识图谱，来自 `useChat.buildAvailableTools()`）
- 上游文本预览（只读，显示连接到此节点的上游 text 内容摘要）
- 高级设置折叠区：max_tokens、top_p、frequency_penalty、presence_penalty
- 「▶ 执行」→ 组装 Knowledge/Skill/Tool/Model → 调 `/v1/chat/completions`
- 「■ 停止」→ AbortController.abort()
- 输出通过 `right-text` Handle 流出到下个节点

**数据模型：**
```ts
interface LlmNodeData extends NodeShellData {
  type: 'llm'
  modelId: string
  temperature: number
  systemPrompt: string
  selectedVaults: string[]        // vaultStore vault IDs
  selectedSkills: string[]        // agentStore skill IDs
  enabledTools: string[]          // ['browser', 'dev', 'office', 'graphify']
  maxTokens: number
  topP: number
  outputText: string
  outputTokens: number
}
```

## 七、执行模型

### 7.1 执行生命周期

```
idle → running → success
         ↓
      stopped (用户点 ■)
         ↓
       error (API 异常)
```

每个节点的状态独立，互不影响。

### 7.2 拓扑执行

当用户点击 LLM 节点的 ▶：
1. `canvasExecutor` 检查上游 text 节点是否已执行成功
2. 如果上游未执行 → 先执行上游（递归）
3. 执行当前节点
4. 结果通过 `right-text` Handle 流出
5. 自动触发下游连接的节点执行（如果开启了「自动执行」）

### 7.3 停止机制

```ts
// 每个节点持有一个 AbortController
const abortControllers = new Map<string, AbortController>()

async function runNode(nodeId: string) {
  const ac = new AbortController()
  abortControllers.set(nodeId, ac)
  // ...传入 ac.signal 到所有 fetch 调用
}

function stopNode(nodeId: string) {
  const ac = abortControllers.get(nodeId)
  ac?.abort()
  abortControllers.delete(nodeId)
  canvasStore.setNodeStatus(nodeId, 'stopped')
}
```
