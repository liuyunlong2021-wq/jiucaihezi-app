# 续写工作流（5 步循环）

第 2 场及之后，严格按以下 5 步执行。**每步都必须等待用户选择或确认后才能继续。**

---

## 步骤一：读取当前状态

**Agent**：调度Agent

**输入**：世界设定、角色档案、关系网、悬念账本、历史记录

**处理逻辑**：
1. 读取压力值，找到压力值最高的角色
2. 读取该角色的完整档案
3. 读取相关角色档案
4. 检查待回收悬念

**输出**：场景分析报告
```
## 场景分析 — 场X-Y
- 下一场主角：[角色名]（压力值：X/100）
- 压力来源：[原因]
- 涉及角色：[列表]
- 最近剧情：[简述]
- 待回收悬念：[列表]
- 方向建议：[1-2句]
```

---

## 步骤二：选择催化剂 🔴 必须询问用户

**Agent**：催化剂设计Agent

**输入**：场景分析报告、主角档案、世界观设定

**交互模板**：
```
# 催化剂选项 — 场X-Y

主角：[名字]，压力值：X/100
上一场：[简述]

### 选项A：[名称]
事件：[描述]
效果：[打破状态/两难/连锁反应]

### 选项B：[名称]
事件：[描述]
效果：[打破状态/两难/连锁反应]

### 选项C：[名称]
事件：[描述]
效果：[打破状态/两难/连锁反应]

### 选项D：用户自定义
请输入您希望的催化剂事件：

请选择（A/B/C/D）：
```

**等待用户选择后**进入步骤二点五。

---

## 步骤二点五：建立场景工作台（内部执行，不打断用户）

在生成场景内容前，内部确定以下要素。**不输出给用户**，只用于指导写作：

```
- 本场中心事件：（一句话，谁/压力/催化事件/选择/结果）
- 催化事件：
- 被触碰的角色红线：
- 施力点（谁来施压）：
- 受力点（谁承受，如何反馈）：
- 冲突升级方式：
- 本场预期反转：
- 结尾钩子：
```

> 用户如果要求查看，可以展示。

---

## 步骤三：生成场景内容

**Agent**：执笔Agent

**输入**：场景分析报告 + 用户选择的催化剂

**处理逻辑**：
1. 设计 3-5 层冲突
2. 每层冲突基于人物性格
3. 保持节奏紧凑
4. 字数控制在 200-400 字

**冲突层次**：
- 第1层：催化剂触发初始冲突
- 第2层：角色反应产生新矛盾
- 第3层：矛盾升级或转向
- 第4层：角色应对，新的冲突
- 第5层：达成临时平衡（保持张力）

生成后，向用户确认：

```
## 场X-Y 已生成，请确认

[展示完整场景内容]

1. 满意 - 继续  2. 不满意 - 重新生成  3. 需要修改

请选择（1/2/3）：
```

**等待用户确认后**进入步骤四。

---

## 步骤四：角色选择（可选）🔴 场景需要决策时必须询问

**Agent**：选择设计Agent

**何时触发**：当场景中角色需要做重要决定时（如：要不要杀反派？要不要说出真相？）

**交互模板**：
```
# 角色选择 — 场X-Y

当前情境：[简述角色面临的两难]

### 选项A：[选择描述]
后果：[对压力值/关系/剧情的影响]

### 选项B：[选择描述]
后果：[对压力值/关系/剧情的影响]

### 选项C：[选择描述]
后果：[对压力值/关系/剧情的影响]

### 选项D：用户自定义
请输入您希望的角色行动：

请选择（A/B/C/D）：
```

**等待用户选择后**进入步骤五。

> 如果场景不需要角色决策（过渡/铺垫场），可跳过此步骤。

---

## 步骤五：更新文件系统

**Agent**：文件更新Agent

**输入**：生成的场景内容 + 用户的选择

**更新顺序**：
1. 剧本正文 → `wiki/作品/第X集.md`（追加本场）
2. 角色档案 → 更新压力值、认知边界、行为记录
3. 关系网 → 更新关系强度
4. 世界快照 → 更新压力值排序、主要冲突
5. 悬念账本 → 新增或回收悬念
6. 历史记录 → 添加本场记录
7. CLAUDE.md → 更新 `## [创作] JC-duanju-shijiemoxing` 区块
8. log.md → 追加一行 `## [日期] 写作 | 第X集……`

**输出**：文件更新报告
```
## 文件更新报告 — 场X-Y
- 更新的文件：[列表]
- 关键变化：[压力值变化/关系变化/悬念新增或回收]
- 下一场主角预判：[角色名]（压力值：X/100）
```

更新完成后，询问用户是否继续下一场。
