# 续写工作流（5 步循环）

第 2 场及之后，严格按以下 5 步执行。**每步都必须等待用户选择或确认后才能继续。**

---

## 步骤一：读取当前状态

**Agent**：调度Agent

**输入**：
- wiki/世界/设定.md
- wiki/世界/调性.md ← ★ 新增：必须读取调性锁
- wiki/角色/
- wiki/世界/关系.md
- wiki/剧情/悬念.md
- wiki/剧情/时间线.md

**处理逻辑**：
1. 读取压力值，找到压力值最高的角色
2. 读取该角色的完整档案（wiki/角色/）
3. 读取相关角色档案
4. 检查待回收悬念（wiki/剧情/悬念.md）
5. **读取调性锁（wiki/世界/调性.md）**，理解本剧的节奏/台词/反转/情绪偏好

**输出**：场景分析报告
```
## 场景分析 — 场X-Y
- 下一场主角：[角色名]（压力值：X/100）
- 压力来源：[原因]
- 涉及角色：[列表]
- 最近剧情：[简述]
- 待回收悬念：[列表]
- 调性约束：[简述当前调性的节奏/台词/反转偏好]
- 方向建议：[1-2句，融入调性考量]
```

---

## 步骤二：选择催化剂 🔴 必须询问用户（调性驱动）

**Agent**：催化剂设计Agent

**输入**：场景分析报告、主角档案、世界观设定、**wiki/世界/调性.md**

**设计原则**（必须遵守）：
1. **先读调性锁**：理解本剧的催化剂设计原则（如无厘头风→优先荒诞误会/身份错位；超英风→优先技术突破/装备升级/强敌降临）
2. **催化事件必须触及红线**：见 `references/核心机制.md` 红线 × 压力值联动规则
3. **选项差异化**：3 个选项分别走不同的催化路径（如 A=外部入侵 B=关系裂变 C=内部觉醒），但**包装方式必须统一在调性之下**
4. **选项描述带调性风味**：不只是冷冰冰的事件描述，而是用调性的语气写出来

**交互模板**：
```
# 催化剂选项 — 场X-Y（调性：[简述当前调性]）

主角：[名字]，压力值：X/100
上一场：[简述]

### 选项A：[带调性风味的名称]
事件：[融入调性特征的催化事件描述]
效果：[打破状态/两难/连锁反应，用调性语言表述]

### 选项B：[带调性风味的名称]
事件：[融入调性特征的催化事件描述]
效果：[打破状态/两难/连锁反应]

### 选项C：[带调性风味的名称]
事件：[融入调性特征的催化事件描述]
效果：[打破状态/两难/连锁反应]

### 选项D：用户自定义
请输入您希望的催化剂事件：

请选择（A/B/C/D）：
```

**正确示例**（调性 = 周星驰无厘头 × 漫威钢铁侠，修仙世界）：
```
### 选项A：法器报废，歪打正着
五爷把林风刚修好的「雷鸣锤」拿去当废铁卖了。林风追到废品站，发现锤子在废铁堆里吸收了三百年的锈铁精华——意外淬炼成了天阶法宝。废品站老板（抠鼻）："这玩意儿刚才还在发光，加五块钱。"

### 选项B：宗门HR挖角
天玄宗HR上门——不是来追杀林风，是来挖角的。"林先生，您修复法器的 KPI 远超我宗炼器堂正编弟子。"林风看着 HR 身后站着的杀父仇人·副宗主："哦？那你们先把离职补偿结一下——我爸的命。"

### 选项C：能量反噬倒计时
林风的机械臂突然失控，抽取他体内灵力——如果不在一炷香内找到寒属性材料冷却核心，机械臂会炸掉他的经脉。最近的材料在「冰渊秘境」——三年前正是那里，林氏满门被困。
```

**错误示例**（没有调性风味，写法通用）：
```
### 选项A：法器损坏
林风修好的法器被五爷弄坏了，林风去追。

### 选项B：宗门来人
天玄宗的人来了，林风要面对仇人。

### 选项C：灵力失控
林风的灵力突然失控，需要找材料救治。
```

**等待用户选择后**进入步骤二点五。

---

## 步骤二点五：建立场景工作台（内部执行，不打断用户）

在生成场景内容前，内部确定以下要素。**不输出给用户**，只用于指导写作：

```
- 本场中心事件：（一句话，谁/压力/催化事件/选择/结果）
- 催化事件：
- 被触碰的角色红线：
- 施力点（谁来施压）：
- 受力点（谁承受，如何反馈）：
- 冲突升级方式：
- 本场预期反转：（必须匹配调性的反转偏好）
- 结尾钩子：（钩子类型匹配调性偏好）
- 调性检查：
  ✓ 节奏是否符合调性？
  ✓ 反转模式是否匹配调性？
  ✓ 台词语气是否贴合调性？
```

> 用户如果要求查看，可以展示。

---

## 步骤三：生成场景内容（调性驱动）

**Agent**：执笔Agent

**输入**：场景分析报告 + 用户选择的催化剂 + **wiki/世界/调性.md**

**处理逻辑**：
1. 设计 3-5 层冲突（冲突升级方式匹配调性）
2. 每层冲突基于人物性格
3. **台词语气/金句密度/对白长度匹配调性**
4. 反转模式匹配调性（如无厘头→身份错位/意外暴露；超英→绝境觉醒/华丽反杀）
5. 保持节奏紧凑

**冲突层次**：
- 第1层：催化剂触发初始冲突
- 第2层：角色反应产生新矛盾
- 第3层：矛盾升级或转向
- 第4层：角色应对，新的冲突
- 第5层：达成临时平衡（保持张力）

生成后，向用户确认：

```
## 场X-Y 已生成，请确认（调性：[简述]）

[展示完整场景内容]

1. 满意 - 继续  2. 不满意 - 重新生成  3. 需要修改

请选择（1/2/3）：
```

**等待用户确认后**进入步骤四。

---

## 步骤四：角色选择（可选）🔴 场景需要决策时必须询问

**Agent**：选择设计Agent

**何时触发**：当场景中角色需要做重要决定时（如：要不要杀反派？要不要说出真相？）

**交互模板**：
```
# 角色选择 — 场X-Y

当前情境：[简述角色面临的两难]

### 选项A：[选择描述]
后果：[对压力值/关系/剧情的影响]

### 选项B：[选择描述]
后果：[对压力值/关系/剧情的影响]

### 选项C：[选择描述]
后果：[对压力值/关系/剧情的影响]

### 选项D：用户自定义
请输入您希望的角色行动：

请选择（A/B/C/D）：
```

**等待用户选择后**进入步骤五。

> 如果场景不需要角色决策（过渡/铺垫场），可跳过此步骤。

---

## 步骤五：更新文件系统

**Agent**：文件更新Agent

**输入**：生成的场景内容 + 用户的选择

**更新顺序**：
1. 剧本正文 → `wiki/剧本/第X集.md`（追加本场）
2. 角色档案 → `wiki/角色/` 更新压力值、成长轨迹
3. 关系网 → `wiki/世界/关系.md` 更新关系强度
4. 世界快照 → `wiki/世界/设定.md` 更新压力值排序、主要冲突
5. 调性 → `wiki/世界/调性.md` 读取（只读，确保本场合规）
6. 悬念 → `wiki/剧情/悬念.md` 新增或回收
7. 时间线 → `wiki/剧情/时间线.md` 追加本场事件
8. CLAUDE.md → 更新 `## [创作] JC-duanju-shijiemoxing` 区块

**输出**：文件更新报告
```
## 文件更新报告 — 场X-Y
- 更新的文件：[列表]
- 关键变化：[压力值变化/关系变化/悬念新增或回收]
- 调性合规：[本场是否贴合调性锁，备注]
- 下一场主角预判：[角色名]（压力值：X/100）
```

更新完成后，询问用户是否继续下一场。
