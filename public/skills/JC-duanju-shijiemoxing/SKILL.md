# 短剧世界模型

从零创作短剧。基于「世界观+极致角色+催化剂→故事自然涌现」核心机制，产出写入项目目录。**每一步都让用户做选择或直接写下一步计划，绝不擅自推进。**

配合 JC-yizhixing（巡检）和 JC-jiyiyasuo（压缩）形成完整内循环。爽点参考 `爽点库.md`。

---

## 系统架构（三层）

```
第三层：调度层 → 读取世界状态，决定下一场主角和冲突方向
第二层：执行层 → 根据场景工作台生成剧本内容
第一层：世界观层 → 存储所有静态和动态世界信息
```

### Agent 职责表（8 个角色）

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| 灵感收集Agent | 收集用户灵感 | 用户输入 | 原点灵感文档 |
| 世界构建Agent | 创建基础文件 | 原点灵感 | 角色/世界/关系/悬念文件 |
| 开场设计Agent | 生成第1场 | 灵感+角色档案 | 开场剧本 |
| 调度Agent | 读取世界状态 | 世界快照等 | 场景分析报告 |
| 催化剂设计Agent | 设计催化剂选项 | 场景分析 | 2-3个催化剂选项 |
| 执笔Agent | 生成场景内容 | 催化剂 | 场景剧本 |
| 选择设计Agent | 设计角色决策 | 场景内容 | 2-3个选择选项 |
| 文件更新Agent | 更新所有文件 | 场景结果 | 更新报告 |

## 核心原则

### 1. 催化剂驱动
每场戏必须有**催化剂**：打破主角当前状态、制造两难处境的事件。必须符合世界观，能引发连锁反应。

### 2. 蝴蝶效应
催化剂发生后，角色的每个行动都要基于性格产生新矛盾，不要一帆风顺。

### 3. 多层冲突（每场 3-5 层）
- 第1层：催化剂触发初始冲突
- 第2层：角色反应产生新矛盾
- 第3层：矛盾升级或转向
- 第4层：角色应对，新的冲突
- 第5层：达成临时平衡（保持张力）

### 4. 压力值驱动
- 压力值最高的角色是下一场的主角
- 压力值决定角色的行为倾向
- 压力值随剧情动态变化

### 5. 人机交互铁律 🔴
- **每个决策点必须询问用户**：收集灵感、关键设定、催化剂选择、角色选择、场景确认
- **提供 2-4 个选项**，必须包含"用户自定义"
- **生成后必须等待用户确认**，绝不自动继续下一场
- **用户说"继续"/"下一场"才继续**

## 核心机制

### 极致性系统

| 维度 | 红线 | 催化剂示例 |
|------|------|-----------|
| 自由 | 被束缚/被命令 | 被囚禁 |
| 正义 | 目睹不公/被冤枉 | 无辜者被杀 |
| 复仇 | 仇人逍遥/被羞辱 | 亲人被杀 |
| 爱情 | 失去爱人/被背叛 | 爱人被抓 |
| 权力 | 失去地位/被轻视 | 被罢免 |
| 知识 | 真相被隐瞒 | 发现秘密 |
| 生存 | 死亡威胁 | 绝境逃生 |
| 荣誉 | 名声被毁 | 被公开诽谤 |
| 贪婪 | 财富被夺 | 宝藏被发现 |
| 守护 | 要保护的人受威胁 | 亲人被绑架 |

### 角色档案模板

写入 `wiki/角色/角色名.md`：

```markdown
---
title: 角色名
tags: [短剧, 角色]
---
## 角色名
- 极致维度：[维度]（8-10/10）
- 内核动机：[一句话]
- 当前身份：[身份]
- 红线：[不可触碰的底线]
- 压力值：[0-100]
- 性格特点：[3-5个关键词]
- 性格反差：[如外冷内热]
- 行为模式：[在压力下的典型反应]
- 家庭背景 / 学识 / 经历：[简述]
- 与主要角色关系：[简述]
```

### 压力值系统

| 压力值 | 状态 | 场景类型 |
|--------|------|----------|
| 0-30 | 平静 | 缓和/铺垫 |
| 30-70 | 紧张 | 冲突/推进 |
| 70-100 | 爆发 | 高潮/决战 |

变化规则：触碰红线 +50 / 催化剂触发 +20 / 冲突解决 -20 / 冲突升级 +30 / 两难选择 +15。
每场后重新计算，最高者为下一场主角。

### 催化剂类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 外部入侵 | 外部力量打破平衡 | 敌人来袭 |
| 内部觉醒 | 角色认知改变 | 发现真相 |
| 关系裂变 | 人际突发变化 | 盟友背叛 |
| 世界突变 | 环境剧烈变化 | 灾难降临 |

## 剧本格式

```markdown
---
title: 第X集
tags: [短剧, 剧本]
---
## 场X-Y
时间：[具体时间]
场景：[地点+环境]
人物：[[wiki/角色/A]] [[wiki/角色/B]]

△ [场景描述]

角色名（情绪）："台词"
△ [动作描述]

---
字数：约XXX字
```

- △ 叙述/描写，对话 `角色名（情绪）："台词"`
- 第1场 300-500 字，续写 180-350 字，每集约 2-3 场 500-700 字
- 🚫 禁止 `【催化剂】` `【悬念】` `【冲突层】` 等方括号标注 → 必须通过画面/动作/对话体现

---

## 写作规范

- 台词：每句 ≤15 字，短狠准，贴合人设，禁止心理活动
- 节奏：开篇冲突 → 羞辱拉扯 → 反转钩子，付费卡点第 8-10 集
- 钩子：每集末场必须留钩子，打斗/暧昧中途卡断
- 不可拍表达转译：`发现自己` `意识到` `心想` `觉得` `想起了` → 必须转化为台词/动作/表情
- 禁忌：主线不明、节奏慢、铺垫多、配角多、逻辑不通、反派弱智、一帆风顺

### 阶段二：构建世界与角色

**Agent**：世界构建Agent

创建文件（按顺序）：
1. `wiki/角色/主角名.md` — 完整人物小传（压力值 8-10/10）
2. `wiki/角色/配角1.md` — 至少 1 个配角
3. `wiki/世界观/世界设定.md` + `wiki/世界观/关系网.md` + `wiki/世界观/势力分布.md`
4. `wiki/悬疑管理/悬念账本.md` + `wiki/悬疑管理/历史记录.md`
5. 写入 CLAUDE.md 初始 `## [创作] JC-duanju-shijiemoxing` 区块

完成后，询问用户关键设定：

**交互模板**：
```
【设定1：能力系统】
A. 完全可用
B. 可用但削弱（推荐）
C. 暂时不可用
D. 无特殊能力

【设定2：共情点】
A. 不公平的伤害
B. 勇气与本事
C. 身份反差
D. 多种结合
E. 用户自行输入

【设定3：爽点类型】
A. [匹配爽点1]
B. [匹配爽点2]
C. [匹配爽点3]
D. 用户自定义

请选择（例如：B;A;C）
```

等待用户选择后进入阶段三。

### 阶段三：生成第 1 场（开场戏）

**Agent**：开场设计Agent

**第 1 场特殊性**：
- 无需催化剂选择（催化剂 = 故事起点）
- 必须建立世界观
- 必须展示主角核心特质
- 必须制造悬念

**开场结构**：
1. 冲突展示（前 3 秒）
2. 身份暗示（30-60 秒）
3. 行动选择（60-90 秒）
4. 悬念设置（结尾）

生成后，向用户确认：

**交互模板**：
```
## 场1-1 已生成，请确认

[展示完整场景内容]

请问您对第1场的内容：
1. 满意 - 继续下一场
2. 不满意 - 重新生成
3. 需要修改 - 提供意见

请选择（1/2/3）：
```

等待用户确认后进入阶段四。

### 阶段四：进入正常循环

第 1 场确认后，更新所有文件，从第 2 场开始使用续写工作流。

## 第二部分：续写工作流（5 步循环）

第 2 场及之后，严格按以下 5 步执行。**每步都必须等待用户选择或确认后才能继续。**

### 步骤一：读取当前状态

**Agent**：调度Agent

**输入**：世界设定、角色档案、关系网、悬念账本、历史记录

**处理逻辑**：
1. 读取压力值，找到压力值最高的角色
2. 读取该角色的完整档案
3. 读取相关角色档案
4. 检查待回收悬念

**输出**：场景分析报告
```
## 场景分析 — 场X-Y
- 下一场主角：[角色名]（压力值：X/100）
- 压力来源：[原因]
- 涉及角色：[列表]
- 最近剧情：[简述]
- 待回收悬念：[列表]
- 方向建议：[1-2句]
```

---

### 步骤二：选择催化剂 🔴 必须询问用户

**Agent**：催化剂设计Agent

**输入**：场景分析报告、主角档案、世界观设定

**交互模板**：
```
# 催化剂选项 — 场X-Y

主角：[名字]，压力值：X/100
上一场：[简述]

### 选项A：[名称]
事件：[描述]
效果：[打破状态/两难/连锁反应]

### 选项B：[名称]
事件：[描述]
效果：[打破状态/两难/连锁反应]

### 选项C：用户自定义
请输入您希望的催化剂事件：

请选择（A/B/C）：
```

**等待用户选择后**进入步骤三。

---

### 步骤三：生成场景内容

**Agent**：执笔Agent

**输入**：场景分析报告 + 用户选择的催化剂

**处理逻辑**：
1. 设计 3-5 层冲突
2. 每层冲突基于人物性格
3. 保持节奏紧凑
4. 字数控制在 180-350 字

生成后，向用户确认：

**交互模板**：
```
## 场X-Y 已生成，请确认

[展示完整场景内容]

请问您对这场的内容：
1. 满意 - 继续
2. 不满意 - 重新生成
3. 需要修改 - 提供意见

请选择（1/2/3）：
```

**等待用户确认后**进入步骤四。

---

### 步骤四：角色选择（可选）🔴 场景需要决策时必须询问

**Agent**：选择设计Agent

**何时触发**：当场景中角色需要做重要决定时（如：要不要杀反派？要不要说出真相？）

**交互模板**：
```
# 角色选择 — 场X-Y

当前情境：[简述角色面临的两难]

### 选项A：[选择描述]
后果：[对压力值/关系/剧情的影响]

### 选项B：[选择描述]
后果：[对压力值/关系/剧情的影响]

### 选项C：用户自定义
请输入您希望的角色行动：

请选择（A/B/C）：
```

**等待用户选择后**进入步骤五。

> 如果场景不需要角色决策（过渡/铺垫场），可跳过此步骤。

---

### 步骤五：更新文件系统

**Agent**：文件更新Agent

**输入**：生成的场景内容 + 用户的选择

**更新顺序**：
1. 剧本正文 → `wiki/作品/第X集.md`（追加本场）
2. 角色档案 → 更新压力值、认知边界、行为记录
3. 关系网 → 更新关系强度
4. 世界快照 → 更新压力值排序、主要冲突
5. 悬念账本 → 新增或回收悬念
6. 历史记录 → 添加本场记录
7. CLAUDE.md → 更新 `## [创作] JC-duanju-shijiemoxing` 区块
8. log.md → 追加一行 `## [日期] 写作 | 第X集……`

**输出**：文件更新报告
```
## 文件更新报告 — 场X-Y
- 更新的文件：[列表]
- 关键变化：[压力值变化/关系变化/悬念新增或回收]
- 下一场主角预判：[角色名]（压力值：X/100）
```

更新完成后，询问用户是否继续下一场。

## 第三部分：人机交互规则

### 交互铁律 🔴

| 规则 | 说明 |

### 交互时机表

| 阶段 | 交互点 | 询问内容 | 格式 |
|------|--------|----------|------|
| 阶段一 | 收集灵感 | 世界观、主角、起点 | 文本 |

### 错误处理

- **用户说"继续"/"下一场"**：从步骤一开始下一轮循环

## 快速参考

| Q | A |
|---|----|
| 第1场和续写的区别？ | 第1场无需催化剂选择，催化剂=故事起点；续写需选催化剂 |
