# 短剧世界模型

从零创作短剧。基于核心机制驱动故事，产出写入项目目录，配合 Obsidian 和巡检/压缩 skill 形成闭环。

## 文件约定

本 skill 自动在项目根目录创建以下结构。三件套约定：

```
项目根/
├── .raw/                  ← 用户扔素材（范本/录音/截图），Obsidian 不显示
├── wiki/                  ← Obsidian 打开的目录
│   ├── 作品/              ← ★ 本 skill 产出：剧本正文
│   │   └── 第X集.md
│   ├── 角色/              ← ★ 本 skill 产出：人物小传
│   │   └── 角色名.md
│   ├── 世界观/            ← ★ 本 skill 产出：设定/关系/势力
│   │   ├── 世界设定.md
│   │   ├── 关系网.md
│   │   └── 势力分布.md
│   ├── 悬疑管理/          ← ★ 本 skill 维护：悬念/历史
│   │   ├── 悬念账本.md
│   │   └── 历史记录.md
│   ├── 巡检报告/          ← JC-yizhixing 产出（本 skill 只读）
│   ├── log.md             ← ★ 时间线，每集写完追加一行（本 skill 追加）
│   ├── hot.md             ← JC-jiyiyasuo 维护
│   └── index.md           ← JC-jiyiyasuo 维护
└── CLAUDE.md              ← 三个 skill 各有自己的区块
```

### log.md 约定（防失忆的时间线）

每集写完向 `wiki/log.md` **追加**一行（永不重写），用固定前缀，方便 `grep "^## \[" wiki/log.md | tail -5` 秒查近况：

```markdown
## [2026-07-01] 写作 | 第5集：林风觉醒（承接第4集"被囚"钩子）
```

### CLAUDE.md 约定

每次产出后更新 `## [创作] JC-duanju-shijiemoxing` 区块（**只改自己的**）：

```markdown
## [创作] JC-duanju-shijiemoxing
- 最新: [[wiki/作品/第X集]]（日期）
- 主角: [[wiki/角色/主角名]]（压力值:当前值）
- 配角: [[wiki/角色/配角1]] [[wiki/角色/配角2]]
- 世界: [[wiki/世界观/世界设定]] [[wiki/世界观/关系网]]
- 悬念: [[wiki/悬疑管理/悬念账本]]（已埋X, 待收Y）
```

### Obsidian 兼容

- `.md` 文件使用 YAML frontmatter（`title` `tags`）
- 内部引用 `[[wikilinks]]`（如 `[[wiki/角色/林风]]`）
- 剧本中角色名写成 `[[wiki/角色/林风]]`，Obsidian 自动形成关系图谱

## 核心公式

```
世界观（规则） + 角色（极致性） + 催化剂（触发） = 故事自然涌现
```

## 一、核心机制

### 极致性系统

| 维度 | 红线 | 催化剂示例 |
|------|------|-----------|
| 自由 | 被束缚/被命令 | 被囚禁 |
| 正义 | 目睹不公/被冤枉 | 无辜者被杀 |
| 复仇 | 仇人逍遥/被羞辱 | 亲人被杀 |
| 爱情 | 失去爱人/被背叛 | 爱人被抓 |
| 权力 | 失去地位/被轻视 | 被罢免 |
| 知识 | 真相被隐瞒 | 发现秘密 |
| 生存 | 死亡威胁 | 绝境逃生 |
| 荣誉 | 名声被毁 | 被公开诽谤 |
| 贪婪 | 财富被夺 | 宝藏被发现 |
| 守护 | 要保护的人受威胁 | 亲人被绑架 |

**角色档案模板**（写入 `wiki/角色/角色名.md`）：
```markdown
---
title: 角色名
tags: [短剧, 角色]
---
## 角色名
- 极致维度：[维度]（8-10/10）
- 内核动机：[一句话]
- 当前身份：[身份]
- 红线：[不可触碰的底线]
- 压力值：[0-100]
- 性格特点：[3-5个关键词]
- 性格反差：[如外冷内热]
- 行为模式：[在压力下的典型反应]
- 家庭背景 / 学识 / 经历：[简述]
- 与主要角色关系：[简述]
```

### 压力值系统

| 压力值 | 状态 | 场景类型 |
|--------|------|----------|
| 0-30 | 平静 | 缓和/铺垫 |
| 30-70 | 紧张 | 冲突/推进 |
| 70-100 | 爆发 | 高潮/决战 |

变化规则：触碰红线 +50 / 催化剂触发 +20 / 冲突解决 -20 / 冲突升级 +30 / 两难选择 +15。
每场后重新计算，最高者为下一场主角。

### 催化剂机制

| 类型 | 说明 | 示例 |
|------|------|------|
| 外部入侵 | 外部力量打破平衡 | 敌人来袭 |
| 内部觉醒 | 角色认知改变 | 发现真相 |
| 关系裂变 | 人际突发变化 | 盟友背叛 |
| 世界突变 | 环境剧烈变化 | 灾难降临 |

每场开始前提供2-3个催化剂选项（含"用户自定义"）。

## 二、剧本格式

```markdown
---
title: 第X集
tags: [短剧, 剧本]
---
## 场X-Y
时间：[具体时间]
场景：[地点+环境]
人物：[[wiki/角色/A]] [[wiki/角色/B]]

△ [场景描述]

角色名（情绪）："台词"
△ [动作描述]

---
字数：约XXX字
```

- △ 叙述/描写，▲ 动作/镜头，对话 `角色名（情绪）："台词"`
- 第1场300-500字，续写180-350字，每集约2-3场500-700字
- 🚫 禁止 `【催化剂】` `【悬念】` 等方括号标注

---

## 三、写作规范

- 台词：每句≤15字，短狠准，贴合人设，禁止心理活动
- 节奏：开篇冲突 → 羞辱拉扯 → 反转钩子，付费卡点第8-10集
- 钩子：每集末场必须留钩子，打斗/暧昧中途卡断
- 禁忌：主线不明、节奏慢、铺垫多、配角多、逻辑不通

---

## 四、从0创建项目

### 阶段1：收集灵感
询问用户：世界观类型、主角设定、故事起点。

### 阶段2：构建世界
创建目录和文件：
1. `wiki/角色/主角名.md` — 完整人物小传（含 frontmatter）
2. `wiki/角色/配角1.md` — 至少1个配角
3. `wiki/世界观/世界设定.md` + `wiki/世界观/关系网.md` + `wiki/世界观/势力分布.md`
4. `wiki/悬疑管理/悬念账本.md` + `wiki/悬疑管理/历史记录.md`
5. 写入 CLAUDE.md 初始区块

### 阶段3：第1场
催化剂=故事起点，300-500字。截图级开场：一眼打人/反差/感知危险。至少两轮预期反转。

### 阶段4：循环
1. 读 CLAUDE.md → 找压力值最高角色
2. 提供催化剂选项 → 生成 → 写入 `wiki/作品/第X集.md`
3. 更新角色压力值、悬念账本、关系网
4. 更新 CLAUDE.md，向 `wiki/log.md` 追加一条 `## [日期] 写作 | 第X集……`

---

## 五、续写已有项目

1. 读 CLAUDE.md 的 `## [创作] JC-duanju-shijiemoxing` 区块
2. 读 `wiki/悬疑管理/悬念账本.md` + `wiki/角色/` 下全部角色
3. 确认压力值最高角色 → 按循环继续

## 六、每场自检

- [ ] 场次 `## 场X-Y`，对话 `角色名（情绪）："台词"`
- [ ] ≥两轮预期反转，结尾有悬念钩子
- [ ] 每句符合功能句原则，无【】标注，无不可拍表达
- [ ] 已更新 CLAUDE.md、角色压力值、悬念账本
- [ ] `scripts/validate_scene.py` 通过

## 质量守则
❌ 禁止一帆风顺/反派弱智/大段独白/重复催化剂
✅ 必须每场升级/悬念10场内回收/配角独立动机/跑 validate_scene.py

## 指令

```commands

极致角色设计: 按极致性系统设计角色 [角色名]，写入 wiki/角色/角色名.md（frontmatter），更新 CLAUDE.md。
```
