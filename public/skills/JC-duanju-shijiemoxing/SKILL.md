# 短剧世界模型

从零创作短剧。基于「世界观+极致角色+催化剂→故事自然涌现」核心机制，产出写入项目目录。**每一步都让用户做选择或直接写下一步计划，绝不擅自推进。**

配合 JC-yizhixing（巡检）和 JC-jiyiyasuo（压缩）形成完整内循环。爽点参考 `爽点库.md`。

---

## 系统架构（三层）

```
第三层：调度层 → 读取世界状态，决定下一场主角和冲突方向
第二层：执行层 → 根据场景工作台生成剧本内容
第一层：世界观层 → 存储所有静态和动态世界信息
```

### Agent 职责表（8 个角色）

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| 灵感收集Agent | 收集用户灵感 | 用户输入 | 原点灵感文档 |
| 世界构建Agent | 创建基础文件 | 原点灵感 | 角色/世界/关系/悬念文件 |
| 开场设计Agent | 生成第1场 | 灵感+角色档案 | 开场剧本 |
| 调度Agent | 读取世界状态 | 世界快照等 | 场景分析报告 |
| 催化剂设计Agent | 设计催化剂选项 | 场景分析 | 2-3个催化剂选项 |
| 执笔Agent | 生成场景内容 | 催化剂 | 场景剧本 |
| 选择设计Agent | 设计角色决策 | 场景内容 | 2-3个选择选项 |
| 文件更新Agent | 更新所有文件 | 场景结果 | 更新报告 |

## 核心原则

### 1. 催化剂驱动
每场戏必须有**催化剂**：打破主角当前状态、制造两难处境的事件。必须符合世界观，能引发连锁反应。

### 2. 蝴蝶效应
催化剂发生后，角色的每个行动都要基于性格产生新矛盾，不要一帆风顺。

### 3. 多层冲突（每场 3-5 层）
- 第1层：催化剂触发初始冲突
- 第2层：角色反应产生新矛盾
- 第3层：矛盾升级或转向
- 第4层：角色应对，新的冲突
- 第5层：达成临时平衡（保持张力）

### 4. 压力值驱动
- 压力值最高的角色是下一场的主角
- 压力值决定角色的行为倾向
- 压力值随剧情动态变化

### 5. 人机交互铁律 🔴
- **每个决策点必须询问用户**：收集灵感、关键设定、催化剂选择、角色选择、场景确认
- **提供 2-4 个选项**，必须包含"用户自定义"
- **生成后必须等待用户确认**，绝不自动继续下一场
- **用户说"继续"/"下一场"才继续**

## 核心机制

### 极致性系统

| 维度 | 红线 | 催化剂示例 |
|------|------|-----------|
| 自由 | 被束缚/被命令 | 被囚禁 |
| 正义 | 目睹不公/被冤枉 | 无辜者被杀 |
| 复仇 | 仇人逍遥/被羞辱 | 亲人被杀 |
| 爱情 | 失去爱人/被背叛 | 爱人被抓 |
| 权力 | 失去地位/被轻视 | 被罢免 |
| 知识 | 真相被隐瞒 | 发现秘密 |
| 生存 | 死亡威胁 | 绝境逃生 |
| 荣誉 | 名声被毁 | 被公开诽谤 |
| 贪婪 | 财富被夺 | 宝藏被发现 |
| 守护 | 要保护的人受威胁 | 亲人被绑架 |

### 角色档案模板

写入 `wiki/角色/角色名.md`：

```markdown
---
title: 角色名
tags: [短剧, 角色]
---
## 角色名
- 极致维度：[维度]（8-10/10）
- 内核动机：[一句话]
- 当前身份：[身份]
- 红线：[不可触碰的底线]
- 压力值：[0-100]
- 性格特点：[3-5个关键词]
- 性格反差：[如外冷内热]
- 行为模式：[在压力下的典型反应]
- 家庭背景 / 学识 / 经历：[简述]
- 与主要角色关系：[简述]
```

### 压力值系统

| 压力值 | 状态 | 场景类型 |
|--------|------|----------|
| 0-30 | 平静 | 缓和/铺垫 |
| 30-70 | 紧张 | 冲突/推进 |
| 70-100 | 爆发 | 高潮/决战 |

变化规则：触碰红线 +50 / 催化剂触发 +20 / 冲突解决 -20 / 冲突升级 +30 / 两难选择 +15。
每场后重新计算，最高者为下一场主角。

### 催化剂类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 外部入侵 | 外部力量打破平衡 | 敌人来袭 |
| 内部觉醒 | 角色认知改变 | 发现真相 |
| 关系裂变 | 人际突发变化 | 盟友背叛 |
| 世界突变 | 环境剧烈变化 | 灾难降临 |

## 剧本格式

```markdown
---
title: 第X集
tags: [短剧, 剧本]
---
## 场X-Y
时间：[具体时间]
场景：[地点+环境]
人物：[[wiki/角色/A]] [[wiki/角色/B]]

△ [场景描述]

角色名（情绪）："台词"
△ [动作描述]

---
字数：约XXX字
```

- △ 叙述/描写，对话 `角色名（情绪）："台词"`
- 第1场 300-500 字，续写 180-350 字，每集约 2-3 场 500-700 字
- 🚫 禁止 `【催化剂】` `【悬念】` `【冲突层】` 等方括号标注 → 必须通过画面/动作/对话体现

---

## 写作规范

- 台词：每句 ≤15 字，短狠准，贴合人设，禁止心理活动
- 节奏：开篇冲突 → 羞辱拉扯 → 反转钩子，付费卡点第 8-10 集
- 钩子：每集末场必须留钩子，打斗/暧昧中途卡断
- 不可拍表达转译：`发现自己` `意识到` `心想` `觉得` `想起了` → 必须转化为台词/动作/表情
- 禁忌：主线不明、节奏慢、铺垫多、配角多、逻辑不通、反派弱智、一帆风顺

## 第一部分：从 0 创建项目

### 建制阶段：逐轮收集灵感 🔴 核心价值

**本 skill 最大的作用就是引导**——用户往往只有一个模糊想法，不够系统。我们用逐轮提问 + 选项的方式，帮用户把故事的每一个关键要素都想清楚、选明白，不遗漏任何关键设定。

**格式铁律**：每个问题提供 3 个具体选项（A/B/C）+ D.用户自定义。

---

#### 第 0 轮：初始探索

**Agent**：灵感收集Agent

先问一句，判断用户想法的完整度：

```
你想写一个什么样的短剧？哪怕只是一个模糊的想法也可以——
比如"修仙世界复仇"、"现代都市逆袭"、"悬疑惊悚"等等。
```

**完整性判断**：
- ✅ 用户已说清世界观 + 主角 + 困境 → 跳过收集，直接进入"关键设定"轮
- ⚠️ 用户只说了模糊方向 → 从第 1 轮开始逐轮收集

## 第二部分：续写工作流（5 步循环）

第 2 场及之后，严格按以下 5 步执行。**每步都必须等待用户选择或确认后才能继续。**

### 步骤一：读取当前状态

**Agent**：调度Agent

**输入**：世界设定、角色档案、关系网、悬念账本、历史记录

**处理逻辑**：
1. 读取压力值，找到压力值最高的角色
2. 读取该角色的完整档案
3. 读取相关角色档案
4. 检查待回收悬念

**输出**：场景分析报告
```
## 场景分析 — 场X-Y
```

## 第三部分：人机交互规则

### 交互铁律 🔴

| 规则 | 说明 |
|------|------|
| 主动询问 | 灵感、设定、催化剂、角色选择、场景确认 → 必须询问 |
| 提供选项 | 2-4 个选项 + "用户自定义" |
| 等待确认 | 生成后等待用户确认，不自动继续 |
| 用户驱动 | 用户说"继续"/"下一场"/"1"才推进 |

### 交互时机表

| 阶段 | 交互点 | 询问内容 | 格式 |
|------|--------|----------|------|
| 第0轮 | 初始探索 | "你想写一个什么样的短剧？" | 文本 |
| 第1轮 | 世界观&基调 | 6种世界观 + 5种基调 | A-F/A-E |
| 第2轮 | 主角核心 | 名字/8种内核/6种性格模板 | 文本/A-H/A-F |
| 第3轮 | 身份&困境&时空 | 6种身份/6种困境/5种时间 | A-F/A-F/A-E |
| 第4轮 | 对立面&规则 | 6种反派/6种世界规则 | A-F/A-F |
| 第5轮 | 关键设定 | 5种能力/5种共情/5种爽点 | A-E/A-E/A-E |
| 建制完成 | 摘要确认 | 确认或修改 | 1/修改意见 |
| 阶段三 | 第1场确认 | 满意/不满意/修改 | 1/2/3 |
| 步骤二 | 催化剂选择 | 3个催化剂 + 自定义 | A/B/C/D |
| 步骤三 | 场景确认 | 满意/不满意/修改 | 1/2/3 |
| 步骤四 | 角色决策 | 3个行动选项 + 自定义 | A/B/C/D |

### 错误处理

- **输入无效**：提示 `输入无效，请重新选择`，重新展示选项
- **用户回退**：读取历史记录，确认后回退到指定场次
- **用户说"继续"/"下一场"**：从步骤一开始下一轮循环

## 快速参考

| Q | A |
|---|----|
| 第1场和续写的区别？ | 第1场无需催化剂选择，催化剂=故事起点；续写需选催化剂 |

| 用户不满意怎么办？ | 问用户要修改意见或重新生成，不改剧情方向只改写法 |
