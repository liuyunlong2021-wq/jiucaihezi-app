# 分镜 → 导演节拍卷轴

> 把 JC-剧本转分镜产出的导演工作本 → 转成 GPT Image 2 分镜大图提示词。不是塞进格子，是画一条流动的视觉时间线。

## 理念

人类分镜表受限于纸面——规整的 3×3 宫格。但 AI 生图没有这个限制。

GPT Image 2 能读懂自由的排版：重要的镜头放大到半页，过渡的镜头缩成邮票大小，画面之间用箭头连接，角色动线像地铁线路图一样贯穿全卷，标注飘在任何需要说明的位置。

这就是 **导演节拍卷轴**——一场戏的视觉时间线，画面大小 = 情绪权重，箭头 = 叙事流向。

## 输入

- `analysis/shot-design.md`（JC-剧本转分镜产出——每场戏的导演工作本）
- `wiki/角色/*.design.json`（JC-角色设计提示词产出）
- `wiki/世界观/*.design.json`（JC-场景提示词 + JC-道具提示词产出）

## 核心规则

1. 不分规整宫格——镜头大小按情绪权重自由变化
2. 流动布局：左→右、上→下的自然阅读流
3. 箭头串联叙事：实线 = 直接切镜，虚线 = 时间跳跃，波浪线 = 情绪断裂
4. 底部角色动线图：不同颜色虚线标注每个角色的空间轨迹
5. 每镜标注：镜号 + 运镜符号 + 时长
6. GPT Image 2 的 @ 语法引用角色/场景/道具设计图

## 阶段 0.5：继承场景风格

不独立选风格——从 `wiki/世界观/*.design.json` 中读取场景的 `visualStyle` 和 `colorPalette`，卷轴自动匹配场景的视觉风格。

目的：分镜大图与场景图风格一致，视频模型拿到后光色不打架。

## 阶段 1：解析导演工作本 → 节拍分组

读取 `analysis/shot-design.md`，对每一场：

1. 列出所有镜头及导演阐述
2. 按情绪曲线将镜头分组为**节拍**——一个节拍 = 一个完整情绪弧
   - 每个节拍 2-6 镜（超过 6 镜则拆成两个节拍）
   - 每个节拍自然时长由导演节奏档案决定（镜数 × 导演单镜节奏）
3. 每个节拍输出一张卷轴

## 阶段 2：生成卷轴提示词

读取 `references/scroll-format.md`，按模板输出一段 GPT Image 2 提示词。

核心要素：
- **全局约束**：卷轴风格 + 无网格线 + 无边框 + 无水印 + 光色一致
- **全局标注**：集号·场号·场景名·总时长 + 底部时间轴
- **每镜画面**：简短视觉节拍（~30字中文）+ 镜号/运镜/时长 + @参考图
- **连接符**：→ 箭头串联 + 标注叙事关系
- **底部动线图**：各角色颜色虚线轨迹

## 输出文件

`analysis/storyboard-scroll.md`——每个节拍一个卷轴提示词，全剧追加到同一文件。

## 约束

1. 卷轴风格继承场景 JSON 的 visualStyle + colorPalette
2. 不分规整宫格——镜头大小 = 情绪权重
3. 每个镜头必须有镜号 + 运镜符号 + @参考图
4. 卷轴底部必须有角色动线图
5. GPT Image 2 提示词直接可用——复制粘贴即生图
