# 视频提示词写作指南

> 什么时候读：用户说「帮我写视频提示词」或要做视频但提示词太短/模糊时。

## 输出格式

**一段中文自然语言。** 按以下结构组织：

```
主体 + 动作 + 镜头运动 + 氛围 + 声音提示
```

## 核心原则

**视频模型是 image-first**：模型从首帧图片获取主体、构图、色调和风格。提示词只补充图片无法传达的东西——运动、镜头行为、氛围、声音。

**不要重复描述图片里已经有的东西。** 如果你的参考图里已经有一个穿红裙子的女人站在森林里，不要写"红裙女人，森林背景"。模型已经看到了。多余的描述浪费 token 且增加噪音。

**指令优先于形容词。** 「镜头缓慢推进」有效。「电影感十足」无效。模型响应具体指令，不响应没有物理参照的美学形容词。

## 各维度要点

| 维度 | 关键词示例 |
|------|-----------|
| **主体动作** | 走路/奔跑/跳舞/转头/挥手、风吹动头发/衣服、雨滴滑落 |
| **镜头运动** | 缓慢推进/拉远/上摇/下摇/跟拍/环绕/静止手持 |
| **氛围** | 暖黄灯光/雾蒙蒙/霓虹闪烁/黄昏余晖/雨夜潮湿 |
| **声音** | 远处车流/鸟鸣/雨声/脚步声/钢琴曲/人群嘈杂 |

## 示例

```
弱提示词:
"一个女人在街上走"

增强后:
"年轻女人缓步走在雨后的石板路上，镜头从脚部缓慢上摇至背影，街灯暖黄光晕在潮湿地面反射，远处隐约车流声和脚步声回响"
```

```
弱提示词:
"海浪"

增强后:
"大浪翻涌拍打礁石，镜头缓慢推进至浪花特写，阴天冷灰调光线，浪花飞沫在镜头前散开，轰鸣的海浪声混着海风声"
```

```
弱提示词（图文生视频，参考图已有一只猫在窗台上）:
"猫在动"

增强后:
"橘猫慢慢转头看向窗外，耳朵轻抖动，镜头静止不动，午后阳光在猫毛上形成暖金色轮廓光，窗外偶尔传来鸟叫声"
```

## 原则

- 一段话，不分行
- 中文
- 只写动作和运动，不重复图片内容
- 具体指令 > 模糊形容词
- 每次只生成 1 版，用户不满意再改
