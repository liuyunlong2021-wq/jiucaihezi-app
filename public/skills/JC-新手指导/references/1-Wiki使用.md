# Wiki 使用完整指南

> 给完全不懂技术的人。看完就能用。

## Wiki 是什么

Wiki 就是你的"第二大脑"。你把小说、剧本、项目文档扔进去，AI 自动帮你整理成能搜索、能画图、能追踪进度的知识库。

打个比方：你有一堆散乱的纸，Wiki 就是 AI 帮你把它们订成一本有目录、有索引、能翻查的书。

## 核心概念（可以不看）

- **raw/** — 原材料。你扔进来的原文放这里，永远不会被修改
- **wiki/** — 整理好的知识。AI 从 raw/ 里提取信息，写成结构化的笔记
- **log.md** — 操作日志。AI 每次干了什么都记一笔

## 五个 Skill 分工

| Skill | 干啥 | 什么时候用 |
|-------|------|-----------|
| Everything转Wiki | 建库 | 刚开始 |
| 记忆压缩 | 填内容 | 建好壳之后 |
| Wiki查询 | 查进度/搜东西/画图 | 随时 |
| 一致性巡检 | 体检，只查不改 | 定期 |
| Wiki修正 | 把体检查出的问题改掉；给 wiki 加新分类 | 巡检之后 / 想加分类时 |

记住一句话就够：**巡检官只挑错不动手，修正官负责把错改掉**。这两个搭配着用，查出来的问题才不会一直堆在报告里没人处理。

## 完整操作：七步走

下面每一步都是**完整指令**，照抄粘贴到对话框，改一下方括号里的内容就能用，不用自己拼词。

### 1. 建库

```
Everything转Wiki：
[粘贴你的内容——可以是小说、剧本、对话记录，或者说"扫描当前项目目录"]
```

### 2. 填内容

```
请用记忆压缩官 Skill 帮我做 Wiki 壳填充：
扫描 wiki/ 里所有标了"> 待摄入"的壳文件，读 raw/ 原文，
把角色/场景/道具信息填进去，补好 [[双链]]，刷新 hot.md。
```

### 3. 查进度

```
请用 Wiki 查询官 Skill 帮我查看知识库现状：
统计 wiki/ 里现在有多少内容、还有多少壳没填、上次操作是什么。
```

### 4. 搜东西

```
请用 Wiki 查询官 Skill 帮我查询知识库：
关键词：[你想查的角色名/设定名，比如"张三"或"世界观"]
```

### 5. 画关系图

```
请用 Wiki 查询官 Skill 帮我生成角色关系图：
读取 wiki/ 里的 [[双链]]，生成 wiki/关系图.canvas。
```
生成后要装「Obsidian Canvas Viewer」插件才能看图。

### 6. 体检

```
请用一致性巡检官 Skill 帮我对知识库做全库巡检：
查换皮漏改、命名漂移、映射撞车、伏笔没收、双链断裂、孤儿笔记，
输出按优先级排序的体检单。
```
查出问题只会列清单，不会自己改。

### 7. 改错

```
请用知识库修正官 Skill 帮我处理最新一份巡检报告：
机械类问题（漏改/断链/命名不一致）直接改；
语义类问题（设定矛盾）先问我哪个对再改；
改完标记✅已修或⏭跳过，更新 log.md。
```
把第 6 步体检单上的问题一条条改掉。改完的问题会打勾，改不确定的会先问你哪个对，不会自己瞎改。

## 项目做大了：加分类

Wiki 刚建的时候分类往往是猜的——比如小说一开始只建了"角色、世界、剧情"，写到一半突然觉得伏笔太多该单独建个本子记，直接粘贴：

```
请用知识库修正官 Skill 帮 wiki 加一个新分类：
新分类：[分类名，比如"伏笔"]
用途：[一句话说明放什么内容，比如"追踪伏笔埋设和回收进度"]
```

修正官会帮你建好新目录。如果这个新分类需要把已经写在别处的内容搬过来重新整理（比如把角色档案里提到的伏笔线索都摘出来），它会转交给记忆压缩官去做——因为那需要理解内容再重写，不是简单建文件夹。

体检的时候如果发现某个分类文件堆得特别多，巡检官也会主动提一句"要不要拆开"，看到这种提示答"好"或者直接说"帮我拆"就行。

## 速查表

| 指令 | 干什么 |
|------|--------|
| `Everything转Wiki：[内容]` | 建知识库 |
| `Wiki壳填充` | 填角色/大纲/日志 |
| `查看Wiki状态` | 看进度 |
| `查询[关键词]` | 搜东西 |
| `生成关系图` | 画关系网 |
| `巡检一下` | 健康检查，出问题清单 |
| `执行修正` | 把清单上的问题改掉 |
| `加个分类` | 给 wiki 建新的子目录 |

## 常见问题

### 小说太长会卡吗？

不会。建壳只建目录不读全文，填充时逐章读。

### 能自己改文件吗？

可以，但建议让 AI 通过指令改——它会自动补双链。

### Wiki 会和代码混在一起吗？

不会。wiki/ 和 raw/ 是独立目录。

### 多个项目怎么切换？

每个项目有自己 wiki/，AI 自动找当前目录的。

### 怎么同步到 GitHub？

wiki/ 和 raw/ 是普通文件夹，git add + commit + push 就行。

### 想重建？

删掉 wiki/ 和 raw/，再跑 `Everything转Wiki`。

### 巡检查出问题，为什么不直接帮我改了？

巡检官的角色就是"体检"，只负责查、不负责治。这是故意分开的：查错和改错混在一起容易查着查着手滑改错东西。查完说一声「执行修正」，交给修正官专门处理。

### 修正官会不会瞎改我的设定？

不会。能直接判断该改成什么的（比如漏改的错别字、断开的链接）它会直接改；但遇到两处设定说法不一样（比如档案说角色左脸有疤，正文写成右脸），它一定会先问你哪个对，不会自己挑一个改掉。
