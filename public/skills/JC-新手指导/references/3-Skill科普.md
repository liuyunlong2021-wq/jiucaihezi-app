# Skill 科普

## 什么是 Skill

Skill 就是给 AI 写的"工作说明书"。你告诉 AI 一段触发词（比如 `Wiki壳填充`），AI 就读对应的说明书，按步骤干活。

打个比方：你请了个助理，Skill 就是你给助理写的"工作流程手册"——第1步干啥、第2步干啥、遇到什么情况怎么处理。

## Skill 的标准结构

每个 Skill 有三个部分：
- **SKILL.md** — 主文件，写清楚干什么、怎么干（<200 行）
- **references/** — 参考资料，需要时再读（模板、规范、指南）
- **scripts/** — 可执行脚本，自动干活用的（可选）

## 怎么创建自己的 Skill

用 skill-creator——它是专门帮你写 Skill 的 Skill。

说「帮我创建一个 Skill，用于 XX」，它会一步一步引导你：
1. 确定 Skill 叫什么名字
2. 写描述和触发词
3. 设计工作流程
4. 生成完整的 SKILL.md 文件

## skill-creator 怎么用

| 想干啥 | 怎么说 |
|--------|--------|
| 创建新 Skill | `帮我创建一个Skill，用于[你想干什么]` |
| 修改 Skill | `帮我改一下 [Skill名]，让它能 [新功能]` |
| 检查 Skill | `审计一下 [Skill名]，看看有没有问题` |
| 优化 Skill | `优化一下 [Skill名] 的描述/触发词` |

## 常见问题

### Skill 和普通对话有什么区别？
普通对话：你说啥 AI 回啥。Skill：AI 先读说明书，再按流程执行——更稳定、更可控。

### Skill 能调用外部工具吗？
能。scripts/ 目录下的 Python/Bash 脚本可以被 Skill 调用。

### Skill 应该拆分多细？
一个 Skill 只做一件事。如果一个 Skill 干了 7 件不同的事，就该拆了。
