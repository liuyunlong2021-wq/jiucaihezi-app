# 写作工作流（阶段5-8）

## 阶段 5：正文写作

### S1 开篇特殊流程

先生成 **3 个不同切入点的开篇版本**（各 ≤180 字）：
- 版本 A：现场冲突切入
- 版本 B：反常设定切入
- 版本 C：结果倒叙切入

用户选一个后，以此为基础展开 S1 全文。

### 逐节写作循环（S2-S10）（三角引擎 + 节奏参考驱动）

1. **读** CLAUDE.md → 进度、压力值、待收伏笔 + **`wiki/世界/文风.md`**
2. ★ **读** 节奏参考（如有）→ `wiki/参考/对标作品/节拍索引.md` 中匹配当前情景的节拍
3. **选** 催化剂 → 提供 A/B/C/D 选项：

```
# 催化剂选项 — S[N]

「我」当前状态：[简述]，压力值：X/100
节奏参考：[无 / 对标作品第N节：身份暴露→被迫面对→自我和解]
当前场景/关键道具：[简述]——没有就省略

### 选项A：三角引擎 ★推荐
[同时击打角色红线 × 场景/道具维度的催化剂]

### 选项B：双引擎
[角色 + 场景/道具]

### 选项C：单引擎（保底）
[传统角色催化剂]

### 选项D：自定义
```

**引擎级别不输出**——三角/双引擎/单引擎是内部质量检查。

4. **写** → 当前节 800-2500 字，对照 `references/写作质量手册.md`
5. **更** 角色压力值、悬念、关系网
6. **追** 时间线，更新 CLAUDE.md

详见 `../JC-短剧-世界模型/references/叙事三角引擎.md`。

### 分批交付
- 第一批：S1-S3
- 第二批：S4-S7
- 第三批：S8-S10

每批完成后用户确认。

**产出**：`wiki/剧本/短故事正文.md`（分批追加）

---

## 阶段 6：改稿润色

**三遍过**：
1. **对话打磨**：逐句检查功能/人设/长度
2. **真实感检查**：事理逻辑/常识/细节精确度
3. **结构改稿**：主线/节奏/伏笔回收 → 输出改稿清单

**产出**：`wiki/剧情/改稿清单.md` + 定稿正文

---

## 阶段 7：投稿包装

**产出**：
- 3 个备选标题
- 一句话主线
- 简介（≤180 字，含钩子）
- 标签（5-8 个）
- 字数统计

**产出**：`wiki/投稿包装/投稿包.md`

---

## 阶段 8：终检

**10 条硬规则逐条校验**：

| # | 规则 |
|---|------|
| 1 | 前50字有冲突/悬念/反差？ |
| 2 | 主线单一清晰？ |
| 3 | 爆款四要素全部落地？ |
| 4 | 第一人称全程一致？无上帝视角？ |
| 5 | 每节末有钩子？ |
| 6 | 对话有功能？ |
| 7 | 真实感自洽？ |
| 8 | 伏笔全部回收？ |
| 9 | 字数在目标范围内？ |
| 10 | 交付物齐全？ |

全部 ✓ → 交付。任一项 ✗ → 回滚修复 → 重新终检。

**产出**：`wiki/终检报告/终检报告.md`
