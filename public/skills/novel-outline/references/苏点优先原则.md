# SKILL-06: 苏点优先原则

## 核心职责

纠正"逻辑大纲"的弊端，从萌点/爽点出发设计大纲

## 核心理念

```
❌ 错误顺序：
先想逻辑（他怎么去那里）→ 再填内容（去了干什么）
结果：逻辑完美但无聊

✅ 正确顺序：
先想萌点（这个场景有什么爽的画面）→ 再构建逻辑（怎么到达这个画面）
结果：有爽点且逻辑自洽
```

## 三步工作法

### Step 1: 列出核心萌点清单

不要直接写大纲，先列出"你最想看的画面"

```markdown
萌点清单（真假千金项目）：
1. 打脸：所有人震惊的瞬间
2. 护短：沈时川霸气宣示主权
3. 反差萌：高冷女主害羞脸红
4. 虐渣：假千金崩溃后悔
5. 甜宠：日常相处的温馨
6. 终极爽：国际大会全世界见证
7. 身份揭露：一层层剥洋葱
8. 能力展示：三针救命、一眼看穿阴谋
9. 独立宣言："我不需要你们的认可"
10. 救赎时刻：程月救程锦，两人和解
```

### Step 2: 用情绪词串联大纲

```
情绪大纲：

憋屈（1-3章）→ 小爽（4-8章）→ 更憋屈（9-11章）
→ 大爽（12-20章）→ 担心（21-25章）→ 守护甜（26-35章）
→ 渣后悔（36-45章）→ 事业线小高潮（46-60章）
→ 马甲掉落爽（61-80章）...
```

### Step 3: 围绕萌点构建逻辑

```
萌点：沈时川霸气宣示主权

逻辑构建：
│
├─ 前置：有人追求程月（创造危机感）
│   └─ 如何安排？→ 医院来了新同事
│
├─ 场景：公开场合（增加张力）
│   └─ 什么场合？→ 医学研讨会
│
├─ 爆发：沈时川突然出现，当众宣示
│   └─ 如何自然？→ 他本来就是大会投资人
│
└─ 后续：程月的反应（害羞/甜蜜）
    └─ 人设一致吗？→ 高冷外表，内心小鹿乱撞
```

## 检查清单

- [ ] 是否列出了10-20个核心萌点？
- [ ] 是否用情绪词串联了整体节奏？
- [ ] 每个大事件都有明确的"爽点/甜点/萌点"吗？
- [ ] 逻辑是否服务于萌点（而非萌点妥协于逻辑）？
- [ ] 萌点分布是否均匀（不是集中在后期）？

## 常见错误

### 错误1: 逻辑绑架萌点

```
❌ "男主不可能这时候出现，违反逻辑"
    → 结果：放弃萌点

✅ "这个萌点必须有，如何让男主合理出现？"
    → 方案：他是大会赞助商/他在找她/巧遇
```

### 错误2: 萌点后置

```
❌ 前50章都是逻辑铺垫，萌点在后面
    → 读者已经弃文了

✅ 前10章就要有萌点，逻辑边走边补
```
