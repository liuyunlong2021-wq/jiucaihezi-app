# SKILL.md 验证指南

创建 SKILL.md 后，在对话中逐条验证以下规则。这些规则来自官方 skill-creator 的验证脚本。

## Frontmatter 检查

- [ ] SKILL.md 以 `---` 开头（YAML frontmatter）
- [ ] frontmatter 是合法的 YAML 字典格式
- [ ] 必须包含 `name` 字段
- [ ] 必须包含 `description` 字段
- [ ] 允许的属性：`name`, `description`, `license`, `allowed-tools`, `metadata`, `compatibility`, `triggers`

## name 规范

- [ ] 使用 kebab-case：小写字母 + 数字 + 连字符
- [ ] 正则匹配：`^[a-z0-9-]+$`
- [ ] 不能以 `-` 开头或结尾
- [ ] 不能包含连续连字符 `--`
- [ ] 最大长度：64 字符

## description 规范

- [ ] 必须是字符串类型
- [ ] 不能包含尖括号 `<` 或 `>`
- [ ] 最大长度：1024 字符
- [ ] 应包含触发场景（不仅描述功能，还要写"何时使用"）
- [ ] 避免过于宽泛导致误触发，也避免过于狭窄导致漏触发

## compatibility 规范（可选）

- [ ] 必须是字符串类型
- [ ] 最大长度：500 字符

## 正文规范

- [ ] 格式为合法的 Markdown
- [ ] 正文 < 500 行（理想值，不是硬限制）
- [ ] 没有引用不存在的 scripts/ 目录或文件
- [ ] 角色定位清晰
- [ ] 有明确的输出格式模板或规范
- [ ] 约束条件覆盖常见边界场景

## 对话中验证

在对话中，你可以通过以下方式执行验证：

1. 逐条对照上面的检查清单
2. 对于 YAML 语法，直接解析 JSON 格式检查（LLM 可以模拟 YAML 解析）
3. 对于长度限制，统计字符数
4. 对于命名规范，用正则匹配
5. 输出验证报告：✅ 通过项 + ⚠️ 警告项 + ❌ 失败项
