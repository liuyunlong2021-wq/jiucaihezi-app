# 影视海报设计

> 剧情介绍 → 海报卖点拆解 → Pinterest 选参考海报 → 反推视觉 DNA → 剧情强连接改写 → 中文生图提示词。

## ⛔ 启动闸门

首次对话必须输出以下欢迎语，等用户选择后再执行：

```
🎬 jc-poster-design，开工！

我能帮你把一段剧情变成一张能直接生图的海报提示词——四步走：

① 拆剧情 —— 把故事压成一个主视觉冲突：谁是主角、什么关系、哪一刻最抓人
② 找参考 —— 去 Pinterest 搜调性匹配的短剧海报，构图/用光/配色拿来当视觉底盘
③ 反推 DNA —— 拆解参考图的每条视觉决策，保留功能，剔除原作信息
④ 替换输出 —— 把人物、冲突、场景、片名全换成你作品的内容，输出中文生图提示词

A 粘贴剧情介绍，开始设计
B 我只有一句话梗概，帮我补齐
C 我已有参考海报，直接发图反推改写
```

选择前不执行任何操作。

## 理念

海报不是一个好看的画面——它是一个**视觉钩子**。三秒内，观众要知道「谁和谁、什么关系、什么情绪、值不值得点进去」。

不让 LLM 凭空想象海报构图——去 Pinterest 搜真实短剧海报，借它的构图骨骼、光线决策和信息层级，换成你作品的肉。参考图提供的是视觉语法，不是视觉内容。

## 流程总览

```
剧情输入
    ↓
第一步：剧情海报拆解（海报设计卡 + A/B/C/D 方向）
    ↓
第二步：Pinterest 搜索匹配海报 → 选图确认
    ↓
第三步：反推参考海报视觉 DNA → 「参考→本剧」替换表
    ↓
第四步：合并输出中文生图 JSON 提示词
```

## 输入

用户提供以下任一内容即可开始：故事梗概、人物小传、分集大纲、剧本节选，或一句话梗概。优先补齐：

```
作品名（暂定名也可）/ 题材 / 主角关系 / 核心冲突 / 时代与场景 / 受众 / 画幅偏好（默认竖屏 3:4）
```

### 影视风格定调联动

若存在 `wiki/世界观/style-design.md`（由 jc-film-style产出），第一步前先读取并提取三项硬约束：

- **媒介**（真人/动漫）：决定 Pinterest 搜索词和最终提示词的风格锚点
- **画幅**（9:16/16:9）：决定最终海报画幅；若与用户偏好冲突，提示并请用户确认
- **色卡库**（3-5 套导演配色）：作为 Pinterest 搜索的颜色线索，也作为最终提示词 `lighting_and_palette` 的调色板候选

没有 style-design.md 时，只基于本次剧情确认媒介、画幅和色调——不强行要求用户先跑 jc-film-style。

## 核心规则

1. 先有剧情判断，再搜参考图；不拿泛题材海报硬套。
2. 参考图借构图、质感、用光、配色和信息层级，不复刻原片人物、片名、Logo 或独特画面。
3. Pinterest 选图必须向用户说明匹配理由，用户确认后才反推。
4. 最终提示词中的人物、冲突、场景、道具和片名必须与用户剧情强连接。
5. 不能访问 Pinterest 或搜不到匹配结果时，说明原因并请用户上传参考海报；没有参考图不凭空输出。

## 工作流

### 第一步：剧情海报拆解

从输入中提取并先展示一张「海报设计卡」：

```markdown
## 海报设计卡

- 一句话钩子：
- 题材与情绪：
- 主视觉冲突：
- 主角关系与站位：
- 必须出现的场景/道具：
- 画幅：
- 片名：
- 海报文案（可选）：
```

给用户 3 个方向并等待选择：

- **A 人物关系型**：突出两到三位主角之间的欲望、对峙或羁绊。
- **B 事件悬念型**：突出关键危机、道具和未揭开的真相。
- **C 情绪氛围型**：突出爱情、遗憾、治愈或宿命感。
- **D 用户自定义方向**。

选择后锁定海报的叙事重心；不要把剧情摘要平铺进一张海报。

### 第二步：Pinterest 搜索与选图

根据已锁定的题材、情绪、媒介和构图方向，构造英文搜索词。搜索词必须包含 `vertical drama poster`、`movie poster design` 或 `series key art` 之一。

示例：

```
urban revenge vertical drama poster red black key art
korean romance series poster warm rain two characters
historical fantasy movie poster gold blue palace key art
```

使用以下流程：

```
open_browser_page(https://jp.pinterest.com/search/pins/?q=关键词)
→ read_page 读取结果
→ 筛选人物站位、文字留白、色彩和题材均匹配的候选图
→ navigate_page(pinURL) 直接进入 Pin 页面
→ screenshot_page 截取清晰大图作为分析参考
```

优先保存浏览器截图；它已足够用于本次分析。需要保留原图时，`gallery-dl` 可用则下载到当前项目的 `./references/`：

```bash
gallery-dl "<pin页面URL>" -d ./references/
```

`gallery-dl` 不可用时截图即可，核心流程不依赖它。下载图仅用于本次视觉分析，不作为最终海报素材或输出内容。

推荐 1 张最佳图并写明 3 条具体匹配理由。用户可选：

- **A 使用这张参考图**
- **B 换一张候选图**
- **C 修改搜索方向**
- **D 上传自己的参考海报**

只在 A 或 D 后进入下一步。

### 第三步：反推参考海报

对已确认的截图或用户上传图，用以下指令做详尽视觉分析：

```
Analyze this image in exhaustive JSON detail.
```

海报分析必须覆盖以下维度：

| 维度           | 具体内容                                                                                   |
| -------------- | ------------------------------------------------------------------------------------------ |
| 画幅与布局     | 比例、视线动线、留白区域、人物占画面比例                                                   |
| 人物层级       | 人数、前后中景分布、动作朝向、关系张力（对峙/背对/凝视/牵拉）                              |
| 场景与道具     | 环境类型、关键道具、材质质感、天气或粒子效果（雨/雪/烟/尘）                                |
| 光影与色彩     | 主光方向、轮廓光、阴影硬度、主色/辅助色/点缀色、整体色温                                   |
| 文字层级       | 片名位置与字号、slogan 位置、演职员小字区域、远距离可读性                                  |
| 可复用与须清除 | 哪些是通用的视觉语法（留白比例、光比、字体层级），哪些是原作的识别元素（片名、角色、Logo） |

拿到完整 JSON 后，输出「参考 → 本剧」替换表。每一条保留的是视觉功能，替换的是叙事内容：

```markdown
| 参考图视觉元素     | 本剧替换内容                       | 替换理由                     |
| ------------------ | ---------------------------------- | ---------------------------- |
| 雨夜巷口背对背双人 | 女刑警与失踪者弟弟隔着审讯玻璃对视 | 落实"追查真相"的核心冲突     |
| 红色霓虹轮廓光     | 冷白审讯灯 + 远处红蓝警灯          | 保留危机感，改成现代刑侦语境 |
| 底部大面积片名区   | "逆光证词"                         | 保留信息层级，不沿用原片文字 |
```

若已读取 style-design.md，替换表中优先采用其色卡库中的配色方案。

### 第四步：输出最终中文生图提示词

把第三步的视觉 DNA 和替换表合并，按以下 JSON 结构输出最终中文提示词：

```json
{
  "canvas": "竖屏 3:4",
  "poster_goal": "一句话说明该海报要卖出的剧情钩子",
  "composition": "人物数量、站位、景别、留白和视线动线",
  "characters": "本剧角色的年龄段、国别感、五官、发型、服装、动作与关系",
  "environment": "与剧情绑定的场景、关键道具和天气/粒子",
  "lighting_and_palette": "主光、轮廓光、阴影、主色和辅助色",
  "typography": "片名最大、slogan、信息小字的位置；所有精确中文文字加引号",
  "render_goal": "摄影/绘制媒介、镜头语境、清晰度和质感",
  "avoid": "原参考片名、原作角色、Logo、乱码文字、塑料感、无关人物"
}
```

提示词必须满足：

1. 画幅和布局放在最前面。
2. 片名、slogan 等精确中文放入引号；不确定文字时留占位符，不编造。
3. 有主要人物时描述国别感、年龄段、五官、发型；不要只写"俊男美女"。
4. 人物、冲突、场景和道具均来自第一步的海报设计卡。
5. 文字层级：片名最大 → 核心文案 → 演职员/档期小字；标明远处可读。
6. `avoid` 必须包含参考海报的原片名、原作人物名、Logo 和乱码文字。

## 输出格式

完成后按固定顺序输出：

1. **海报设计卡**：剧情如何被压缩成一个主视觉冲突。
2. **参考海报**：Pinterest Pin 链接或用户图片来源，以及 3 条选择理由。
3. **视觉 DNA 与替换表**：说明借用了什么视觉功能、替换成了什么剧情内容。
4. **最终中文生图提示词**：可直接投喂 GPT Image 2 的完整 JSON 提示词。

> 💡 复制提示词到韭菜盒子创作面板，选 GPT Image 2 就能出海报。想换构图或色调直接说。
