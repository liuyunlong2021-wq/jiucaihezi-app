# 建制阶段（阶段0-4）

## 阶段 0：对标拆解（可选）

**目标**：选一篇知乎高赞短篇，拆解它为什么爆。

**流程**：
1. 用户提供一篇知乎高赞短篇链接/标题/正文
2. 细读拆解：小节切分、人物速写、事件链、情绪曲线、高亮段落
3. 提炼一句话主线 + 总结可复用策略

**产出**：`wiki/剧情/对标拆解.md`

---

## 阶段 1：选题定调

**目标**：结构化抽取用户需求 + 3-5 个选题方案。

**交互模板**：
```
请告诉我：
1. 题材偏好：[悬疑/情感/惊悚/奇幻/现实主义/复仇/成长…]
2. 情绪基调：[冷/暖/压抑/燃…]
3. 目标字数（默认15000-25000）
4. 有没有想避开的话题？
```

用户回复后，输出 3-5 个选题方案，每个含：
- 标题草案
- 一句话主线
- **爆款四要素映射**：话题性/设定反差/主线清晰度/开篇钩子
- 预估字数 + 风险提示

推荐 1 个最优选题，用户确认。

**确认后 → 立即进入阶段 1.5 文风锁定**：

```
阶段1.5 文风锁定流程：
1.5a 文风分类（悬疑冷峻/温情治愈/黑色幽默/社会派纪实/诗意文学/自定义）
1.5b 对标作家+作品推荐（A/B/C/D/E，模型从训练数据搜索）
1.5c 五维文风DNA锁定 → 写入 wiki/世界/文风.md
```

详细协议见 `references/文风分析框架.md`。文风锁是一切后续的基础——催化剂、反转、语言风格、叙述者声音都以此驱动。

**产出**：`wiki/剧情/选题方案.md` + `wiki/世界/文风.md`

---

## 阶段 2：核心梗（文风约束）

**目标**：把选题提炼为 Logline + 输出选题专属禁区。

**Logline 公式**：
```
当一个[极致角色]遇到[催化剂]，
必须在[两难选择]中[做什么]，
否则[代价]。
```

生成 2-3 个 Logline，用户选一个。
输出 `do_not_do_list`（选题专属禁区，3-5 条）。

**产出**：`wiki/剧情/核心梗.md`

---

## 阶段 3：角色设计（文风约束）

**目标**：建立主角「我」和配角（≤5个）的完整档案。

**文风约束**：主角「我」的叙述者声音（叙述距离/不可靠程度/语癖）必须匹配 `wiki/世界/文风.md` 的「叙述者声音 DNA」。

**主角「我」**：极致维度 + 内核动机 + 第一人称叙事特质 + 角色弧光 + 压力曲线
**配角**（≤5）：每人身份/欲望/恐惧/关系

**产出**：
- `wiki/角色/主角名.md`
- `wiki/角色/配角名.md`（每个配角）
- `wiki/世界/关系.md`

---

## 阶段 4：大纲（文风约束）

**目标**：10节结构 + 节奏曲线 + 伏笔布局。

**文风约束**：节奏分配（开篇20%/攀升30%/高潮30%/收束20%）需匹配 `wiki/世界/文风.md` 的「叙事节奏 DNA」和「悬念 DNA」。

按知乎节奏模型拆 10 节：
- 每节：目标/冲突/动作/信息增量/悬念 + 字数预估 + 压力值目标
- 标注 3 个关键转折点
- 列出伏笔清单（埋在哪节/在哪节回收）

**产出**：
- `wiki/剧情/大纲.md`
- `wiki/世界/设定.md`
- `wiki/剧情/悬念.md`
