# 真实感检查清单

> 知乎读者对真实感极度敏感。一个常识错误可以毁掉整篇的沉浸感。

## 一、事理逻辑

| # | 检查项 | 常见错误 |
|---|--------|----------|
| L1 | 因果链完整？ | A 的发生有没有合理的 B 作为前提？ |
| L2 | 角色行为合理？ | 角色做的事符合他的身份/性格/处境吗？ |
| L3 | 时间线自洽？ | 不会出现「三天前是周一，但周一那天在下周四才下的雨」 |
| L4 | 空间关系正确？ | 从 A 地到 B 地的时间/距离合理 |
| L5 | 信息获取合理？ | 角色知道的信息，是他能合理知道的吗？ |

## 二、常识精确度

| # | 检查项 | 精确写法 |
|---|--------|----------|
| C1 | 时间 | 「下午两点半」>「下午」；「2019 年 3 月」>「那年春天」 |
| C2 | 地点 | 「解放路 137 号 3 楼」>「一条老街上的一栋楼」 |
| C3 | 称谓 | 「骨外科副主任医师」>「医生」；「刑侦支队二大队」>「警察」 |
| C4 | 流程 | 报案流程、医院挂号流程、离婚手续——写之前查一下 |
| C5 | 职业细节 | 程序员不会说「修电脑」；律师不会说「打赢官司就行」 |
| C6 | 数字 | 具体数字 > 模糊形容：「37.2 度」>「有点烫」 |

## 三、细节功能检查

每个细节必须至少满足以下一项功能，否则删掉：

| 功能 | 说明 | 示例 |
|------|------|------|
| 塑造人物 | 细节透露角色的身份/性格/状态 | 「她喝咖啡不放糖。离婚之后开始的。」 |
| 推动情节 | 细节是后面剧情的伏笔或工具 | 「我注意到他左手无名指有一圈白印。」（后面发现他摘了婚戒） |
| 调动情绪 | 细节触发读者的共情或不安 | 「冰箱里只有一瓶老干妈和半袋挂面。」 |

## 四、共情点检查

真实感不只是「不出错」，更是「让读者觉得这是真的」。最好的方法是让读者在某个细节上看见自己。

| 共情类型 | 触发方式 | 示例 |
|----------|----------|------|
| 身体记忆 | 写人人都有过的身体感受 | 「指甲掐进掌心的那种疼，又钝又清醒。」 |
| 社交尴尬 | 写人人都经历过的社交不适 | 「我举着杯子站了十秒，没有人跟我碰。」 |
| 隐秘羞愧 | 写人人都有但不说的心理 | 「我把那条消息删了又打，打了又删，最后发了个『好的』。」 |
| 微小仪式 | 写人人都有但不自觉的习惯 | 「每次锁门之后，我都要拧三下把手才放心。」 |

## 五、真实感审查流程

1. 通读全文，标记所有具体细节（时间/地点/称谓/数字/流程）
2. 逐条检查常识正确性
3. 逐条检查细节是否有功能（塑造人物/推动情节/调动情绪）
4. 无功能细节 → 删除或替换为有功能版本
5. 空洞细节 → 用「精确写法」替换
6. 检查是否有 1-2 处共情点
