# 短故事创作引擎

九阶段管线，从对标拆解到终检交付，产出一篇完结、第一人称、知乎调性的短故事（1-3万字）。**介入阶段 1.5「文风锁定」**——以对标作家+作品锁定全篇叙事节奏/语言风格/悬念密度/情绪基调/叙述者声音，后续一切据此驱动。配合 JC-yizhixing（巡检）和 JC-jiyiyasuo（压缩）形成「创作→养护→质检」内循环。

## 短故事定义

- 篇幅 10000-30000 字，一篇完结，10 节左右
- 第一人称「我」视角，零距离代入
- 知乎/严选调性：强话题性、强设定反差、强情绪张力

## 核心公式

```
世界观（规则） + 角色（极致性） + 催化剂（触发） = 故事自然涌现
```

## 爆款四要素

| 要素 | 说明 |
|------|------|
| 话题性 | 素材自带讨论度 |
| 设定反差 | 出人意料的身份反转 |
| 主线清晰 | 一条主线到底 |
| 开篇高信息量 | 前50字冲突/悬念/反差 |

---

## 九阶段管线

```
阶段0: 对标拆解 → 选一篇爆款拆骨架（可选）
阶段1: 选题定调 → 结构化需求 + 3-5方案 + 爆款四要素映射
★阶段1.5: 文风锁定 → 作家+作品对标 → 五维文风DNA → `references/文风分析框架.md`
阶段2: 核心梗   → 一句话 Logline + 选题专属禁区（文风约束）
阶段3: 角色设计 → 主角「我」+ 配角(≤5) + 第一人称叙事特质（文风约束）
阶段4: 大纲     → 10节结构 + 节奏曲线 + 伏笔布局（文风约束）
阶段5: 正文写作 → 开篇三选一 + 逐节文风驱动催化剂 → `references/写作工作流.md`
阶段6: 改稿润色 → 对话打磨 + 真实感检查 + 结构改稿
阶段7: 投稿包装 → 标题/简介/标签
阶段8: 终检     → 10条硬规则逐条校验
```

详见 `references/建制阶段.md`（阶段0-4）和 `references/写作工作流.md`（阶段5-8）。

## 自检清单

- [ ] 前50字有冲突/悬念/反差（且符合文风基调）
- [ ] 第一人称全程一致，无上帝视角
- [ ] 主线单一清晰
- [ ] 每节推进主线 + 留钩子（钩子类型匹配文风偏好）
- [ ] 「我」的情绪具象化，用身体反应
- [ ] 对话有功能（贴人设/推剧情/调情绪/合文风）
- [ ] 伏笔全部回收
- [ ] 字数在目标范围
- [ ] 博伊尔检查通过：叙事转向 ✅ / 鸟食节奏 ✅ / 冲突升级 ✅
- [ ] 文风脱轨自检通过（`references/写作质量手册.md` §文风驱动）
- [ ] 终检10条全部通过
- [ ] 已更新 CLAUDE.md + 时间线

## 质量守则
✅ 开篇即冲突 / 第一人称到底 / 一节一推进 / 收尾有回响 / 文风一锁到底
🚫 上帝视角 / 人称漂移 / 塌腰 / 主线发散 / 伏笔不回收 / 文风漂移
