# 输出规范

> 被 SKILL.md 第三步引用。回答「输出什么格式」「wiki/ 不存在怎么办」「怎么补双链」。

## 什么时候读

用户选了 A/B/C 分类方案后，读本文件，按本规范执行沉淀。

## wiki/ 不存在时的处理

```
1. 检查 {projectDir}/wiki/ 是否存在
2. 不存在 → 创建目录：
   mkdir {projectDir}/wiki/
   mkdir {projectDir}/wiki/对话记录/
3. 创建 index.md 骨架：
   # {项目名} · Wiki 目录

   ## 对话记录
   （待沉淀）
```

## 单条对话 .md 模板

```markdown
# {对话标题}

- **日期**: 2026-07-12
- **来源**: OpenCode 对话记录
- **标签**: #角色设计 #主角性格

## 对话概要

{3-5 句话提炼核心讨论内容和最终结论}

## 关键结论

- {结论1}
- {结论2}

## 相关对话

- [[另一条对话的标题]]
```

## 双链规则

- 同一分类文件夹内的对话，互相不加链接（它们已经在同一文件夹，不需要）
- 跨文件夹的对话，如果提到相同的角色/概念/文件名，用 `[[目标对话标题]]` 链过去
- 不要硬塞链接——只链真正相关的
- 链接目标文件名要去掉 `.md` 后缀

## 完成报告模板

```
✅ {N} 条对话全部沉淀完成！

📁 输出: wiki/对话记录/{结构}/
📝 共沉淀: {N} 条对话（0 条遗漏）
🔗 新增双链: {M} 处
📋 已刷新: wiki/index.md
🏗️ wiki/ 新建: {是/否}

建议：调用 jc-jian-wiki 巡检一下，确认没有断链或矛盾。
```
