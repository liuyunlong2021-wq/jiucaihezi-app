# 扫描规范

> 被 SKILL.md 第一步引用。回答「怎么扫」「扫多少」「怎么确保不遗漏」。

## 创模式项目账本（优先扫描）

创模式不依赖 OpenCode 数据库。当前项目里每个创模式会话都有一份追加账本：

```text
{projectDir}/.raw/sessions/jcses_*.jsonl
```

执行方式：先 `glob` `.raw/sessions/*.jsonl`，逐个 `read`；一行一个 JSON 事件。按 `sessionId` 分组，按 `at` 升序排列，不设条数上限。`turn_finished` 用来判断一轮是否完成、失败或取消；之前已经产生的用户文本、工具结果和回答仍必须保留。

若 `.raw/sessions/` 不存在或为空，报告“当前项目没有创模式账本”，继续扫描文/武，不把它当成错误。

## 文/武数据库位置（跨平台，不猜）

```
macOS / Linux:  $HOME/.jiucaihezi/opencode-runtime/data/jiucaihezi-opencode.db
Windows:        %USERPROFILE%\.jiucaihezi\opencode-runtime\data\jiucaihezi-opencode.db
```

源码确认（`src-tauri/src/commands/opencode.rs:user_home_dir()`）：
```rust
// 先读 $HOME，再回退 %USERPROFILE%
std::env::var_os("HOME")
    .or_else(|| std::env::var_os("USERPROFILE"))
```

不管从 VS Code Copilot 还是韭菜盒子内部聊天调用，路径规则不变。

## SQL 查询（替代 SDK）

### 列出项目所有根会话

```bash
# macOS/Linux
DB="$HOME/.jiucaihezi/opencode-runtime/data/jiucaihezi-opencode.db"
# Windows (Git Bash / WSL):
# DB="$USERPROFILE/.jiucaihezi/opencode-runtime/data/jiucaihezi-opencode.db"

sqlite3 "$DB" "SELECT id, title, datetime(time_created/1000,'unixepoch','localtime') FROM session WHERE directory = '{projectDir}' AND parent_id IS NULL ORDER BY time_created DESC"
```

- `WHERE directory = '{projectDir}'` — 精确匹配项目路径
- `AND parent_id IS NULL` — 排除 fork/子会话
- 不设 LIMIT — 返回全部，不存在 SDK 默认 50 条的坑
- `projectDir` 里的单引号需转义：`'` → `''`

### 读取会话消息

```bash
# 前 3 条（用于主题分析）
sqlite3 "$DB" "SELECT data FROM message WHERE session_id = '{id}' ORDER BY time_created LIMIT 3"

# 全部消息（用于沉淀）
sqlite3 "$DB" "SELECT data FROM message WHERE session_id = '{id}' ORDER BY time_created"
```

`data` 列是 JSON：`{"role":"user","time":{...},"agent":"build","model":{...},"parts":[{"type":"text","text":"..."}]}`

### 读消息文本

```bash
# 提取所有文本内容（用 jq 解析 JSON）
sqlite3 "$DB" "SELECT data FROM message WHERE session_id = '{id}' ORDER BY time_created" | while read row; do echo "$row" | python3 -c "import sys,json; d=json.load(sys.stdin); [print(p.get('text','')) for p in d.get('parts',[]) if p.get('type')=='text']" 2>/dev/null; done
```

## 主题标签提取

对每条 session，读前 3 条消息的 `data` JSON，找 `role: "user"` 的 `parts[].text` 内容，提取 1-3 个主题标签（如：角色设计、世界观、Bug修复、市场分析）。

## 隔离保证

| 保证 | 机制 |
|------|------|
| 只扫当前项目 | `WHERE directory = '{projectDir}'` 精确匹配 |
| 不遗漏 | 不设 LIMIT，返回全部 |
| 不串项目 | `directory` 列存的是绝对路径，不同项目路径不同 |
| 已验证 | 2026-07-12 实测：jiucaihezi-app=30条，JC-manju-skills=0条 ✅ |
| 不遗漏 | `limit: 500` 覆盖绝大多数项目 |
| 不截断 | 扫描后输出总数，超过 500 条时提示 |
| 不跨项目 | `roots: true` + `directory` 精确匹配 |
