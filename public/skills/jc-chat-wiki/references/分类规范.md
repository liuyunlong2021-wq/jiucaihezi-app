# 分类规范

> 被 SKILL.md 第二步引用。回答「怎么分类」「什么时候读」「方案怎么定」。

## 什么时候读

Skill 完成第一步扫描并拿到主题分布后，读本文件，按本规范生成 A/B/C 选项。

## 分类方案生成规则

### 必须基于实际内容

- 不能凭空编造方案。必须基于第一步产出的「主题标签分布」和「时间跨度」。
- 如果 80% 的对话都围绕 2-3 个主题，方案 A 就用这几个主题做分类。
- 如果对话时间跨度超过 3 个月，方案 B 就按月份归档。

### 方案命名

| 方案 | 名字模板 | 什么时候用 |
|------|---------|-----------|
| A | 按 {主题1}、{主题2}、{主题3} 分类 | 主题分布有 3-6 个明显聚类 |
| B | 按 {阶段1} → {阶段2} → {阶段3} 分阶段 | 对话有明显先后顺序 |
| C | 按月份归档 | 对话时间跨度大且没有明显主题聚类 |
| D | 自定义 | 用户自己指定 |

### 选项格式（严格遵守）

```
━━━ A. {方案名} ━━━
适合：{一句话说清什么场景用这个方案}
结构预览：
wiki/对话记录/{分类1}/
wiki/对话记录/{分类2}/
wiki/对话记录/{分类3}/
（共 {N} 条对话，已分配到各个文件夹）

━━━ B. {方案名} ━━━
适合：{同上}
结构预览：{同上}

━━━ C. 自定义 ━━━
你自己说说想怎么分。比如「把角色设计和世界观放一起，剧情单独放」「按 Agent 分」都行。

选哪个？（A / B / C）
```

### 选项后必须等用户回复

输出 A/B/C 后，**停止一切操作**，等用户明确回复 A/B/C。不要把选项输出和执行混在一起。
