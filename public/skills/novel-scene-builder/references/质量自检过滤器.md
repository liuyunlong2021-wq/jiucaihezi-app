# SKILL-05: 质量自检过滤器 (Quality Filters)

> 并在输出前做最后一道“防烂片”工序。

---

## 核心功能

对生成的场景进行四维度的质量审计，发现并标记问题，甚至拒绝输出劣质场景。

---

## 过滤器清单

### 1. 冲突审计 (The Conflict Audit)

* **Check**: 这个场景里，是否有人/物在**积极地阻碍**主角？
* **Pass**: 主角想买票，售票员说"没票了" / 取票机坏了。
* **Fail**: 主角想买票，售票员说"好的给您"，主角顺利买票。（这是流水账）
* **Action**: 如果 Fail，强制加入障碍（Obstacle）。

### 2. "这有什么关系"测试 ("So What?" Test)

* **Check**: 如果直接把这个场景整段删除，后续剧情还能连得上吗？
* **Pass**: 连不上，因为在这个场景里主角获得了关键道具/受伤了/得知了秘密。
* **Fail**: 完全连得上，这个场景只是在展示他们吃饭聊天刷日常。
* **Action**: 如果 Fail，建议删除该场景，或将其功能合并到其他场景。

### 3. 逻辑与名字检查 (Logic & Names)

* **Check**:
  * 角色名字是否发音太像（John/Jane, 这里的 李伟/李威）？
  * 反派是否过于弱智？
  * 环境是否会有既然反应？（如：枪战现场警察多久到？）
* **Action**: 标记不合理处。

### 4. 格式清洗 (Format Cleaner)

* **Check**:
  * Sluglines 是否标准？（去除 `内 充满悲伤气息的卧室` 这种废话）。
  * 是否指导了导演？（去除 `镜头推近`、`特写` 等术语，除非极有必要）。
  * 是否出现了非视觉描写？（去除 `他此时内心五味杂陈`）。
* **Action**: 自动修正为标准剧本格式。

---

## 执行指令

**Input**:
生成好的场景文本。

**Process**:

1. Run Conflict Audit.
2. Run "So What?" Test.
3. Run Logic Check.
4. Run Format Check.

**Output**:

* 如果通过：输出最终场景。
* 如果未通过：输出包含 [WARNING] 的修改建议报告。
