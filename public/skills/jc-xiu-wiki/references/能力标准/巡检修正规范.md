# 巡检修正规范

> 定义"执行修正"模式下，怎么拿到问题清单、怎么判断机械/语义、怎么落地、怎么留痕。

## 第一步：拿到要修的问题清单

### 来源 A：读巡检报告

优先用已定位 Wiki 根目录下的 `巡检报告/` 里最新一份（按文件名日期排序取最新）。开发项目的根目录可以是 `docs/wiki/`，其他项目可以是 `wiki/`；找不到再请用户给出报告位置或直接提供问题。

### 来源 B：用户口述问题

用户可能没跑过巡检，直接说"第12章孙八应该是钱七"之类的具体问题。视为一条❌级问题处理，其余流程一致。

### 来源 C：先盘点欠了多少活

读取最近一份报告或交给 [[jc-jian-wiki]] 重新巡检，列出尚未标记“✅已修”或“⏭ 跳过”的条目。用户已指定修正目标时直接处理，不把优先级选择推回用户。

## 第二步：分类——机械类直接改，语义类先问

最重要的判断：**巡检报告本身有没有给出唯一答案**。

### 机械类（报告已给答案，直接改，不用再问用户）

| 问题类型 | 报告里的答案 | 你怎么改 |
|---------|-------------|---------|
| 换皮漏改 | "残留范本旧名「X」，应换成「Y」" | 在对应章节文件里把 X 全部替换成 Y |
| 命名不一致/漂移 | "旧名「X」对应了多个新名，全篇只能用一个" | 先问用户定哪个新名为准（报告没说该留哪个，这一步要问），定下后全篇统一替换 |
| 映射撞车 | "新名「X」被多个范本元素共用" | 问用户给其中一个改一个新名，更新映射表 + 全篇替换 |
| 断链（报告未给出唯一目标） | "[[X]] 指向的笔记不存在" | 找到最接近的正确笔记名改链接；如果压根没这篇笔记，转交 [[jc-raw-wiki]] 新建，不是你直接编内容 |
| 孤儿笔记 | "没有任何笔记链入/链出" | 问用户是"该补链"还是"该废弃"，不要自己删——废弃与否是内容决策 |

注意"命名不一致"和"映射撞车"看起来是机械类，但报告只指出了矛盾本身，没指出该留哪个——这一层还是要问用户，只是问完之后的替换动作是机械的（字符串替换，不涉及编内容）。

### 语义类（报告只报告矛盾，必须先问用户）

| 问题类型 | 为什么必须问 | 你怎么做 |
|---------|-------------|---------|
| 设定前后矛盾（"档案说左脸疤，第18章写右脸疤"） | 可能是正文笔误，也可能是档案记错——只有用户知道哪个是他想要的 | 把两个矛盾点原样呈现给用户，问"哪个对"，用户答复后统一改成用户确认的版本 |
| 关系矛盾（前文兄弟、后文仇敌无交代） | 到底是该补一段转变情节，还是本来就是笔误 | 问用户想怎么处理：补交代 or 改回一致 |
| 命运走向偏离（换皮库改动了范本原有的输赢/生死） | 这类改动可能是用户故意为之 | 先确认是不是故意的，故意则更新映射表/大纲记一条备注，不是故意则改回范本走向 |
| 伏笔未回收 | 可能是真遗漏，也可能用户改了计划推迟到后面章 | 问用户是"真漏了要补"还是"计划变了，更新账本预计回收章" |

**核心纪律：矛盾要报告不要替用户定夺。** 两处设定打架时，列出来问用户哪个对，绝不自己挑一个改掉——这是你和巡检官共享的纪律，巡检官不擅自改，你替用户拍板前也不擅自定。

## 第三步：落地修改

- 字符串替换类（漏改/命名不一致/撞车统一后的全篇替换）：用 `scripts/apply_fix.py replace` 做精确替换并报告替换处数，比手改可靠。默认 dry-run 预览，确认无误后加 `--apply`。
- 双链修补类（断链改指、孤儿笔记补链）：用 `scripts/apply_fix.py link` 追加 `[[双链]]`。
- 语义内容类（矛盾改成用户确认的版本、伏笔账本更新回收章）：直接编辑对应文件，因为这类改动要写自然语言，脚本做不了。

改完每一条，立刻回去把体检单原文那一条标记：

```markdown
1. **命名不一致** · 第12章
   「孙八」与第1-5章的「钱七」是同一角色，映射表登记的是「钱七」。
   → ✅ 已修（2026-07-18）：第12章「孙八」→「钱七」，用户确认以映射表为准。
```

跳过的条目也要标记原因，不能留空当作忽略：

```markdown
4. **孤儿笔记** · wiki/角色/配角C.md
   → ⏭ 跳过：用户确认是废弃角色，暂不处理，留作后续判断。
```

## 第四步：记 log + 汇总修正报告

追加 `wiki/log.md`：

```
## [2026-07-18] 修正 | 处理巡检报告 2026-07-17：❌3已修3 🔻2已修1跳过1 ⚠️4已修2跳过2
```

如果本次处理的问题较多（5条以上），额外写一份汇总，追加在原体检单文件末尾或另存 `wiki/巡检报告/[日期]-修正记录.md`：

```markdown
## 修正记录（2026-07-18）

| 问题 | 处理 | 结果 |
|------|------|------|
| 命名不一致·第12章 | 统一为「钱七」 | ✅ 已修 |
| 换皮漏改·第17章 | 「洛阳」→「西安」 | ✅ 已修 |
| 设定矛盾·左脸疤/右脸疤 | 问用户，确认档案对，改正文 | ✅ 已修 |
| 孤儿笔记·配角C | 用户确认废弃 | ⏭ 跳过（保留观察） |

**下一步**：建议让 [[jc-jian-wiki]] 复跑一次巡检，确认本轮修正后问题清零。
```
