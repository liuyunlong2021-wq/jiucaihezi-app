## 职责边界（Scope）

### ✅ 只做

1. **识别导演视角标志词**
   - 搜索并标记："看到"/"听到"/"觉得"/"认为"/"似乎"/"看似"/"整个人"

2. **删除导演视角表达**
   - 删除标志词，保留实际画面
   - 例："他看到门打开了" → "门打开了"

3. **转换为演员视角**
   - 让读者直接"看到"画面，而非通过主角的眼睛"看到"
   - 强化代入感

### ❌ 禁止

1. **禁止改变句子的实际内容**
   - ❌ "他看到小明走进来" → "小红走进来"（改了人物）
   - ✅ "他看到小明走进来" → "小明走进来"（只删标志词）

2. **禁止删除必要的感知动词**
   - ❌ "他尝到了甜味" → "甜味"（"尝到"是必要的感知）
   - ✅ "他觉得饭很甜" → "饭很甜"（"觉得"是冗余的）

3. **禁止修改对话和心理活动**
   - ❌ 不修改引号内的内容
   - ❌ 不删除合理的"心想"/"暗道"（除非是旁白式描写）

4. **禁止过度删减导致语义不清**
   - ❌ "他隐约听到远处传来声音" → "声音"（丢失了"远处""隐约"的信息）
   - ✅ "他隐约听到远处传来声音" → "远处隐约传来声音"

## 输入格式（Input Schema）

```json
{

  "preserve_ambiguity": false  // 可选，是否保留模糊感知（如"似乎""好像"）
```

**输入要求：**

- text不能为空
- text长度建议 50-5000 字
- 超过5000字建议分段处理

## 核心算法

### 导演视角识别规则

```javascript
// 1. 高频标志词（必须删除）
const mustRemove = [
  "看到", "听到", "觉得", "认为", "感到",
  "发现", "注意到", "意识到", "察觉到",
  "看似", "似乎", "好像", "仿佛"
];

// 2. 结构标志（需要重构）
const structurePatterns = [
  /整个人.{0,5}(僵|呆|愣|怔)/,  // "整个人僵住了" → "他僵住了"
  /他\/她.{0,3}(看到|听到).{1,20}[，。]/  // 主语+感知动词
];

// 3. 保留情况（不删除）
const preserve = [
  // 具体感官动词
  ["看见", "看清", "看穿"],  
  ["听见", "听清"],
  ["闻到", "闻见"],
  ["尝到"],
  ["摸到", "触到"],
  
  // 合理的认知动词
  ["发现真相", "发现秘密"],  // 有明确宾语时保留
  ["意识到危险"]
];
```

### 修正流程

```
Step 1: 分句
└─ 按标点符号分割文本

Step 2: 逐句扫描
├─ 识别是否包含标志词
├─ 判断是否属于保留情况
└─ 标记需要修正的句子

Step 3: 执行修正
├─ 删除标志词
├─ 调整句子结构
└─ 确保语义完整

Step 4: 合并结果
└─ 重新组合句子，保持原有段落结构
```

## 修正示例库

### 示例1: 基础删除

```
原文: 他看到门打开了，小明走了出去。
修正: 门打开了，小明走了出去。
理由: 删除"看到"，直接呈现画面
```

### 示例2: 结构调整

```
原文: 他听到外面传来声音。
修正: 外面传来声音。
理由: 删除"听到"，声音本身就是客观存在
```

### 示例3: 保留必要感知

```
原文: 他尝到了苦涩的味道。
修正: 不修改（保留）
理由: "尝到"是具体感官动词，删除后语义不完整
```

### 示例4: 处理"整个人"

```
原文: 他整个人僵住了。
修正: 他僵住了。
理由: "整个人"是冗余强调，删除更简洁
```

### 示例5: 处理"觉得"

```
原文: 她觉得很紧张。
修正: 她很紧张。
理由: "觉得"是旁白式表达，直接陈述更有力
```

### 示例6: 处理"似乎"

```
原文: 他似乎有些不高兴。
修正: 他有些不高兴。（如果preserve_ambiguity=false）
保留: 他似乎有些不高兴。（如果preserve_ambiguity=true）
理由: "似乎"制造模糊感，根据参数决定是否保留
```

### 示例7: 复杂句处理

```
原文: 他看到她站在门口，觉得她似乎有话要说。
修正: 她站在门口，有话要说的样子。
理由: 删除"看到""觉得""似乎"，用"样子"保留推测语气
```

## 边界情况处理

### 情况1: 全是对话的文本

```
输入: "你好。" "你好。" "最近怎么样？"
处理: 返回status=422，error="文本全是对话，无叙述需要修正"
```

### 情况2: 心理活动保留

```
原文: 他心想：这家伙在搞什么？
修正: 不修改（保留）
理由: "心想"引出的是内心独白，不是旁白
```

### 情况3: 特殊文学表达

```
原文: 他仿佛看到了母亲的身影。
修正: 慎重处理
├─ 如果是幻觉/回忆 → 保留"仿佛"（强调虚幻）
└─ 如果是比喻 → 删除"仿佛"
```

## 典型错误与修正

### 错误1: 过度删减

```
❌ 错误修正:
原文: 他隐约听到远处传来枪声。
错误: 枪声。
问题: 丢失了"远处""隐约"的信息

✅ 正确修正:
远处隐约传来枪声。
```

### 错误2: 删除必要动词

```
❌ 错误修正:
原文: 他闻到了烧焦的气味。
错误: 烧焦的气味。
问题: "闻到"是必要的感知，删除后语义不完整

✅ 正确修正:
保留原文或改为：空气中弥漫着烧焦的气味。
```

### 错误3: 改变句意

```
❌ 错误修正:
原文: 他觉得她不会来了。
错误: 她不会来了。
问题: 从"他的推测"变成了"确定的事实"

✅ 正确修正:
她大概不会来了。（保留推测语气）
```

## 完整指令

当你被调用为SKILL-03时：

1. 接收输入的text和可选参数
2. 分段处理（如果文本过长）
3. 逐句扫描，识别导演视角标志
4. 按规则执行修正
5. 记录所有改动到changes数组
6. 统计修正数量
7. 返回标准格式的输出

**核心原则：**

- 删除旁观者视角，强化代入感
- 保留语义完整性
- 不改变剧情内容
- 记录所有修改以供审查
