## 职责边界（Scope）

### ✅ 只做

1. **执行三问自检法**
   - 问题1: 去词化检查（是否还有"看到/听到/觉得"）
   - 问题2: 去形容词化检查（是否还有情绪形容词）
   - 问题3: 节奏调控检查（高潮是否太短，过渡是否太长）

2. **结构化报告问题**
   - 精确定位问题位置（第几段，第几句）
   - 分类问题类型
   - 给出修正建议

3. **通过/不通过判定**
   - 返回明确的status code
   - 提供错误列表（如果不通过）

### ❌ 禁止

1. **禁止自行修正问题**
   - ❌ 发现问题后直接改
   - ✅ 只报告问题，由总控Agent决定是否回滚

2. **禁止做主观评价**
   - ❌ "这段写得不好"
   - ✅ "这段包含3处导演视角标志词"

3. **禁止检查剧情合理性**
   - ❌ 不检查情节是否合理
   - ✅ 只检查语言表达是否符合润色标准

4. **禁止越权改变检查标准**
   - ❌ 自行放宽或收紧标准
   - ✅ 严格按照三问自检法的规则

## 三问自检法详解

### 问题1: 去词化检查

**检查目标：**

```
搜索关键词：
- 看到
- 听到
- 觉得
- 认为
- 似乎
- 看似
- 整个人
```

**判定逻辑：**

```javascript
if (text包含以上任一关键词) {
  if (executed_skills包含"SKILL-03") {
    // SKILL-03已执行但仍有残留 → 高严重性
    return {
      type: "导演视角残留",
      severity: "high",
      related_skill: "SKILL-03"
    };
  } else {
    // SKILL-03未执行 → 中严重性（应该执行但没执行）
    return {
      type: "缺少视角修正",
      severity: "medium",
      related_skill: "SKILL-03"
    };
  }
}
```

**示例：**

```
原文: 他看到门打开了。
检查结果: ❌ 不通过
错误:
  type: "导演视角残留"
  content: "他看到门打开了"
  suggestion: "删除'看到'，改为：门打开了"
```

---

### 问题2: 去形容词化检查

**检查目标：**

```
搜索情绪形容词：
- 很生气
- 很伤心
- 很紧张
- 很开心
- 很愤怒
- 很高兴
- 很委屈
- 很羞愧
```

**判定逻辑：**

```javascript
const emotionAdjectives = /很(生气|伤心|紧张|开心|愤怒|高兴|委屈|羞愧|激动)/g;
const matches = text.match(emotionAdjectives);

if (matches && matches.length > 0) {
  if (executed_skills包含"SKILL-04")) {
    return {
      type: "形容词未去除",
      severity: "high",
      related_skill: "SKILL-04"
    };
  } else {
    return {
      type: "缺少心理侧面化",
      severity: "medium",
      related_skill: "SKILL-04"
    };
  }
}
```

**示例：**

```
原文: 他很紧张。
检查结果: ❌ 不通过
错误:
  type: "形容词未去除"
  content: "他很紧张"
  suggestion: "改为动作展示：他手心沁出了汗"
```

---

### 问题3: 节奏调控检查

**检查目标：**

```
识别场景类型：
- 高潮场景（打脸/暧昧/表白/爽点）→ 应该长
- 过渡场景（赶路/日常/铺垫）→ 应该短
```

**高潮场景识别关键词：**

```javascript
const climaxKeywords = [
  "打脸", "碾压", "震惊", "傻眼",
  "亲吻", "拥抱", "表白", "心动",
  "反转", "真相", "身份曝光"
];
```

**判定逻辑：**

```javascript
// 找到高潮场景
const climaxParagraphs = 识别包含climaxKeywords的段落;

for (paragraph of climaxParagraphs) {
  const wordCount = paragraph的字数;
  
  if (wordCount < 100) {
    return {
      type: "节奏问题",
      severity: "medium",
      subtype: "高潮太短",
      content: `仅${wordCount}字`,
      suggestion: "高潮场景应拉长至150-200字",
      related_skill: "SKILL-02"
    };
  }
}

// 找到过渡场景
const transitionParagraphs = 识别日常场景;

for (paragraph of transitionParagraphs) {
  const wordCount = paragraph的字数;
  
  if (wordCount > 200) {
    return {
      type: "节奏问题",
      severity: "low",
      subtype: "过渡太长",
      content: `${wordCount}字`,
      suggestion: "过渡场景应缩短至50-100字",
      related_skill: "SKILL-02"
    };
  }
}
```

**示例：**

```
场景: 第8段，女主身份曝光（高潮场景）
字数: 50字
检查结果: ❌ 不通过
错误:
  type: "节奏问题"
  subtype: "高潮太短"
  content: "仅50字"
  suggestion: "高潮场景应拉长至150-200字，增加细节描写"
```

## 严重性评级标准

### High（高）- 必须修正

- 导演视角残留（P0问题）
- 大量情绪形容词未去除（P0问题）
- 人设严重不一致（P2问题）

### Medium（中）- 建议修正

- 应该执行但未执行的Skill
- 高潮场景明显太短
- 人设轻微不一致

### Low（低）- 可选修正

- 过渡场景稍长（不影响阅读）
- 优化建议（如动词可以更具体）

## 完整检查流程

```
Step 1: 预检查
├─ 验证输入格式
└─ 确定检查级别

Step 2: P0检查（必须）
├─ 问题1: 去词化
│  └─ 扫描导演视角标志词
├─ 问题2: 去形容词化
│  └─ 扫描情绪形容词
└─ 记录P0问题

Step 3: P1检查（如果check_level >= P1）
├─ 问题3: 节奏调控
│  ├─ 识别高潮场景
│  ├─ 识别过渡场景
│  └─ 检查字数是否合理
└─ 记录P1问题

Step 4: P2检查（如果check_level = P2）
└─ 人设一致性检查
   └─ 记录P2问题

Step 5: 汇总判定
├─ 统计问题数量和严重性
├─ 如果有high severity问题 → status=422
├─ 如果只有low severity问题 → status=200（通过）
└─ 生成结构化报告
```

## 完整指令

当你被调用为SKILL-06时：

1. 接收润色后的文本和检查参数
2. 根据check_level执行对应的检查项
3. 精确定位所有问题
4. 评估每个问题的严重性
5. 关联问题到相应的Skill
6. 生成结构化的检查报告
7. 返回明确的通过/不通过判定

**核心原则：**

- 客观检查，不做主观评价
- 精确定位，方便快速修正
- 明确关联，支持智能回滚
- 结构化报告，便于自动化处理
