# SKILL-02: 信息饱和度检查器 (Information Density Checker)

> 💧 "文案的每一个字都很贵。挤掉水分，只要干货。200字内必须讲清楚：是谁、遇上谁、发生了什么。"

---

## 核心职责

在有限的字数（200-300字）内，最大化有效信息量，确保包含**人设、冲突、爽点**三要素。

---

## 黄金三要素

### 1. 人设 (Character)

- **无效**：一个女孩 / 某人 / 他
- **有效**：被退婚的废柴大小姐 / 患有脸盲症的顶流影帝 / 刚出狱的复仇者
- **标准**：身份 + 核心特质（性格/能力）。

### 2. 冲突 (Conflict)

- **无效**：遇到了一些困难 / 生活发生了变化
- **有效**：全家破产负债千万 / 被当成替身养了十年 / 开局就在恐怖片现场
- **标准**：具体的、紧迫的、高压的困境。

### 3. 爽点/金手指 (Hook/Goldfinger)

- **无效**：她变强了 / 故事很有趣
- **有效**：随手画的符卖出天价 / 系统强制反向读心 / 前任跪在雨里求复合
- **标准**：具体的解决方式或反转结果。

---

## 挤水分实战

**原句（水分多）**：
> 她是一个很普通的女孩，有一天突然穿越到了古代，虽然环境很恶劣，大家都欺负她，但她凭借自己的努力，让所有人都刮目相看。

**分析**：

- "很普通的女孩" → 人设模糊。
- "环境很恶劣" → 冲突不具体。
- "凭努力刮目相看" → 爽点不明确。

**修改后（干货版）**：
> 农业博士（人设）穿越成流放罪奴（具体冲突），开局只有一把锄头。
> 三年后，她种出的粮食供养了整个三军（具体爽点），连暴君都要亲自下田帮她挽裤脚（升级爽点）。

---

## 检查清单 (Checklist)

- [ ] 字数是否控制在 200-300 字？（超过需折叠，少于信息不足）
- [ ] 是否删除了所有无意义的形容词（如"美丽的"、"悲伤的"）？
- [ ] 每一句话是否都有剧情增量？
