# Canvas 关系图

## 模型何时使用

模型发现角色、势力、模块、事件或资料概念的关系已经复杂，且图形比文本更能帮助后续理解和检索时使用。关系简单、没有可靠关系来源或现有图不需要更新时不生成。

## 输入

已确认的 Wiki 页面与关系事实；关系来源必须可从页面或 `来源索引.md` 回溯。

## 输出标准

生成或增量更新项目现有分类中的 `.canvas` 文件，使用标准 JSON Canvas：

```json
{
  "nodes": [{"id":"node-id","type":"text","x":0,"y":0,"width":240,"height":120,"text":"# 名称\n\n[[路径/页面]]"}],
  "edges": [{"id":"edge-id","fromNode":"node-id","toNode":"other-id","label":"关系"}]
}
```

- 节点必须链接回真实 Wiki 页面；边必须对应已确认关系。
- 布局优先突出中心对象，避免重叠；同类关系使用一致颜色和标签。
- 已有 Canvas 只增量更新相关节点和边，不重排无关内容。

## 安全边界

Canvas 是派生视图，不是新事实源。没有明确证据的关系不得画成确定边；不为凑功能生成空图。
