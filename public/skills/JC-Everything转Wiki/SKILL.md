# Everything 转 Wiki

> 输入内容 → 判断类型 → 建目录 → 拆章节 → 列角色名壳 → 写入 `wiki/`。不管填内容。填内容交给 JC-Wiki-记忆压缩。

## ⛔ 启动闸门

首次对话输出以下欢迎语，等用户选择：

```
🗂️ Everything 转 Wiki

输入内容，自动判断类型，建 Wiki 目录：
• 漫剧/短剧 → 场次/角色/场景/道具
• 小说 → 章节/角色/世界观/伏笔
• 短故事 → 核心梗/角色/正文
• 对话记录 → 主题分类/关键结论

怎么给内容？
A 粘贴文本
B 指定本地文件夹路径
C 扫描当前项目目录
```

## 阶段 1：扫描判断

定位 wiki：优先用当前工作目录下的 `wiki/`。不存在则扫描子目录找，列出让用户选。

### 1.1 已是 Wiki？

同时满足 3 项判定为已是 Wiki：

- 存在 `wiki/` 目录
- wiki 下至少 2 个子目录
- 文件含 `[[双链]]` 或 YAML frontmatter

→ 如满足：「✅ 已是 Wiki，可直接用。」跳过后续阶段。

### 1.2 类型指纹匹配

按顺序匹配，命中即停：

| 顺序 | 类型 | 指纹 | reference |
|------|------|------|-----------|
| 1 | 漫剧/短剧 | `第X场`/`Scene` + `△`动作 + `角色：台词` 格式 | `漫剧-Wiki标准.md` |
| 2 | 小说 | `第X章` + 叙事体>60% + 引号对话 | `小说-Wiki标准.md` |
| 3 | 短故事 | 第一人称 + 1-3万字 + 知乎调性 | `短故事-Wiki标准.md` |
| 4 | 对话记录 | `用户：`/`AI：` 交替 + 回合>5 | `对话记录-Wiki标准.md` |
| 5 | 开发项目 | `.git/`/`package.json` + 代码结构为主 + AI协作标记 | `开发项目-Wiki标准.md` |
| 6 | 未知 | — | `通用转换协议.md` |

输出：「🔍 判断为 **[类型]**，指纹：[命中的特征]」

**A 确认 / B 不对，我指定类型 / C 再分析**

## 阶段 2：按标准建壳

1. 读对应 `references/` 下的标准文件
2. 调用 `scripts/everything_to_wiki.py scaffold <type> <目标目录>` 建空目录
3. **原文存 `raw/`**：按章节/场次拆分原文，存入 `raw/<来源名>/第X章.txt`。原文不修改、不重写
4. **从 raw 编译到 `wiki/`**：将原文写入 `wiki/剧情/` 或 `wiki/剧本/`，标注 `> 来源：raw/<来源名>/第X章.txt`
5. 扫描原文，识别角色/场景/道具名称列表，建壳文件：

```markdown
# 张三

> 待摄入。来源：第 3 章首次出场。
```

5. 调用 `scripts/everything_to_wiki.py validate <目标目录>` 确保壳格式统一
6. 生成 `wiki/方向.md`：

```markdown
# 研究方向

- 来源：外部导入，[类型]
- 待摄入时细化
```

7. 生成 `wiki/使用指南.md`，内容参考 `references/使用指南模板.md`——按 wiki 类型输出完整指令清单（小说/开发/通用），每条指令附一句话说明。

**不出大纲、不做角色小传、不补双链。** 这些归 JC-Wiki-记忆压缩。

### 转换方案展示

```
📋 转换方案：[类型] | 源：XX字/XX场/XX章

raw/
└── [来源名]/
    └── 第1章.txt      ← 原文，不可变

wiki/
├── 角色/
│   ├── 主角A.md       ← 壳，待摄入
│   └── 主角B.md
├── 世界/
│   └── 世界观.md       ← 壳
├── 剧情/（或剧本/）
│   ├── 第1章.md        ← 编译自 raw/
│   └── ...
├── 方向.md
└── index.md

A 确认，建壳
B 调整结构

完成后提示：壳建好了，raw/ 存了原文。用 JC-Wiki-记忆压缩 深度摄入。
```

## 阶段 3：LLM 自由设计

未匹配类型时，读 `references/通用转换协议.md`。

1. 分析内容特征
2. 设计 Wiki 架构示意图
3. 展示 + 用户讨论确认后写入

## 约束

1. 先判断，再建壳。已是 Wiki 跳过
2. 只管建壳。不出大纲、不填角色、不补双链
3. 全部 A/B/C/D 选项推进
