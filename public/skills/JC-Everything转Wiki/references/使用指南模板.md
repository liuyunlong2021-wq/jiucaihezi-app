# 使用指南模板

> 给完全不懂编程的小白。只要会复制粘贴就能用。

## 你刚建好了一个 Wiki，然后呢？

你现在有了一个「知识库」。它就像一本会自动更新的笔记本，帮你记住项目里的一切。

下面这些指令，每次打开项目后，**直接复制发给 AI 就行**。

---

## 小说/漫剧 Wiki

### 第 1 步：填充角色和出大纲

```
Wiki壳填充
```

这会让 AI 做两件事：
- 从你的小说原文里提取所有角色的信息（身份、外貌、性格）
- 生成故事大纲

AI 会问你「想填充哪些角色」，你可以选全部，也可以指定几个。

### 第 2 步：查看进度

```
查看Wiki状态
```

AI 会告诉你：大纲出了没、多少个角色填好了、还有多少没填。

### 第 3 步：查角色

```
查询[角色名]是什么角色
```

比如：`查询张三是什么角色`。AI 会从 wiki 里找到并告诉你。

### 第 4 步：看角色关系图

```
生成关系图
```

AI 会自动画一张角色关系网。装一个叫「Obsidian Canvas Viewer」的 VS Code 插件就能看图。

### 第 5 步：检查有没有错误

```
巡检一下
```

AI 会检查你的 wiki 有没有断链、矛盾、遗漏。能自动修的就自动修，修不了的会列出来告诉你。

---

## 开发项目 Wiki

### 每次跟 AI 聊完开发后

```
Wiki壳填充
```

AI 会把你这次跟它聊的关键决策、学到的东西，自动写进开发日志和学习笔记。

### 查看项目进度

```
查看Wiki状态
```

AI 告诉你：总共有多少文件、记了多少条功能、写了多少篇开发日志。

### 想看项目结构长什么样

```
生成关系图
```

AI 画一张项目的关系网图。用「Obsidian Canvas Viewer」就能看。

### 检查 wiki 健康

```
巡检一下
```

## 常用指令速查

| 想干什么 | 发什么给 AI |
|---------|------------|
| 填内容 | `Wiki壳填充` |
| 看进度 | `查看Wiki状态` |
| 查东西 | `查询[关键词]` |
| 看关系图 | `生成关系图` |
| 检查错误 | `巡检一下` |
| 重新建 Wiki | `转Wiki` |

## 装一个看图插件（可选）

在 VS Code 扩展商店搜 `Obsidian Canvas Viewer`，装上就能看关系图。
