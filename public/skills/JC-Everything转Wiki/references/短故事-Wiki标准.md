# 短故事 Wiki 标准

## 判断指纹

- 第一人称叙事（`我` 为主视角）
- 单篇 1-3 万字（非连载、非分章）
- 知乎/严选调性（开篇高信息量、反转、情感张力）
- 有明确的核心梗/一句话梗概

## 标准 Wiki 目录树

```
wiki/
├── 角色/
│   ├── _index.md
│   ├── 我（主角）.md
│   └── ...
├── 世界/
│   ├── 世界观.md
│   └── 文风.md
├── 剧情/
│   ├── 核心梗.md            # 一句话梗概
│   ├── 大纲.md              # 故事梗概
│   └── 正文.md              # 完整正文（不拆分）
├── 参考/
│   └── （写作中引用的资料）
└── index.md
```

## 各文件内容要求

### 角色/我（主角）.md

```markdown
---
name: 主角名
type: 主角（第一人称）
---

# 主角名

## 身份
## 性格
## 动机
## 弧线（开头→结尾的变化）
```

### 世界/世界观.md

短故事世界观通常较轻：

```markdown
# 世界观

## 时代/地点
## 核心设定（如有超自然元素）
## 规则/限制
```

### 世界/文风.md

```markdown
# 文风

## 叙事腔调
## 节奏特点
## 语言风格
```

### 剧情/核心梗.md

一句话核心梗 + 扩展梗概（200 字）。

### 剧情/正文.md

完整正文，不拆分。顶部加 YAML frontmatter 标注字数、完成日期。

## 拆分规则

短故事 1-3 万字，全文可读，不走壳模式：

1. 正文不拆分，一个文件完整保留
2. 大纲从全文直接生成，标注「来源：全文」
3. 角色可直接做完整小传（全文可读，上下文不爆）
4. 文风分析从正文提取
5. 文件间用 `[[双链]]` 互连

> 长篇小说/漫剧走壳模式：角色只建标题+出处，后续由 JC-Wiki-记忆压缩 逐章摄入填充。短故事是例外。

## 读取时机

SKILL.md 阶段 1 匹配到短故事 → 阶段 2 读本文件 → 按目录树和内容要求执行转换。
