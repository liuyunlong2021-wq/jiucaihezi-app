# 开发项目 Wiki 标准

> AI 协作开发项目：Git 仓库 + 功能迭代 + AI 对话驱动。面向不懂编程的 AI 协作开发者。

## 判断指纹

- 包含 `.git/` 目录或 `package.json`/`pyproject.toml` 等项目文件
- 文件结构以代码/配置为主（非叙事文本）
- 可能有 `SKILL.md`、`CLAUDE.md` 等 AI 协作标记文件
- 用户意图包含「开发」「项目」「迭代」

## 标准 Wiki 目录树

```
wiki/
├── 📋 项目总览/
│   ├── _index.md              ← 一句话介绍 + 架构图 + 技术栈
│   └── 开发目标.md             ← 为什么要做、做到什么程度
│
├── 🧩 功能清单/
│   ├── _index.md              ← 功能状态表（✅已完成 🚧进行中 📋计划中）
│   └── [功能名].md            ← 每功能：做什么、依赖谁、当前状态
│
├── 📝 开发日志/
│   ├── _index.md              ← 按日期倒序
│   └── 2026-07-14.md          ← 今天改了什么、为什么、commit hash
│
├── 🤖 AI对话精华/
│   ├── _index.md              ← 关键决策摘要（非全文）
│   └── [主题].md              ← "为什么选X方案""某次重要讨论结论"
│
├── 🔗 GitHub/
│   ├── _index.md
│   ├── 仓库信息.md             ← 地址、分支策略、发布流程
│   └── 同步记录.md             ← 每次 push 的内容摘要
│
├── 📖 学习笔记/
│   ├── _index.md
│   └── [主题].md              ← 开发中学到的新知识
│
└── index.md
```

## 各文件内容要求

### 开发日志/[日期].md

极简格式，每次开发会话后追加一条：

```markdown
## [2026-07-14] 简短标题

**改了**：描述
**为什么**：原因
**关键决策**：需要记住的结论
**参考对话**：[[AI对话精华/相关主题]]
**commit**：`abc1234`
```

### AI对话精华/[主题].md

记结论不记全文：

```markdown
# 主题

> 对话日期：2026-07-14

## 结论
- 决定了 X，因为 Y
- 放弃了 Z，因为 W

## 待办
- [ ] 后续跟进项

## 相关
[[开发日志/2026-07-14]] · [[功能清单/相关功能]]
```

### 功能清单/[功能名].md

```markdown
# 功能名

- 状态：🚧 进行中
- 依赖：[[其他功能]]
- 描述：一句话
- 进度：做了什么、还剩什么
```

## raw/ 结构

```
raw/
├── 对话记录/
│   └── 2026-07-14-Skill拆分.md  ← AI 对话原文，按日期归档
├── 参考资料/
│   └── karpathy-llm-wiki理念.md  ← 外部学习资料
└── 设计稿/
    └── (截图、草图等)
```

## 拆分规则

1. 源文件不进 raw/——开发项目 Wiki 是「元认知层」，原始代码在项目目录本身
2. 项目本身已有的 CLAUDE.md/README.md 等信息 → 提取到 wiki
3. 开发日志按会话追加，不拆章节
4. 对话精华从 raw/对话记录/ 提炼

## 读取时机

SKILL.md 阶段 1 匹配到开发项目 → 阶段 2 读本文件 → 按目录树和内容要求执行转换。
