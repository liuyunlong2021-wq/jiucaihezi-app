# 道具原型激发 —— 从世界经典道具库挪移道具引擎

> **核心洞察**：LLM 训练数据覆盖人类所有经典道具——神话法器、文学信物、电影武器、历史文物。创作时说「一把厉害的剑」——没人知道厉害在哪。说「石中剑式的身份认证之剑，只有真正的继承人能拔出」——戏剧张力立刻出来了。

## 一、道具原型的特殊之处

道具跟角色、场景不同——道具的核心不是「长什么样」，而是**它能做什么、它不能做什么、它为什么推动剧情**。

好道具 = 物理法则 + 叙事功能 + 视觉签名 的三合一。

**经典案例**：
- 金箍棒：可大可小、重三万六千斤、只听孙悟空的话 → 物理法则（尺寸/重量）+ 叙事功能（身份绑定/战力增幅）+ 视觉签名（两端金箍、中间乌铁）
- 魔戒：隐身/腐化心智/只有末日火山能毁掉 → 物理法则（隐身但有代价）+ 叙事功能（诱惑/道德考验）+ 视觉签名（金色圆环、火焰显铭文）
- 圣杯：实现愿望/只有最纯洁的人能找到 → 物理法则（许愿）+ 叙事功能（终极目标/人性试炼）+ 视觉签名（朴素的金杯）

## 二、道具引擎三维提取协议

### 维度 1：物理法则（这个道具的游戏规则）

| 问自己 | 产出 |
|--------|------|
| 这个道具能做什么？ | 核心能力 |
| 这个道具有什么限制/代价？ | 使用壁垒/副作用 |
| 这个道具的物理属性有什么反常之处？ | 尺寸/重量/材质/变形规则 |

**示例**：
- 金箍棒 → 能力：随意大小变化 / 限制：只听主人 / 反常：绣花针大小却有万斤重量
- 雷神之锤 → 能力：召唤雷电+飞行 / 限制：只有「配得上」的人能举起 / 反常：看似不大，无人能移
- 死亡笔记 → 能力：写名字即死 / 限制：需知道长相+真名 / 反常：一本普通笔记本=终极武器

### 维度 2：叙事功能（这个道具在剧情中扮演什么角色）

| 问自己 | 产出 |
|--------|------|
| 道具是目标还是工具？ | 终极目标（麦高芬）/ 日常工具 / 身份信物 |
| 道具制造什么类型的戏剧冲突？ | 争夺/诱惑/道德困境/身份揭穿 |
| 道具跟角色是什么关系？ | 绑定（只有特定人能用）/ 诅咒（用了就倒霉）/ 交易（有代价）|

**示例**：
- 魔戒 → 功能：麦高芬（谁持有谁是故事中心）→ 冲突：诱惑（每个人都被腐化风险）→ 关系：诅咒（持有越久越扭曲）
- 圣杯 → 功能：终极目标 → 冲突：争夺+道德试炼 → 关系：绑定（只有纯洁者）
- 月光宝盒 → 功能：工具（穿越时空）→ 冲突：滥用导致更糟结果 → 关系：交易（改变过去的代价）

### 维度 3：视觉签名（让人一眼认出的外观特征）

| 问自己 | 产出 |
|--------|------|
| 这个道具最标志性的视觉元素？ | 形状/颜色/材质/符号 |
| 有什么独特的识别标记？ | 铭文/伤痕/光芒/特殊质感 |
| 激活和休眠状态外观不同？ | 两种状态的视觉差异 |

## 三、交互模板

在剧本创作中，当剧情需要一个关键道具时触发——用到时激发。

```
你这里用到了一个关键道具：[道具名/功能描述]。
要不要做一次道具原型激发？

我直接从神话/文学/影视/历史的经典道具库里提取几个引擎：

### 原型 A：金箍棒式 [道具类型]
- 引擎提取：可随意大小变化 + 极重 + 只认一个主人。
  物理法则 = 可变尺寸/恒定重量。叙事功能 = 身份信物（拿着它=被认可）。
- 移植方案：不是「一根棒子」——是「只有继承人能举起的一枚戒指/一支笔/一把钥匙」。
  保留「尺寸可变+只认一人」的核心物理，换成你的世界观材质。
- 戏剧可能：反派拿到了它，但它在他手里就是一根普通的[道具]——
  因为「它不认你」。这个场景本身就是身份揭穿。

### 原型 B：魔戒式 [道具类型]
- 引擎提取：强大的能力 + 恐怖的代价。用得越多，被腐蚀越深。
  物理法则 = 隐身/增强，但有代价。叙事功能 = 道德试炼（好人会变坏）。
- 移植方案：你的[道具]每次使用都让使用者失去一点「人性」——
  可能是记忆、情感、寿命。保留「用=被诅咒」的核心，换掉外观。
- 戏剧可能：主角发现自己已经记不清母亲的脸了。
  他用[道具]太多次了。还要继续吗？

### 原型 C：石中剑式 [道具类型]
- 引擎提取：只有「配得上」的人才能使用。道具本身是一个资格认证系统。
  物理法则 = 身份锁定。叙事功能 = 命运/身份的揭晓时刻。
- 移植方案：不是一把剑——是任何「只有特定条件才能激活」的道具。
  一滴血、一句话、一个日期。保留「谁能用=谁就是天选之人」的核心。
- 戏剧可能：所有人都试过了，纹丝不动。主角走过去，还没碰到——
  [道具]自己亮了。

### 原型 D：自定义
你想用什么道具原型？雷神之锤、死亡笔记、月光宝盒、圣杯、
哆啦A梦口袋、哈利波特的魔杖、倚天剑屠龙刀……
或者告诉我这个道具在你的故事里要起什么作用，我推荐原型。
```

## 五、使用时机

道具跟场景一样——用到时才激发：

| 时机 | 触发 |
|------|------|
| 建制阶段 | 如果用户提到了核心道具（如「复仇用的那把刀」），提议激发 |
| 续写时出现关键道具 | 「这个道具是什么样的？要不要做原型激发？」 |
| 用户说「一件祖传的XX」「一把特殊的XX」 | 主动提议 |
| 下游做道具设计前 | 先确保 wiki/道具/ 里有档案 |

## 六、零知识模式 — 当用户说不出原型也说不清文化领域

用户说「主角需要一个厉害的武器」「一个关键道具」——没有原型名、没有文化领域。

### 触发条件

用户提出道具需求但：
- 没说原型名（如「金箍棒式」「魔戒式」）
- 没说文化领域（如「东方法宝」「北欧神器」）
- 只说了道具功能（如「一把武器」「一个信物」）或零信息（「来一个酷的道具」）

### 执行逻辑

LLM 自动用故事上下文当过滤器，跨全领域匹配道具原型：

```
已建立的故事上下文            →  过滤器            →  道具原型候选
─────────────────────────────────────────────────────────
世界观（修仙/都市/末世）      →  过滤出该世界规则兼容的道具类型
调性（燃/悲/荒诞/暗黑）       →  过滤出该情绪基调的道具风格
使用者的角色引擎              →  找最能放大角色弱点的道具
道具的叙事功能                →  找最能制造戏剧张力的物理法则
```

### 交互模板（零知识模式）

```
你需要一个 [道具类型/功能]。基于你目前的设定——
使用者是 [角色名]（[角色引擎简述]），当前 [剧情处境]。

我扫描了世界经典道具库，这几个引擎最适合你的故事：
```

### 零知识模式示例

```
用户（已设定：都市暗黑复仇 · 主角哈姆雷特式总裁 · 刚拿下第一场胜仗）：
「主角需要一件他父亲留给他的遗物，很厉害的那种」

LLM：
你的主角是哈姆雷特式——思辨拖延，需要「百分百确定」才出手。
他刚赢了一场，但靠的是运气不是决断。
他父亲留下的遗物，应该推他一把——逼他突破自己的致命弱点。

### A：石中剑式
核心：只有「配得上」的人才能使用。
你的版本：父亲留下的企业印章。但有一个生物锁——
只有检测到使用者心率平稳（真正的冷静决策），才能盖章生效。
主角第一次用时，印章不亮——他在发抖。
戏剧性：这枚印章就是他父亲在说：「你现在还不够格。」

### B：魔戒式
核心：强大的能力 + 恐怖的代价。
你的版本：父亲留下的黑卡，里面有花不完的钱——
但每次使用，银行会自动向警方匿名举报一笔「疑似洗钱交易」。
用了它等于把自己暴露给法律风险。
戏剧性：父亲在说「用我的钱可以，但你要有承担后果的觉悟。」

### C：月光宝盒式
核心：能看到过去，但不能改变——越看越痛苦。
你的版本：父亲的手机，云端存着他死前三个月的所有通话录音。
主角可以听到真相——但每听一段，录音自动删除一段。
戏剧性：真相是有限的资源。他必须决定「先听哪一段」——
而他的性格（思辨拖延）让他痛苦万分。

### D：自定义
```

**关键**：用户从头到尾不需要知道石中剑、魔戒、月光宝盒是什么——
他只需要看「哪个道具的戏剧效果最对我的故事」。
原型引擎是 LLM 在后台匹配用的，不是用户需要认识的。
