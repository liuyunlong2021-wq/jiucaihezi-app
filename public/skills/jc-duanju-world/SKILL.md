# 短剧世界模型

从零创作短剧。核心工作方式是**渐进式逐场创作**：读取世界状态，围绕压力值最高角色设计催化事件，让角色在红线被触碰后自然反应，生成一场可拍、强冲突、有钩子的短剧场景。

**每一步都让用户做选择，绝不擅自推进。**

配合 jc-jian-wiki（巡检）和 jc-raw-wiki（填充）形成「创作→养护→质检」内循环——你负责写，它们负责让记忆不丢、剧情不崩。

本 skill 最大的作用是**引导**——帮用户把模糊想法变成完整故事建制，锁定调性，不遗漏任何关键设定。

**新增「人物原型激发」**：在创作最早期，从世界经典角色库（文学/影视/历史/神话）中提取叙事引擎，挪移到用户的故事语境中。不是贴标签（「哈姆雷特式」），而是提取角色的决策算法、内核矛盾、关系化学式——让角色自带冲突生成能力。详见 `references/人物原型激发.md`。

**新增「场景原型激发」**：用到新场景时，从世界建筑/影视/历史空间库提取空间引擎。说「豪华宫殿」不如说「卢浮宫式的宫殿但缩小到私人住宅」——立刻有了空间逻辑、情绪锚点、材质语言。详见 `references/场景原型激发.md`。

**新增「道具原型激发」**：关键道具出现时，从神话/文学/影视经典道具库提取物理法则+叙事功能。不是「一把厉害的剑」，是「石中剑式的身份认证之剑」——道具自带的游戏规则驱动剧情。详见 `references/道具原型激发.md`。

**新增「叙事三角引擎」**：角色引擎 × 道具引擎 × 场景引擎 三体联动。催化剂同时击打三个维度时，戏剧张力指数级增长。详见 `references/叙事三角引擎.md`。

爽点参考 `references/爽点库.md`，调性分析参考 `references/风格库.md`（五维拆解协议，非封闭作品列表）。

## 系统架构

```
调度层 → 读取世界状态，决定下一场主角和冲突方向
执行层 → 根据场景工作台生成剧本内容
世界观层 → 存储所有静态和动态世界信息
```

8 个 Agent 角色：灵感收集 → 世界构建 → 开场设计 → 调度 → 催化剂设计（调性驱动） → 执笔（调性驱动） → 选择设计 → 文件更新

---

## 核心原则

1. **催化剂驱动** — 每场必须有打破主角状态、制造两难的事件
2. **蝴蝶效应** — 每个行动基于性格产生新矛盾，不一帆风顺
3. **多层冲突** — 每场 3-5 层：触发→反应→升级→应对→临时平衡
4. **压力值驱动** — 最高者为下一场主角（变化规则见 `references/核心机制.md`）
5. **人机交互铁律** — 每个决策点询问用户，提供选项+自定义，等待确认才继续
6. **中心事件先行** — 每场生成前内部确定一句话中心事件（谁/压力/催化/选择/结果）
7. **催化事件够狠** — 必须触碰红线、制造两难或升级冲突（标准见 `references/写作质量手册.md` §八）
8. **台词服务冲突** — 每句台词只服务当前冲突，不闲聊、不解释、不绕弯
9. **正文只写可拍内容** — 不写心理活动、不写作者解释、不写设定说明
10. **★ 三角引擎优先** — 催化剂设计优先考虑角色×道具×场景三角联动，详见 `references/叙事三角引擎.md`

## 文件约定

```
项目根/
├── .raw/                      ← ★ 原始素材（Obsidian 隐藏）
│   ├── 参考资料/               ← 书籍/文献（jiyiyasuo 摄入模式用）
│   ├── 对话记录/               ← 创作对话（jiyiyasuo 压缩模式用）
│   └── 灵感笔记/               ← 随手灵感
├── wiki/
│   ├── 剧本/第X集.md          ← 剧本正文（逐场追加）
│   ├── 角色/角色名.md         ← 人物小传（模板见 references/核心机制.md）
  ├── 场景/场景名.md         ← 场景档案（用到时激发+落袋，下游 jc-scene-prompt读取）
  ├── 道具/道具名.md         ← 道具档案（用到时激发+落袋，下游 jc-prop-prompt读取）
│   ├── 世界/
│   │   ├── 设定.md            ← 时代/地点/规则/等级体系（建制时创建）
│   │   ├── 调性.md            ← ★ 全剧调性锁（建制后立即锁定，后续一切据此驱动）
│   │   └── 关系.md            ← 角色关系图（关系变化时更新）
│   ├── 剧情/
│   │   ├── 悬念.md            ← 已埋悬念 + 待回收（每集更新，yizhixing 检查）
│   │   ├── 时间线.md          ← 每场事件按时间排列（每场追加，yizhixing 检查）
│   │   └── 大纲.md            ← 分集梗概（建制时创建，后续按需更新）
│   ├── hot.md                 ← 记忆小抄 ~500字（jiyiyasuo 维护，续写前必读）
│   ├── index.md               ← 全库目录（jiyiyasuo 维护）
│   ├── 交付手册.md            ← 提炼交付手册 输出（后置，不参与创作）
```

## 工作流程

### 从 0 创建 → `references/建制阶段.md` + `references/调性阶段.md` + `references/人物原型激发.md` + `references/场景原型激发.md` + `references/道具原型激发.md`

1. **第 0 轮**：先问"你想写什么？"，判断完整性
2. **★ 阶段 0.5：人物原型激发**（可选但推荐）——如果用户提到了角色类型，提议从世界经典角色库做原型挪移（详见 `references/人物原型激发.md`）：
   - 用户说出角色类型（如「霸道总裁」「复仇女主」）
   - 提供 3-4 个原型候选人（A/B/C/D）+ 自定义，每个原型附带完整的**叙事引擎五维分析**
   - 用户选定（或融合多个原型）→ 生成标准人物小传（🔒锁定区 + 🌱生长区）→ ✅ 写入 `wiki/角色/角色名.md`
   - 跳过原型激发 → 直接进入第 1 轮
3. **第 1-5 轮**：逐轮收集——世界观/主角/身份/反派/关键设定
4. **建制摘要**：输出全部设定让用户确认
5. **★ 调性阶段**：媒介（真人/动漫）→ 风格推荐（A/B/C/D/E）→ 锁定 `wiki/世界/调性.md`
6. **构建文件**：创建角色/世界/悬念/调性文件 + 写入 CLAUDE.md
7. **第 1 场**：催化剂选项**必须带调性风味** → 生成 → 确认 → 保存

**★ 创作过程中，随时触发场景/道具原型激发**：

- 续写中出现新场景 → 提议场景原型激发（详见 `references/场景原型激发.md`）
- 续写中出现关键道具 → 提议道具原型激发（详见 `references/道具原型激发.md`）
- 激发后 → 生成标准档案 → ✅ 写入 `wiki/场景/场景名.md` 或 `wiki/道具/道具名.md`

### 续写循环 → `references/续写工作流.md` + `references/叙事三角引擎.md`

5 步循环（三角引擎驱动）：

1. 读取状态（压力值最高角色 + **读取 `wiki/世界/调性.md`** + ★ **扫描 wiki/道具/ 和 wiki/场景/ 三角引擎**）
2. 催化剂选择（A/B/C + D，**A=三角引擎★推荐 B=双引擎 C=单引擎保底**）
3. 生成场景（**台词/反转/冲突按调性生成，结合道具物理法则和场景空间逻辑**） → 用户确认
4. 角色选择（可选，A/B/C + D 自定义）
5. 更新文件系统 → 报告 → 继续？

### 三 Skill 协作（不失忆循环）

本 skill 负责**创作**，写完后需要另外两个 skill 养护和质检：

```
创作 skill 写 3-5 集
       ↓
jc-raw-wiki 整理记忆（沉淀素材进 wiki/，刷新 hot.md）
       ↓
创作 skill 召回 hot.md → 接着写（记忆不丢失）
       ↓
写满 10 集 → jc-jian-wiki 全库巡检（查跨章矛盾/伏笔未收/断链）
       ↓
创作 skill 根据体检报告修复 → 继续循环
```

**衔接点**：
| 时机 | 建议调用 | 说明 |
|------|---------|------|
| 每写完 3-5 集 | jc-raw-wiki | 整理记忆，避免写到后面忘前面 |
| 每写完 10 集 | jc-jian-wiki | 全库体检，抓跨章穿帮 |
| 每次续写前 | 读 `wiki/hot.md` | 这是 jc-raw-wiki 维护的记忆小抄，快速唤起上下文 |
| 巡检发现问题后 | 本 skill 修复 | yizhixing 只挑错不改错，修复归你 |

## 自检清单

- [ ] 场次 `## 场X-Y`，对话 `角色名（情绪）："台词"`
- [ ] 每集有明确的中心事件（这一集到底讲什么）
- [ ] 开篇 3 秒有冲突，跳过无用过渡（不写打车/问路/找房间）
- [ ] 台词短（≤15字）、爆（有爆发力）、贴（贴合人设+调性）
- [ ] 每句台词都有目的（塑造/推动/强化/交代/递进）
- [ ] 起承转合结构：起10% 承80% 转 合10%
- [ ] ≥ 两轮预期反转，反转模式匹配调性（见 `wiki/世界/调性.md`）
- [ ] 结尾有悬念钩子/卡点，钩子类型匹配调性偏好
- [ ] 冲突强度够：施力点越狠，受力点反馈越强
- [ ] 危机感到位：信息差 / 时间锁 / 生死局
- [ ] 无 `【】` 标注，无不可拍表达，无心理活动
- [ ] 情绪强烈、直接、不绕弯，贴合调性的情绪基调
- [ ] 已更新 CLAUDE.md、角色压力值、道具激活状态、场景未用元素、wiki/剧情/悬念.md、wiki/剧情/时间线.md
- [ ] 已对照 `references/写作质量手册.md` §九点五「博伊尔检查」+ §十六「调性驱动写作」逐条自检
- [ ] ★ 三角检查：本场催化剂是否用到了可用的道具/场景维度？
- [ ] `scripts/validate_scene.py` 通过
- [ ] 建制阶段 13 项 + 调性阶段全部覆盖（清单见 `references/建制阶段.md` + `references/调性阶段.md`）

## 质量守则

❌ 一帆风顺 / 反派弱智 / 大段独白 / 重复催化剂 / 主线不明
✅ 每场升级冲突 / 悬念 10 场内回收 / 配角独立动机 / 跑 validate_scene.py
