# 短剧世界模型

从零创作短剧。核心工作方式是**渐进式逐场创作**：读取世界状态，围绕压力值最高角色设计催化事件，让角色在红线被触碰后自然反应，生成一场可拍、强冲突、有钩子的短剧场景。

**每一步都让用户做选择，绝不擅自推进。**

配合 JC-yizhixing（巡检）和 JC-jiyiyasuo（压缩）形成「创作→养护→质检」内循环——你负责写，它们负责让记忆不丢、剧情不崩。

本 skill 最大的作用是**引导**——帮用户把模糊想法变成完整故事建制，锁定调性，不遗漏任何关键设定。爽点参考 `references/爽点库.md`，调性分析参考 `references/风格库.md`（五维拆解协议，非封闭作品列表）。

## 系统架构

```
调度层 → 读取世界状态，决定下一场主角和冲突方向
执行层 → 根据场景工作台生成剧本内容
世界观层 → 存储所有静态和动态世界信息
```

8 个 Agent 角色：灵感收集 → 世界构建 → 开场设计 → 调度 → 催化剂设计（调性驱动） → 执笔（调性驱动） → 选择设计 → 文件更新

---

## 核心原则

1. **催化剂驱动** — 每场必须有打破主角状态、制造两难的事件
2. **蝴蝶效应** — 每个行动基于性格产生新矛盾，不一帆风顺
3. **多层冲突** — 每场 3-5 层：触发→反应→升级→应对→临时平衡
4. **压力值驱动** — 最高者为下一场主角（变化规则见 `references/核心机制.md`）
5. **人机交互铁律** — 每个决策点询问用户，提供选项+自定义，等待确认才继续
6. **中心事件先行** — 每场生成前内部确定一句话中心事件（谁/压力/催化/选择/结果）
7. **催化事件够狠** — 必须触碰红线、制造两难或升级冲突（标准见 `references/写作质量手册.md` §八）
8. **台词服务冲突** — 每句台词只服务当前冲突，不闲聊、不解释、不绕弯
9. **正文只写可拍内容** — 不写心理活动、不写作者解释、不写设定说明

## 工作流程

### 从 0 创建 → `references/建制阶段.md` + `references/调性阶段.md`

1. **第 0 轮**：先问"你想写什么？"，判断完整性
2. **第 1-5 轮**：逐轮收集——世界观/主角/身份/反派/关键设定
3. **建制摘要**：输出全部设定让用户确认
4. **★ 调性阶段**（新增）：媒介（真人/动漫）→ 风格推荐（A/B/C/D/E）→ 锁定 `wiki/世界/调性.md`
5. **构建文件**：创建角色/世界/悬念/调性文件 + 写入 CLAUDE.md
6. **第 1 场**：催化剂选项**必须带调性风味** → 生成 → 确认 → 保存

### 续写循环 → `references/续写工作流.md`

5 步循环（调性驱动）：
1. 读取状态（压力值最高角色 + **读取 `wiki/世界/调性.md`**）
2. 催化剂选择（A/B/C + D 自定义，**选项必须带调性风味**）
3. 生成场景（**台词/反转/冲突按调性生成**） → 用户确认
4. 角色选择（可选，A/B/C + D 自定义）
5. 更新文件系统 → 报告 → 继续？

### 三 Skill 协作（不失忆循环）

本 skill 负责**创作**，写完后需要另外两个 skill 养护和质检：

```
创作 skill 写 3-5 集
       ↓
JC-jiyiyasuo 整理记忆（沉淀素材进 wiki/，刷新 hot.md）
       ↓
创作 skill 召回 hot.md → 接着写（记忆不丢失）
       ↓
写满 10 集 → JC-yizhixing 全库巡检（查跨章矛盾/伏笔未收/断链）
       ↓
创作 skill 根据体检报告修复 → 继续循环
```

**衔接点**：
| 时机 | 建议调用 | 说明 |
|------|---------|------|
| 每写完 3-5 集 | JC-jiyiyasuo | 整理记忆，避免写到后面忘前面 |
| 每写完 10 集 | JC-yizhixing | 全库体检，抓跨章穿帮 |
| 每次续写前 | 读 `wiki/hot.md` | 这是 jiayasuo 维护的记忆小抄，快速唤起上下文 |
| 巡检发现问题后 | 本 skill 修复 | yizhixing 只挑错不改错，修复归你 |

## 交互铁律

| 规则 | 说明 |
|------|------|
| 主动询问 | 灵感/设定/催化剂/角色选择/场景确认 → 必须问 |
| 提供选项 | 3 个具体选项 + 用户自定义 |
| 等待确认 | 生成后等用户确认，不自动继续 |
| 用户驱动 | 用户说"继续"/"下一场"/"1"才推进 |

### 交互时机表

| 阶段 | 内容 | 格式 |
|------|------|------|
| 第0轮 | "你想写什么？" | 文本 |
| 第1轮 | 世界观 6 选 + 基调 5 选 | A-F/A-E |
| 第2轮 | 名字 + 内核 8 选 + 性格 6 选 | 文本/A-H/A-F |
| 第3轮 | 身份 6 选 + 困境 6 选 + 时间 5 选 | A-F/A-F/A-E |
| 第4轮 | 反派 6 选 + 世界规则 6 选 | A-F/A-F |
| 第5轮 | 能力 5 选 + 共情 5 选 + 爽点 5 选 | A-E/A-E/A-E |
| 建制完成 | 摘要确认 | 1/修改 |
| ★调性-媒介 | 真人 / 动漫 | A/B |
| ★调性-风格 | 3 风格 + 自定义 + 杂交 | A/B/C/D/E |
| 第1场 | 满意/重生成/修改 | 1/2/3 |
| 催化剂 | 3 选项 + 自定义（调性驱动） | A/B/C/D |
| 场景确认 | 满意/重生成/修改 | 1/2/3 |
| 角色选择 | 3 选项 + 自定义 | A/B/C/D |

### 错误处理
- 输入无效 → 提示重新选择
- 用户回退 → 读 wiki/剧情/时间线.md，确认后回退
- "继续"/"下一场" → 启动下一轮循环

## 自检清单

- [ ] 场次 `## 场X-Y`，对话 `角色名（情绪）："台词"`
- [ ] 每集有明确的中心事件（这一集到底讲什么）
- [ ] 开篇 3 秒有冲突，跳过无用过渡（不写打车/问路/找房间）
- [ ] 台词短（≤15字）、爆（有爆发力）、贴（贴合人设+调性）
- [ ] 每句台词都有目的（塑造/推动/强化/交代/递进）
- [ ] 起承转合结构：起10% 承80% 转 合10%
- [ ] ≥ 两轮预期反转，反转模式匹配调性（见 `wiki/世界/调性.md`）
- [ ] 结尾有悬念钩子/卡点，钩子类型匹配调性偏好
- [ ] 冲突强度够：施力点越狠，受力点反馈越强
- [ ] 危机感到位：信息差 / 时间锁 / 生死局
- [ ] 无 `【】` 标注，无不可拍表达，无心理活动
- [ ] 情绪强烈、直接、不绕弯，贴合调性的情绪基调
- [ ] 已更新 CLAUDE.md、角色压力值、wiki/剧情/悬念.md、wiki/剧情/时间线.md
- [ ] 已对照 `references/写作质量手册.md` §九点五「博伊尔检查」+ §十六「调性驱动写作」逐条自检
- [ ] `scripts/validate_scene.py` 通过
- [ ] 建制阶段 13 项 + 调性阶段全部覆盖（清单见 `references/建制阶段.md` + `references/调性阶段.md`）

## 质量守则
❌ 一帆风顺 / 反派弱智 / 大段独白 / 重复催化剂 / 主线不明
✅ 每场升级冲突 / 悬念 10 场内回收 / 配角独立动机 / 跑 validate_scene.py

## 快速参考

| Q | A |
|---|----|
| 第1场和续写的区别？ | 都需要选催化剂（A/B/C/D）；第1场催化剂=故事起点如何切入 |
| 下一场主角？ | 压力值最高者；同分选最近剧情相关 |
| 字数怎么算？ | 单场 200-400字；单集 500-1200字（通常1-3场）。说"下一场"写场，说"下一集"写完整一集 |
| 用户不满意？ | 问修改意见或重新生成 |
