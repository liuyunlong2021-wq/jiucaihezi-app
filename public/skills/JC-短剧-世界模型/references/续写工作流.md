# 续写工作流（5 步循环）

第 2 场及之后，严格按以下 5 步执行。**每步都必须等待用户选择或确认后才能继续。**

---

## 步骤一：读取当前状态

**Agent**：调度Agent

**输入**：
- wiki/世界/设定.md
- wiki/世界/调性.md
- wiki/角色/
- wiki/场景/ ← ★ 三角引擎：读取场景档案
- wiki/道具/ ← ★ 三角引擎：读取道具档案
- wiki/世界/关系.md
- wiki/剧情/悬念.md
- wiki/剧情/时间线.md

**处理逻辑**：
1. 读取压力值，找到压力值最高的角色
2. 读取该角色的完整档案（wiki/角色/）
3. 读取相关角色档案
4. 检查待回收悬念（wiki/剧情/悬念.md）
5. **读取调性锁（wiki/世界/调性.md）**，理解本剧的节奏/台词/反转/情绪偏好
6. ★ **扫描已激活道具**（wiki/道具/*.md），找出当前剧情中可用的道具及其物理法则
7. ★ **定位当前场景**（wiki/场景/*.md），提取场景的空间逻辑和情绪锚点

**输出**：场景分析报告
```
## 场景分析 — 场X-Y
- 下一场主角：[角色名]（压力值：X/100）
- 压力来源：[原因]
- 涉及角色：[列表]
- 当前场景：[场景名]（[空间逻辑简述]，未用元素：[列表]）
- 可用道具：[道具名]（[物理法则简述]，激活状态：[已激活/未激活]）
- 最近剧情：[简述]
- 待回收悬念：[列表]
- 调性约束：[简述当前调性的节奏/台词/反转偏好]
- 三角评估：角色=[简述] | 道具=[简述或无] | 场景=[简述或无]
- 方向建议：[1-2句，融入三角联动考量]
```
```

## 步骤二：选择催化剂 🔴 必须询问用户

**Agent**：催化剂设计Agent

**输入**：场景分析报告（含三角评估）、主角档案、世界观设定、**wiki/世界/调性.md**、`references/叙事三角引擎.md`

**设计原则**（必须遵守）：
1. **先读调性锁**：理解本剧的催化剂设计原则
2. **催化事件必须触及红线**：见 `references/核心机制.md`
3. ★ **三角思维**（详见 `references/叙事三角引擎.md`）：设计时多想两个维度——有没有可用道具？场景空间能否放大冲突？选项A优先三角、B/C按需降级
4. **选项差异化**：A=三角（推荐）、B=双引擎、C=单引擎保底、D=自定义
5. **引擎级别不输出**：三角/双引擎/单引擎是 LLM 内部质量检查，不标注在选项中

**交互模板**：
```
# 催化剂选项 — 场X-Y

主角：[名字]，压力值：X/100
节奏参考：[无 / wiki/参考/《书名》/第N章 / 用户指定]
当前场景：[场景名]（[空间简述]）—— 如无场景档案则省略此行
可用道具：[道具名]（[简述]）—— 如无可用道具则省略此行

### 选项A：[名称] ★推荐
[优先设计三角引擎——同时击打角色红线 × 道具物理法则 × 场景空间逻辑]

### 选项B：[名称]
[双引擎或单引擎]

### 选项C：[名称]
[单引擎保底]

### 选项D：自定义
请输入您希望的催化剂事件：

请选择（A/B/C/D）：
```

**三角引擎示例**（都市暗黑 · 哈姆雷特式总裁 · 石中剑式印章 · 阿房宫式办公室）：
```
### 选项A：印章审判 ★推荐
仇人把印章推过来：「签了转让协议。哦——得用你爸的印章。」
主角拿起印章。走这条走廊时他还在做心理建设——现在手在抖，印章不亮。

### 选项B：走廊里的录音
主角走这条无尽走廊时，手机响了——父亲的云端录音推送了一条新片段。
他站在走廊正中间，进退都是三分钟的路。

### 选项C：董事会的突袭
反派提出动议：以「精神不稳定」为由暂停主角CEO职务。
所有人举手——有三个是他今早刚谈好的盟友。
```

**等待用户选择后**进入步骤二点五。

## 步骤二点五：建立场景工作台（内部执行，不打断用户）

在生成场景内容前，内部确定以下要素。**不输出给用户**，只用于指导写作：

```
- 本场中心事件：（一句话，谁/压力/催化事件/选择/结果）
- 催化事件：
- 被触碰的角色红线：
- 道具如何参与：（物理法则如何在本场驱动/约束行动）—— 如无可用道具则写「无」
- 场景如何参与：（空间逻辑如何在本场放大冲突）—— 如无场景档案则写「无」
- 施力点（谁来施压）：
- 受力点（谁承受，如何反馈）：
- 冲突升级方式：
- 本场预期反转：（必须匹配调性的反转偏好）
- 结尾钩子：（钩子类型匹配调性偏好）
- 调性检查：
  ✓ 节奏是否符合调性？
  ✓ 反转模式是否匹配调性？
  ✓ 台词语气是否贴合调性？
- 三角检查：
  ✓ 道具的物理法则在本场是否被用到？
  ✓ 场景的空间逻辑在本场是否被用到？
  ✓ 如果本场没用到的引擎元素，是否记录下来供后续场次使用？
```

> 用户如果要求查看，可以展示。

---

## 步骤三：生成场景内容（调性驱动）

**Agent**：执笔Agent

**输入**：场景分析报告 + 用户选择的催化剂 + **wiki/世界/调性.md**

**处理逻辑**：
1. 设计 3-5 层冲突（冲突升级方式匹配调性）
2. 每层冲突基于人物性格
3. **台词语气/金句密度/对白长度匹配调性**
4. 反转模式匹配调性（如无厘头→身份错位/意外暴露；超英→绝境觉醒/华丽反杀）
5. 保持节奏紧凑

**冲突层次**：
- 第1层：催化剂触发初始冲突
- 第2层：角色反应产生新矛盾
- 第3层：矛盾升级或转向
- 第4层：角色应对，新的冲突
- 第5层：达成临时平衡（保持张力）

生成后，向用户确认：

```
## 场X-Y 已生成，请确认（调性：[简述]）

[展示完整场景内容]

1. 满意 - 继续  2. 不满意 - 重新生成  3. 需要修改

请选择（1/2/3）：
```

**等待用户确认后**进入步骤四。

---

## 步骤四：角色选择（可选）🔴 场景需要决策时必须询问

**Agent**：选择设计Agent

**何时触发**：当场景中角色需要做重要决定时（如：要不要杀反派？要不要说出真相？）

**交互模板**：
```
# 角色选择 — 场X-Y

当前情境：[简述角色面临的两难]

### 选项A：[选择描述]
后果：[对压力值/关系/剧情的影响]

### 选项B：[选择描述]
后果：[对压力值/关系/剧情的影响]

### 选项C：[选择描述]
后果：[对压力值/关系/剧情的影响]

### 选项D：用户自定义
请输入您希望的角色行动：

请选择（A/B/C/D）：
```

**等待用户选择后**进入步骤五。

> 如果场景不需要角色决策（过渡/铺垫场），可跳过此步骤。

---

## 步骤五：更新文件系统

**Agent**：文件更新Agent

**输入**：生成的场景内容 + 用户的选择

**更新顺序**：
1. 剧本正文 → `wiki/剧本/第X集.md`（追加本场）
2. 角色档案 → `wiki/角色/` 更新压力值、成长轨迹
3. 关系网 → `wiki/世界/关系.md` 更新关系强度
4. 世界快照 → `wiki/世界/设定.md` 更新压力值排序、主要冲突
5. 调性 → `wiki/世界/调性.md` 读取（只读，确保本场合规）
6. 悬念 → `wiki/剧情/悬念.md` 新增或回收
7. 时间线 → `wiki/剧情/时间线.md` 追加本场事件
8. CLAUDE.md → 更新 `## [创作] JC-duanju-shijiemoxing` 区块

**输出**：文件更新报告
```
## 文件更新报告 — 场X-Y
- 更新的文件：[列表]
- 关键变化：[压力值变化/关系变化/悬念新增或回收]
- 调性合规：[本场是否贴合调性锁，备注]
- 下一场主角预判：[角色名]（压力值：X/100）
```

更新完成后，询问用户是否继续下一场。
