# 场景原型激发 —— 从世界建筑/空间库挪移场景引擎

> **核心洞察**：LLM 训练数据覆盖人类所有经典建筑、影视场景、历史空间。创作时说「豪华宫殿」——大模型和用户都 get 不到。说「卢浮宫式的宫殿，但缩小到私人住宅尺度」——立刻就懂了。

## 一、为什么不直接描述

「一个阴暗的地下室」——每个人脑子里想的不一样。是刑房？是酒窖？是末日避难所？描述越抽象，下游场景设计提示词搜 Pinterest 越难命中。

「巴士底狱式的地下囚室」——立刻锁定：石墙、铁链、狭小窗口、潮湿、绝望感。Pinterest 搜 `bastille dungeon environment design` 精准命中。

**场景原型 = 空间逻辑 + 情绪锚点 + 材质语言**的三合一快速索引。

## 二、场景引擎四维提取协议

### 维度 1：空间逻辑（这个空间如何组织人的行为）

| 问自己 | 产出 |
|--------|------|
| 空间如何控制人的移动路径？ | 强制路线 / 自由穿行 / 迷宫式 |
| 空间如何体现权力/等级？ | 谁在高处/谁在深处/谁在中心 |
| 空间的「看与被看」关系？ | 谁能看到谁，谁被谁看到 |

**示例**：
- 阿房宫：覆压三百余里 → 以「大」压人 → 人在其中是蝼蚁
- 蝙蝠洞：地下/高穹顶/暗 → 既是庇护所也是囚笼 → 人在其中被自己的影子追赶
- 千与千寻汤屋：垂直分层 → 底层锅炉房→中层浴场→顶层汤婆婆 → 每一层是不同的权力等级

### 维度 2：情绪锚点（这个空间让人产生什么感受）

| 问自己 | 产出 |
|--------|------|
| 进入这个空间的第一情绪是什么？ | 敬畏/压抑/温暖/不安/自由/被困 |
| 这种情绪通过什么空间手段达成？ | 尺度/光线/材质/声音 |
| 情绪是否有转折？ | 刚进入时__ → 待久后__ |

**示例**：
- 卢浮宫玻璃金字塔：现代入侵古典 → 第一眼「冲突」，走进去「和解」
- 酒池肉林：极致的感官刺激 → 第一眼「震撼」，待久「恶心/虚无」
- 哈尔的移动城堡：混乱中的温暖 → 第一眼「这是什么鬼」，推开门的瞬间「我想住这里」

### 维度 3：材质语言（这个空间用什么材料说话）

| 问自己 | 产出 |
|--------|------|
| 标志性材质是什么？ | 石/木/玻璃/金属/织物/水 |
| 材质的对比关系？ | 粗粝 vs 光滑 / 冷 vs 暖 / 旧 vs 新 |
| 材质透露了什么信息？ | 时代/财富/性格/历史痕迹 |

### 维度 4：尺度签名（大还是小，为什么）

| 问自己 | 产出 |
|--------|------|
| 这个空间的标志性尺度是什么？ | 巨型/正常/微型 |
| 尺度制造了什么效果？ | 压迫感/亲密感/荒诞感 |
| 短剧/小说的场景适配？ | 哪些特征需要放大，哪些需要缩小 |

## 三、交互模板

在剧本创作过程中，当用户提到一个新场景时触发——不是在建制阶段全做完，而是「用到时再激发」。

```
你提到了 [场景名]。要不要做一次场景原型激发？

跟角色一样——我直接从世界建筑/影视/历史空间库里提取几个经典场景的「空间引擎」：

### 原型 A：阿房宫式 [场景类型]
- 空间引擎：以「大」压人。覆压三百余里的尺度制造绝对权力感。
  人在其中不是使用者，是装饰品。
- 转译要点：保留「走廊无尽延伸/天花板极高/所有家具都比正常尺寸大一号」的尺度逻辑。
  去掉「皇家专属色」——换成目标时代的色彩体系。
- 招牌画面：总裁的办公室在走廊尽头，访客要走三分钟才能从门口走到他桌前。
  这段路没有椅子，没有窗户，只有两排空白的墙。
  走这段路的人，气势已经矮了一半。

### 原型 B：巴比伦空中花园式 [场景类型]
- 空间引擎：在不可能的地方造绿洲。反重力、反常识、反环境。
  核心矛盾：沙漠中的花园 = 权力可以改写自然法则。
- 转译要点：保留「植物从非地面位置生长/水从不应有水的地方流出」的反逻辑。
  替换为现代材料（钢结构+玻璃+垂直绿化）。
- 招牌画面：总裁的顶楼公寓，整面墙是一块垂直森林。
  在市中心40层高空，他赤脚踩在草地上。

### 原型 C：对角巷式 [场景类型]
- 空间引擎：隐藏在日常背后的魔法世界。入口越不起眼，内部越疯狂。
  核心矛盾：表象 vs 真相的空间化。
- 转译要点：保留「入口隐蔽/内部颠覆外部认知/空间折叠感」。
  替换魔法为科技或社会隐喻。
- 招牌画面：从破旧便利店后门推开，走进一个地下拍卖行——
  这里交易的，是上市公司不知道的秘密。

### 原型 D：自定义
你想用什么场景原型？故宫、比萨斜塔、哈尔的移动城堡、千与千寻汤屋、
浣熊市警察局、泰坦尼克号……任何你想得到的空间。
```

## 五、使用时机

与角色原型激发不同——场景不像角色那样一开始就全部确定。场景是**用到时才激发**：

| 时机 | 触发 |
|------|------|
| 建制阶段 | 没有具体场景不需要激发 |
| 续写时提到新场景 | 「这个场景是什么样的？要不要做原型激发？」 |
| 用户说「XX的住所/办公室/秘密基地」 | 主动提议 |
| 下游做场景设计前 | 先确保 wiki/场景/ 里有档案，没有就补激发 |

## 六、零知识模式 — 当用户说不出原型也说不清文化领域

用户说「主角需要一个住的地方」「反派的老巢是什么样的」——没有原型名、没有文化领域。

### 触发条件

用户提出场景需求但：
- 没说原型名（如「卢浮宫式」「对角巷式」）
- 没说文化领域（如「东方宫殿」「哥特式」）
- 只说了场景功能（如「一个办公室」「一个藏身处」）

### 执行逻辑

LLM 自动用故事上下文当过滤器，跨全领域匹配空间原型：

```
已建立的故事上下文            →  过滤器            →  空间原型候选
─────────────────────────────────────────────────────────
世界观（修仙/都市/末世）      →  过滤出该世界的空间类型
调性（燃/悲/荒诞/暗黑）       →  过滤出该情绪基调的空间
使用者的角色引擎              →  找最能外化角色内心的空间
场景的戏剧功能                →  找最能为这场戏制造冲突的空间
```

### 交互模板（零知识模式）

