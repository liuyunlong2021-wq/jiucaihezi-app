## 压力值系统

| 压力值 | 状态 | 场景类型 |
|--------|------|----------|
| 0-30 | 平静 | 缓和/铺垫 |
| 30-70 | 紧张 | 冲突/推进 |
| 70-100 | 爆发 | 高潮/决战 |

**变化规则**：触碰红线 +50 / 催化剂触发 +20 / 冲突解决 -20 / 冲突升级 +30 / 两难选择 +15。
每场后重新计算，最高者为下一场主角。

## 催化剂类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 外部入侵 | 外部力量打破平衡 | 敌人来袭 |
| 内部觉醒 | 角色认知改变 | 发现真相 |
| 关系裂变 | 人际突发变化 | 盟友背叛 |
| 世界突变 | 环境剧烈变化 | 灾难降临 |

每场开始前提供 3 个催化剂选项（A/B/C）+ D.用户自定义。

## 红线 × 压力值 × 催化剂 联动规则

三者不是独立运作的，必须联动：

```
催化事件 → 触碰红线 → 压力值飙升 → 角色被逼到墙角 → 自然产生强烈反应
```

**设计催化事件时**：
1. 先看角色的红线是什么（不可触碰的底线）
2. 设计一个事件精确触碰到这条红线
3. 触碰红线 → 压力值 +50 → 角色必须立刻行动
4. 角色的行动产生新的矛盾 → 推动下一场

**反例**（无效催化）：
- 催化事件不触碰任何红线 → 角色不痛不痒 → 压力值不变 → 剧情原地踏步
- 催化事件触碰红线但角色忍了 → 角色人设崩塌 → 观众弃剧

**正例**（有效催化）：
- 林风的红线是"家人被伤害" → 催化事件：反派当众侮辱林风母亲 → 压力值 +50 → 林风暴起反击 → 引发更大冲突

## ★ 叙事三角引擎

角色不是唯一的冲突来源。道具的物理法则和场景的空间逻辑，本身就是冲突生成器。详见 `references/叙事三角引擎.md`。

**核心原则**：设计催化剂时多想两个维度——有没有可用道具？场景的空间逻辑能否放大冲突？能多咬合一个维度 = 催化剂更强。不建追踪系统，不标引擎级别，LLM 每场重读 wiki 自然判断。
