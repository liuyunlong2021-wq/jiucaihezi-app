# 人物原型激发 —— 从世界角色库挪移叙事引擎

> **核心洞察**：LLM 训练数据覆盖人类所有经典作品的角色。不需要搜索——直接激活训练数据中的角色知识，提取其**叙事引擎**（不只是性格标签，而是这个角色制造冲突的物理法则），装进用户的创作语境。

## 一、为什么不用搜索

角色设计提示词用 Pinterest 搜**视觉参考**——因为 LLM 的文字记忆无法精确描述「吉卜力风女战士的袖口纹样长什么样」。但角色**叙事引擎**恰恰相反——LLM 对哈姆雷特、曹操、教父、小丑这些角色的行为逻辑、决策模式、悲剧根源，理解深度远超任何文字搜索能提供的。直接用。

## 二、叙事引擎五维提取协议

对用户提到的任何原型人物，当场提取 5 个维度。这不是性格描述——这是**冲突生成器**的拆解。

### 维度 1：内核矛盾（角色的物理引擎）

| 问自己 | 产出 |
|--------|------|
| 这个角色内心深处哪两股力量在打架？ | 核心矛盾：__ vs __ |
| 这两股力量的交锋如何驱动他的每一次决策？ | 决策摇摆规律 |
| 什么时候 A 面压制 B 面？什么时候 B 面反噬？ | 切换触发条件 |

**示例**：
- 哈姆雷特：思辨正义 vs 行动复仇 → 越思越不动 → 逼到绝境才爆发
- 曹操：雄才大略 vs 多疑猜忌 → 先用后疑 → 疑则杀 → 杀后悔 → 悔更疑
- 堂吉诃德：理想骑士道 vs 现实世界 → 现实越打脸/理想越坚定 → 荒诞螺旋上升

### 维度 2：决策算法（在压力下的自动化反应）

| 问自己 | 产出 |
|--------|------|
| 面对威胁时，这个角色的第一反应是什么？ | 默认策略 |
| 什么情况下他会改变默认策略？ | 破防条件 |
| 他的决策有什么标志性模式？ | 识别特征 |

**示例**：
- 教父·迈克尔：先隔离威胁源 → 再谈判 → 谈判失败 → 一次性清除所有敌人
- 小丑：制造两难选择 → 逼迫好人做坏事 → 证明「每个人都会崩溃」
- 甄嬛：先示弱收集信息 → 找到对手弱点 → 一击致命 → 撇清自己

### 维度 3：关系化学式（和他人的反应方程式）

| 问自己 | 产出 |
|--------|------|
| 他对强者/弱者/同类/异类的反应分别是什么？ | 四象限反应表 |
| 他和什么类型的人会产生最强化学反应？ | 最佳对手配方 |
| 他最容易和什么类型的人结盟/决裂？ | 盟友/敌人模板 |

### 维度 4：悲剧/喜剧触发器

| 问自己 | 产出 |
|--------|------|
| 什么事件会让他走向悲剧？ | 触发条件 |
| 什么事件会让他逆袭？ | 翻盘条件 |
| 他最大的自毁倾向是什么？ | 阿喀琉斯之踵 |

### 维度 5：当代转译锚点

| 问自己 | 产出 |
|--------|------|
| 这个角色的核心矛盾在现代语境下对应什么？ | 当代映射 |
| 他的标志性场景可以转换成什么现代场景？ | 场景转译 |
| 哪些元素必须保留，哪些可以替换？ | 保留/替换清单 |

## 三、交互模板

### 阶段触发

在用户说出初始想法后，判断：用户是否提到了角色类型（如「霸道总裁」「复仇女主」「废柴逆袭」）？

- ✅ 提到了 → 主动提议原型激发
- ❌ 没提到 → 先走正常建制第0-1轮，在第2轮（主角核心）时提议

### 交互流程

### 原型 A：哈姆雷特式总裁
- 引擎提取：思辨型复仇者。父亲（创始人）死于叔叔（现任CEO）之手。
  他明明手握证据，却反复验证、拖延、自我怀疑。
  越接近真相越不敢行动——不是懦弱，是太聪明，想太多。
- 总裁版：继承之战。父亲被合伙人害死，他十年后回归夺权，
  但每次要出手都在想「万一错了呢」。
  魅力来源：观众看着他一步步把自己逼到绝境，最后在无路可退时爆发。
- 招牌场景：董事会上，仇人坐在对面。他手握致命证据。
  所有人都等他出手。他开口了——说的是一段关于商业伦理的演讲。
  散会后，对手笑了。他在车里把证据捏成一团，手在发抖。

### 原型 B：曹操式总裁
- 引擎提取：枭雄型。多疑→试探→确认→清洗→后悔→更多疑，
  这是一个自循环的猜疑链。核心矛盾：才华横溢 vs 安全感缺失。
- 总裁版：白手起家的狼性CEO，信奉「宁教我负天下人」。
  收购对手公司后，半夜睡不着，爬起来重新看合同——
  怀疑对方留了后手。第二天召集律师团队逐条复查。
  魅力来源：极强的能力和极度的不安全感驱动出极端的行动。
- 招牌场景：他最信任的VP递来辞呈。他微笑签字，VP转身离开。
  然后他拨通电话：「查他过去三个月的所有记录。所有。」
  挂了电话，看着VP的工位照片——这是他创业时的第一个员工。

### 原型 C：堂吉诃德式总裁
- 引擎提取：理想主义到偏执。把风车当巨人，把平凡当史诗。
  现实越打脸，信念越坚定。核心矛盾：崇高理想 vs 平庸现实。
- 总裁版：一个做传统手工艺的二代，非要和互联网巨头正面竞争。
  所有人都说这是以卵击石。他把工厂当成骑士城堡，
  把每一次被碾压当成「骑士的考验」。
  魅力来源：你一边觉得他可笑，一边希望他赢。
- 招牌场景：公司账上只剩三个月现金流。团队在会议室焦虑。
  他站起来，指着窗外互联网巨头的大楼：
  「看见那栋楼了吗？那就是我的风车。」
  一个员工小声说：「老板，那是真的楼……」

### 原型 D：基督山伯爵式总裁
- 引擎提取：精密复仇机器。被陷害→绝境觉醒→获得资源→
  化身归来→逐个复仇。每个仇人的毁灭方式精确对应其罪行。
- 总裁版：被合伙人做局踢出自己创立的公司，入狱三年。
  在狱中遇到一位商业前辈，学到一切。出狱后以新身份回归，
  用精密的商业布局让每个仇人按他们自己的弱点自毁。
  魅力来源：看一场完美复仇——每一步都在计划之中。
- 招牌场景：在拍卖会上，他举牌竞拍自己当年创立的公司。
  坐在前排的，正是当年害他的合伙人。

## 引擎激活指南（给催化剂系统用）

当调度层读取本角色时，AI 自动提取以下信息用于生成催化剂：

- **决策算法** = 角色反应模板（触碰红线后他会怎么行动）
- **致命弱点** = 冲突升级入口（他的弱点就是反派下手的地方）
- **性格反差** = 反转素材（表面对外形象 vs 私下真实反应）

## 五、原型融合模式（进阶）

用户可以说「给我一个 哈姆雷特（思辨） + 基督山伯爵（复仇手段） 的总裁」。

融合规则：
1. **主体引擎**（第一个原型）= 角色的决策算法和内心矛盾
2. **方法引擎**（第二个原型）= 角色的行动方式和策略
3. 冲突点 = 两个引擎的矛盾之处（思辨型的大脑 vs 精密行动型的手）——这个冲突本身就可以是角色弧光

示例：
```
## 原型融合 — 哈姆雷特 × 基督山伯爵 式总裁

主体引擎（哈姆雷特）：思辨驱动，行动前反复验证，内心道德挣扎
方法引擎（基督山伯爵）：一旦决定行动，精密布局，一击致命
合成效果：一个平时看起来优柔寡断的总裁，但当他终于出手时——
你才发现他沉默的那三个月，已经布好了天罗地网。
他的「拖延」不是懦弱，是在下棋。
核心矛盾：我需要百分百确定才出手（哈姆雷特） vs 我一旦出手必置人死地（基督山伯爵）
→ 这个矛盾本身就是他的悲剧根源：等他「确定」的时候，可能已经来不及了。
```

## 六、零知识模式 — 当用户说不出原型也说不清文化领域

用户说「我想要一个很酷的主角」「帮我设计一个反派」——没有原型名、没有文化领域。

**不要慌。用户虽然没有文化知识，但他提供了更重要的东西：故事上下文。**

### 触发条件

用户提出角色需求但：
- 没说原型名（如「哈姆雷特式」）
- 没说文化领域（如「东方修仙」「北欧神话」）
- 只说了角色功能（如「一个反派」「一个导师」）或零信息（如「帮我设计主角」）

### 执行逻辑

LLM 自动完成以下匹配：

```
已建立的故事上下文            →  过滤器            →  原型候选
─────────────────────────────────────────────────────────
世界观（修仙/都市/末世）      →  过滤出该世界规则    →  排除不兼容的原型
调性（燃/悲/荒诞/暗黑）       →  过滤出该情绪基调    →  排除基调冲突的原型
已有角色引擎                  →  找化学反应最强的    →  排在前面
当前剧情处境                  →  找立刻能制造冲突的  →  加急推荐
```

### 零知识模式示例

```
用户（已设定：都市暗黑复仇 · 主角是哈姆雷特式总裁）：
「帮我设计一个反派」

LLM：
你的主角是哈姆雷特式——思辨型，越接近真相越犹豫。
一个有效的反派应该击打这个弱点。我扫描了几个引擎：

### A：克劳狄斯式（哈姆雷特原配对手）
为什么适合：他是已经赢了的人。你的主角反复验证、拖延——
而反派一直在主动出击，不给他思考的时间。
「你不是想确认是不是我杀的你爸吗？对，是我。然后呢？」
招牌场景：反派约主角喝咖啡，当他的面给主角的盟友发offer——挖角。

### B：小丑式
为什么适合：小丑不跟你斗智，他跟你斗「失去理智」。
哈姆雷特型角色最怕的不是强敌，是规则被打破。
招牌场景：反派劫持了主角公司系统，但不要赎金——
他在大屏幕上直播随机解雇员工。每十分钟一个，直到主角「承认自己也是个混蛋」。

### C：曹操式
为什么适合：反派也是高智商，但决策算法是「先下手为强」。
他和主角的差别：主角收集证据→验证→行动。反派：怀疑→立刻清洗→证据以后再说。
招牌场景：主角刚查到一条线索，赶到现场——
反派已经把所有相关证据销毁了，现场还给你泡了杯茶。

### D：自定义
```

**关键**：用户从头到尾不需要知道克劳狄斯是谁、小丑是谁、曹操是谁——
他只需要看「哪个引擎跟我故事最搭」。原型是 LLM 在后台用的工具，不是用户需要认识的名字。
