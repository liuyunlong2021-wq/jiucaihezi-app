# SKILL-01: 情绪算法引擎 (Emotion Algorithm Engine)

> 🎢 "剧情只是载体，情绪才是目的。不要问'发生了什么'，要问'读者感觉到了什么'。"

---

## 核心职责

负责从**目标情绪流**（Emotional Flow）反推**具体情节**。将抽象的"爽"或"甜"转化为可执行的情节触发器。

---

## 情绪曲线模型

### 1. 爽文曲线 (The Cool Curve)

- **核心逻辑**：压抑 → 爆发
- **公式**：
  1. **起（憋屈）-8分**：被误解、被轻视、被抢夺。
  2. **承（沉潜）-5分**：默默积蓄力量，准备反击。
  3. **转（爆发）+10分**：实力碾压，当众打脸，真相大白。
  4. **合（余韵）+5分**：他人震惊，主角淡定离场。

### 2. 甜文曲线 (The Sweet Curve)

- **核心逻辑**：试探 → 确认
- **公式**：
  1. **起（懵懂）+2分**：有些好感，但未察觉。
  2. **承（推拉）+5分**：暧昧试探，吃醋，眼神拉丝。
  3. **转（高甜）+10分**：双向奔赴，亲密接触，确认心意。
  4. **合（温馨）+8分**：日常发糖，确定的幸福感。

### 3. 虐文曲线 (The Angst Curve)

- **核心逻辑**：美好 → 破碎
- **公式**：
  1. **起（美好）+8分**：曾经拥有，岁月静好。
  2. **承（裂痕）-2分**：误会产生，信任动摇。
  3. **转（破碎）-10分**：彻底决裂，真相迟到，生离死别。
  4. **合（意难平）-5分**：回不去了，无尽遗憾。

## 情绪触发器库 (Emotion Triggers)

| 情绪目标 | 常见触发器 (Plot Devices) |
|:---|:---|
| **愤怒/憋屈** | 被最信任的人背叛、被污蔑且无法辩解、成果被窃取、替人顶罪 |
| **爽快/解气** | 身份/马甲曝光、当众展示实力、反派跪地求饶、舆论反转 |
| **心动/暧昧** | 意外的身体接触、由于误会产生的亲密、危急时刻的保护、双标宠溺 |
| **紧张/悬念** | 身份即将暴露、时间倒计时、二选一困境、杀手逼近 |
| **感动/治愈** | 绝境中的援手、跨越时空的守护、无条件的信任、自我牺牲 |

## 生成指令

**输入**：我想写一个[爽文]，主角是[真假千金]。

**处理流程**：

1. 调用**爽文曲线**：憋屈 → 爆发。
2. 匹配**憋屈触发器**：被亲生父母嫌弃，被假千金陷害。
3. 匹配**爆发触发器**：马甲掉落（其实是神医/首富），众人后悔莫及。

**输出**：

- **开篇情绪**：极度憋屈（家人偏心假千金）。
- **转折情节**：在豪门宴会上，假千金装病，主角直接出手治好（展示神医马甲）。
- **高潮情绪**：全场震惊，家人想认回但被拒（爽）。

## 检查清单 (Checklist)

- [ ] 设定好的情绪曲线是否有起伏？（数值差是否 > 10？）
- [ ] 选用的触发器是否符合当前的人设？
- [ ] 情绪转化是否生硬？（需要铺垫）
