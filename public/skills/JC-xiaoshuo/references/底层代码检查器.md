# SKILL-05: 底层代码检查器 (Core Code Logic Checker)

> 🧬 "很多梗看似热闹，其实缺少'先天优越性'。检查底层代码，确保它天生能火。"

---

## 核心职责

检查核心梗是否包含**热题材的底层逻辑**（先天优越设定），如信息差、金手指或极致情感。

---

## 三大底层代码

### 代码1：信息差优势 (Information Gap)

- **定义**：主角知道别人不知道的信息。
- **表现**：穿书（知道剧情）、重生（知道未来）、读心（知道想法）。
- **爽点**：降维打击，开启上帝视角。

### 代码2：金手指优势 (Cheat Code)

- **定义**：主角拥有不劳而获的绝对优势。
- **表现**：系统、空间、绝世天赋、隐形富豪身份。
- **爽点**：碾压，容错率极高。

### 代码3：极致情感优势 (Extreme Emotion)

- **定义**：主角获得的情感浓度远超常人。
- **表现**：团宠（所有人爱我）、独宠（大佬只爱我）、火葬场（渣男跪求我）。
- **爽点**：被偏爱，安全感，情感补偿。

---

## 诊断与优化

**案例：一个普通的娱乐圈文**

- **诊断**：女主通过努力演戏拿奖。 → **无底层代码**（太累，爽点延迟）。
- **优化**：
  - 加信息差：女主重生，知道哪些剧会火。
  - 加金手指：女主有"演技系统"或"只要演戏就能通过弹幕看到观众反应"。
  - 加极致情感：影帝男主是她隐婚老公。

**合格标准**：至少包含 **1-2** 个底层代码。

---

## 检查清单 (Checklist)

- [ ] 主角是否有"不用努力就能赢"的先天优势？
- [ ] 读者代入主角后，是否能获得"特权感"？
- [ ] 如果没有底层代码，是否有足够强的人设反差来弥补？
