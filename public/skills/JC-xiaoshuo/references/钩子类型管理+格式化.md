# SKILL-06: 钩子类型管理+格式化 (Type Manager & Formatter)

> 🎣 "不仅要有钩子，还要知道这是什么钩子。标准化输出，让钩子一目了然。"

---

## 核心职责

1. **类型管理**：识别并标记钩子的具体类型（剧情/感情/身份/大钩子）。
2. **格式输出**：生成标准化的钩子设计方案。
3. **最终质检**：确保钩子有效且无毒。

---

## 钩子类型库

### A. 剧情钩子 (Plot Hooks)

- **A1 即将打脸**：承诺下章反转/复仇。
- **A2 危机降临**：危险逼近，生死存亡。
- **A3 真相揭露**：谜题解开的前一秒。

### B. 感情钩子 (Romance Hooks)

- **B1 关系突破**：亲吻、表白、确认关系。
- **B2 误会消融**：解除隔阂的前夜。
- **B3 醋意大发**：修罗场预警。

### C. 身份钩子 (Identity Hooks)

- **C1 马甲掉落**：身份曝光。
- **C2 身份对峙**：不知道对方是马甲的情况下相遇。

---

## 标准输出模板

```markdown
### 🎣 钩子设计方案：第X章

**断章位置**：[引用原文最后一段]

**钩子类型**：【A1 即将打脸 / B1 关系突破 / ...】
**核心承诺**：[下章读者能看到什么爽点/甜点]

**技法应用**：
- [ ] 确定性承诺
- [ ] 希区柯克悬念
- [ ] 未完成态

**极性检查**：
- [x] 正面期待 (✅)
- [ ] 负面憋屈 (❌ → 需修改)

**大钩子状态**：[当前主线与大钩子在大纲中的状态简述]
```

---

## 终检清单 (Final Checklist)

- [ ] **必须有**：章末必须有明确的钩子，不能自然平淡结束。
- [ ] **必须爽**：承诺的内容必须是读者想看的。
- [ ] **必须准**：类型标记准确，便于后续（如012复盘）检查。
