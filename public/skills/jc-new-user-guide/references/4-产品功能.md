# 韭菜盒子 Studio · 页面介绍

> 打开韭菜盒子你会看到这些区域。每个都能干啥，往下看。

## Chat（聊天区）

页面正中间最大的区域。你跟 AI 说话的地方。

**能干啥**：
- 打字问问题，AI 回答
- 粘贴一大段内容（小说、剧本、代码），让 AI 帮你处理
- @ 引用项目里的文件，让 AI 基于文件回答
- 表达目标（比如“填充 Wiki”），启动 AI 的对应能力
- 桌面端：所有对话走 OpenCode Agent，AI 能直接改你的文件

## Rail（左侧导航栏）

页面最左边那一竖条图标。

**能干啥**：
- 切换工作区：创作面板 / 对话 / 历史记录 / 用户中心
- 手机端精简到 3 个按钮

## 项目

页面左上角，显示当前项目名。

**能干啥**：
- 切换项目：换个文件夹干活
- 一个项目 = 一个文件夹，所有文件、对话、Wiki 都在里面
- 桌面端直接操作本地文件夹，Web 端在浏览器里管理

## 会话

Chat 区域上方的历史对话列表。

**能干啥**：
- 查看之前跟 AI 聊过的对话
- 切换回某个旧对话继续聊
- 删除不需要的对话

## 编辑区

代码和文件的查看/编辑面板。

**能干啥**：
- 打开项目里的文件看内容
- 编辑 Markdown、代码、配置文件
- 语法高亮（不同语言显示不同颜色，方便阅读）
- Markdown 预览

## 创作面板

桌面端专属。一键生成图片/视频/音频。

**能干啥**：
- 选一个模型（图片/视频/语音）
- 填一段提示词描述你想要什么
- 提交后 AI 生成媒体文件
- 生成的图或视频存在 `jc-media/` 文件夹里
- 支持的模型：GPT Image、Banana、Seedance、Veo、LTX、Qwen TTS

## 审查变更

AI（OpenCode Agent）改完你的文件后，所有改动在这里显示。

**能干啥**：
- 绿底 = AI 加的内容
- 红底 = AI 删的内容
- 逐条看，对的点**接受**，不对的点**拒绝**
- 拒绝后文件恢复原样

## Skill 仓库

管理所有 Skill 的地方。

**能干啥**：
- 看已经装了哪些 Skill
- 从 GitHub 安装新的 Skill
- 从本地文件夹安装 Skill
- 禁用暂时不想用的 Skill
- 更新 Skill 到最新版本

> Skill 存在 `~/.agents/skills/` 目录下，没有内置 Skill，全自己装。

## 工具仓库

管理外部工具和 MCP 集成。

**能干啥**：
- 安装 MCP 工具（让 AI 连接外部服务）
- 管理 API 密钥
- 配置媒体生成后端

## 设置

**能干啥**：
- 填 API Key（各模型平台的通行证）
- 选默认模型（平时用哪个 AI）
- 切换文/武模式（文=省钱本地模型，武=强模型）
- 改语言（中文/English）
- 账号：手动 Key → Keychain → 账号登录 → Worker 代理
- 存储位置：`~/.jiucaihezi/data/`（数据库）+ `~/.jiucaihezi/output/`（媒体文件）

## 常见问题

### 桌面端和 Web 端有什么区别？
桌面端有完整能力（OpenCode Agent、Tauri 本地文件、创作面板、画布）。Web 端只有聊天和基础功能。

### 我想用的功能找不到？
可能你在 Web 端——创作面板、画布等是桌面端专属。或者手机端——只保留了最核心的功能。
