# SKILL-03: 弹簧理论节奏控制器

## 核心职责

确保每个事件的"爽感"合格，通过A-B-C模型验证

## 弹簧公式

```
A（基准状态）：主角原本的正常生活/能力
    ↓ 压抑
B（压抑值）：遇到的危机、反派的羞辱
    ↓ 反弹
C（逆袭值）：爆发后的状态

关键公式：C > A （必须比之前更强！）
```

## 数值化评分系统

```
A状态（基准）：0分
B状态（压抑）：-1到-10分
C状态（反弹）：必须>0分

合格标准：C - A > 0
优秀标准：C - A >= |B|（反弹值≥压抑值）
```

## 应用示例

### ❌ 错误示例（C=A）

```
A：程月在乡下，平静生活（基准值：0）
B：被豪门嫌弃羞辱（压抑值：-5）
C：手术后豪门态度好转，程月回到平静生活（逆袭值：0）

检查：C(0) = A(0) ❌ 不合格！
问题：只是解决了麻烦，没有获得奖励
```

### ✅ 正确示例（C>A）

```
A：程月在乡下，平静但无名（基准值：0）
B：被豪门嫌弃羞辱（压抑值：-5）
C：M医生身份曝光，成为医学界传奇（逆袭值：+7）

检查：C(+7) > A(0) ✅ 合格！
计算：C - A = +7，|B| = 5，反弹值 > 压抑值 ✅ 优秀！
```

## 每个事件必填表格

```markdown
### 事件X：[事件名称]

| 状态 | 描述 | 数值 |
|------|------|------|
| A（基准） | XXX | 0 |
| B（压抑） | XXX | -5 |
| C（反弹） | XXX | +8 |

**检查**：C > A？ ✅ 是 / ❌ 否
**反弹度**：C - A = +8，|B| = 5，✅ 反弹值 > 压抑值
```

## 关键原则

1. **实质性奖励**: C必须带来地位/能力/资源/关系突破
2. **递进原则**: 后面事件的C值必须 > 前面事件的C值
3. **压抑匹配**: B的压抑值越大，C的反弹值必须越高
4. **禁止原地**: 不能只是"恢复原状"
