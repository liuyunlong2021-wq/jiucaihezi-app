# 写作工作流（阶段8-9）

## 写作单位：逐章

每章 2000-4000 字，一章是一个完整的写作单元。一章内部分四阶段推进，每个阶段调用不同引擎协作完成。**写一章、确认一章、更新一章。**

---

## 阶段 8：正文写作

### 第1章特殊流程

开篇三选一（各 ≤300 字预览），用户选定后进入四阶段：

```
A. 冲突切入 — 战斗/危机中开场
B. 反常设定 — 独特世界观直接展示
C. 悬念倒叙 — 精彩片段前置
```

### 逐章四阶段（S2-全书完，每章循环一次）

```
┌─────────────────────────────────────────────────────────┐
│ 阶段一：构思（内部，不输出）                              │
│                                                         │
│ 读 CLAUDE.md → 压力值/待收伏笔/上章钩子                  │
│ 选催化剂（A/B/C/D）                                     │
│ 建场景工作台                                            │
│                                                         │
│ 调用引擎：                                              │
│   场景核心逻辑构建器  → 确定本章"一个中心事件"           │
│   弹簧理论节奏控制器  → 节奏分布（起10%/承80%/转/合10%）│
│   期待感管理引擎      → 读者会在哪里心跳加速？           │
│   钩子预埋系统        → 章末用什么钩子？                 │
├─────────────────────────────────────────────────────────┤
│ 阶段二：起草（生成正文 2000-4000字）                     │
│                                                         │
│ 调用引擎：                                              │
│   情绪先行引擎        → 本章情绪基调 + 情绪曲线          │
│   动作与对白生成器    → 每句对话有功能、贴合人设         │
│   冲突三位一体构建器  → 3-5层冲突，层层升级              │
│   视角修正器          → 视角一致，不跳上帝视角           │
│   视觉化锚点设计器    → 关键画面的视觉描写               │
│   潜文本设计器        → 台词背后的潜台词                  │
├─────────────────────────────────────────────────────────┤
│ 阶段三：润色（内部打磨）                                 │
│                                                         │
│ 调用引擎：                                              │
│   人设化润色器        → 每句检查是否贴人设               │
│   风格化节奏控制器    → 长短句交替、叙述节奏             │
│   微观修辞升级器      → 比喻/排比/通感进阶               │
│   心理描写侧面化器    → 不写"他很生气"，写动作           │
│   信息饱和度检查器    → 是否太空？是否太满？             │
│   跳笔剪辑引擎        → 跳过无用过渡                     │
├─────────────────────────────────────────────────────────┤
│ 阶段四：检查（质量把关）                                 │
│                                                         │
│ 调用引擎：                                              │
│   章纲自检过滤器      → 是否符合章纲？                   │
│   冲突自检雷达        → 冲突够不够？                     │
│   人设保护器          → 人设有没有崩？                   │
│   主角中心检查器      → 主角有没有边缘化？               │
│   伏笔管理器          → 新埋/回收伏笔记录                │
│   连压预警器          → 是否连续多章无高潮？             │
│                                                         │
│ 全部通过 → 用户确认 → 更新 wiki/ → 下一章                │
│ 任一不通过 → 回到对应阶段修复                            │
└─────────────────────────────────────────────────────────┘
```

### 分批节奏

```
第1-3章：黄金三章，开篇定调（每章用户确认）
第4-10章：建立节奏，每3章一确认
第11-30章：稳定输出，每5章一确认 + jiyiyasuo 压缩记忆
第31章+：每10章一确认 + yizhixing 巡检
```

## 阶段 9：复盘分析

### 全稿终检

```
调用引擎：
  终检器              → 10条硬规则逐条通过
  结构排毒自检器      → 主线/节奏/逻辑
  四线对齐诊断器      → 情节线/情感线/主题线/人物线
  质量自检过滤器      → 语言质量
  风格一致性检查器    → 前后风格是否统一
```

全部通过 → 交付。任一不通过 → 标注章节 → 回滚修复。
