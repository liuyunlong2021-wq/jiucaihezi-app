# 小说世界模型

长篇小说创作引擎。九阶段管线：灵感→市场→核心梗→角色→大纲→章纲→文案→正文→复盘。**基于极致角色+催化剂驱动，逐章推进，每一步让用户做选择。**

配合 JC-yizhixing（巡检）和 JC-jiyiyasuo（压缩）形成「创作→养护→质检」内循环。小说写作工具箱见 `JC-xiaoshuo`（独立 Skill，提供 100+ 写作引擎辅助）。

---

## 核心公式

```
世界观（规则） + 角色（极致性） + 催化剂（触发） = 故事自然涌现
```

## 核心原则

1. **催化剂驱动** — 每章必须有打破主角状态的事件
2. **蝴蝶效应** — 每个行动基于性格产生新矛盾
3. **多层冲突** — 每章 3-5 层：触发→反应→升级→应对→临时平衡
4. **压力值驱动** — 最高者为下一章主角
5. **中心事件先行** — 每章生成前内部确定一句话中心事件
6. **台词服务冲突** — 每句对话/叙述都有目的
7. **人机交互铁律** — 每个决策点询问用户，提供选项+自定义，等待确认

---

## 九阶段管线

```
阶段1: 灵感策划 → 3-10个创意方案（建制阶段）
阶段2: 市场分析 → 赛道定位、对标作品
阶段3: 核心梗   → 一句话高概念 Logline
阶段4: 角色设计 → 主角/配角/反派/关系网
阶段5: 大纲     → 全书结构、情节点、节奏曲线
阶段6: 章纲     → 逐章详细规划
阶段7: 文案包装 → 书名/简介/标签
阶段8: 正文写作 → 逐章催化剂选择 + 写作循环
阶段9: 复盘分析 → 全稿诊断报告
```

详见 `references/建制阶段.md`（阶段1-7）和 `references/写作工作流.md`（阶段8-9）。

## 工作流程

### 建制阶段 → `references/建制阶段.md`

阶段1-7：灵感策划 → 市场分析 → 核心梗 → 角色设计 → 大纲 → 章纲 → 文案包装

### 正文写作 → `references/写作工作流.md`

阶段8：逐章写作循环
1. 读 CLAUDE.md + hot.md
2. 本章催化剂选择（A/B/C + D 自定义）
3. 生成章节（对照 `references/写作质量手册.md`）
4. 用户确认
5. 更新角色/悬念/时间线/CLAUDE.md

### 三 Skill 协作

```
写 5-10 章 → jiyiyasuo 压缩记忆 → 读 hot.md 接着写
写满 30 章 → yizhixing 全库巡检 → 修复 → 继续
```

## 自检清单

- [ ] 每章有明确的中心事件
- [ ] 开章有冲突/悬念
- [ ] 对话/叙述有功能（塑造/推动/强化/交代/递进）
- [ ] 起承转合节奏正确
- [ ] 伏笔有埋有收
- [ ] 压力值动态驱动剧情
- [ ] 每章末有钩子
- [ ] 已更新 CLAUDE.md + 时间线
- [ ] 写满30章建议跑 yizhixing 巡检

## 质量守则
✅ 每章升级冲突 / 伏笔必收 / 配角独立动机
🚫 主线不明 / 节奏慢 / 铺垫多 / 配角多 / 逻辑不通
