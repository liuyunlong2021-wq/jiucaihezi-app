## 职责边界（Scope）

### ✅ 只做

1. **识别导演视角标志词** - 搜索并标记："看到"/"听到"/"觉得"/"认为"/"似乎"/"看似"/"整个人"
2. **删除导演视角表达** - 删除标志词，保留实际画面
3. **转换为演员视角** - 让读者直接"看到"画面，而非通过主角的眼睛"看到"

### ❌ 禁止

1. **禁止改变句子的实际内容** - 只删标志词，不改人物/事件
2. **禁止删除必要的感知动词** - "尝到""触到"等具体感知必须保留
3. **禁止修改对话和心理活动** - 不修改引号内的内容

---

## 核心算法（精简版）

### 必须删除的高频标志词

```
看到、听到、觉得、认为、感到、发现、注意到、
察觉到、看似、似乎、好像、仿佛、整个人
```

### 基本修正流程

```
Step 1: 分句（按标点符号）
Step 2: 逐句扫描，识别标志词
Step 3: 删除标志词，调整句子结构
Step 4: 合并结果，保持原有段落结构
```

### 核心示例（3个最常见）

```markdown
原文: 他看到门打开了。
修正: 门打开了。

原文: 他觉得很紧张。
修正: 他很紧张。

原文: 他整个人僵住了。
修正: 他僵住了。
```

## 输入格式（Input Schema）

```json
{
  "text": "[待修正的文本]",
  "preserve_ambiguity": false  // 可选，是否保留"似乎""好像"
}
```

---

## 输出格式（Output Schema）

```json
{
  "status": 200,
  "text_after_fix": "[修正后的文本]",
  "changes": [
    {
      "original": "他看到门打开了",
      "fixed": "门打开了",
      "reason": "删除导演视角标志词'看到'"
    }
  ],
  "stats": {
    "total_fixes": 12,
    "removed_words": ["看到", "听到", "觉得"]
  }
}
```

## 失败返回（Error Schema）

| 状态码 | 含义 |
|--------|------|
| 400 | 输入格式错误（text为空） |
| 422 | 输入逻辑错误（文本全是对话，无需修正） |
| 500 | 执行失败（正则匹配异常） |

### 执行后必须确认

- [ ] 所有"看到/听到/觉得"都已处理

- [ ] 未删除必要的感知动词（如"尝到""触到"）
- [ ] 未改变原文的剧情内容
