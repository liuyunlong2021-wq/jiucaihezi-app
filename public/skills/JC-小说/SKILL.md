# 小说世界模型

长篇小说创作引擎。九阶段管线：灵感→市场→核心梗→角色→大纲→章纲→文案→正文→复盘。**基于极致角色+催化剂驱动故事自然涌现，逐章推进。**

本 skill 自带 **100+ 写作引擎**（`references/engines/`），覆盖结构/节奏/角色/对话/冲突/润色/终检——写小说需要的全部工具都在这里。

配合 JC-yizhixing（巡检）和 JC-jiyiyasuo（压缩）形成完整内循环。

---

## 核心公式

```
世界观（规则） + 角色（极致性） + 催化剂（触发） = 故事自然涌现
```

## 核心原则

1. **催化剂驱动** — 每章必须有打破平衡的事件；每幕有大局催化剂
2. **蝴蝶效应** — 角色行动基于性格产生连锁反应
3. **多层冲突** — 每章 3-5 层，每幕有高潮
4. **压力值驱动** — 最高者为下一章主角（长篇需要压力曲线管理）
5. **长线记忆** — 50万字+ 必须依赖外置 Wiki + hot.md，否则人设崩、伏笔烂
6. **引擎辅助** — 写角色调 `references/engines/主角中心检查器`，写大纲调 `BS2宏观骨架生成器`，写冲突调 `冲突三位一体构建器`，终检调 `终检器`

## 九阶段管线

```
阶段1: 灵感策划 → engines/发散穷举器 + engines/爆款迁移引擎
阶段2: 市场分析 → engines/竞品爬虫分析器 + engines/读者画像侧写师
阶段3: 核心梗   → engines/核心梗公式化生成器 + engines/一句话梗概生成器
阶段4: 角色设计 → engines/原型角色分配器 + engines/人设保护器 + engines/20个出圈反差人设库
阶段5: 大纲     → engines/BS2宏观骨架生成器 + engines/三层矛盾叠加器 + engines/四线对齐诊断器
阶段6: 章纲     → engines/章纲自检过滤器 + engines/弹簧理论节奏控制器 + engines/期待感管理引擎
阶段7: 文案包装 → engines/三段式文案结构器 + engines/卖点数值化引擎
阶段8: 正文写作 → 逐章催化剂循环（见下方）

```

详见 `references/建制阶段.md`（阶段1-7）和 `references/写作工作流.md`（阶段8-9）。

## 正文写作方法

### 写作单位：逐章

每章 2000-4000 字，一章四阶段：构思→起草→润色→检查。详见 `references/写作工作流.md`。

```
构思阶段：场景核心逻辑构建器 + 弹簧节奏 + 期待感 + 钩子预埋
起草阶段：情绪先行 + 动作对白 + 冲突三位一体 + 视角修正 + 潜文本
润色阶段：人设润色 + 风格节奏 + 微观修辞 + 心理侧面化 + 信息饱和度
检查阶段：章纲过滤 + 冲突雷达 + 人设保护 + 主角中心 + 伏笔 + 连压预警
                                  ↓
                            全部通过 → 用户确认
```

### 分批交付
- 每 5-10 章一批，用户确认
- 每批结束建议调用 JC-jiyiyasuo 压缩记忆
- 每 30 章建议跑 JC-yizhixing 巡检
