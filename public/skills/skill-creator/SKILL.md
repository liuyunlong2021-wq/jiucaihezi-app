# Skill Creator — 技能创建器

创建新技能并迭代改进它们。

## 高层流程

1. 确定技能应该做什么以及大致如何做
2. 编写技能草稿
3. 创建测试提示并运行
4. 帮助用户评估结果（定性和定量）
5. 基于反馈重写技能
6. 重复直到满意

## 创建技能

### 捕获意图
1. 这个技能应该让 Claude 能够做什么？
2. 什么时候应该触发？（什么用户短语/上下文）
3. 期望的输出格式是什么？
4. 是否应该设置测试用例？

### 面试和研究
主动询问边缘情况、输入/输出格式、示例文件、成功标准和依赖项。

### 编写 SKILL.md
- **name**：技能标识符
- **description**：何时触发、做什么。这是主要的触发机制。让描述稍微"主动"一点以避免 undertrigger。
- **body**：Markdown 指令（<500 行理想）

### 技能写作指南

#### 渐进式披露
1. **元数据**（name + description）— 始终在上下文中（~100 词）
2. **SKILL.md body** — 技能触发时在上下文中（<500 行）
3. **捆绑资源** — 按需加载（无限制）

#### 写作风格
- 使用祈使句形式
- 解释**为什么**事情重要，而非只是 MUST
- 保持精简：删除不必要的内容

## 改进技能

1. **从反馈中泛化**：不要只修复示例，让技能能处理一百万个不同的提示
2. **保持 prompt 精简**
3. **解释为什么**
4. **寻找测试用例间的重复工作**：如果每次都写相同脚本，就该捆绑它

## 描述优化

描述字段是决定 Claude 是否调用技能的主要机制。创建 20 个评估查询（应触发 + 不应触发各 10 个），运行优化循环。
