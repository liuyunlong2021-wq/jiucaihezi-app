# 创作型 Skill 生成模板

> 当用户说"创建一个XX创作Skill"时，按此模板生成。

## 生成流程

### 第0轮：领域探路

```
你想创建什么类型的创作 Skill？
A. 小说
B. 论文/学术写作
C. 演讲稿
D. 短故事
E. 剧本（已有 jc-duanju-world 可参考）
F. 用户自定义
```

### 第1轮：建制设定收集

根据领域，确定需要收集的关键设定。**短剧的 5 轮是模板，其他领域据此调整**：

| 领域 | 建制轮次建议 |
|------|------------|
| 小说 | 世界观→主角→配角→主线→章纲→关键设定 |
| 论文 | 领域→论点→论据框架→参考文献→方法论→格式要求 |
| 演讲稿 | 场合→受众→核心信息→情感曲线→时长→风格 |
| 短故事 | 世界观→主角→困境→转折→结局走向→篇幅 |

每轮格式：A/B/C 具体选项 + 用户自定义。

### 第2轮：Wiki 目录确定

根据领域确定信息类型，生成 Wiki 目录。模板：

```
wiki/
├── 作品/           ← 创作正文
├── 角色/           ← 主体档案（小说）/ 论点档案（论文）/ 素材库（演讲）
├── 世界/           ← 设定/规则/关系（小说）/ 文献库（论文）
├── 剧情/           ← 悬念/时间线/大纲（小说）/ 逻辑链（论文）/ 情感曲线（演讲）
├── hot.md          ← 记忆小抄
├── index.md        ← 全库目录
└── 交付手册.md     ← 后置产出
```

### 第3轮：质量约束确定

根据领域确定写作质量规则要包含哪些章节。短剧的 12 章是模板：

| 短剧（已完成） | 小说 | 论文 | 演讲 |
|--------------|------|------|------|
| 中心事件 | 章节目标 | 段落论点 | 段落信息点 |
| 节奏结构 | 章节节奏 | 论证节奏 | 情感节奏 |
| 台词铁律 | 对话规则 | 引用规范 | 修辞规则 |
| 情绪法则 | 情感张力 | 逻辑强度 | 情绪曲线 |
| 钩子法则 | 章节钩子 | 论证钩子 | 记忆点 |

### 第4轮：命名和触发词

```
Skill 名称建议：JC-[领域拼音缩写]
触发词建议：包含领域核心词 + 常见变体

示例：
  小说 → JC-xiaoshuo | 写小说/开新书/续写/章节
  论文 → JC-lunwen   | 写论文/学术写作/论文大纲/引用
  演讲 → JC-yanjiang | 写演讲稿/TED/路演/致辞
```

## 生成清单

生成以下文件：

- [ ] `SKILL.md`（统帅，<200行）：定位/架构/原则/文件约定/流程概览/交互铁律/时机表/自检/质量守则/快参/指令/ref索引
- [ ] `references/建制阶段.md`：逐轮交互收集模板（最重要）
- [ ] `references/创作工作流.md`：核心创作循环
- [ ] `references/核心机制.md`：领域特有的规则/公式
- [ ] `references/写作质量手册.md`：质量控制规则
- [ ] `references/素材库.md`：参考素材（可选，小说=爽点库/论文=引用格式/演讲=修辞手法）

## 三 Skill 协作（必须集成）

无论什么领域，都推荐配合 jc-jian-wiki 和 jc-raw-wiki：

```
创作 Skill 写几个单元
       ↓
jc-raw-wiki 整理记忆（沉淀进 wiki/，刷新 hot.md）
       ↓
写满一定数量 → jc-jian-wiki 巡检
       ↓
创作 Skill 修复 → 继续
```

在 SKILL.md 中注明衔接时机即可，不需要在创作 Skill 里实现巡检和压缩的功能。
