# 漫剧项目

用于长篇漫剧、系列漫剧和连续分集作品。按“集纲 → 集 → 正文”组织。所有角色、场景、道具和伏笔都是平等实体，不预先判断主次；信息越完整、引用越多、出场越频繁，戏份自然越清楚。

这是可承载长篇漫剧的目录标准：先稳定故事事实和导航，再让内容随创作自然增长。目录可预建；具体角色、场景、道具、伏笔、集纲和集必须有真实材料才创建。

## 分类标准

| 位置 | 放什么 | 不放什么 |
|---|---|---|
| `作品/正文/{集名}/` | 一集的正式正文 | 大纲、素材、人物设定 |
| `作品/大纲/集纲/{集纲名}/` | 一份集纲的事件、角色、冲突、钩子和伏笔计划 | 多集共用的大阶段计划 |
| `角色/{角色名}/` | 六类角色资料；任何说话或影响剧情的角色都建立目录 | 用主角、配角、反派等人为分类 |
| `场景/{场景名}/` | 空间简介、外观、内部结构、尺寸、布局、空间关系、细节和状态 | 只有一句且永不复用的背景描述 |
| `道具/{道具名}/` | 尺寸、材质、外观、功能、故事作用、归属和流转 | 无法识别的泛称物品 |
| `伏笔/{伏笔名}/` | 一个伏笔的一生：埋设、承诺、当前状态、回收 | 只凭猜测写出的未来剧情 |
| `.raw/参考资料/` | 对标作品、外部资料、图片、网页摘录等原始素材 | 已确认的漫剧事实和正文 |
| `.raw/创作笔记/` | 灵感、草稿、未确认判断和过程记录 | 已确认的漫剧事实和定稿结论 |

## 建库与增长规则

- 建库时创建导航目录、`_index.md`、`.raw/参考资料/` 和 `.raw/创作笔记/`；不要预建任何以实际名称命名的角色、场景、道具、伏笔、集或集纲目录。
- 所有角色都直接创建在 `角色/{角色名}/`。角色资料少，代表当前可知信息少或戏份少；无需人为降级分类。后续用户或模型可依据目录内容、引用和出场记录审查其重要性。
- 每个场景、道具、伏笔各自一个目录；目录名必须是可检索的实体名。需要补资料时，只补该实体缺少的分类，不新建重复实体。
- 每份集纲和每集正文，必须双链到实际出现的角色、场景、道具和伏笔；相关实体页面也回链到出现它们的集纲和正文。
- `剧情/时间线.md` 是全局事件顺序；人物经历、势力历史、道具流转和场景状态分别链接回实体目录，不重复维护。
- 原始参考和过程笔记只放 Raw。它们经过判断后，已确认的世界规则写入 `世界/`，人物结论写入 `角色/`，剧情结论写入 `剧情/`，定稿文案写入 `作品/`；未确认内容始终留在 Raw。
- 外部原始文件不复制、不移动；`.raw/参考资料/` 与 `.raw/创作笔记/` 只存项目内产生或用户明确放入的原始素材。每个正式条目在 `来源索引.md` 记录它对应的 Raw 会话、原始文件或正文位置。
- `hot.md` 只保留续写前必须知道的当前进度、角色状态、活跃冲突、未回收伏笔和下一步；`log.md` 只追加已发生的事实；`index.md` 是全库导航。
- 语义内容、双链、关系图、统计表和热缓存更新由 `jc-raw-wiki` 根据实际材料完成。

## Raw 填充标准

`jc-raw-wiki` 填充漫剧项目时，以上骨架是唯一落位标准。

### 绝对边界

- Raw 是证据和完整过程，只读；不复制、不移动、不删除。
- 不创建 `对话记录/`、会话摘要页或聊天转录页，也不把完整会话、整段聊天原文或参考原文写进 Wiki。
- `作品/正文/` 只保存用户明确交付或明确确认的一集正式正文。它是作品正式内容，不是对话副本；不要从一段包含讨论的会话中整段复制进去。
- 讨论、提议、未确认设定只留在 Raw，不能伪装成漫剧事实。

### 输入与增量

读取本轮指定内容或尚未处理的 Raw 范围，以及现有 Wiki、`来源索引.md`、`hot.md`、`log.md`。先从来源索引确认该会话或文件的已处理范围，只处理新增范围。

### 落位标准

| 确认内容 | 写入位置 | 必须关联 |
|---|---|---|
| 一集的事件、人物、冲突、钩子、伏笔计划 | `作品/大纲/集纲/{集纲名}/集纲.md` | 本集角色、场景、道具、伏笔 |
| 明确交付或确认的一集正式正文 | `作品/正文/{集名}/正文.md` | 本集实际出现的实体与集纲 |
| 角色信息、关系、成长或状态变化 | `角色/{角色名}/` 的对应六类资料页，必要时更新 `关系网.md`、`状态追踪.md` | 相关集和角色 |
| 世界规则、势力、地点、历史变化 | `世界/设定/`、`世界/规则/`、`世界/势力/`、`世界/地点/`、`世界/关系/` | 受影响角色、剧情、场景 |
| 主线、支线、冲突、主题或事件顺序变化 | `剧情/主线/`、`剧情/支线/`、对应总页、`剧情/时间线.md` | 相关集、实体、伏笔 |
| 可复用场景的空间事实或状态变化 | `场景/{场景名}/` 的对应七类资料页 | 出现它的集、角色、道具 |
| 可复用道具的尺寸、材质、功能或流转变化 | `道具/{道具名}/` 的对应五类资料页 | 出现它的集、角色、场景 |
| 伏笔埋设、状态变化或回收 | `伏笔/{伏笔名}/埋设与回收.md` 与 `伏笔/伏笔管理.md` | 埋设和回收所在集 |

没有合适实体时，按本文件的目录在正确父目录创建最小新页面；不创建平行分类或重复实体。

### 写入顺序

1. 判断新增范围中的内容是正式正文、确认事实还是未确认讨论。
2. 将正式正文或确认事实写入唯一的语义位置；同一事实只维护一份。
3. 按表中“必须关联”创建双链和回链，链接目标必须存在或本轮同步创建。
4. 在 `来源索引.md` 登记来源路径和已处理范围；只写定位信息，不摘抄 Raw。
5. 仅在当前状态改变时更新 `hot.md`；向 `log.md` 追加本轮已确认事实。
6. 自检：Wiki 中没有完整会话、没有原文副本、没有断链、没有同一事实的平行页面。
