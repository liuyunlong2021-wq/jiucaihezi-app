# 电视剧项目

用于连续电视剧、网剧和多季剧集。按“集纲 → 集 → 正文”组织。所有角色、场景、道具和伏笔都是平等实体，不预先判断主次；信息越完整、引用越多、出场越频繁，戏份自然越清楚。

这是可承载长篇连续剧的目录标准：先稳定故事事实和导航，再让分集正文与实体资料自然增长。目录可预建；具体角色、场景、道具、伏笔、集纲和集必须有真实材料才创建。

## 分类标准

| 位置 | 放什么 | 不放什么 |
|---|---|---|
| `作品/正文/{集名}/` | 一集的正式正文 | 大纲、素材、人物设定 |
| `作品/大纲/集纲/{集纲名}/` | 一份集纲的事件、角色、冲突、钩子和伏笔计划 | 多集共用的大阶段计划 |
| `角色/{角色名}/` | 六类角色资料；任何说话或影响剧情的角色都建立目录 | 用主角、配角、反派等人为分类 |
| `场景/{场景名}/` | 空间简介、外观、内部结构、尺寸、布局、空间关系、细节和状态 | 只有一句且永不复用的背景描述 |
| `道具/{道具名}/` | 尺寸、材质、外观、功能、故事作用、归属和流转 | 无法识别的泛称物品 |
| `伏笔/{伏笔名}/` | 一个伏笔的一生：埋设、承诺、当前状态、回收 | 只凭猜测写出的未来剧情 |
| `.raw/参考资料/`、`.raw/创作笔记/` | 原始资料、灵感、草稿和过程记录 | 已确认的电视剧事实和定稿结论 |

## 建库与增长规则

- 建库时创建导航目录、`_index.md` 和 Raw 入口；不要预建以实际名称命名的角色、场景、道具、伏笔、集或集纲目录。
- 所有角色都直接创建在 `角色/{角色名}/`。资料少表示当前可知信息少或戏份少；无需人为降级分类。
- 每个场景、道具、伏笔各自一个目录；目录名必须是可检索的实体名。需要补资料时，只补该实体缺少的分类，不新建重复实体。
- 每份集纲和每集正文，必须双链到实际出现的角色、场景、道具和伏笔；相关实体页面也回链到出现它们的集纲和正文。
- `剧情/时间线.md` 是全局事件顺序；人物经历、势力历史、道具流转和场景状态分别链接回实体目录，不重复维护。
- 原始参考和过程笔记只放 Raw。确认后的世界规则、人物结论、剧情结论和定稿文案分别写入 `世界/`、`角色/`、`剧情/`、`作品/`；每个正式条目在 `来源索引.md` 记录来源。

## Raw 填充标准

`jc-raw-wiki` 填充电视剧项目时，以上骨架是唯一落位标准。

### 绝对边界

- Raw 只读，不复制、不移动、不删除；不创建 `对话记录/`、会话摘要页或聊天转录页。
- 不把完整会话、整段聊天原文或参考原文写进 Wiki。`作品/正文/` 只保存用户明确交付或确认的一集正式正文，不从含讨论的会话中整段复制。
- 未确认的讨论、提议和设定只留在 Raw，不能伪装成电视剧事实。

### 输入与增量

读取本轮指定内容或尚未处理的 Raw 范围，以及现有 Wiki、`来源索引.md`、`hot.md`、`log.md`。先从来源索引确认该会话或文件的已处理范围，只处理新增范围。

### 落位标准

| 确认内容 | 写入位置 | 必须关联 |
|---|---|---|
| 一集的事件、人物、冲突、钩子、伏笔计划 | `作品/大纲/集纲/{集纲名}/集纲.md` | 本集角色、场景、道具、伏笔 |
| 明确交付或确认的一集正式正文 | `作品/正文/{集名}/正文.md` | 本集实际出现的实体与集纲 |
| 角色、世界、剧情变化 | 对应 `角色/`、`世界/`、`剧情/` 的唯一页面 | 相关集和实体 |
| 场景、道具、伏笔的事实或状态变化 | `场景/{场景名}/`、`道具/{道具名}/`、`伏笔/{伏笔名}/埋设与回收.md` | 出现它的集和实体 |

没有合适实体时，只在正确父目录创建最小新页面；不创建平行分类或重复实体。

### 写入顺序

1. 判断新增范围是正式正文、确认事实还是未确认讨论。
2. 将正式正文或确认事实写入唯一语义位置；同一事实只维护一份。
3. 创建表中要求的双链和回链，链接目标必须存在或本轮同步创建。
4. 在 `来源索引.md` 登记来源路径和已处理范围，只写定位信息，不摘抄 Raw。
5. 当前状态变化时更新 `hot.md`，向 `log.md` 追加已确认事实。
6. 自检：没有完整会话、原文副本、断链或重复实体。
