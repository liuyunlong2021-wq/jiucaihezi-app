# AI-first 微信小程序开发完整指南

> 你不会编程？不懂架构？不会用 GitHub？没关系。这篇指南教你用"对 AI 说话"的方式，从零做出一个能跑的小程序。

---

## 核心原则：AI-first 大白话

### 你不是在学编程，你是在指挥一个程序员

把你的角色想清楚：

- **你是产品经理**：你决定做什么、长什么样、好不好用
- **AI 是你的工程师**：你把需求说清楚，它给你写出代码
- **微信开发者工具是你的眼睛**：你在里面看效果，看到问题就告诉 AI

你需要具备的**唯一三个能力**：

1. **描述需求**：用大白话说清楚你想做什么
2. **检查结果**：打开开发者工具看看 AI 做的对不对
3. **给出反馈**："这里不对，改一下"——就这么简单

---

## 阶段 0：需求梳理 — 你究竟想做什么？

### 这步是干嘛的

很多人一上来就让 AI 写代码，结果发现做出来的跟想的不一样——因为 AI 根本没理解你的需求。这一步就是让你和 AI 先"对齐"，确保 AI 完全理解你想做什么，再开始写代码。

### 你需要准备什么

只是你的想法。用大白话说出来就行。

### 对 AI 这么说

```
我想做一个微信小程序，功能是 [用大白话描述，越具体越好]。

请帮我做以下梳理：
1. 这个小程序的核心功能是什么（一句话）
2. 需要哪些页面（列出所有页面名和页码路径）
3. 每个页面展示什么内容、有什么交互（用户点了哪里会怎样）
4. 适合用什么风格的 UI（简洁/活泼/专业）
5. 推荐用什么组件库（vant-weapp / WeUI / TDesign / iView Weapp / 纯原生），并说明理由

先不要写代码，我们先把方向确认清楚。
```

### 示例

**用户说**：

> 我想做一个小区二手交易的小程序，邻居可以在上面发布闲置物品，有人看到想要的可以联系发布者。

**AI 应该输出类似这样的分析**：

- 核心功能：小区邻里二手物品发布与浏览平台
- 需要页面：首页（商品列表）、发布页、详情页、我的页面、搜索页
- 交互：首页卡片列表 → 点击进详情 → 详情页联系发布者或收藏；发布页填表单；我的页面看发布历史和收藏
- 风格：简洁温馨，邻里社区感
- 推荐组件库：vant-weapp（卡片、搜索、表单、TabBar 都很完善，适合这种商品展示型应用）

### 检查清单

- [ ] AI 是否完整理解了你想要的功能？
- [ ] 页面列表有没有遗漏？
- [ ] 推荐的组件库你有没有异议？

**确认无误后，告诉 AI"没问题，我们开始做"**，然后进入阶段 1。

## 阶段 1：环境准备 — 把工具装好

### 1.1 注册微信小程序账号

1. 打开 [微信公众平台](https://mp.weixin.qq.com/)
2. 点击右上角"立即注册"→ 选择"小程序"
3. 按提示填写信息（用你的微信扫码即可）

**重要提醒**：

- **个人主体**限制较多：不能做电商（含支付功能）、外卖、社交、资讯、直播等。如果你要做这些，需要注册企业主体。
- **一个邮箱只能注册一个小程序**，一个身份证只能注册 5 个小程序（个人）。
- 注册免费，审核快（通常几分钟到半天）。

### 1.2 获取 AppID

注册通过后：

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 左侧菜单：开发 → 开发管理 → 开发设置
3. 找到 **AppID（小程序 ID）**，记下来

> **对 AI 说**："我已经注册好微信小程序，AppID 是 wx[xxxxxxxx]。下一步怎么做？"

### 1.3 下载安装微信开发者工具

1. 去 [微信开发者工具下载页](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 下载对应你电脑系统（Windows / Mac / Linux）的稳定版
3. 安装后打开，用微信扫码登录

### 1.4 新建项目

> **对 AI 说**："请告诉我在微信开发者工具里如何创建新项目，选择什么模板、填什么参数。我的 AppID 是 wx[xxx]。注意事项有哪些？"

AI 会告诉你：

- 点击"+"或"新建项目"
- 项目名称随意填
- 目录选一个空文件夹
- AppID 填你的
- 开发模式选"小程序"
- **模板选"不使用模板"**（用 AI 从零搭建）或选"JavaScript 基础模板"

### 1.5 如果后面要下载组件库代码到本地

> **对 AI 说**："帮我把 [组件库] 的 GitHub 仓库 clone 到我本地电脑上"（参考本 Skill 的 [场景 2：下载代码]）

## 阶段 2：选型 — 选什么组件库？

### 大白话：组件库是什么？

你玩过乐高吗？组件库就像一盒乐高积木——别人已经把砖块（按钮、列表、弹窗、导航栏……）做好了，你只需要拼起来就行。不需要自己从零捏一块积木。

### 五选一详细对比

| 对比维度 | vant-weapp         | WeUI              | TDesign           | iView Weapp       | 纯原生       |
| -------- | ------------------ | ----------------- | ----------------- | ----------------- | ------------ |
| 组件数量 | 70+                | 30+               | 50+               | 40+               | 0            |
| 适合类型 | 电商/生活服务/通用 | 极简工具/官方风格 | 企业应用/专业工具 | 后台管理/数据展示 | 极简 1-2 页  |
| 文档质量 | ★★★★★              | ★★★★              | ★★★★★             | ★★★               | —            |
| 社区活跃 | 非常活跃           | 官方维护          | 活跃              | 一般              | —            |
| 学习成本 | 低                 | 极低              | 低                | 中                | 高（全手写） |
| 风格     | 清新现代           | 微信原生          | 大气专业          | 企业风格          | 完全自定义   |
| 安装方式 | npm                | npm / 下载        | npm               | npm               | —            |
| Star 数  | 17k+               | 5k+               | 1.5k+             | 5k+               | —            |

### 决策树：你适合哪个？

```
你的小程序类型是？
├── 电商/商城/生活服务 → vant-weapp（组件最丰富）
├── 极简工具/希望和微信风格一致 → WeUI
├── 企业应用/专业工具/注重品牌调性 → TDesign
├── 后台管理/数据面板/表格多 → iView Weapp
└── 就一两页，也没什么复杂交互 → 纯原生
```

### 对 AI 这么说（以选定 vant-weapp 为例）

**第一步：让 AI 帮你选**

```
我想做一个小程序，类型是 [电商/工具/展示/管理]，大概 [X] 个页面，风格偏好 [简洁/专业/活泼]。
根据这些条件，帮我从 vant-weapp / WeUI / TDesign / iView Weapp / 纯原生 中推荐最合适的组件库，并说明推荐理由。
```

**第二步：让 AI 帮你装**

```
我要用 [选定的组件库] 开发微信小程序。请帮我：
1. 用 npm 方式安装
2. 修改 app.json 和 project.config.json 配置文件
3. 确保所有组件可以在项目中正常引用

先只做配置，不写页面代码。
```

**安装完后必须做**：在微信开发者工具里，点击顶部菜单 **"工具" → "构建 npm"**，等构建完成。如果没有这步，组件不会生效！

## 阶段 3：搭建项目骨架

### 大白话：什么是项目骨架？

就像盖房子先搭框架——先把所有房间（页面）的位置定好、门（导航）的位置定好。但房间里面还没装修。在这个阶段，你能在开发者工具里看到一个有导航栏的空壳。

### 对 AI 这么说

```
基于阶段 0 整理的功能清单，帮我搭建小程序的项目骨架：

1. 配置 app.json：
   - 列出所有页面的路径（pages 数组）
   - 配置底部 TabBar（4 个以内图标导航）+ 顶部标题栏
   - 如果用组件库的导航组件，请说明如何在 app.json 里配置

2. 为每个页面创建四个文件：
   - [页面名].wxml（页面结构——先放一个占位文字）
   - [页面名].wxss（页面样式——空的也行）
   - [页面名].js（页面逻辑——基础 Page({}) 即可）
   - [页面名].json（页面配置——空的 {}）

3. 写好 app.wxss 全局样式：
   - 基础字体、颜色变量
   - 页面背景色
   - 常用间距

只做骨架，每个页面放一行占位文字就行，不要写具体内容。
```

### 做完后确认

在开发者工具里点击"编译"，检查：

- [ ] 底部导航栏能显示吗？
- [ ] 点击导航栏每个图标，页面能切换吗？
- [ ] 每个页面标题栏文字正确吗？

如果有任何不对：

> "TabBar 没有显示 / 第三个 Tab 点不了 / 页面标题不对，帮我修一下。"

## 阶段 4：逐页开发 — 核心循环

### 核心原则：一次只做一页！

很多人想让 AI 一次性生成所有页面——结果容易出 bug，而且出了问题你不知道是哪页引起的。

**逐页标准流程（每页重复一遍）：**

```
第 1 步：告诉 AI 本页需求  →  AI 生成代码
第 2 步：在开发者工具预览   →  看看效果
第 3 步：不满意？描述问题   →  AI 修改
第 4 步：满意了？告诉 AI    →  做下一页
```

### 每页的 Prompt 模板

```
现在开发 [页面名] 页面（路径 pages/[路径]/[文件名]）。

页面上需要展示：
- [从上到下列出每一块内容]

交互要求：
- 用户点击 [某处] 后 → [发生什么：跳转/弹窗/切换/展开]
- 用户下拉 → [刷新/不做操作]
- [其他交互]

请用 [组件库] 来实现。参考组件：
- 列表用 [van-cell / t-cell / ...]
- 按钮用 [van-button / t-button / ...]
- [其他组件对应]

请生成这个页面的四个文件：
- [页面名].wxml — 页面结构
- [页面名].wxss — 页面样式
- [页面名].js   — 页面逻辑
- [页面名].json — 页面配置

不要生成其他页面的代码。
```

### 示例：做一个商品列表首页

```
现在开发首页（路径 pages/index/index）。

页面上需要展示：
- 顶部搜索框（点击跳转搜索页）
- 商品分类横向滚动标签（全部/家电/数码/服饰/图书）
- 商品列表（两列网格布局），每个卡片显示：
  - 商品图片（先用灰色占位方块代替）
  - 商品名称
  - 价格（红色大字）
  - 发布者头像+昵称

交互要求：
- 点击搜索框 → 跳转搜索页
- 点击分类标签 → 切换当前分类（先做 UI 切换效果，暂时不联数据）
- 点击商品卡片 → 跳转详情页
- 下拉 → 触发刷新（先做动画效果）

请用 vant-weapp 来实现。参考组件：
- 搜索框用 van-search
- 标签用 van-tabs
- 商品卡片用 van-card 或自定义 grid 布局
- 价格用 van-price（如果有的话）

请生成首页的四个文件：index.wxml / index.wxss / index.js / index.json。
不要生成其他页面的代码。
```

### 不满意时怎么改

```
# 样式类问题
"首页的商品卡片间距太大了，帮我缩小一点。"
"价格的颜色不够醒目，改成红色 #FF0000，字号加大。"

# 功能类问题
"点击商品卡片没有反应，帮我检查一下跳转逻辑。"
"分类标签切换后列表没有变化，帮我加上切换效果。"

# 布局类问题
"手机上看列表只显示了一列，但我想要两列。帮我改成双列网格。"
"搜索框太靠上了，往下移一点。"

# 数据类问题
"列表是空的，没有示例数据。请帮我在 js 里加一些模拟的商品数据。"
```

## 阶段 5：打磨细节

### 5.1 页面跳转和传参

**大白话**：页面 A 跳转到页面 B，同时把一些信息（比如商品 ID）传给 B，让 B 知道该展示什么。

> **对 AI 说**："帮我把首页和详情页之间的跳转加上，点击首页的某个商品时把商品 ID 传给详情页，详情页根据 ID 显示对应商品的信息。"

### 5.2 表单处理

如果你的小程序有表单（发布页、登录页等）：

> **对 AI 说**："帮我实现发布页的表单：有文字输入框、图片上传、分类选择器、提交按钮。用户填写后点击提交，把数据保存到本地缓存（storage），然后跳转回首页并显示提交成功提示。"

### 5.3 常见视觉问题速查

| 我看到的       | 我想改成 | 对 AI 说                                         |
| -------------- | -------- | ------------------------------------------------ |
| 文字太小看不清 | 大一点   | "把 [位置] 的文字放大到 [XX] 号字"               |
| 按钮太小不好点 | 大一点   | "把按钮加到至少 44px 高，小程序最小点击区域要求" |
| 颜色太素/太花  | 换个配色 | "帮我换一套暖色调/冷色调配色方案"                |
| 间距太挤       | 松一点   | "各个模块之间加一些间距"                         |
| 图片变形了     | 正常比例 | "图片用 mode='aspectFill' 保持比例裁剪"          |
| 页面对不齐     | 对齐     | "这个 [元素] 和左边的 [元素] 没有对齐，帮我调整" |
| 滚动不流畅     | 顺畅     | "长列表加一下优化，用虚拟列表或分页加载"         |

## 阶段 6：上线发布

### 6.1 提交前检查

> **对 AI 说**："我的小程序要提交审核了，帮我做全面检查：
>
> 1. 所有页面路径在 app.json 里是否正确
> 2. 有没有硬编码的测试数据（比如'测试用户'、'123456'）
> 3. 有没有用到微信不允许的 API
> 4. 页面的标题、图标是否完整
> 5. 有没有明显的崩溃风险
>    列出所有需要修改的问题。"

### 6.2 上传代码

1. 在微信开发者工具右上角，点击"上传"
2. 版本号填如 `1.0.0`，项目备注写"第一版发布"
3. 点击上传

### 6.3 提交审核

1. 打开 [微信公众平台](https://mp.weixin.qq.com/) → 管理 → 版本管理
2. 在"开发版本"区域找到你刚上传的版本
3. 点击"提交审核"
4. 填写：
   - 配置功能页面（至少要勾上一个功能页面）
   - 测试账号（如果需要登录的话）
   - 备注说明
5. 提交，等待审核（通常 1-7 个工作日）

### 6.4 审核通过后

审核通过后，在版本管理页面点击"发布"，小程序就正式上线了！

### 6.5 审核被驳回怎么办

驳回很正常，通常是些小问题。微信会给出驳回原因。

> **对 AI 说**："我的小程序审核被驳回了。驳回原因是 [贴驳回原因原文]。请帮我分析问题并修改代码。修好后告诉我具体改了哪些文件。"

常见驳回原因及对应操作：

- "功能不完整"→ 检查是不是有空页面，补充内容
- "类目不合适"→ 换一个服务类目
- "含测试数据"→ 让 AI 把测试数据清除
- "需加用户协议"→ 让 AI 加一个用户协议页面

### 6.6 后续更新

以后要更新小程序，流程一样：

1. 告诉 AI 你要改什么 → AI 修改代码
2. 在开发者工具里确认效果
3. 上传新版本（版本号递增，如 1.0.1）
4. 提交审核 → 通过 → 发布

> **对 AI 说**："我要更新小程序，修改 [具体要改的内容]。改完后帮我把版本号从 X.X.X 升级到 X.X.X。"

## 附录 B：微信小程序常见字段大白话解释

| 术语         | 大白话                                                   |
| ------------ | -------------------------------------------------------- |
| **wxml**     | 页面的"骨架"——决定页面长什么样、有什么元素（类似 HTML）  |
| **wxss**     | 页面的"皮肤"——决定颜色、大小、间距、字体（类似 CSS）     |
| **js**       | 页面的"大脑"——决定数据怎么处理、用户点了之后发生什么     |
| **json**     | 页面的"配置单"——这个页面能不能下拉刷新、用什么标题       |
| **app.json** | 整个小程序的"总配置"——有哪些页面、导航栏什么样、窗口标题 |
| **app.wxss** | 全局样式——所有页面共享的颜色、字体                       |
| **app.js**   | 小程序入口——小程序启动时执行，可以放全局数据             |
| **组件**     | 一个可复用的"部件"——比如按钮、列表、弹窗，写好一次到处用 |
| **TabBar**   | 底部导航栏——通常 2-4 个图标，点了切换页面                |
| **Storage**  | 手机上的本地存储——就像浏览器 cookie，存了关了还有        |
| **npm**      | Node.js 的"应用商店"——下载别人写好的代码包               |

## 附录 C：小白常见坑

### 1. AppID 用测试号

测试号只能在开发者工具里看，真机扫不了、发布了不。**一定要用正式注册的 AppID**。

### 2. 个人主体类目限制

个人主体不能选：电商平台、外卖、社交、直播、资讯、金融等类目。注册前先确认你要做的类型个人能不能做。

### 3. npm 没构建

装了 npm 包后，一定要点 **"工具 → 构建 npm"**。否则 `usingComponents` 引用的组件路径找不到白屏。

### 4. 真机预览白屏

先看开发者工具的"控制台"有没有报错。常见原因：没配置合法域名（开发阶段可以勾选"不校验合法域名"）。

### 5. 不要把敏感信息写进代码

AppSecret、API Key 等敏感信息**永远不要直接写在代码里**上传。用云函数或后端来处理。

### 6. 审核不通过怕什么

审核不通过很正常，改完重新提交就行。不影响之后审核。

### 7. 图片不显示

检查图片链接是不是 https（小程序要求 https）。开发阶段可以用 http，真机不行。

### 8. 样式在电脑上好看手机上不对

开发者工具的模拟器和真机有差异。**以真机为准**。不确定时就扫预览码在手机上确认。

### 9. 每个页面必须有 .wxml 文件

如果 app.json 里注册了某个页面路径，但那个路径下面没有对应的 .wxml 文件，编译会报错。

### 10. TabBar 的 iconPath 图片路径

TabBar 图标必须是本地图片（放项目文件夹里），不能是网络图片。图标尺寸 81px × 81px。

## 附录 D：完整 Prompt 速查卡

以下是各阶段最常用的 prompt，直接复制用：

```
# 阶段 0：梳理需求
"我想做一个微信小程序，功能是 [描述]。请帮我梳理功能清单和页面列表，先不写代码。"

# 阶段 1：环境确认
"我的 AppID 是 wx[xxx]。请告诉我在微信开发者工具里如何创建新项目。"

# 阶段 2：组件库推荐
"我想做 [类型] 小程序，[X] 个页面，风格 [偏好]。推荐什么组件库？"

# 阶段 2：组件库安装
"我要用 [组件库]，请帮我用 npm 安装并完成所有配置。"

# 阶段 3：搭建骨架
"基于功能清单，帮我搭建项目骨架：app.json + 全部页面空文件 + 全局样式。"

# 阶段 4：逐页开发
"现在开发 [页面名]。需要展示 [内容]。用户点击后 [交互]。用 [组件库] 实现。"

# 阶段 4：修改样式
"[页面名] 的 [元素] 需要调整：[具体问题]。帮我修改。"

# 阶段 5：加功能
"帮我给 [页面名] 加上 [功能描述]。"

# 阶段 6：提交前检查
"我的小程序要提交审核了，帮我全面检查有没有问题。"

# 阶段 6：审核驳回
"小程序审核被驳回了，原因是 [原文]。帮我修改。"

# 阶段 6：更新版本
"我要更新小程序，改 [内容]。版本号升级到 [新版本号]。"
```
