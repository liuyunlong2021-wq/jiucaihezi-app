# Coverage Patterns

Use this file only when the scene clearly matches one of these patterns.

The purpose of these patterns is not to force a fixed number of shots.

The purpose is to help the engineering book decide what reusable material units normally exist in an editable scene.

## 1. Two-person confrontation dialogue

Typical material order:

1. `geography_wide` or `two_shot`
2. `over_shoulder_A_to_B`
3. `over_shoulder_B_to_A`
4. `single_medium_A` or `single_close_A`
5. `single_medium_B` or `single_close_B`
6. `reaction_pickup` at emotional escalation points

Use this pattern when:

- two characters are in direct exchange
- axis clarity matters
- the editor may need to re-time the exchange

Do not force every role to appear.

Choose only the units that create real cut flexibility.

## 2. Intrusion / discovery / shock

Typical material order:

1. `trigger_wide`
2. `reaction_pickup`
3. `reveal_subject`
4. optional `master_reset`
5. response dialogue unit

Do not collapse reaction and reveal into one coarse unit if both are dramatically important.

## 3. Evidence prop beat

Typical material order:

1. hand reaches or draws prop
2. prop move or throw
3. `insert_prop`
4. pickup or touch
5. reaction

Use separate units when:

- the prop itself carries decisive information
- the editor needs a clean insert
- the hand action and the prop result are both important

## 4. Action exchange

Typical material order:

1. `action_prep`
2. `action_release`
3. `action_travel`
4. `action_impact`
5. victim reaction
6. optional geography reset or aftermath

Do not merge preparation and impact by default.

If the weapon or attack path matters, keep travel as its own unit.

## 5. Entry / exit / seal beat

Typical material order:

1. turn or commit to leave
2. movement toward exit
3. response or pursuit
4. `exit_seal`, such as the door closing
5. aftermath hold

This pattern is useful when the scene needs a strong closing punctuation.

## 6. Long dialogue line split

Split one spoken line into several dialogue units when:

- the line contains several emotional steps
- the editor would naturally cut during the line
- the line needs alternating reaction inserts

Keep only the exact dialogue fragment in each unit.

Do not repeat the entire line in every fragment.

## Support coverage rule

Support coverage is allowed only when it solves one of these problems:

- axis is unclear
- geography is unclear
- emotional escalation has no clean cut point
- prop information has no clean insert
- action continuity would be muddy

Support coverage is not allowed just to "have more options".
