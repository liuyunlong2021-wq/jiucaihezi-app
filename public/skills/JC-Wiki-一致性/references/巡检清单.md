# 巡检清单（检查项定义）

> 本文件定义巡检官的所有检查项、判定标准、以及 `scan_vault.py` 的退出码。巡检时按库类型选用对应清单。

## 文件约定（与写作 skill 共享的事实源）

巡检官不内置任何写作 skill 的知识，只认这套**目录约定**。库里有哪些文件，就启用哪些检查：

| 文件/目录 | 谁维护 | 巡检官怎么用 |
|-----------|--------|--------------|
| `wiki/映射表.md` | [[JC-linmoxiaoshuo]] | 当作换皮对错的事实源 |
| `wiki/伏笔账本.md` / `悬念账本.md` | 写作 skill | 查"埋了没收" |
| `wiki/改写稿/` | linmo | 扫漏改/撞车/漂移 |
| `wiki/角色/` 或 `角色档案/` | 写作 skill | 查角色设定矛盾 |
| `.raw/` | JC-jiyiyasuo 整理 | 巡检一般不扫（是待整理素材，不是定稿） |
| `hot.md` / `CLAUDE.md` | JC-jiyiyasuo | 仅检查是否过期，不当事实源 |

## A. 换皮专项检查（库里有 `映射表.md` 时启用）

这组对应临摹/换皮库，是 [[JC-linmoxiaoshuo]] 单章自查的全库放大版。

### A1. 漏改（最高优先级 ❌）
范本旧专名残留在改写稿里。映射表里每个"范本名"都不该出现在任何改写稿正文中。
- 判定：改写稿正文中 `text.count(旧名) > 0` → 漏改。
- 为什么最严重：读者看到原作人名/地名，一眼识破换皮，作品废掉。

### A2. 撞车（❌）
多个不同范本元素映射到了同一个新名。
- 判定：映射表中两个不同"范本名"对应同一"新名"。
- 后果：新作里两个本该不同的角色/地点变成同名，读者混乱。

### A3. 命名不一致 / 漂移（❌）
同一范本元素，在映射表里登记了多个新名，或不同章节用了不同新名。
- 判定：同一"范本名"对应多个"新名"；或某新名在前 N 章稳定、后某章突变。
- 这是跨章漂移的典型，单章自查抓不到。

### A4. 时代/世界观穿帮（⚠️→❌，看程度）
换了大背景（古代→现代等），改写稿残留旧世界专名。
- 判定：用户提供的旧世界词表（如 `灵气 剑气 客栈 圣旨 内力`）在现代背景改写稿中出现。
- 通过 `--extra-old` 传入要扫的旧世界词。

## B. 通用角色一致性检查（原创小说/剧本库）

库里没有映射表、但有角色档案时启用。

### B1. 设定前后矛盾（❌）
同一角色的固定属性（外貌、年龄、籍贯、能力）在档案与正文、或不同章节间打架。
- 脚本能抓的：档案 frontmatter 字段缺失/为空。
- 靠人脑读判断的：语义级矛盾（"左脸疤"vs"右脸疤"），脚本抓不到，巡检官读档案+正文比对。

### B2. 关系矛盾（🔻）
角色关系前后变化但中间无交代（兄弟→仇敌却没有反目情节）。靠读判断。

### B3. 能力体系穿帮（🔻）
力量/等级体系前后逻辑不通（越级打怪没解释、境界倒退）。靠读判断。

## C. 伏笔回收检查（有伏笔账本时启用）

### C1. 埋了没收（🔻）
账本记录"第X章埋、预计第Y章收"，但扫到当前最新章仍未回收。
- 判定：账本条目标"已埋"未标"已收"，且当前最新章号 ≥ 预计回收章。
- 报告时附"预计回收章"，让用户判断是遗漏还是故意延后。

---

## D. wiki 健康检查（任何库都跑）

### D1. 断链（🔻）
`[[某笔记]]` 指向的目标文件不存在。
- 判定：正文中 `[[X]]` 的 X 在 wiki 里找不到对应 .md。

### D2. 孤儿笔记（⚠️）
某笔记既没有被任何笔记链接，自己也不链出去。
- 判定：文件名不出现在任何其他文件的 `[[ ]]` 里，且自身无 `[[ ]]`。

### D3. frontmatter 缺字段（⚠️）
角色档案缺关键 frontmatter（如模板要求的"姓名/年龄"等）。
- 判定：与模板要求的 metadata_required 比对。

## scan_vault.py 退出码

| 退出码 | 含义 |
|:------:|------|
| 0 | 全部通过，无问题 |
| 1 | 发现问题（正常的"有错"出口，体检单已输出） |
| 2 | 用法错误（参数缺失、路径不存在） |
| 3 | 库结构异常（找不到 wiki/ 目录等） |

巡检官读退出码：0 报"健康"，1 把脚本输出整理成体检单，2/3 提示用户检查库路径。
