# 角色参考图 → 提示词

> 输入人物小传 → Pinterest 搜索真实参考图 → 分析图片提取视觉DNA → 融合人物小传输出生图提示词。参考图是像素，不是回忆。

## 理念

「吉卜力风格的女战士长什么样？」——不让 LLM 回忆，直接去 Pinterest 搜给你看。

找到合适的参考图后，用 `view_image` 分析这张图的每一个视觉要素，提取「视觉 DNA」。再用人物小传里的具体信息（身高、年龄、伤疤、服装偏好）覆盖 DNA 中不匹配的部分。最终输出一个既有参考图精度、又有角色个性的提示词。

不满意参考图？翻下一页搜索结果，换一张。不满意提示词？换一张参考图再试。纯像素驱动的迭代。

## 输入

人物小传，至少含：

```
姓名 / 年龄 / 身高(cm) / 身份 / 世界观 / 外貌特征 / 风格偏好
```

## 核心规则

1. 不用 LLM 回忆风格——用 Pinterest 搜真实参考图，或用户自备参考图
2. 直接推荐最佳参考图，不满意就换
3. 每张参考图必须用 `view_image` 做详尽视觉分析
4. 最终提示词必须融合参考图 DNA + 人物小传差异
5. 身高必须出现在最终提示词中
6. 输出为 JSON 格式——机器可解析，直接投喂下游 AI 工具

## 阶段 0.5：项目风格确认（全剧只确认一次）

搜索参考图之前，先确认项目整体风格。确认后所有角色共用。

**A. 媒介类型**：真人剧 / 动漫剧 / 漫剧(webtoon)

**B. 题材类型**：悬疑 / 爱情 / 奇幻 / 科幻 / 末世 / 武侠 / 都市 / 校园 / 恐怖 / 其他___

**C. 视觉风格**：偏写实 / 偏风格化 / 偏卡通 / 黑暗成人向 / 明亮少年向 / 其他___

**D. 特殊设定（可多选，无则跳过）**：兽人 / 机甲 / 修仙 / 异能 / 蒸汽朋克 / 赛博朋克 / ___

确认后，所有角色的搜索词自动拼接：`[题材] [媒介] [视觉风格] [特殊设定] [角色类型] character turnaround sheet`

## 阶段 1：获取参考图

先问用户：「有自备参考图吗？有的话直接发我，跳过搜索。」

**有自备图** → `view_image` 直接分析，进阶段 3。

**无自备图** → 从人物小传 + 阶段 0.5 确认信息中提取 3-5 个英文关键词构造搜索 URL。

关键词组合逻辑：
- 题材+媒介+风格（来自阶段 0.5 确认）
- 角色类型（英文）：`female warrior`, `elder mage`, `cyberpunk doctor`, `wasteland scavenger`
- 关键特征（英文）：`scarred`, `tattered cloak`, `mechanical arm`, `white hair`
- **必加限定词**：`character turnaround sheet` 或 `character model sheet`

用 `+` 连接，URL 模板：
```
https://jp.pinterest.com/search/pins/?q=关键词1+关键词2+character+turnaround+sheet
```

备选搜索源：
```
https://www.artstation.com/search?q=关键词1+关键词2+turnaround
https://www.deviantart.com/search?q=关键词1+关键词2+model+sheet
```

## 阶段 2：Pinterest 搜索 + 选图

```
open_browser_page(搜索URL)
→ read_page 读取搜索结果
→ 从列表中找到最有潜力的参考图
→ navigate_page(pinURL) 直接导航到 Pin 页面（Pinterest 弹窗常拦截点击）
→ screenshot_page 截取大图
→ 推荐：「这张最贴合——[具体理由，3句话]」
```

用户 OK → 进阶段 3。不满意 → 翻页或换搜索词再找。

## 阶段 3：提取视觉 DNA

```
view_image 查看选定的参考图
→ 详细分析以下维度：

面部与头部：
- 脸型轮廓、颧骨、下颌
- 眼型（大小/间距/眼角走向/单双眼皮）
- 鼻型（鼻梁高度/鼻翼宽度/鼻尖形状）
- 唇型（厚度/唇峰/嘴角走向）
- 年龄感（皮肤纹理/眼袋/法令纹/肤色）
- 发型（长度/质地/发色/分线/碎发处理）

体型与比例：
- 头身比、肩宽、腰线位置
- 体态倾向（挺拔/慵懒/紧绷/不对称）
- 四肢比例（手臂长度/腿部比例/手部大小）

服装与材质：
- 层叠逻辑（从内到外几层/每层面料/颜色）
- 材质表现（布料垂感/皮革反光/金属锈迹/磨损位置）
- 鞋履（类型/材质/磨损）

配件清单（逐一列出）：
- [配饰1]：[位置/材质/尺寸/功能]
- [道具/武器]：[位置/材质/尺寸]

多角度辨识特征：
- 正面：最显眼的特征（脸/胸/腰带扣）
- 侧面：只有侧面能看到的（发型后侧轮廓/背部曲线/腰带侧面）
- 背面：背面独有的（发型后部/背部纹身/腰带扣位置）
```

## 阶段 4：人物小传融合

对照人物小传，逐一比对差异：

```
参考图 DNA          vs    人物小传要求
─────────────────────────────────────────
年轻女性、光滑皮肤   →    52岁退役老兵 → 加深法令纹、眼袋、添加伤疤
华丽抛光盔甲         →    末世拾荒者   → 褪去光泽、废旧金属拼接、磨损
7头身               →    8头身185cm   → 头身比+1、拉长四肢比例
短发                 →    长发马尾     → 替换发型、保留发色
```

输出调整清单，逐项融入最终提示词。比例用头身比和具体数值（cm）。

## 阶段 5：输出最终提示词

读取 `references/prompt-format.md`，按 JSON 模板输出。模板含：

- 角色基础信息 + 性格特质
- 面部/体型/服装/道具结构化数据
- 纹身/特殊标志/辨识特征（含特写标注）
- 5 视角描述
- 图片上需生成的文字标注（角色名/身高/性格关键词/特写说明）
- 纯色背景、均匀柔光、合理排版

## 阶段 6：保存参考图

优先用浏览器截图——`screenshot_page` 截取的大图足以分析。

gallery-dl 可用时（安装说明见 `references/gallery-dl-setup.md`）：
```
gallery-dl "<pin页面URL>" -d ./references/
```
不可用时截图即可，核心流程不依赖它。
