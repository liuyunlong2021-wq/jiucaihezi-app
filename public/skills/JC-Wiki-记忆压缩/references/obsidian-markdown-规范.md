# Obsidian Markdown 规范 —— Wiki 文件书写标准

> 所有 wiki/ 下的 .md 文件遵循此规范。确保在 Obsidian 中打开时 [[双链]] 可点击、frontmatter 可检索、嵌入可预览。

## Frontmatter（必须）

每个 wiki 文件顶部必须有 YAML frontmatter：

```yaml
---
title: 标题
tags: [标签1, 标签2]
类型: 角色/场景/道具/剧本/参考/索引/目录
创建日期: YYYY-MM-DD
---
```

**必填字段**：`title`、`tags`、`类型`
**选填字段**：`原型`、`原型引擎`、`涉及集数`、`最后操作`（目录文件专用）

## Wikilink（[[双链]]）

**文件链接**：
```markdown
[[角色/韭菜盒子]]           → 链接到 wiki/角色/韭菜盒子.md
[[催化/第3章]]              → 链接到 wiki/参考/书名/催化/第3章.md
[[../世界/调性]]            → 相对路径
```

**别名链接**（用于显示名≠文件名时）：
```markdown
[[角色/韭菜盒子|那个社恐宅男]]   → 显示「那个社恐宅男」，链接到韭菜盒子
```

**标题锚点**：
```markdown
[[角色/韭菜盒子#性格引擎]]       → 链接到该文件的「性格引擎」标题
```

**块引用**：
```markdown
[[角色/韭菜盒子#^decision]]      → 链接到特定段落块
```

## 嵌入

**嵌入笔记**：
```markdown
![[角色/韭菜盒子]]               → 完整嵌入该文件内容
```

**嵌入图片**（下游设计Skill产出）：
```markdown
![[角色/韭菜盒子.design.png|300]] → 嵌入图片，宽度300px
```

## Callout（标注）

用于重要提示：
```markdown
> [!note] 引擎激活
> 本角色的红线=模型安全。催化剂设计优先攻击此红线。
```

可用类型：`note`、`warning`、`tip`、`info`、`danger`、`example`、`quote`

折叠标注：
```markdown
> [!example]- 三角引擎示例
> 角色红线 × 道具物理法则 × 场景空间逻辑
```

## 标签

```markdown
#角色 #主角 #费马原型
```

用于 Obsidian 的标签搜索和图谱视图。
