# 小说催化提取 —— 两步走：拆分原文 → 提取催化

> 当用户把一本参考小说放入 `.raw/参考资料/` 时，两步处理：①拆分原文为单章文件 ②逐章提取催化事件表单。主 Skill 只读催化表单。

## 目录结构

```
.raw/参考资料/
└── 书名.txt                      ← 原始书，不动

wiki/参考/《书名》/
├── 目录.md                       ← 全章索引（标题+行号）
├── 节拍总览.md                   ← 全书节奏DNA（积累足够催化后生成）
├── 原文/                         ← 拆分后的单章原文
│   ├── 第1章.md
│   ├── 第2章.md
│   └── ...                       ← 用户说拆几张就拆几张
└── 催化/                         ← 催化事件表单 ★ 主Skill只读这个目录
    ├── 第1章.md
    ├── 第3章.md
    └── ...                       ← 用户说拆哪章就拆哪章
```

## 流程

### 第一步：拆分小说

用户说「拆分这本小说」或「把这书拆成章节」。

1. 扫描 `.raw/参考资料/书名.txt`，识别章节标题
2. 生成 `wiki/参考/书名/目录.md`——**带状态追踪**：
```markdown
---
title: 书名 · 目录
类型: 参考/目录
总章节: 1247
已拆原文: 5
已拆催化: 3
最后操作: 拆第3章催化 · 2026-07-11
---
# 书名 · 目录

| 章节 | 标题 | 原文 | 催化 |
|------|------|:----:|:----:|
| 第1章 | 穿越成废柴 | ✅ | ✅ |
| 第2章 | 金手指激活 | ✅ | ✅ |
| 第3章 | 第一次战斗 | ✅ | ✅ |
| 第4章 | 宗门入门 | ✅ | — |
| 第5章 | 秘境试炼 | ✅ | — |
| 第6章 | 隐藏实力 | — | — |
| ... | ... | | |
```

**一个月后回来，看一眼目录.md 的 frontmatter 就知道**：`最后操作: 拆第3章催化`。从第4章催化继续。

jiyasuo 每次处理完一章，自动更新目录.md 的对应行 + frontmatter 计数。

### 第二步：提取催化（逐章 + 同步更新索引）

用户说「拆第X章的催化」。

#### 2a. 输出章节催化表单

`wiki/参考/书名/催化/第X章.md`：

```markdown

tags: [冲突:信任危机, 关系:情人争吵, 反转:认知反转]
---

## 催化事件1
```

#### 2b. ★ 同步更新节拍索引

**每拆完一章催化，立刻往 `wiki/参考/书名/节拍索引.md` 追加条目。**

这是 LLM 的检索入口——只要读这一个文件，瞬间定位所有相关节拍。

## 主 Skill 如何瞬间检索（关键）

### 旧方式（按章号检索）—— 慢

```
调度层：「本场是韭菜盒子被镇元子认出红云身份」
  → 读 催化/第1章.md → 是穿越醒来，不匹配
  → 读 催化/第3章.md → 是信任危机，部分相关
  → 读 催化/第5章.md → 是拍卖会，完全不匹配
  → ...大海捞针
```

### 新方式（节拍索引）—— 瞬间

```
调度层：「本场是韭菜盒子被镇元子认出红云身份」
  → 分析当前情景 → 标签：「冲突：身份暴露」「关系：故人重逢」
  → 读 wiki/参考/书名/节拍索引.md
  → 瞬间定位到「冲突：身份暴露」区块：
      [[催化/第8章]]：被认出→假装不是→对方拿证据→被迫承认→试探立场
      [[催化/第31章]]：偶遇→双方装没看见→第三方戳破→尴尬对峙
  → 选第8章的模式，点开 [[催化/第8章]] 看细节
  → 适配韭菜盒子：
     「被认出=镇元子感应到红云气息
      假装不是=韭菜盒子：『你认错人了，我只是路过』
      对方拿证据=镇元子：『你散发的红云本源，天地间独一无二』
      被迫承认=韭菜盒子：『...行吧。但我不是你以为的那个人』」
```

### CLAUDE.md 里的入口

用户的创作项目 `wiki/` 下的 CLAUDE.md 中加一行：

```markdown
## [节奏参考] 
- 参考书: [[wiki/参考/《我师兄实在太稳健了》/节拍索引]]
```

主 Skill 续写前读 CLAUDE.md → 看到节奏参考 → 读节拍索引 → 当前场景匹配情景标签 → 找到对应节拍 → 读具体催化章节 → 适配。

### 不需要参考时

```
节奏参考：无
```

就跳过索引检索，正常用三角引擎设计催化剂。完全不影响没有参考书的创作流程。

## 核心原则

1. **两步分开**：拆分原文和提取催化是两个独立操作，用户分别控制
2. **按需拆分**：不全拆——一千章全拆是浪费。用户说拆哪章就拆哪章
3. **提取节拍，不抄内容**：可迁移的节奏模式，不是具体情节
4. **催化表单独立**：催化/ 目录是主 Skill 的唯一读取入口，原文/ 只是原材料
