# 通用摄入协议 —— 任何参考资料 → 结构化 Wiki

> jiayasuo 的摄入模式不限于小说。用户扔进 `.raw/参考资料/` 的任何东西，都能拆解成原子笔记 + 概念索引。这是 Zettelkasten 的核心：按概念组织，按[[双链]]连接，按索引检索。

## 通用流程

```
用户提供资料
    ↓
如果是 URL → defuddle parse <url> --md → .raw/参考资料/文件名.md
如果是文件 → 直接放入 .raw/参考资料/
    ↓
jiyasuo 判断类型 → 选择拆解协议 → 原子笔记进 wiki/参考/
    ↓
按概念建索引（不按来源组织）
    ↓
更新目录.md（状态追踪）
    ↓
（可选）生成 .canvas 关系图 / .base 统计表
    ↓
CLAUDE.md 一行指向索引入口
```

## 已支持的拆解协议

### 协议A：小说 → 催化节拍

详见 `references/小说催化提取.md`。

- 输出：`wiki/参考/书名/催化/第X章.md` + `节拍索引.md`（按情景标签组织）
- 索引标签：冲突类型、关系类型、反转类型、爽点类型

### 协议B：论文/文献 → 概念卡片

用户扔进一篇研究论文或参考书籍。

- 输出：`wiki/参考/文献名/概念/概念名.md`（一条概念一张卡片）+ `概念索引.md`
- 索引标签：概念层级（核心/支撑/延伸）、相关角色、相关世界观

```markdown
# 概念索引 · 《山海经》

## 神兽
- [[概念/凤凰]]：火德之鸟，涅槃重生，五百年一劫
- [[概念/九尾狐]]：善变化，食人，其音如婴儿

## 地理
- [[概念/昆仑墟]]：帝之下都，百神之所在
- [[概念/不周山]]：天柱，共工撞断→天倾西北
```

### 协议C：电影/剧集 → 场景结构

用户扔进电影剧本或分集大纲。

- 输出：`wiki/参考/作品名/场次/场X-X.md` + `场景索引.md`
- 索引标签：冲突类型、空间类型、情绪类型、反转类型

### 协议D：通用 → 自由拆解

用户说「拆这个，按XX方式拆」。

jiyasuo 问：
```
你想怎么拆？
A. 按章节/段落拆 → 原子笔记
B. 按概念/关键词拆 → 概念卡片
C. 按人物/角色拆 → 角色档案
D. 自定义
```

然后按用户选择执行。不管哪种方式，最终都生成**概念索引**。

## 索引的通用格式

无论什么协议，索引文件都遵循同一个结构：

```markdown
---
title: [来源] · [类型]索引
类型: 参考/索引
来源: [[wiki/参考/来源名/]]
最后更新: YYYY-MM-DD
---
# [来源] · 索引

## [概念标签1]
- [[原子笔记A]]：[一句话摘要]
- [[原子笔记B]]：[一句话摘要]

## [概念标签2]
- [[原子笔记C]]：[一句话摘要]
```

**核心原则**：
1. 索引按「概念标签」组织——不按来源章节号
2. 每条目一句话摘要——不需要点开就知道是什么
3. [[双链]] 到原子笔记——需要细节时点进去
4. 标签体系由 jiyasuo 根据内容自动判断——不做预设穷举

## Obsidian 兼容

所有 wiki 文件遵循 Obsidian Markdown 规范（`obsidian-markdown`）：

- **[[双链]]**：`[[角色/韭菜盒子]]` `[[催化/第3章]]`
- **Frontmatter**：YAML 元数据（title, tags, 类型）
- **嵌入**：`![[场景/办公室.png]]`（用于设计图引用）
- **标注**：`> [!note]` 用于重要提示

这对主Skill的意义：LLM 读 wiki 文件时，[[双链]] 是天然的「下一步检索建议」——看到 `[[催化/第3章]]`，LLM 知道如果需要细节就去读那个文件。

## CLAUDE.md 的索引入口

用户的创作项目 wiki/ 下 CLAUDE.md 中，格式统一：

```markdown
## [参考资料]
- 小说催化: [[wiki/参考/《我师兄实在太稳健了》/节拍索引]]
- 世界观参考: [[wiki/参考/《山海经》/概念索引]]
- 角色参考: [[wiki/参考/《封神演义》/概念索引]]
```

主Skill续写前读这一行 → 需要哪个索引就读哪个 → 全都不需要就跳过。

## 渐进披露总结

```
CLAUDE.md（一行指向索引）
    ↓
[类型]索引.md（概念标签 → [[双链]]）
    ↓
原子笔记.md（单条详细信息）
    ↓（可选）
原文/（原始材料）
```

每一层只披露你需要的信息。这就是 Zettelkasten 在 AI 时代的用法——不是给人类翻的，是给 LLM 瞬间检索用的。
