# 关系图画布生成协议 —— JSON Canvas (.canvas)

> jiayasuo 整理完角色关系后，自动生成 `wiki/世界/关系.canvas`。用户在 Obsidian 中打开可直接看到角色关系可视化图谱。

## 生成时机

创作 3-5 集后，jiayasuo 整理记忆时：
- 扫描 `wiki/角色/` 下所有角色文件
- 读取每个角色的「🔒 四、关系网络」
- 生成 `.canvas` 文件

## 画布布局规则

```
主角居中（x=0, y=0）
盟友右侧（x=+500）
对手左侧（x=-500）
导师/上级上方（y=-400）
待加入角色下方（y=+400）
```

## 节点颜色

| 关系 | 颜色预设 | 含义 |
|------|---------|------|
| 主角 | `"4"` (绿) | 中心节点 |
| 盟友 | `"5"` (青) | 正向关系 |
| 对手/敌人 | `"1"` (红) | 对抗关系 |
| 暧昧/复杂 | `"6"` (紫) | 模糊关系 |
| 中立/未知 | `"3"` (黄) | 待发展 |

## 边样式

| 关系 | 样式 |
|------|------|
| 同盟 | 双箭头、青色、标签「同盟」 |
| 对抗 | 单箭头(敌对→主角)、红色、标签「对抗」 |
| 暧昧 | 虚线、紫色、标签「？」 |
| 师徒 | 单箭头(师→徒)、黄色、标签「师徒」 |

## jiayasuo 集成

jiayasuo 整理记忆时增加一步：

```
扫描 wiki/角色/*.md → 提取关系网络 → 生成 wiki/世界/关系.canvas
```

规则：
- 每个角色一个节点（type=text，内容=名字+一句话+[[双链]]）
- 每对关系一条边
- 已有 canvas → 更新而非覆盖（新增角色/关系时追加节点和边）
