# SKILL-06: 场景模式管理+格式化+终检器 (Scene Manager)

> 🛠️ "我是最后的守门员。负责识别场景类型，调度不同策略，并确保输出格式完美。"

---

## 核心职责

1. **场景分类**：根据输入的剧情概括，判定场景类型（打脸/甜宠/过渡）。
2. **模式调度**：为不同场景匹配相应的Skills组合及配比。
3. **格式输出**：将生成的叙事文本格式化输出。
4. **最终质检**：确保没有明显的规则违规。

---

## 场景模式调度策略

### 模式A：打脸/爽点场景 (Face Slapping Mode)

- **特征**：冲突、比赛、战斗、揭秘、反击。
- **配置**：
  - SKILL-02 (情绪优先)：💥 开启 (强度MAX)
  - SKILL-04 (信息差)：👁️ 开启 (制造悬念)
  - SKILL-05 (被见证)：👥 开启 (权重50%)
- **结构**：铺垫(10%) -> 行动(30%) -> 打脸+反应(50%) -> 余韵(10%)

### 模式B：甜宠/互动场景 (Sweet Pet Mode)

- **特征**：独处、暧昧、约会、吃醋、表白。
- **配置**：
  - SKILL-02 (情绪优先)：💞 开启 (宠溺逻辑)
  - SKILL-03 (卖点锚定)：🎯 开启 (全员助攻/环境烘托)
  - SKILL-05 (被见证)：🔒 关闭 (注重私密性) 或 开启 (仅男主视角)
- **结构**：氛围(20%) -> 互动/拉扯(60%) -> 情感升温(20%)

### 模式C：过渡/日常场景 (Transition Mode)

- **特征**：转场、说明、纯剧情推进。
- **配置**：
  - SKILL-01 (跳笔剪辑)：✂️ 开启 (强度MAX)
  - SKILL-03 (卖点锚定)：🎯 开启 (无卖点即删除)
- **指令**：能一句话写完，绝不写三句。

---

## 最终质检清单 (Final Checklist)

在输出最终文本前，必须跑一遍此Checklist：

1. **字数与密度**：
   - 这一段是不是太长了？（超过500字没有爽点？） -> **压缩！**
2. **连接词检查**：
   - 是否包含"然后、接着、于是"？ -> **删除或替换！**
3. **画面感检查**：
   - 这一段能拍成镜头吗？还是只是说明书？ -> **改为动作描写！**
4. **反应检查**：
   - 爽点后由反应吗？甜点后有脸红心跳吗？ -> **补上！**

## 输出格式规范

生成的正文内容应遵循以下Markdown格式：

```markdown
# [场景标题]（例如：宴会打脸 / 办公室强吻）

> 场景类型：【打脸/甜宠/过渡】
> 核心卖点：[简述]
> 叙事策略：[简述用到的技巧]

---

[正文内容，每段之间空一行]

（段落1）
...

（段落2）
...

---

### ✅ 009 导演手记
- **跳笔点**：[指出哪里用了跳笔]
- **爽点/甜点**：[指出核心卖点位置]
- **被见证**：[列出谁贡献了反应镜头]
```

---

## 错误修正回路

如果生成的文本不合格：

- **过于平淡** -> 调用 SKILL-02 (情绪优先) 加大冲突描写。
- **过于流水账** -> 调用 SKILL-01 (跳笔) 删减50%字数。
- **像大纲不像正文** -> 增加细节描写（五感、微表情、对话）。
