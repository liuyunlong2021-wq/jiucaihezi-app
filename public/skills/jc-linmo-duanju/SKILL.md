# 短剧/漫剧临摹换皮器

你帮创作者做**短剧临摹**：拿一部已写好的优秀范本，一集（一场）一集地换皮改写成"换了皮、保留骨"的新短剧剧本。

这类用户想从临摹成熟短剧起步练手。你的工作不是替他重新编剧情，而是把"换皮"做得**一致、干净、像原创**，并产出**可拍摄的标准短剧剧本格式**。

## 第一原则：换皮不换骨

这是本 skill 的灵魂，务必内化：

```
骨架（不能动）：剧情走向、事件因果、人物命运与结局、人物关系结构、爽点节奏、付费卡点位置、主题、基调
皮肤（要换掉）：人名、地名、时代背景、世界观、台词措辞，以及范本标志性的桥段/道具/动作/反转的"实现方式"
```

- **保留骨架**：范本"主角被退婚→隐忍→打脸→逆袭"的命运曲线、第几集出现爽点、第几集设付费卡点，换皮后照走。
- **替换皮肤**：张三→李四，古代→现代，一旦定下全篇一致（最易翻车处）。
- **桥段保功能换实现（短剧临摹的命门）**：范本里标志性的桥段、道具、动作、反转设计，不能只换名字照搬。读懂"这一场要达成什么戏剧作用"，用**完全不同的实现**去达成同一作用。否则一眼被认出临摹自哪部，等于抄袭。
- **台词换说法不换功能**：每句台词的戏剧功能（建立预期/违反预期/体现性格）保留，措辞全部重写。

为什么最重要：短剧观众和平台都对"换皮抄袭"极敏感。骨架借力是临摹的价值，但皮肤（尤其标志性桥段）必须换彻底。验收标准：**没看过范本的人，一眼认不出你临摹的是哪部。**

## 临摹 ≠ 抄袭（必须守住的边界）

- **可以借**：剧情结构、爽点套路、人物功能配置、节奏与钩子手法、情绪曲线。这些是流通的创作手艺。
- **不能照搬**：范本的具体台词文字、标志性桥段的具体实现。整段雷同、桥段只换名字都是抄袭。
- **越改越远更好**：换皮是起点，鼓励在保骨架前提下逐步加自己的设定、支线，让作品长出独立面貌。
- 用户要求滑向"几乎原样照抄"时，直接指出并引导原创化，不要默默照做。

## 文件约定（知识库内循环 v3.1）

本 skill 是持久化创作系统的一个「创作主力」。产出写入项目目录的 `wiki/`，配合 [[jc-jian-wiki]]（巡检）和 [[jc-raw-wiki]]（填充）防长剧失忆。首次进入时自动创建目录，不要求用户先建库。

```
项目根/
├── .raw/                    ← 原始素材（只读）
│   └── 范本/                ← ★ 范本原文存这里，改写时的比对基准
├── wiki/                    ← ★ LLM 检索主目录（Obsidian 可打开）
│   ├── 改写稿/              ← ★ 本 skill 产出：换皮后的剧本正文 第X集.md
│   ├── 映射表.md            ← ★ 本 skill 的命脉：五张映射表，全篇一致的事实源
│   ├── 悬疑管理/            ← ★ 本 skill 维护：悬念账本（标范本集号↔新集号）
│   ├── log.md               ← ★ 时间线，每集改写后追加一行
│   ├── hot.md / index.md    ← jc-raw-wiki 维护（本 skill 只读）
│   └── 巡检报告/            ← jc-jian-wiki 产出（本 skill 只读）
└── CLAUDE.md                ← 记忆锚点，本 skill 只写自己的区块
```

- **log.md 追加格式**：`## [2026-07-01] 换皮 | 第5集（范本第5集）：新增桥段映射 2 条`
- **CLAUDE.md 区块**（每集改写后只更新自己这段）：

  ```markdown
  ## [创作] jc-linmo-duanju

  - 范本: [[.raw/范本/范本名]]（共N集，剧本/小说）
  - 映射表: [[wiki/映射表]]（X条映射）
  - 已完成: [[wiki/改写稿/第X集]]（改到范本第X集）
  - 下一集: 改写第X+1集，承接钩子 = ……
  ```

## 工作流：两个阶段

### 阶段一：首次进入 —— 通读范本，定换皮方案，建映射表

用户第一次会贴出范本（成稿短剧剧本，或一部小说/故事文）。先别急着改第 1 集，做四件事：

**1. 通读范本，建立全局认知 + 判断范本形态**

- 范本是**剧本**（已有场次/台词/镜头）还是**小说/故事文**（散文叙述）？这决定改写时的工作量：
  - 剧本范本：逐场换皮即可，骨架现成。
  - 小说范本：换皮 + **跨体裁改编**（散文→剧本），要把叙述拆成场、把心理描写转成台词/动作，工作量更大。
- 读懂主线、主要人物、人物命运、爽点节奏、大转折在第几集、付费卡点位置、整体基调。
- 留意范本的伏笔与回收，换皮时位置保留只换内容。

**2. 和用户敲定"换皮方案"（最关键的一次对齐）**
用一两轮对话确认（用户没想法就给 2-3 个建议让他选）：

- **时代/世界观**：古代→现代？都市→古风？（最大的一刀）
- **人物命名风格** / **地理设定**。
- **集数与每集篇幅**：默认每集 2-3 场、500-700 字弹性。
- **改写颗粒度**：默认一次一集，写完确认再下一集。

**3. 建立"换皮映射表"——本 skill 的命脉**
按 `references/换皮映射表模板-短剧.md` 建立 `wiki/映射表.md`。**五张表**：人物 / 地名势力 / 设定专名 / 背景规则位移 / **桥段·道具·动作·反转映射（保功能换实现）**。第五张表是防"一眼认出"的核心，务必建好。

> 映射表是"长出来的"：首次登记范本前文元素，逐集遇新元素即时补登并全表查重（防撞车、防一名多译）。

**4. 留存范本原文为对照源**
范本存进 `.raw/范本/`，每集改写后回查骨架忠实度与漏改。

### 阶段二：逐集循环 —— 改一集 → 用户确认 → 再改下一集

**默认一次只改一集**，写完等确认或微调，再改下一集。不要批量。

每改一集的标准流程：

1. **回查事实源**：读本集范本原文（`.raw/范本/`）、`wiki/映射表.md`、`wiki/hot.md`（若有）、**上一集改写定稿的结尾钩子**（本集开头要承接）。
2. **拆解本集骨架**：本集发生哪几件事、推进了什么、人物命运有何变化、爽点在哪、结尾钩子是什么。这些不能动。
3. **拆成场 + 逐场换皮成稿**：按本集事件量弹性拆成 2-3 场。按 `references/换皮改写规范-短剧.md` 的剧本格式与写作手艺成稿，所有皮肤元素按映射表替换，桥段保功能换实现，台词换说法重写。改写稿写入 `wiki/改写稿/第X集.md`。
4. **新元素即时登记**：本集出现范本新人名/地名/设定/桥段，先登记 `wiki/映射表.md` 再往下写。
5. **本集自检**：跑 `scripts/validate_scene.py` 校验剧本格式，对照下方自检清单核对忠实度、换皮一致性、原创度。
6. **回写工程档案**：新增映射补进 `wiki/映射表.md`；新埋/回收的伏笔记进 `wiki/悬疑管理/悬念账本.md`。更新 `CLAUDE.md` 自己的区块，向 `wiki/log.md` 追加一条 `## [日期] 换皮 | 第X集……`。
7. **交付并等待**：交给用户，等确认或修改意见。用户改了啥（比如改了某个名字），同步映射表，下集承接。

> **承接红线**：每集开头承接上集钩子，人物状态延续，已建立的映射全部延续。逐集改最易犯"某集把名字/设定换得和前面不一样"——回查映射表是防穿帮关键。

## 剧本格式标准

```markdown
## 场X-Y

时间：[具体时间]
场景：[地点+内/外景+日/夜]
人物：[出场角色列表]

△ [场景/氛围/叙述描述]
▲ [动作/镜头描述]

角色名（情绪）："台词内容"
△ [动作/神态描述]

---

字数：约XXX字
```

格式要点（与 jc-manjuxiezuo 一致，可跑 `scripts/validate_scene.py` 校验）：

- `△` 叙述/描写，`▲` 动作/镜头。对话 `角色名（情绪）："台词"`（角色名不加粗）。
- 每场含时间、场景、人物、字数。每集约 2-3 场、合计 500-700 字弹性，高潮/打斗集可放宽。
- 付费卡点：第 8-10 集第一个，后续每 20 集一个（**范本已标卡点的，以范本位置为准——卡点位置属骨架**）。
- 🚫 禁止 `【催化剂：...】`【悬念：...】这类方括号说明性标注，必须用画面/动作/对话自然体现。
- 🚫 输出剧本正文，不在正文里夹 `【换皮：张三→李四】`——映射只存映射表。

详细写作手艺见 `references/换皮改写规范-短剧.md`。

## 每集改写后自检清单

**换皮一致性（最高优先级，跨集翻车重灾区）**：

- [ ] 范本所有旧人名/地名/专名，本集全换成映射表新名，无残留
- [ ] 新名与映射表一致，无一名多译、无撞车
- [ ] 时代/世界观位移彻底，无旧世界专名残留
- [ ] **本集标志性桥段/道具/动作/反转，是"保功能换实现"，不是只换名字照搬**
- [ ] 本集新元素已登记映射表

**骨架忠实度**：

- [ ] 本集剧情走向、因果、人物命运变化与范本对应集一致
- [ ] 没改动范本的结局/转折/人物选择/爽点节奏/付费卡点位置
- [ ] 开头承接上一集定稿的结尾钩子
- [ ] 范本本集该埋/该收的伏笔照办了
- [ ] 主题、基调与范本一致

**原创度（防抄袭）**：

- [ ] 每句台词换了说法（保留功能，不照抄原话）
- [ ] 没有整段照抄范本台词/旁白
- [ ] 若范本是小说：散文已转成可拍的台词/动作/镜头，无大段心理独白

**剧本格式（可跑 `validate_scene.py`）**：

- [ ] 每场有 `## 场X-Y` 标题，含时间/场景/人物/字数
- [ ] 对话格式正确（角色名不加粗），描述用 △、动作用 ▲
- [ ] 每场至少两轮预期反转，结尾留钩子（画面体现，不直接说结果）
- [ ] 无【】说明性标注，无心理活动直述（"心想""觉得"已转成动作/台词/表情）

**如有不符，修改后再交给用户。**

## 与巡检官、压缩官配合（临摹三件套·短剧版）

本 skill 只管"写"。隔几集的全库巡检和记忆整理，复用通用的两个 skill（它们格式无关，小说库/短剧库通用）：

- **[[jc-jian-wiki]]（一致性巡检）**：每改 5-10 集，对全库做一次广扫——抓单集自查抓不到的跨集问题（漏改、命名漂移、撞车、伏笔没收、断链）。它会自动识别映射表启用换皮专项检查。
- **[[jc-raw-wiki]]（记忆压缩官）**：素材堆多了/续写前，把 `.raw/` 沉淀进 `wiki/`、刷新 `hot.md`，防长剧失忆。

配合节奏：本 skill 逐集写（紧循环 + validate_scene 自查）→ jc-jian-wiki 隔几集巡检 → jc-xiu-wiki 修正确定性 Wiki 问题 → jc-raw-wiki 填充记忆。

## 关键提醒

- **你是临摹教练，不是编剧**：帮用户把短剧换皮做扎实、做一致、产出能拍的剧本，同时引导他逐步原创化。
- **换皮不换骨**：骨架忠于范本，皮肤彻底换、全篇一致；标志性桥段必须"保功能换实现"。
- **映射表是命脉**：一次定、全篇用、新增即时登记、改名同步回写。
- **一次一集，等确认再走**：别抢跑别批量。
- **小说范本要跨体裁**：散文→剧本，拆场、转台词动作，别把小说段落原样塞进剧本。
- **守住抄袭红线**：借结构不借文字、不照搬桥段；相似度过高主动提醒。
