## 4. 常用操作

### 连接数据库

```bash
docker exec $(docker ps -q -f name=postgres) bash -c \
  "PGPASSWORD='JiucaiheziPostgres_2026!' psql -h localhost -U newapi -d new-api"
```

### 查询渠道状态

```sql
SELECT id, name, status, models, "group" FROM channels ORDER BY id;
-- status: 1=启用, 2=禁用, 3=自动禁用
```

### 查询用户 Token

```sql
SELECT id, name, "group", remain_quota, unlimited_quota FROM tokens WHERE status = 1 AND deleted_at IS NULL;
```

### 启用/禁用渠道

```sql
UPDATE channels SET status = 1 WHERE id = 33;  -- 启用
UPDATE channels SET status = 2 WHERE id = 33;  -- 禁用
```

### 重启 NewAPI（不丢数据）

```bash
cd /root/new-api-new
docker compose restart new-api
```

### 完整重建（会短暂中断服务！）

```bash
cd /root/new-api-new
docker compose down
docker compose up -d
```

### 重载 Nginx

```bash
nginx -t && nginx -s reload
```

### 查看 NewAPI 版本

```bash
docker exec $(docker ps -q -f name=new-api) /new-api --version
```

### 拉取最新 NewAPI 镜像

```bash
docker pull calciumion/new-api:latest
# 然后 docker compose down && up -d 生效
```

## 5. 已知问题 & 解决方案

### NewAPI `progress` 字段解析 bug

**现象**: Seedance 视频提交返回 500，日志报 `cannot unmarshal object into Go struct field responseTask.progress of type int`

**原因**: Seedance 上游 `progress` 返回 `{"message": "..."}` 对象，NewAPI 期望 int

**解决**: 前端 Seedance 请求走 Nginx 直连代理 `/api/seedance/`，绕过 NewAPI

### NewAPI 模型名自动转换

**现象**: 请求 `veo3.1-fast` 时，NewAPI 内部转换为 `veo_3_1-fast`（点号变下划线），导致找不到渠道

**解决**: 渠道 models 列表中两种写法都要加上，或用模型重定向

### 渠道自动禁用

**现象**: 渠道连续报错后被自动禁用 (status=3)

**解决**: `UPDATE channels SET status = 1 WHERE id = <id>;`

## 7. 分组 & 计费

- **AutoGroup**: `["0.5","1","1.2","1.5","2"]` — Token group 为空时自动匹配
- **GroupRatio**: `{"0.5":0.5, "1":1, "1.2":1.2, "1.5":1.5, "2":2, "9":9, "川普特供":8}`
- 计费公式: `费用 = 分组倍率 × 模型倍率 × (输入tokens × 基准单价 + 输出tokens × 基准单价 × 补全倍率)`
- 模型如无 `ModelRatio` 或 `ModelPrice` 配置，`/v1/models` 接口不会返回该模型
