## 5. 已知问题 & 解决方案

### NewAPI `progress` 字段解析 bug（历史问题）

**现象**: Seedance 视频提交返回 500，日志报 `cannot unmarshal object into Go struct field responseTask.progress of type int`

**原因**: Seedance 上游 `progress` 返回 `{"message": "..."}` 对象，NewAPI 期望 int

**当前策略**: 创作面板主链路不再依赖 Seedance 直连代理；模型是否展示和是否允许执行由 `/api/creation/models` 结合 NewAPI 渠道状态决定。`/api/seedance/` 保留为历史/备用代理，若后续重新启用 Seedance，优先在 NewAPI/RH 适配链路里闭环。

### NewAPI 模型名自动转换

**现象**: 请求 `veo3.1-fast` 时，NewAPI 内部转换为 `veo_3_1-fast`（点号变下划线），导致找不到渠道

**解决**: 渠道 models 列表中两种写法都要加上，或用模型重定向

### 渠道自动禁用

**现象**: 渠道连续报错后被自动禁用 (status=3)

**解决**: 先查看 NewAPI 渠道错误日志和上游余额/密钥，再执行 `UPDATE channels SET status = 1 WHERE id = <id>;`。创作面板会通过 `/api/creation/models` 把自动禁用的模型标记为不可用。

### 创作面板点击后转圈消失

**现象**: 用户点击生成，任务短暂转圈后没有结果，也没有明确失败反馈。

**排查顺序**:

```bash
# 1. 看创作可用性服务是否正常
curl https://api.jiucaihezi.studio/api/creation/models

# 2. 看 NewAPI 是否返回模型
curl -H "Authorization: Bearer <用户 NewAPI Token>" \
  https://api.jiucaihezi.studio/v1/models

# 3. 看 rh-adapter / creation-models 日志
cd /opt/rh-adapter && docker compose logs -n 100 rh-adapter
journalctl -u creation-models -n 100 --no-pager

# 4. 看 NewAPI 渠道状态
docker exec $(docker ps -q -f name=postgres) bash -c \
  "PGPASSWORD='<从服务器环境变量获取>' psql -h localhost -U newapi -d new-api -c 'SELECT id,name,status,models FROM channels ORDER BY id;'"
```

**当前前端行为**: 不可用模型应在创作面板显示不可用状态；运行中任务如果失败，会回写创作画廊失败卡片，避免“无声失败”。

## 7. 分组 & 计费

- **AutoGroup**: `["0.5","1","1.2","1.5","2"]` — Token group 为空时自动匹配
- **GroupRatio**: `{"0.5":0.5, "1":1, "1.2":1.2, "1.5":1.5, "2":2, "9":9, "川普特供":8}`
- 计费公式: `费用 = 分组倍率 × 模型倍率 × (输入tokens × 基准单价 + 输出tokens × 基准单价 × 补全倍率)`
- 模型如无 `ModelRatio` 或 `ModelPrice` 配置，`/v1/models` 接口不会返回该模型

## 9. RunningHub 适配器 (rh-adapter) — 2026-06-03 更新

### 架构

```
提交计费:
Studio → NewAPI /v1/* → http://rh-adapter:8789 → RH 原生 API

轮询:
Studio → Nginx /rh/tasks/{task_id} → http://172.17.0.1:8789/tasks/{task_id}
```

适配器是 Python FastAPI Docker 服务，将标准 OpenAI 格式请求翻译为 RunningHub 原生 API 调用。POST 提交走 NewAPI 渠道并产生一次计费；GET 轮询走 Nginx `/rh/tasks/` 直连 adapter，避免重复计费。NewAPI 负责用户鉴权和计费，rh-adapter 只在 Docker/内网可达，不额外增加 adapter 鉴权。

### 文件位置

| 内容 | 路径 |
|------|------|
| 适配器源码 | `/opt/rh-adapter/` |
| 环境变量 | `/opt/rh-adapter/.env` |
| Docker Compose | `/opt/rh-adapter/docker-compose.yml` |
| 部署配置产物 | `scripts/rh-deploy/` |
| 日志 | `docker compose logs -f rh-adapter` |

### 环境变量

```
PORT=8789
HOST=0.0.0.0
RUNNINGHUB_API_KEY=32位RH_API_KEY
LOG_LEVEL=info
MAX_POLL_SECONDS=600
POLL_INTERVAL_IMAGE=5
POLL_INTERVAL_VIDEO=10
```

不要在 adapter 上新增独立密钥。隔离方式是 Docker/内网绑定 + Nginx 只公开 `/rh/tasks/` 轮询路径。

### 常用操作

```bash

docker compose up -d

# 查看状态

# 查看日志

# 测试轮询路由（task 不存在时 404 也说明路径到达 adapter）

```

### Nginx 轮询代理

```nginx
location /rh/tasks/ {
    proxy_pass http://172.17.0.1:8789/tasks/;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 30s;
}
```

本仓库提供幂等安装脚本：`scripts/rh-deploy/install-nginx-rh-tasks.py`。执行后必须 `nginx -t && systemctl reload nginx`。

## 10. 创作模型可用性服务 (creation-models) — 2026-05-31 新增

### 架构

```
Nginx /api/creation/models → http://127.0.0.1:8790/api/creation/models
creation-models → 只读查询 NewAPI PostgreSQL channels 表
```

这个服务只提供“创作面板模型是否可用”的只读状态，前端用它隐藏或拦截不可用模型。它不保存上游供应商 Key，不提交生成任务，不替代 NewAPI。

2026-05-31 更新：前端创作面板会在挂载时读取该接口；`media-generation.ts` 在真正发起生成前也会再次检查运行时可用性，避免其他调用方绕过 UI 直接执行已禁用模型。

### 文件位置

| 内容 | 路径 |
|------|------|
| 服务源码 | `/opt/creation-models/server.mjs` |
| 环境变量 | `/opt/creation-models/.env` |
| systemd | `/etc/systemd/system/creation-models.service` |
| Nginx 配置 | `/etc/nginx/sites-enabled/api.jiucaihezi.studio.conf` |
| Nginx 备份 | `/etc/nginx/backups/` |
| 日志 | `journalctl -u creation-models -f` |

### 环境变量

```
CREATION_MODELS_HOST=127.0.0.1
CREATION_MODELS_PORT=8790
NEWAPI_PSQL_PASSWORD=<PostgreSQL newapi 用户密码>
NEWAPI_PSQL_USER=newapi
NEWAPI_PSQL_DB=new-api
```

`.env` 必须是 `600 root root` 权限。systemd 使用 `EnvironmentFile=-/opt/creation-models/.env`，文件不存在时服务会启动但查询会失败；生产环境必须配置 `NEWAPI_PSQL_PASSWORD`。不要把数据库密码写入源码、脚本输出或 Git。
