## 📋 项目快速理解

### 项目定位

**韭菜盒子 Studio** 是一个**本地优先的纯手动 AI 工作台桌面应用**，重点是给用户完全的控制权。不同于通用 AI Agent，这个应用严格遵循用户显式控制原则：

- **用户选择 Skill**（谁来做） → 官方 Skill 或自定义 Skill
- **用户选择 Knowledge**（参考什么） → 本地知识库
- **用户选择 Tools**（动手做什么） → 本地工具或云端工具
- **用户选择 Model**（用哪个模型） → Claude/GPT/Gemini/Grok 等

LLM 只按这套显式配置执行，不能自动决策。

### 核心能力

| 能力 | 说明 |
|------|------|
| 💬 多模型对话 | Claude/GPT/Gemini 通过 NewAPI 中转 |
| 🎯 Skill 系统 | 官方预设 Skill + 用户自定义 Skill |
| 📚 知识库 | 本地 Wiki 知识库，仅接受用户手动添加 |
| 🎨 画布节点系统 | 31 个节点类型的工作流编辑器 |
| 📷 创作面板 | 图片/视频/音频/文档生成（13 个模型） |
| 🔧 本地工具 | 浏览器、文件处理、项目操作 |
| 🖥️ OpenCode 集成 | 完整复刻官方 OpenCode 的项目目录能力 |

## 🎯 核心模块详解

### 1. 对话引擎（useChat.ts）

**位置**: `src/composables/useChat.ts` (2227 行)

**流程**：
```
sendMessage(text)
  ├─ 检测贴图 → 非 vision 模型触发图片桥接（imageBridge.ts）
  ├─ buildApiMessages() 组装消息（按 102K token 预算）
  ├─ recallKnowledge() 知识库注入
  ├─ jinaWebSearch() 可选网络搜索
  ├─ POST /v1/chat/completions (stream=true)
  ├─ readSSEStream() 解析 SSE delta
  └─ runToolLoop() 工具循环（最多 10 轮）
       ├─ 执行工具调用
       ├─ 收集结果
       └─ 推回消息继续循环
```

**关键特性**：
- Token 水位计：`≈2.4K / 200K ▓░░░ 1.2%`
- 消息编辑 + 重新生成
- 搜索互斥：Jina API 搜索 ON → 隐藏浏览器工具；OFF → 暴露浏览器工具
- 知识库注入：top 4 wiki 页 + 钉选知识 + CLAUDE.md
- 禁用自动沉淀：`ingestAssistantOutput()` 已移除

### 2. Skill 系统（agentStore.ts）

**位置**: `src/stores/agentStore.ts` (1151 行)

**特点**：
- 30+ 官方预设 Skill（来自 anthropics/skills 等）
- 用户自定义 Skill（SKILL.md 格式）
- Skill 进化：从对话历史 → 生成 diff → 用户确认
- 官方 Skill 锁定（只读）；用户 Skill 可编辑
- SkillConfig 存储格式支持版本控制 + 来源追踪

### 3. 知识库系统（vaultStore.ts）

**位置**: `src/stores/vaultStore.ts`

**三层结构**：
```
Vault/
├── CLAUDE.md              # 配置（编译规则、主题、分类）
├── raw/                   # 原始素材（用户上传）
│   ├── topic1/
│   └── topic2/
└── wiki/                  # 编译后的知识页
    ├── index.md           # 索引
    ├── log.md             # 变更日志
    └── pages/
```

**原则**：
- 🚫 **AI 不能自动写入知识库** — 防止幻觉污染
- ✅ **仅接受用户手动添加** — 3 条路径（FileTree、VaultWizard、创建时）
- 📖 **知识召回只读** — recallKnowledge() 搜索 wiki/ + 钉选
- 🔧 **用户手动提炼** — 右键「提炼」触发 LLM 分析

### 4. 画布节点系统（canvas）

**位置**: `src/components/canvas/`

**31 个节点类型**：
- **生成**: text, llm, imageGen, videoGen, audioGen, seedance (6)
- **RunningHub**: runninghub, runninghubWallet, rhTools, rhConfig (4)
- **素材**: upload, output, materialSet (3)
- **流程控制**: loop, pickFromSet, textSplit, framePair (4)
- **图像处理**: resize, combine, removeBg, upscale, gridCrop, imageCompare, drawingBoard (7)
- **工具箱**: cinematic, videoMotion, multiAngleVisual (3)
- **辅助**: idea, bp, relay, edit, videoOutput (5)
- **结果**: imageResult, videoResult, audioResult (3)
- **其他**: file, tool, group (3)

**执行流程**：
```
用户点击 ▶ → jc-canvas-run-node 事件
  → canvasExecutor.runCanvasNode(nodeId)
    → 拓扑排序 + 依赖检查
    → 分发到 runtime:
        ├─ canvasLlmRuntime (LLM 节点)
        ├─ canvasMediaRuntime (图片/视频/音频/RunningHub)
        └─ canvasToolRuntime (本地工具)
    → 更新节点状态 (idle→running→success/error)
    → 刷新文件列表
```

### 5. 存储系统（idb.ts）

**位置**: `src/utils/idb.ts` (37KB)

**单后端设计**：
- **数据库**: SQLite at `~/.jiucaihezi/data/jiucaihezi.db`
- **内存缓存**: 启动时全量加载，读写即时同步
- **4 个表**: kv_store, conversations, messages, documents
- **浏览器降级**: `pnpm dev` 时用 localStorage + 内存模拟

**API**：
```ts
getItem(key) / setItem(key, value)           // kv_store
getRecord(store, id) / setRecord(store, data) // 结构化
getAll(store) / getAllByIndex(...)           // 查询
runStorageBatch(() => { ... })               // 事务
```

### 6. 本地工具运行层

| 工具域 | 文件 | 功能 |
|--------|------|------|
| **格式转换** | `localContentTools.ts` | Markdown/媒体元信息/ffmpeg/Whisper |
| **可见浏览器** | `browserTools.ts` | Chrome 搜索/打开/读取/截图/点击 |
| **源码项目** | `devProjectTools.ts` | 文件读写/搜索/git diff/命令执行 |
| **本地模型** | `localMlxRuntime.ts` | MLX 模型管理 |

**执行模式**：工具调用 → `useChat.ts` tool loop → 分发到对应 runtime

## 🔄 HTTP 通信模式

### Fetch 劫持 (patchFetch)

```
Frontend fetch() 调用
  ↓
patchFetch() 全局劫持
  ├─ body.stream:true
  │   → Tauri http_request_stream
  │   → Rust reqwest.bytes_stream()
  │   → JS ReadableStream
  │   → SSE 解析正常工作
  │
  ├─ 外部 HTTPS/HTTP
  │   → Tauri http_request
  │   → Rust reqwest
  │   → 一次性返回
  │
  └─ 内部 localhost
      → 原生 fetch
```

**修复历史**：API 请求 "Load failed" 已通过 Rust HTTP 桥接完全绕过 WKWebView fetch 限制。
