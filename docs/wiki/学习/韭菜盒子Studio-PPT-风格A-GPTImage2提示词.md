# 韭菜盒子 Studio

## 创作 Agent 产品介绍

**封面金句**  
创作的人，终于不用在「浪费」和「失忆」之间二选一。  
也终于不用为每一个模型，再下载一套产品。

---

## 01｜先把今天的创作现实说清楚

现在几乎所有内容团队都在用 AI。  
问题不在「用没用」，而在**怎么用**。

现实里，创作者同时卡在两道死结上。

### 死结一：能力路径二选一，都很疼

**路 A：用编程 Agent 硬扛创作**  
Codex、Claude Code 一类工具，能力很强。  
但它们天生为代码仓库设计：

- 为了「稳」，大量读写、反复确认、上下文堆叠  
- Token 烧得快  
- 换到剧本、角色、分镜、世界观时，路径笨拙  
- 细节记得住工程结构，却很难像主创一样记住「这部戏的人设真相」

**强，但贵；能跑，但不像创作者。**

**路 B：直接跟大模型聊**  
GPT、Claude Opus、Gemini、DeepSeek……打开就写，确实省。  
但省下来的代价是：

- 聊完就淡  
- 角色前后漂移  
- 设定散落在聊天记录里  
- 上一集定过的东西，下一集要重新喂  
- 项目越长，越像在跟一个聪明但健忘的助理反复对齐

**快，但飘；省，但失忆。**

> **要么笨拙地贵，要么轻巧地忘。**

### 死结二：模型越强，工具越碎

今天想用 Grok，得进 Grok 的产品。  
想用 GPT，得进 OpenAI 体系：ChatGPT、Codex……  
想用 Claude，得进 Anthropic 那一套。  
想用国内模型，再换另一批 App、网页、账号、额度、工作流。

结果是：

- 账号一堆  
- 界面一堆  
- 历史一堆  
- 资产一堆  
- 同一个角色，在 A 工具画完，在 B 工具接着写，在 C 工具再生成视频——**上下文全断**

模型没有变少，创作却被切碎了。

所以今天真正的痛点不是「AI 不够聪明」。  
是：

> **能力分散在无数产品里；  
> 记忆消散在无数对话框里；  
> 生产线断裂在无数切换里。**

---

## 02｜韭菜盒子解决的，就是这组死结

**韭菜盒子 Studio**  
面向内容创作的 AI 工作台。  
**桌面 APP（macOS / Windows）+ Web 端**。

它不是又一个聊天框，也不是把编程 Agent 换皮成创作界面。  
它同时做三件关键的事：

1. **一个账号，打通中外主流模型** —— 不再为每个模型单独换产品  
2. **第二大脑，记住项目细节** —— 不再在「省 token」和「能记住」之间妥协  
3. **完整流水线，从小说到影像** —— 不再把写作、设定、生图、生视频拆成孤岛

一句话：

> **用户主导，AI 执行。  
> 模型负责聪明，系统负责统一入口、长期记忆、全链路落地。**

---

## 03｜卖点一：一个账号，中外模型全开

这是最直观、也最能立刻被感受到的能力。

### 以前怎么用模型

| 你想用的能力 | 你通常要做的事 |
| --- | --- |
| Grok | 下载 / 打开 Grok 相关产品 |
| GPT | 进入 OpenAI 产品体系（ChatGPT、Codex 等） |
| Claude / Opus | 进入对应官方产品 |
| Gemini | 再换一套 Google 侧产品 |
| 国内模型 | 再开一批国内 App / 网页 / 账号 |

每一次切换，都不只是换一个 logo。  
是在换：

- 账号体系  
- 交互习惯  
- 历史记录  
- 文件位置  
- 项目上下文

### 韭菜盒子怎么做

**一个账号登录，即可调用中外主流模型。**  
写、画、拍、改，都在同一工作台完成。  
不用为了换模型，再换一整套产品。

对创作者意味着：

- **选型自由**：哪个模型适合剧本就用哪个，适合生图就用哪个  
- **路径统一**：同一项目里切换模型，不离开工作流  
- **资产不断**：结果回到同一项目，而不是散落在十个 App  
- **团队友好**：不要求每个人装齐全网工具，只要求进入一个系统

> **模型可以很多。  
> 创作入口，只需要一个。**

---

## 04｜核心引擎：创模式

创模式是韭菜盒子真正不同于「普通模型聊天」和「编程 Agent」的地方。

它不是套壳问答，而是一条创作运行时：

1. **直连大模型决策** —— 用最强模型的判断力，不绕远路  
2. **按需加载专业 Skill** —— 编剧、小说、角色、场景、分镜、媒体提示词……需要哪个上哪个  
3. **读写当前项目** —— 结果落进文件，不是飘在对话框  
4. **生成图片 / 视频 / 音频** —— 媒体任务进同一项目链路  
5. **自动沉淀事实** —— 说过什么、改过什么、生成过什么，可追溯

所以它既不像编程 Agent 那样为了「工程闭环」把流程做笨；  
也不像裸聊模型那样写完就散。

> **创模式 = 聪明的执行层 + 专业的能力包 + 可沉淀的项目手脚。**

---

## 05｜真正的破局点：第二大脑

第二大脑，是韭菜盒子的命门能力。  
不是 slogan，是工程。

### 别人为什么会忘？

- 聊天 AI：上下文有限，项目真相不在文件里  
- 编程 Agent：记得住仓库，不天然理解「角色圣经 / 场景资产 / 剧情连续性」  
- 多产品切换：上下文在工具之间彻底断裂  
- 网盘 + 微信 + 文档：人记得住入口，系统接不上

### 韭菜盒子怎么记？

| 层 | 做什么 | 价值 |
| --- | --- | --- |
| **项目 Wiki** | 角色 / 场景 / 道具 / 世界 / 剧情结构化沉淀 | 项目有单一真相源 |
| **热记忆 hot** | 当前最关键事实优先注入模型 | 每次开聊都先对齐主线 |
| **事实账本 .raw** | 用户意图、AI 动作、文件变更可追溯 | 不是“好像说过”，而是有据可查 |
| **巡检与修正** | 查穿帮、漏改、断链、设定冲突 | 长项目不靠人肉回忆 |
| **对话转 Wiki** | 把过程沉淀成可复用知识 | 聊过的不会白聊 |

结果是：

> **不笨拙：不必用编程 Agent 的重路径硬扛创作；  
> 不健忘：关键细节写进项目，而不是赌模型上下文；  
> 不碎片：模型切换发生在系统内，记忆仍在项目里。**

对影视与内容团队，这意味着：  
一部戏最贵的不是生成一张图，  
是**角色不飘、世界观不散、多集不断、资产可复用。**

## 06｜完整流水线：从文字到影像

大多数 AI 工具停在单点：  
会写，或会画，或会动一下。

韭菜盒子复现的是内容工业真正需要的链路：

```
灵感 / IP
   ↓
小说（长线叙事）
   ↓
剧本（分集、成稿、爽点、钩子）
   ↓
工程化素材（角色、场景、道具、分镜）
   ↓
生图（设定图、空镜、海报、物料）
   ↓
生视频（镜头、动态、成片素材）
   ↓
第二大脑持续记忆与巡检
```

**不是五个散装功能。**  
是一条能从头跑到尾、并且中途不丢记忆的生产线。  
再叠加「一个账号调用中外模型」，整条链路终于不用在十几个产品之间跳。

---

## 07｜五个必须被看见的能力

### ① 写小说：长线叙事可续写

- 灵感到成稿全流程  
- 核心梗、角色网、大纲、章纲、正文、复盘  
- 进入 Wiki，可续写、可巡检、可修正

小说不是一次聊完就散。  
它变成**可生长的 IP 资产。**

---

### ② 写剧本：从梗概到可拍摄成稿

- 短剧 / 漫剧 / 爽剧专业流  
- 世界观 + 极致角色 + 催化剂  
- 分集大纲 → 逐集成稿  
- 反转、卡点、钩子、临摹换皮 / 从零涌现

不是「帮你写个剧情」。  
是按内容生产逻辑组织的**剧本生产线。**

---

### ③ 生图：出的是资产，不是灵感截图

- 文生图 / 图生图  
- 多模型可选（同一账号内切换）  
- 角色设定、场景空镜、道具多视图、海报物料  
- 结果回写项目，可进画布继续用

对团队真正有价值的是：  
**可复用、可对齐、可进入下一环节。**

---

### ④ 生视频：嵌在项目流里，不是孤立按钮

- 多引擎视频生成  
- 图生视频 / 镜头驱动  
- 与分镜、角色、场景提示词衔接

重点不是「能动一下」，  
而是镜头结果仍然活在同一项目记忆里，  
后面还能继续改、继续接、继续生产。

---

### ⑤ 第二大脑：让整条流水线不崩

没有第二大脑，上面四项都会在第二周开始变形。  
有了第二大脑：

- 角色脸和性格不漂  
- 场景规则不乱  
- 前后集不断档  
- 团队换人也能接上

> **第二大脑不是附加功能。  
> 它是整条创作流水线的稳定器。**

## 08｜APP + Web：同一套创作系统

| 端 | 形态 | 价值 |
| --- | --- | --- |
| **桌面 APP** | macOS / Windows | 全能力工作台：创模式、项目文件、Skill、媒体生成、画布、深度创作 |
| **Web 端** | 浏览器即开 | 轻量进入创模式，降低启动成本，随时继续创作 |

双端共享同一产品哲学：

- 同一个项目思维  
- 同一套 Skill 能力  
- 同一条媒体任务链路  
- 同一个账号下的模型能力  
- 同一种「用户主导、AI 执行」

**不是两个产品，是一个创作系统的两种入口。**

---

## 09｜为什么这不是套壳

| 维度 | 裸聊大模型 | 编程 Agent | 各家官方单点产品 | **韭菜盒子 Studio** |
| --- | --- | --- | --- | --- |
| 创作门槛 | 低 | 高 | 中 | **低（说人话）** |
| Token / 路径成本 | 相对省 | 常偏贵、偏重 | 单点计费 | **创作路径更直接** |
| 项目记忆 | 差 | 偏工程记忆 | 基本不跨产品记 | **Wiki + hot + 事实账本** |
| 模型入口 | 一家一个 | 偏开发场景 | 一家一个 App/网页 | **一个账号，中外模型统一调用** |
| 小说 / 剧本专业流 | 泛写 | 无 | 弱 / 无 | **专业 Skill 生产线** |
| 图视频是否回项目 | 通常否 | 无 | 下载即断 | **资产回写 + 画布** |
| 全链路闭环 | 无 | 无 | 无 | **小说→剧本→制作完整流水线** |

最关键的三句：

> **别人给你的是模型入口。**  
> **韭菜盒子给你的是统一入口 + 长期记忆 + 创作操作系统。**  
> **模型再多，也不该逼创作者活在切换里。**

---

## 10｜对内容团队意味着什么

1. **不再在「贵且笨」和「省且忘」之间妥协**

既用得上强模型，又留得住项目细节。

1. **不再为每个模型单独养一套工具**

一个账号覆盖中外主流模型，选型在系统内完成。

1. **从灵感工具升级为生产能力**

小说、剧本、设定、画面、视频在同一系统内推进。

1. **长项目第一次真正适合 AI**

多集、多角色、多场景、多版本，靠第二大脑稳住。

1. **资产可复用，而不是聊天记录可翻找**

项目文件就是真相，知识可以沉淀、巡检、迭代。

---

## 11｜收束页

**聪明，从来不是创作的全部。**  
**统一入口，才配谈效率。**  
**记得住，才配谈生产。**

韭菜盒子 Studio

- **一个账号** —— 中外模型统一调用  
- **创模式** —— 负责执行  
- **Skill** —— 负责专业  
- **第二大脑** —— 负责不遗忘  
- **流水线** —— 负责从文字走到影像

**打开就能用。**  
**用了就记得住。**  
**切换模型，不用切换人生。**  
**记得住，才能持续生产。**
