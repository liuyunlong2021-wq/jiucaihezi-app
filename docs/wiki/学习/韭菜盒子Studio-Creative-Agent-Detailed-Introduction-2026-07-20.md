## 1. What Jiucaihezi Studio Is

**Jiucaihezi Studio is an AI creative workspace that brings international and domestic AI models, creative Agents, project memory, image/video/audio generation, and downstream production tools into one connected environment.**

It is not merely a chatbot, and it is not limited to generating one image or one piece of text. Its purpose is to help users move an idea forward until it becomes a usable creative deliverable.

Within one account and one project, users can:

- discuss ideas, organize requirements, and create plans;
- write novels, screenplays, episode outlines, and production-ready drafts;
- build character, location, prop, and world-setting materials;
- generate images, videos, audio, and visual assets;
- review, revise, align, and organize assets on a canvas;
- combine images, video, audio, subtitles, and other materials into a more complete work;
- preserve important project information for future search and reuse.

In simple terms, Jiucaihezi Studio helps users avoid moving between many models, accounts, applications, and disconnected chat histories while carrying out one creative project.

## 2. The Problems It Solves

### 2.1 Powerful models are scattered across separate products

Today, each model often comes with its own product, account, interface, subscription, history, files, and workflow. A character may be designed in one tool, written into a script in another, and turned into video in a third. Project context is easily lost between steps.

**Jiucaihezi Studio provides a unified model entry point, so model selection happens inside the same creative project.**

### 2.2 Using multiple AI services can be difficult for everyday users

Different AI products may require different registration methods, network conditions, payment methods, quotas, and operating habits. Users without a technical background can spend more time configuring tools than creating.

Jiucaihezi Studio aims to reduce this barrier through one account and one workspace. Available models, services, pricing, and access conditions depend on the current product page and the user’s account.

### 2.3 Coding Agents are powerful, but are not designed primarily for creative production

Coding Agents are usually centered on code repositories and engineering files. They are excellent for software development, but creative work needs a workflow built around stories, characters, scenes, shots, images, video, and audio.

Jiucaihezi Studio brings Agent-style execution into creative production rather than treating every task as a programming task.

### 2.4 Direct chat is fast, but long projects easily lose context

A normal chat is useful for quick questions, but long projects can suffer from changing character details, forgotten world rules, scattered references, and assets that do not connect naturally to the next production step.

**Jiucaihezi Studio is designed to balance speed, continued execution, and preservation of project facts.**

## 3. Product Positioning and Working Logic

Jiucaihezi Studio is not a standalone chatbot, a standalone image generator, or a coding Agent with a different interface. It is an AI production system for creative work, composed of:

1. **Unified model access:** multiple models inside one account and project;
2. **Creative Mode:** a controlled execution layer that can use project files, Skills, and tools;
3. **Professional Skills:** structured workflows for novels, scripts, characters, scenes, props, storyboards, and media prompts;
4. **Project knowledge base and Second Brain:** reusable, searchable project facts and settings;
5. **Creation Panel:** a place to submit image, video, audio, and other media tasks;
6. **Canvas:** a workspace for reviewing, comparing, editing, and arranging generated results;
7. **Project files and production tools:** a bridge from text and generated assets to further production.

The basic logic is:

> **Users define goals, make judgments, and choose among options. AI understands, executes, generates, and organizes. The system provides the unified entry point, preserves the project, and connects each stage.**

## 5. Creative Mode: An Agent That Executes Creative Work

### 5.1 What Creative Mode is

Creative Mode is Jiucaihezi Studio’s core working mode. Users can describe a goal in natural language, for example:

- I want to make a particular kind of novel or short drama;
- turn this script into character, scene, and prop materials;
- convert this scene into storyboard prompts for image or video generation;
- read the project settings and check this episode for contradictions;
- organize the generated assets and continue the production process.

Within the project scope, Creative Mode can read relevant materials, call Skills, use tools, write results into project files, or submit tasks to the Creation Panel.

### 5.2 How it differs from ordinary chat

Ordinary chat mainly answers one question at a time. Creative Mode focuses on moving a project forward. Instead of leaving everything in the conversation window, it can turn long-term results into project documents, knowledge-base entries, or media assets.

### 5.3 How it differs from coding Agents

A coding Agent centers on code and an engineering repository. Creative Mode centers on a creative project and its production workflow. It is as direct as chat, as executable as an Agent, and designed for creative work rather than only programming.

## 6. The Second Brain: Project Memory

Long projects need a stable source of truth: character identity and appearance, scene rules, prop details, completed story events, settings that must not change, unresolved plot threads, and the currently valid version.

Jiucaihezi Studio organizes project materials in a Wiki-like knowledge base, giving important information five useful properties: it can be searched, structured, updated, reused, and traced back to project facts where appropriate.

### The five-layer memory cycle

1. **Everything to Wiki:** organize conversations, documents, scripts, and notes into a maintainable structure;
2. **Raw to Wiki:** preserve confirmed source facts and evidence;
3. **Query:** retrieve information by keyword, character, location, plot, or project topic;
4. **Consistency check:** detect naming drift, setting conflicts, continuity errors, missed replacements, broken links, and unresolved setups;
5. **Correction:** write confirmed corrections back into the source files and knowledge base.

> **Collect → preserve → query → inspect → correct.**

The goal is not to make the model pretend to remember more. The goal is to give the project maintainable knowledge assets.

## 7. From Words to Images, Video, and Audio

```text
Idea / IP → Story or novel → Script and episode drafts → Character / scene / prop assets
→ Storyboards and shot design → Image generation → Video and audio generation
→ Canvas refinement → Subtitles, editing, and final delivery
```

### Step 1: Clarify the creative plan

Use Creative Mode and professional Skills to define visual direction, characters, locations, props, story beats, camera instructions, prompts, and reference materials before generating media.

### Step 2: Submit a media task

In the Creation Panel, select an image, video, audio, or other task; choose a model; enter the prompt; attach reference images, start frames, end frames, or audio when needed; check ratio and duration; then submit.

### Step 3: Refine and organize on the canvas

Generated results are usually drafts. Users can compare outputs, repair details, adjust composition, organize multiple assets, and reuse selected results as references for later generation.

### Step 4: Assemble and export

Return to Creative Mode to combine images, video, and audio, add subtitles or titles, join clips, convert formats, and prepare a deliverable. The exact tool range depends on the current version and device environment.

> **Prompt → generate → refine → assemble.**

## 10. Detailed Getting-Started Guide

### Step 1: Open the website or App

Visit **https://jiucaihezi.studio/** in a browser, or open the installed Jiucaihezi Studio App.

### Step 2: Register or sign in

Follow the on-screen instructions. After signing in, enter your workspace and projects. If available, use Settings to manage account details, model options, and App downloads.

### Step 3: Create a project

Give each project a clear goal, such as *Short Drama Project*, *Brand Film*, *E-commerce Product Images*, or *Personal Content Series*. The project is the shared scope for files, conversations, knowledge, and media assets.

### Step 4: Describe the goal before asking for output

Explain what you are making, who it is for, the desired final result, available source materials, style/size/duration/platform requirements, and what must remain fixed.

Example:

> I want to make a 60-second brand introduction video for customers who are hearing about Jiucaihezi Studio for the first time. Please first organize the product positioning, audience, video structure, and shot requirements. Do not generate the video yet.

### Step 5: Choose a mode and model

- **Chat Mode:** quick discussion and questions;
- **Coding Mode:** software development and code projects;
- **Creative Mode:** novels, scripts, characters, scenes, prompts, media, and long-term creative projects.

For content and visual production, choose Creative Mode. Select a model based on the task. You do not need to study every model before starting.

### Step 6: Use Skills for specialized work

Ask directly for tasks such as creating a character prompt, breaking a script into shots, building scene and prop references, reverse-engineering an image prompt, checking continuity, or organizing a conversation into the project Wiki.

### Step 7: Save confirmed results

When a character, scene, plot point, style, or production plan is confirmed, ask the Agent to save it to the project, create or update the relevant material, preserve the previous version if needed, and record the changes.

### Step 8: Generate media

In the Creation Panel, select the task type and model, enter the prompt, add references, check ratio, duration, price, and account balance, then submit. Media generation may consume credits or incur charges.

### Step 9: Review and refine

Check character and setting consistency, composition, ratio, unwanted elements, and delivery quality. Decide whether to repair, extend, regenerate, or replace the result. Use the canvas for continued editing and comparison.

### Step 10: Assemble and export

Ask the system to arrange clips by shot order, add dialogue subtitles, create a vertical preview, or convert formats. Review and confirm the scope when files, commands, or media processing are involved.

### Step 11: Maintain project memory

For long projects, periodically query project materials, preserve new facts, inspect names and timelines, correct source files, organize versions, and make backups.

## 11. A Simple First Project Example

Suppose you want to make a 30-second short-drama trailer:

1. **State the goal:** an urban revenge story for short-video viewers, with strong conflict, fast pacing, vertical video, and subtitles.
2. **Ask for a plan:** have Creative Mode provide the selling points, timing, shot list, dialogue, and visual prompts before generating media.
3. **Confirm and save:** once the relationships and structure are approved, save the character, shot, and style requirements to the project.
4. **Generate:** create first-frame images from the shot list, then turn selected first frames into video.
5. **Refine and deliver:** correct inconsistent shots, assemble the clips, add subtitles, and export a 9:16 preview.

The basic method is:

> **State the goal → plan → save the settings → generate assets → refine → assemble and deliver.**
