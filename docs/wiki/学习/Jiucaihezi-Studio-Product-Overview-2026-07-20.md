## 1. What Is Jiucaihezi Studio?

Jiucaihezi Studio is an AI-powered creative workspace designed to help people move from ideas to finished creative work.

It brings models, creative workflows, project memory, media generation, editing, and production tools into one system.

The product is built around two core promises:

> **One account. Every model.**  
> **A Second Brain that never forgets.**

This is not simply another chatbot, image generator, or coding agent. It is a **Creative Agent** designed for continuous, structured creative production.

Users can work with leading models from China and around the world without constantly switching between separate products, accounts, interfaces, and histories.

---

## 2. Why Existing AI Workflows Break Down

AI has become part of everyday creative work, but the current experience is still fragmented.

### One model usually means one product

Want to use Grok? Open Grok.  
Want GPT? Go to an OpenAI product.  
Want Claude? Switch to another platform.  
Want a China-based model? Open another app, website, or account.

Every switch can break the workflow:

- Different accounts
- Different interfaces
- Different billing systems
- Different histories
- Disconnected files and assets
- No shared project memory

The models keep getting better, but the creative process keeps getting split apart.

### Accessing overseas models can also be difficult

For users in mainland China, accessing overseas AI products may require:

- A VPN or other network setup
- An overseas email address
- Identity verification
- An international payment method
- Ongoing concern about account restrictions or suspension

The setup is time-consuming, uncertain, and difficult for ordinary users.

### The two common AI paths both have weaknesses

#### Coding Agents

Coding agents are powerful, but they are built around software repositories and programming workflows.

They often:

- Use a heavy read-write-confirmation process
- Consume large amounts of tokens
- Focus on engineering structure rather than creative structure
- Forget creative details even while preserving parts of the code workflow

They are powerful, but heavy, expensive, programming-oriented, and still forgetful.

#### Direct Model Chat

Directly chatting with a model is fast and accessible, but it often lacks the structure needed for a long creative project.

It can:

- Drift away from the intended direction
- Lose character, world-building, and continuity details
- Provide weak support for Skills, MCP, and plugins
- Leave the user responsible for manually organizing the project memory

It is fast, but loose, limited, and still forgetful.

> **The current choice is often between a heavy system that forgets and a lightweight system that also forgets.**

Jiucaihezi Studio is designed to remove that tradeoff.

## 4. Creative Mode: Fast Like Chat, Structured Like an Agent

Creative Mode is the core operating mode of Jiucaihezi Studio.

It is not a coding agent with a different visual interface. It is also not a basic chat wrapper.

Creative Mode combines the speed of direct model access with the structure, permissions, and tool capabilities of an agent.

It can:

- Read and write to the current project
- Use built-in Skills
- Call MCP tools
- Use plugins and other production tools
- Organize creative materials inside the project
- Generate images, video, and audio through the connected workflow

This makes the system both responsive and controlled:

- The model can respond directly instead of following a heavy programming path
- The project can still be read, updated, and organized through defined tools
- Creative Skills can guide the work instead of leaving every decision to an unconstrained conversation

> **Fast, but not uncontrolled.  Powerful, but not clumsy.**

## 5. The Second Brain: Memory That Can Be Looked Up

The idea behind the Second Brain is simple:

> **Let the model look up project memory the way it looks up Wikipedia.**

This concept was strongly discussed in the AI community through the work and public writing of **Andrej Karpathy**, an OpenAI co-founder. The idea quickly attracted widespread attention, rapid GitHub growth, and broad discussion about how AI should manage long-term memory.

The key insight is that memory should not depend only on a model's temporary context window.

Project memory should be:

- Searchable
- Structured
- Updateable
- Reusable
- Traceable

Jiucaihezi Studio combines this idea with the knowledge-base approach popularized by tools such as Obsidian to build an AI-Wikipedia-style memory system for creative projects.

The goal is straightforward: the AI should not merely sound as if it remembers. It should be able to retrieve the project's actual source of truth.

---

## 6. Five Integrated Skills Behind the Second Brain

The Second Brain is supported by five integrated Skills that work together as one memory system.

### 1. Everything to Wiki

Turn conversations, documents, scripts, notes, and other material into structured project knowledge.

### 2. Raw to Wiki

Trace confirmed knowledge back to source facts and preserve the original evidence behind the project memory.

### 3. Query

Search and retrieve project memory as if consulting an encyclopedia.

### 4. Consistency Check

Detect continuity errors, missed changes, broken links, naming drift, and conflicts across the project.

### 5. Correction

Apply verified corrections to the source of truth so the project can continue from a clean, consistent foundation.

Together, these five Skills create a complete memory loop:

> **Capture. Preserve. Retrieve. Check. Correct.**

The result is designed for long-form work. Even in a one-million-word project, the AI can still answer a detail such as:

> What weapon was a minor character carrying in Chapter 100?

That is the difference between a model that merely remembers a conversation and a system that maintains project knowledge.

## 7. From Story to Finished Film

Jiucaihezi Studio is designed to connect the major stages of AI-assisted creative production.

A long-form project can move through a connected path:

```text
Idea / IP
  → Novel and long-form story development
  → Script and episode development
  → Characters, scenes, props, and storyboards
  → Image generation
  → Video and audio generation
  → Canvas refinement
  → Final assembly and production
```

The current core workflow is organized into four connected steps.

### Step 1 — Creative Mode + Skills

Use Creative Mode and built-in Skills to develop the text and production instructions:

- Lock the visual style
- Build character prompts
- Build scene prompts
- Build prop prompts
- Build storyboard and shot prompts
- Convert creative decisions into precise, usable generation prompts

### Step 2 — Creation Panel

Move the prompts into the Creation Panel and select the appropriate model from the unified model system.

Generate:

- Images
- Video
- Audio

The prompt does not need to be copied into another product. The model does not require a separate account or a separate production history.

### Step 3 — Canvas Refinement

Use the Canvas to refine the generated material:

- AI image editing
- Detail repair
- Local changes
- Extension and adjustment
- Visual alignment

Generation is the starting point. Refinement turns a generated result into usable production material.

### Step 4 — Creative Mode + Tools

Bring the refined assets back into the project and use Creative Mode with production tools to complete the work.

The toolchain can support tasks such as:

- Combining image, video, and audio assets
- Adding subtitles and other production elements
- Using tools such as FFmpeg to assemble clips
- Building a complete video or film sequence

> **Prompt. Generate. Refine. Assemble.**  
> **One connected creative production loop.**

## 10. How to Get Started

### Step 1 — Open the product

Go to:

[https://jiucaihezi.studio/](https://jiucaihezi.studio/)

You can begin through the Web version.

### Step 2 — Register and sign in

Open **Settings** to create an account or sign in.

### Step 3 — Choose a model

After signing in, use the model selector in the upper-right corner of the conversation panel.

Choose the model that best fits the task and start working.

### Step 4 — Choose a mode

Jiucaihezi Studio currently provides three main modes:

- **Chat Mode** — discuss ideas and ask questions without directly operating the computer
- **Code Mode** — work with programming tasks and software projects
- **Creative Mode** — create and manage scripts, stories, prompts, media, and creative projects

### Step 5 — Open the creative tools

- Click **Creation Panel** on the left to generate images, video, and audio
- Open **Editor** to store and organize scripts, stories, and other project materials
- Use the Canvas to refine visual assets
- Return to Creative Mode to continue the production workflow

### Step 6 — Download the desktop app if needed

Open **Settings** and use the **Download App** option in the upper-right corner.

The basic workflow is simple:

> **Sign in. Choose a model. Choose a mode. Start creating.**
