## 06｜痛点三：两条路，都废

今天大家用 AI 创作，只剩两条路：

### 路 A｜编程 Agent

- 浪费 Token  
- 也会忘  
- 全套设计偏编程，不贴创作

**又贵，又重，还记不住你的戏。**

### 路 B｜直接聊大模型

- 对模型几乎没约束  
- 容易跑偏  
- 不能稳用 Skill / MCP / 插件  
- 能力砍一半，照样忘

**又飘，又弱，还是记不住。**

**两条路：一条笨且忘，一条飘且忘。**

---

## 07｜解法：创模式

创模式，不是编程 Agent 换皮，也不是裸聊套壳。

它同时拿走两边的优点：

- **像直连模型一样快**  
- **像编程 Agent 一样有约束、有工具**

创模式能：

- 读写当前项目  
- 调用 Skill  
- 调用 MCP  
- 使用插件与工具  
- 直接驱动生图 / 生视频 / 生音频

**快，但不野。强，但不笨。**

## 08｜第二大脑·源头

第二大脑的源头，来自 OpenAI 联合创始人 **Andrej Karpathy（卡帕西）** 的公开讨论与方案。

核心观点一句话：

> **让模型像查维基百科一样，查询你的记忆。**

文章与方案一经放出：

- GitHub 瞬间爆火  
- 星标暴涨  
- 引发全民讨论

**记忆不该赌上下文。记忆该像百科，可查、可更新、可复用。**

---

## 09｜第二大脑·五层 Skill

我们把卡帕西的思路，加上 Obsidian 知识库方法，做成了：

**AI 维基百科式项目记忆。**

后端是独创 **五层 Skill，五位一体**：

1. **Everything 转 Wiki** —— 任何内容，先沉淀进库  
2. **Raw 转 Wiki** —— 原始事实，压缩成可复用知识  
3. **查询** —— 像查百科一样取回记忆  
4. **一致性巡检** —— 查穿帮、漏改、断链、冲突  
5. **修正** —— 发现问题，直接改回真相源

达到的效果：

> **就算你写到 100 万字，**  
> **AI 也能知道第 100 章里，一个龙套男人手里拿的是什么武器。**

**不笨拙。不健忘。细节真的留得住。**

## 10｜流水线总览：四步成片

AI 做影视，不该在十几个软件里跳。

韭菜盒子创作流水线，四步：

1. **创模式 + Skill** → 生成提示词  
2. **创作面板** → 生成图片 / 视频 / 音频  
3. **画布** → 精修  
4. **创模式 + 工具** → 拼装完整影片

**提示词、生成、精修、成片，一条链路走完。**

---

## 11｜第一步：创模式 + Skill，生成提示词

文本侧工作，创模式 + Skill 最强。

内置一整套精细 Skill，帮你把影片拆成可生成的提示词：

- 定风格  
- 写角色提示词  
- 写场景提示词  
- 写道具提示词  
- 写分镜提示词

不是泛泛聊天写两句。  
是按影视生产逻辑，把该写的提示词一次写全、写准。

**先把“怎么拍”写清楚，后面才生成得稳。**

---

## 12｜第二步：创作面板，生成资料

提示词写好，直接进创作面板。

在同一套系统里：

- 选中外模型  
- 生成图片  
- 生成视频  
- 生成音频

不用再复制提示词去另一个 App。  
不用再换账号、换界面、换历史记录。

**左边想清楚，右边直接出料。**

---

## 13｜第三步：画布精修

生成出来，不等于能直接用。

画布负责精修：

- 对细节图片做 AI P 图  
- 调、改、补、对齐  
- 把能用的留下，把不对的修准

媒体不是下完就散。  
继续留在项目里，随时复用。

**生成是起点，精修才进生产。**

---

## 14｜第四步：创模式 + 工具，拼装成片

精修好的图片、视频、音频，回到创模式。

用工具链继续往后走，例如：

- FFmpeg 等拼装能力  
- 把素材接成完整影片  
- 在同一项目里完成最后一公里

不是“素材生成器”。  
是能把结果真正拼起来的创作系统。

**从提示词，到素材，到精修，到成片。**  
**四步闭环。**

---

## 15｜下一步：贴近小白的表单式

核心能力已经有了：创模式 + 创作面板 + 画布 + 工具成片。

下一步，是把复杂能力做成小白也能直接填的表单：

- **电商模式**：已有，直观填写即可生成  
- **漫剧 / 小说 / 电影 / 电视剧**：后续按场景上线

用户不必先懂 Skill 原理。  
选场景，填表单，直接出结果。

**专业能力在底层。小白入口在表层。**
