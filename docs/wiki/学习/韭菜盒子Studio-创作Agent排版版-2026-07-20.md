## 1. 韭菜盒子 Studio 是什么

**韭菜盒子 Studio，是一个把中外 AI 模型、创作 Agent、项目记忆、图片/视频/音频生成，以及后续制作工具组织在一起的 AI 创作工作台。**

它不是只陪用户聊天，也不是只生成一张图片或一段文字，而是帮助用户把一个想法持续推进为可以使用的创作成果。

用户可以在同一个账号和同一个项目中完成：

- 讨论创意、整理需求和制定方案；
- 写小说、剧本、分集大纲和拍摄成稿；
- 创建角色、场景、道具和世界观资料；
- 生成图片、视频、音频和各种视觉物料；
- 使用画布继续修改、对齐和整理素材；
- 通过工具把图片、视频、音频、字幕等素材组合成更完整的作品；
- 将项目中的重要信息沉淀下来，供后续持续查询和复用。

简单说，韭菜盒子 Studio 解决的是：**让用户不用在很多模型、很多账号、很多软件和很多聊天记录之间来回切换，就能持续完成一项创作工作。**

## 2. 它主要解决什么问题

### 2.1 模型很多，但入口被切碎了

现在使用 AI，通常是一个模型对应一个产品：想用 Grok，就打开 Grok；想用 GPT，就进入 OpenAI 的产品；想用 Claude、Gemini 或国内模型，又要换到另一套应用、网页或账号。

每次切换，用户还要面对不同的账号体系、界面、订阅方式、历史记录、文件素材和项目背景。模型越来越强，创作过程却被切成许多互不相连的片段。

**韭菜盒子 Studio 提供统一的模型入口，让模型选择发生在同一个创作项目内部。**

### 2.2 普通用户使用不同模型的门槛较高

不同 AI 产品往往有不同的注册、网络、支付、额度和操作要求。没有技术背景的用户，容易把时间消耗在配置工具上，而不是实际创作上。

韭菜盒子 Studio 的方向，是通过统一账号和统一工作台，降低用户接触不同模型的使用门槛。具体可用模型、服务范围和使用条件，以产品当前页面和实际账户状态为准。

### 2.3 编程 Agent 强大，但不完全适合创作

编程 Agent 通常围绕代码仓库和工程文件设计，适合软件开发；当任务变成剧本、角色、场景、分镜、长篇故事或媒体制作时，用户需要的是更贴近创作流程的工具。

韭菜盒子 Studio 借用 Agent 的执行能力，但把中心放在创作项目上：角色、场景、道具、剧情、分镜、图片、视频和音频都可以成为同一个项目里的材料。

### 2.4 直接聊天很快，但项目容易失忆

普通聊天适合快速问答，却容易出现角色前后变化、世界观被忘记、重要资料散落、素材无法衔接等问题。

**韭菜盒子 Studio 想解决的，就是“又快、又能持续推进、还能够保留项目事实”这一组矛盾。**

## 3. 产品定位与工作逻辑

韭菜盒子 Studio 不是单独的聊天机器人、单独的生图工具，也不是把编程 Agent 换一个界面。它是一套面向创作工作的 AI 生产系统，主要包括：

1. **统一模型入口**：在同一个账号和项目中调用不同模型；
2. **创模式**：让模型理解创作任务，并在受控范围内使用项目文件、Skill 和工具；
3. **Skill 专业能力包**：针对小说、剧本、角色、场景、道具、分镜和媒体提示词提供专业流程；
4. **项目知识库和第二大脑**：沉淀重要事实、设定和过程，支持查询和复用；
5. **创作面板**：提交图片、视频、音频等媒体生成任务；
6. **画布**：查看、比较、修改和整理生成结果；
7. **项目文件与制作工具**：让文字、图片、视频和音频继续进入后续制作。

基本逻辑是：

> **用户负责提出目标、做判断和做取舍；AI 负责理解、执行、生成和整理；系统负责统一入口、保存项目和连接各个环节。**

## 4. 核心能力：一个账号，统一调用中外模型

韭菜盒子 Studio 将不同类型的模型放在同一套创作工作台中。用户可以根据任务选择模型，而不必每次换模型就重新打开产品、登录账号或建立新流程。

### 语言模型

当前接入或规划接入的语言模型包括 Claude、GPT、Grok、Gemini、DeepSeek、GLM，以及其他国内外模型。

### 图片、视频和音频生成

创作面板可根据当前接入情况提供图片、视频、音频或 AI 应用类任务，例如 GPT Image 2、Banana Pro、Sora、Grok 相关媒体能力、CDA 及其他媒体引擎。

模型目录和具体名称会随接入情况、版本更新和账户权限变化。本文中的模型名称仅用于说明能力范围，不代表所有模型在任何时间都可用。

### 统一入口带来的价值

- **选型更自由**：写作、推理、出图、做视频可以选择不同模型；
- **项目不断裂**：切换模型仍在同一个项目中进行；
- **素材更集中**：生成结果可以回到当前项目继续使用；
- **团队更容易协作**：不必分别搭建一整套模型工具；
- **减少重复配置**：不用为每个模型单独学习一套产品。

> **模型可以很多，创作入口只需要一个。**

## 5. 核心能力：创模式——会执行的创作 Agent

### 5.1 创模式是什么

创模式是韭菜盒子 Studio 的核心工作模式。用户可以用自然语言告诉它想做什么，例如：

- 我想做一部什么样的小说或短剧；
- 请把这份剧本整理成角色、场景和道具资料；
- 请把这一场戏转成可用于生图或生视频的分镜提示词；
- 请读取项目设定，检查这一集有没有前后矛盾；
- 请把已经生成的素材整理并继续制作。

创模式会在项目范围内，根据任务需要读取资料、调用 Skill、使用工具，并将结果写入项目或提交到创作面板。

### 5.2 与普通聊天的区别

普通聊天主要回答一轮问题；创模式更关注“把项目继续往前推进”。普通聊天的结果可能停留在对话框里，创模式则会尽量把长期使用的内容变成项目文件、知识库资料或媒体资产。

### 5.3 与编程 Agent 的区别

编程 Agent 的中心是代码和工程仓库；创模式的中心是创作项目和生产流程。它像直接聊天一样快，像 Agent 一样能执行，但面向的是内容创作而不只是编程。

## 6. 核心能力：第二大脑——项目记忆不是靠模型碰巧记住

长篇项目需要稳定的事实来源：角色身份和外貌、场景规则、道具细节、已发生剧情、不可修改的设定、伏笔状态以及当前有效版本。这些内容不能只存在临时上下文里，也不能只靠用户翻聊天记录。

韭菜盒子 Studio 将项目资料组织成类似 Wiki 的知识库，让重要内容具备可查询、可结构化、可更新、可复用和可追溯的特点。

### 五层记忆循环

1. **Everything 转 Wiki**：把对话、文档、剧本和笔记整理成可维护结构；
2. **Raw 转 Wiki**：保留已确认的原始事实和证据；
3. **查询**：按关键词、人物、场景、剧情和主题查找资料；
4. **一致性巡检**：检查命名漂移、设定冲突、前后不一致、漏改、断链和伏笔未收；
5. **修正**：把确认后的修正结果写回源文件和知识库。

> **收集 → 沉淀 → 查询 → 检查 → 修正。**

它的价值不是让模型“假装记得更多”，而是让项目拥有可持续维护的知识资产。

## 7. 从文字到影像的创作流水线

```text
灵感 / IP → 小说或故事 → 剧本与分集成稿 → 角色/场景/道具资料
→ 分镜与镜头设计 → 图片生成 → 视频与音频生成 → 画布精修
→ 字幕、剪辑和成片制作
```

### 第一步：先把方案写清楚

使用创模式和专业 Skill 明确项目风格、角色、场景、道具、剧情节拍、镜头运动、生成提示词和所需参考素材，避免只提出模糊的“帮我生成一张图”。

### 第二步：提交媒体生成任务

在创作面板选择图片、视频、音频或其他任务类型，选择模型，填入提示词，按需要添加参考图、首帧、尾帧或声音，检查比例和时长后提交任务。

### 第三步：画布精修和整理

生成结果通常只是初稿。用户可以在画布中查看、比较、局部修复、调整构图、整理多个素材，并把选中的结果作为下一次生成的参考。

### 第四步：继续组装和输出

回到创模式，继续组合图片、视频和音频，添加字幕、片头片尾，拼接视频片段，转换格式并输出可交付文件。具体工具范围以当前版本和设备环境为准。

> **提示词 → 生成 → 精修 → 组装。**

## 8. 用户可以用它做什么

### 小说和长篇故事

从题材、核心梗、世界观、人物关系，到故事大纲、章节大纲、正文续写和一致性检查，逐步完成长期项目。

### 漫剧、短剧和影视剧本

构建故事、设计分集梗概和付费卡点，将梗概扩写为可拍摄正文，设计反转、爽点、钩子、动作链，并继续生成角色、场景、道具和分镜资料。

### 角色、场景和道具资产

建立角色控制表、场景空镜、道具多视图和关键标记，让视觉要求能够跨章节、跨镜头复用。

### 图片、视频和音频内容

制作故事配图、角色概念图、场景图、分镜图、短视频镜头、口播、对白、配音、海报、封面和电商视觉物料。

### 电商和场景化创作

通过更简单的表单和引导步骤整理商品图需求、参考图和交付目标。未来还可扩展到漫剧、小说、电影、电视剧、自媒体和其他业务场景。

> **专业能力放在底层，简单入口放在表层。**

## 10. 详细使用说明：第一次如何开始

### 第 1 步：打开网站或 App

在浏览器访问 **https://jiucaihezi.studio/**，或打开已安装的韭菜盒子 Studio。

### 第 2 步：注册或登录

按页面提示注册或登录。登录后进入自己的工作区和项目；如有设置入口，可在此管理账号、模型选项和 App 下载。

### 第 3 步：创建一个项目

建议一个项目对应一个明确目标，例如《某某短剧》《品牌宣传片》《电商商品图》或《个人自媒体栏目》。项目是文件、对话、知识库和媒体素材的共同范围。

### 第 4 步：先说明目标

第一次进入项目时，说明内容类型、受众、最终结果、已有资料、风格/尺寸/时长/平台要求，以及必须保留和可以发挥的部分。

示例：

> 我想做一条 60 秒的品牌介绍视频，面向第一次听说韭菜盒子 Studio 的客户。请先帮我整理产品定位、受众、视频结构和镜头需求，不要直接生成视频。

### 第 5 步：选择模式和模型

- **聊天模式**：快速讨论和问答；
- **编程模式**：软件开发和代码工程；
- **创模式**：小说、剧本、角色、场景、提示词、媒体和长期创作项目。

内容创作优先选择创模式，再按任务选择文本、图像、视频或音频模型。不必一开始研究所有模型，先用熟悉的模型完成任务即可。

### 第 6 步：调用 Skill 完成专业任务

可以直接说：“帮我设计角色设定图提示词”“把剧本拆成分镜”“建立场景和道具资料”“检查章节冲突”或“把对话整理进项目 Wiki”。系统会根据当前可用能力组织更完整的流程。

### 第 7 步：保存已经确认的结果

对于已经确定的角色、场景、剧情、风格或制作方案，明确告诉 AI 保存到项目、建立资料、更新 Wiki、保留旧版本并记录修改内容，不要只留在聊天记录里。

### 第 8 步：生成媒体

在创作面板中选择任务类型和模型，填入提示词，添加参考素材，检查比例、时长、价格和账户余额，再提交任务。媒体生成可能消耗额度或产生费用。

### 第 9 步：检查、精修和重新生成

查看人物与设定是否一致、构图和比例是否正确、是否出现不希望出现的内容，并决定是补图、改图、延展还是重新生成。需要时把结果放入画布继续处理。

### 第 10 步：组装并输出

可以要求系统按镜头顺序拼接视频、加入对白字幕、输出竖屏预览版或进行格式转换。涉及文件写入、命令执行和媒体处理时，应查看并确认实际操作范围。

### 第 11 步：定期维护项目记忆

长期项目应定期查询资料、沉淀新事实、巡检命名和时间线、修正源文件，并对阶段成果进行版本整理和备份。

## 11. 一个最简单的上手示例

假设要制作一条 30 秒的短剧宣传片：

1. **说明目标**：都市复仇题材，面向短视频用户，强冲突、快节奏，输出竖屏视频和字幕。
2. **整理方案**：请创模式先给出核心卖点、节奏、镜头表、台词和画面提示词，暂不生成媒体。
3. **确认并保存**：确认人物关系和结构后，请系统保存角色、镜头和风格要求。
4. **生成素材**：根据镜头表生成首帧图片，再把选定首帧转成视频。
5. **精修交付**：修正不符合设定的镜头，按顺序拼接，加入字幕，输出 9:16 预览版。

基本方式就是：

> **先说目标 → 再做方案 → 保存设定 → 生成素材 → 精修素材 → 组装交付。**

## 12. 适合哪些用户

适合小说作者、漫剧/短剧/影视编剧、导演和制片、AI 图片与视频创作者、自媒体团队、电商和品牌团队，以及需要多人共同维护内容资产的小型团队。

尤其适合：项目周期长、需要持续修改续写、要求视觉一致、同时使用文本/图片/视频/音频，或希望把创意逐步变成交付成果的用户。

## 13. 用户能够获得什么

用户获得的不只是某一个模型的回答，而是一套完整的创作工作方式：统一模型入口、项目创作空间、可执行的创模式、可维护的项目记忆、从文字到媒体的生产链路、可继续修改的画布，以及 Web 和桌面 App 两种入口。

实际变化是：少切换产品、少重复解释、少丢失设定、少散落素材，多一些可复用的项目资产和从想法走到成品的可能。

## 14. 与其他常见 AI 使用方式的区别

| 对比维度 | 单一模型产品 | 编程 Agent | 直接聊天 | 韭菜盒子 Studio |
| --- | --- | --- | --- | --- |
| 模型入口 | 单一产品 | 以开发场景为主 | 通常一次一个模型 | 一个账号进入多种模型 |
| 工作重点 | 单点对话或生成 | 代码和工程仓库 | 快速问答和生成 | 长期创作项目 |
| 项目记忆 | 通常有限 | 偏工程结构 | 依赖上下文 | Wiki、事实记录、巡检 |
| 专业创作流程 | 用户自行组织 | 不是主要目标 | 通用且容易跑偏 | 创模式与 Skill 组织 |
| 媒体与后处理 | 往往分开 | 通常不是核心 | 项目连接较弱 | 同一项目继续生成和制作 |

韭菜盒子 Studio 不是要取代所有模型，而是把模型、项目、专业流程和后续制作连接起来。

## 15. 产品发展方向与愿景

产品从创作场景开始，因为小说、剧本和影视制作最能体现 AI 面临的长期记忆、一致性、多轮修改和多媒体衔接问题。未来会把复杂的专业能力进一步做成更容易使用的场景化入口，服务漫剧、短剧、小说、影视、电商、品牌、自媒体和更多需要长期项目记忆的工作。

AI 不应该只属于懂代码、懂模型和懂设计的人。韭菜盒子 Studio 希望让普通人也能完成真实工作：**只要会表达，就能用上好的 AI。**
