# 文件树五期

> 日期：2026-07-18
> 分支：已合入 `main`（原 `0718-wenjianshu&bianjiqu` 已删除）
> 总原则：每期只交付一个可验收的边界；所有文件变更继续经过 [[开发/文件系统/文件树一期资源身份与文件安全SDD]] 定义的 `ProjectFileService` 和资源事件。

## 第一期：资源身份与文件安全（已完成）

已交付：统一 `ProjectResource`、Desktop/Web 文件服务和打开路由；文本安全读取；改名/删除同步编辑 Tab 与画布；音频画布；当前目录新建；Desktop/Web 菜单对齐；Desktop 上传/导入/导出与系统废纸篓。

关键经验：路径不能做 `trim()`。首尾空格是合法文件名的一部分，安全校验只拒绝绝对路径、`..` 和空字节。

详情见 [[开发/文件系统/文件树一期资源身份与文件安全SDD]] 和 [[开发/文件系统/编辑区与创作面板基础能力升级SDD]]。

## 第二期：批量文件操作（已完成）

只做用户直接需要的批量操作：

1. 多选。
2. 复制、剪切、粘贴。
3. 项目内拖放移动。
4. 重名处理、批量删除和批量另存为。
5. Desktop/Web 使用同一批量动作协议，完整发出资源映射事件。

完成标准：一次多文件操作后，文件树、已打开 Tab 和画布都落在同一结果，不存在旧路径复活或只更新部分后代。

详情见 [[开发/文件系统/文件树二期批量文件操作SDD]]。

## 第三期：Explorer 状态与性能（已完成）

1. 用真正的增量文件事件替换 Desktop 5 秒全量轮询；轮询只保留为异常兜底。
2. 目录按需加载，解除渲染树的 1000 项静默上限。
3. 补排序、展开状态恢复、自动定位当前资源和筛选。

完成标准：大项目不丢节点，切换编辑区或画布后能可靠反向定位到文件树。

详情见 [[开发/文件系统/文件树三期Explorer状态与性能SDD]]。

## 第四期：编辑区生命周期基础（已完成）

一期已落地每个 Tab 的资源、保存基线、脏状态、条件保存、改名/删除处理和外部修改冲突基础。第四期只保留这些生命周期基础的双端验收：

1. 外部修改、冲突、关闭前确认的完整双端人工矩阵。
2. 临时草稿与项目文件的视觉区分和明确入口。
3. 已打开项目文件的脏状态与文件树同源显示。

“全部保存”不再是任何一期的目标。第五期统一实现“保存当前文件”，并把它接回文件树的资源规则。

完成标准：编辑器的资源生命周期可靠，且不引入多编辑组或复杂合并编辑器。

详情见 [[开发/文件系统/文件树四期编辑区收尾SDD]]。

## 第四点五期：项目资源服务唯一文件总管（已完成）

文件树不只是左侧展示，而是项目资源服务的可视化入口。项目资源服务才是唯一文件总管，拥有资源身份、打开、新建、保存、导出、改名、删除、批量、定位和 Desktop/Web 差异；编辑区和画布不得继续各自管理项目文件。

本期先抽出共享文件动作层，让文件树、编辑区和画布都调用它；画布保留 JSON 编解码、保存队列和 lifecycle gate，但所有实际项目文件提交交给项目资源服务。完成后才能开始第五期 UI 接入。

详情见 [[开发/文件系统/文件树四点五期文件总管统一SDD]]。

## 第五期：编辑区与画布最终对齐（已完成）

第五期只接入四点五期完成的文件总管：删除编辑区、画布旧旁路，并完成保存当前文件、按当前资源原格式导出、中文编辑器右键、安全文本按资源路由打开，以及编辑区/文件树/画布共用新建项目文档。

直接交付五项：保存当前文件、按当前资源原格式导出、中文编辑器右键、安全文本按文件树路由打开，以及编辑区/文件树/画布共用的新建项目文档。编辑区的旧格式转换导出、预览、分片和关闭时 SQLite 自动保存已删除；导出统一委托文件树命令宿主，以继承同一份 Desktop/Web 目标目录、冲突和取消规则。

详情见 [[开发/文件系统/文件树五期编辑区与项目文档统一SDD]]。

## 后续可选 Explorer 增强

紧凑目录、文件嵌套、多根项目和 Git/任务状态装饰。

Desktop 废纸篓已在第一期完成，不再列为后续目标。后续能力只能建立在第五期统一后的文件树事实上，不能重新向编辑区或画布分叉文件规则。
