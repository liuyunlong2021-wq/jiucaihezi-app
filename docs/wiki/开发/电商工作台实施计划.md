# 电商工作台 Phase 1-3 实施计划

> **完成状态（2026-07-17）**：Task 1-7 已完成并进入审计。实现范围为 Desktop，因为现有创模式直连工具循环和 `.raw` 写入本身只在 Desktop 可用；Web Rail 不暴露电商入口，避免出现无法执行的半成品工作台。

> **执行方式**：逐项 TDD；每个行为先写失败测试，再写最小实现。  
> **目标**：把电商商品图工作台接入同一创模式会话和现有 CreationPanel，完成 Rail 入口；不改媒体 API、轮询器或其他 Rail 项。

## 文件边界

| 文件 | 职责 |
|---|---|
| `src/runtime/workbench/mediaPlan.ts` | 受限图片媒体计划的类型、JSON 提取和 Creation 模型/参数校验 |
| `src/runtime/workbench/mediaPlanBridge.ts` | 将合法计划交给现有 `useCreation` 状态和 `mediaTaskStore` 的唯一桥 |
| `src/runtime/workbench/ecommercePlanner.ts` | 将电商表单事实组织为创模式规划请求；不执行媒体任务 |
| `src/stores/ecommerceWorkbenchStore.ts` | 当前工作台/协作记录视图、商品图草稿、已审阅计划和关联任务 ID |
| `src/components/workbench/EcommerceWorkbench.vue` | 第三列商品图工作台、方案卡、开始生成和协作记录切换 UI |
| `src/components/chat/ChatPanel.vue` | 复用创模式直连循环为工作台生成计划，并将结果交回工作台；不新增第二套会话 |
| `src/components/creation/CreationPanel.vue` | 接收合法媒体计划并继续走已有 `submitTask({ source: 'creation' })` |
| `src/layouts/WorkspaceLayout.vue` | 保持 Chat 挂载，切换第三列工作台/协作记录，保留第四列 CreationPanel |
| `src/components/rail/ActivityRail.vue` | 仅新增 `ecommerce` Rail 入口 |
| `public/skills/JC-电商商品图/SKILL.md` | 内置策划 Skill：只输出 `JC_MEDIA_PLAN`，禁止直接执行媒体任务 |

## Task 1：媒体计划桥合同

**文件**：新建 `src/runtime/workbench/mediaPlan.ts`、`src/runtime/workbench/__tests__/mediaPlan.test.ts`

1. 先写失败测试：合法 fenced JSON 可解析；缺 `prompt`、未知模型、该模型不支持的比例或分辨率均被拒绝。
2. 定义最小 `MediaPlan`：`kind: 'image'`、`prompt`、`modelId`、`ratio?`、`resolution?`、`title`、`referenceImages`。
3. 实现 `parseMediaPlan(text)`，只接受最终回复中的 ` ```jc-media-plan ` JSON 块；实现 `validateMediaPlan(plan)`，使用 `getCreationModelSpec` 和 `buildCreationRunPlan` 验证。
4. 跑定向测试，确认 RED 后 GREEN；提交合同和测试。

## Task 2：电商工作台状态与规划请求

**文件**：新建 `src/stores/ecommerceWorkbenchStore.ts`、`src/runtime/workbench/ecommercePlanner.ts`，测试对应文件

1. 先写失败测试：新建草稿默认 `surface: 'workbench'`；切到 `collaboration` 再回来保留商品、参考素材、交付目标、市场和自由补充。
2. 实现最小 Pinia store：一个创模式会话对应一份商品图草稿和一个已审阅 `MediaPlan`；不保存第二份消息历史。
3. 实现 `buildEcommercePlannerPrompt(draft)`：以 XML/JSON 事实块给模型，要求加载 `JC-电商商品图` 并只输出可解析的媒体计划；不允许它调用 `JC-瞬间创作` 或媒体 API。
4. 跑定向测试并提交。

## Task 3：内置电商商品图 Skill

**文件**：新建 `public/skills/JC-电商商品图/SKILL.md`，更新 `public/skills/index.json`，测试 `src/utils/__tests__/skillContentResolver.test.ts`

1. 用 `skill-creator` 的普通 Skill 规范写 Skill：读取商品/参考素材和用户事实，产品图必须保留、参考图只借鉴画面语言、缺关键事实才追问。
2. 明确最终输出唯一格式：一个 `jc-media-plan` JSON fenced block；禁止运行 CLI、`jc_media.py`、轮询、下载或声称已生成。
3. 将 GPT Image 2 的提示词方法和瞬间创作的模型能力规则作为“规划参考”，不加载其执行指令。
4. 运行 `node scripts/build-skills-index.mjs` 与相关 catalog 测试后提交。

## Task 4：创模式规划接力

**文件**：修改 `src/components/chat/ChatPanel.vue`，新增/修改 `src/components/__tests__/chatMessagePresentation.test.ts`

1. 先写失败测试：收到电商规划请求时，ChatPanel 仍使用当前 `creative_*` 会话和既有 `sendCreative`；请求不会进入媒体模型拦截或 `mediaTaskStore.submitTask`。
2. 增加仅内部使用的 `ecommerce-plan-request` 事件处理：复用现有创模式发送路径，临时强制加载 `JC-电商商品图`，完成后用 `parseMediaPlan` 校验最终文本。
3. 校验成功派发 `ecommerce-media-plan-ready`；校验失败保留模型原文并派发真实错误，允许用户从协作记录修正后重试。
4. 不改变普通 Chat 的发送、Skill 选择、三按钮审批、文/武路径；跑定向测试并提交。

## Task 5：CreationPanel 执行桥与完成回流

**文件**：新建 `src/runtime/workbench/mediaPlanBridge.ts`、测试文件；修改 `src/components/creation/CreationPanel.vue`、`src/stores/mediaTaskStore.ts`

1. 先写失败测试：合法计划只以 `source: 'creation'` 提交；未知模型/非法参数不提交；完成事件带项目相对路径和关联会话 ID。
2. 实现 `submitMediaPlan(plan, context)`：调用已有 `switchTask('image')`、`switchModel`、比例/分辨率 setter 和 `buildCreationRunPlan`；不复制 HTTP、轮询或文件保存代码。
3. CreationPanel 监听 `ecommerce-media-plan-approved`，调用桥；桥记录 `taskId -> creativeSessionId`。
4. `media-task-settled` 时派发 `ecommerce-media-plan-settled`，把状态、项目路径或真实错误关联回工作台；不让模型轮询。
5. 跑媒体任务、CreationPanel 和桥的定向测试后提交。

## Task 6：电商第三列 UI 和同会话切换

**文件**：新建 `src/components/workbench/EcommerceWorkbench.vue`，修改 `src/layouts/WorkspaceLayout.vue`，新增组件/布局测试

4. WorkspaceLayout 用 `v-show` 保持 ChatPanel 挂载，在创模式 + `ecommerce` surface 时显示工作台；非创模式保持原布局。
5. 跑 UI 定向测试、TypeScript 检查后提交。

## 实施核对

| 目标 | 实现 | 结果 |
|---|---|---|
| 受限媒体计划 | `mediaPlan.ts` + Creation 注册表 | 只接受 fenced JSON，未知模型和非法参数在提交前拒绝 |
| 同会话工作台 | `ecommerceWorkbenchStore` + `v-show` | 不新建第二份消息历史，切换协作记录不卸载 Chat |
| 商品图专业规则 | `JC-电商商品图` | 只策划，禁止 CLI/媒体 API/轮询/下载 |
| 实际生成 | `mediaPlanBridge` + `CreationPanel` | 唯一走 `source: 'creation'` 的现有任务引擎 |
| 结果回流 | `ecommerce-media-plan-settled` | 回写会话、`.raw`、项目结果和画布 |
| Rail | `ecommerce` | 仅新增入口；旧项保持 |
