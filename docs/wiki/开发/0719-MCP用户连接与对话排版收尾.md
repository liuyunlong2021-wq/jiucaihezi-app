# 0719-MCP 用户连接与对话排版收尾

> 日期：2026-07-20
> 范围：`a1c1fea3`、`93d8972b`、`01a1d85c` 三次已完成提交的开发结论。

## 自定义 MCP

- 设置的 MCP 扩展增加“添加 MCP”。
- 支持 Streamable HTTP、SSE、Desktop stdio；Web 不显示 stdio。
- 新配置复用既有 `mcpStore -> mcpClient -> listTools -> mcpBridge`，连接成功后的工具进入文、武、创共用工具池。
- OAuth 和密钥不进入自定义表单；GitHub、Obsidian的内置流程不改。
- UI 改为产品内三段连接类型选择，按钮统一尺寸，窄窗口可换行。

## Markdown 正文

- 根因：助手 Markdown 已渲染为 HTML，容器仍使用 `pre-wrap`，于是 HTML 标签间的源码换行被显示为额外空白。
- 修复：正文容器改为 `normal`；段落和列表间距收紧。
- 不改用户消息、代码块、表格、标题、引用和模型输出。

## 验证与边界

- 三次提交均已运行对应合同测试、`pnpm run test:focused` 和 `git diff --check`。
- 用户已人工确认 MCP 表单与 UI。
- 真实自定义远程 MCP 调用，以及 Web、Windows、Intel 的人工矩阵仍待补；不得把它们写成已验收。
