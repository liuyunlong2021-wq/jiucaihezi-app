# v1.3.0 遗留测试对齐 SDD

> 日期：2026-07-20
> 状态：已完成
> 触发：正式 Web 发布前，focused suite 暴露 Workspace 与创作模型两组过期断言。

## 1. 根因

不是生产功能回退，而是两个已落地产品决定没有同步测试：

1. `74830725` 将 Skill 与 Review 移入设置。Web 当前只不支持 `files` 和 `context`，但旧测试仍要求不支持 `tools`、`review`，并要求已移走的 Rail 面板继续留在 `WorkspaceLayout`。
2. `f1509019` 清理固定的 AI App、火山 Seedance 和旧渠道模型，只保留动态发现节点的通用 `runninghub/aiapp/rh-aiapp`。旧测试仍按五个固定 AI App 的静态字段、模型 ID、参数边界和运行时请求断言，导致一个“模型不存在”引发多条连锁失败。

当前生产合同：

```text
Web -> WorkspaceLayout 只屏蔽 files/context
Skill / Review -> SettingsPanel
AI App -> 一个 rh-aiapp 注册项 -> 用户选应用 -> 运行时发现 nodeInfoList -> 提交
```

不能通过恢复旧模型、恢复旧 Rail 面板或给通用模型伪造静态字段让测试通过。这会与当前产品架构冲突。

## 2. 目标与验收

1. Workspace 测试验证当前 Web 可用性和 Settings 承接关系，不再约束已删除的 Layout 实现。
2. 创作模型测试验证通用 `rh-aiapp` 的 partial workflow 合同、动态参数透传与 warning；明确已退休固定 AI App 和火山 Seedance 不可查找。
3. 不修改 `WorkspaceLayout.vue`、`SettingsPanel.vue`、`creationModelRegistry.ts`、`creationMediaPlan.ts` 或 `creationMediaRuntime.ts`。
4. 目标测试、`pnpm run test:focused`、`pnpm exec vue-tsc -b`、`pnpm run build`、`pnpm run build:desktop` 全部通过。

## 3. 实施步骤

1. 修改 `src/layouts/__tests__/workspaceLayoutSizing.test.ts`：以 `WorkspaceLayout.vue` 和 `SettingsPanel.vue` 为事实来源，断言 Web 仅屏蔽 `files/context`、右栏只处理 editor/creation/settings，Skill/Review 由 Settings 管理；运行该测试确认通过。
2. 修改 `src/runtime/creation/__tests__/creationMediaPlan.test.ts`：删除对已退休模型的存在性、固定字段和静态参数范围断言；替换为通用 `rh-aiapp` 的注册、查找、workflow 参数、媒体引用计数和 partial warning 合同。保留仍在册的正式模型与真实参数校验。
3. 修改 `src/runtime/creation/__tests__/creationMediaRuntime.test.ts`：移除依赖已退休模型名称的三条 AI App 运行时测试；以通用 `rh-aiapp` 和动态 `nodeId:fieldName` 参数验证 `nodeInfoList` 提交及 `ai_app` 轮询。
4. 先运行三个目标测试，再运行完整发布门禁。任一失败先回到实际生产合同，不恢复历史模型。
