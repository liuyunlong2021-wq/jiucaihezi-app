## 关键证据

1. `sdkContract.test.ts` 原先使用 `Authorization: Bearer <password>`；Rust 启动代码和 OpenCode 官方服务端使用 Basic Auth。修复后以真实 `1.17.18` 二进制验证通过。
2. 当前 SDK `session.promptAsync()` 返回 HTTP 204；`event.subscribe()` 返回 `{ stream }`。测试已按官方生成 SDK 形状验证，不再把旧同步 prompt 当作 Desktop 发送合同。
3. `chatMessagePresentation.test.ts` 同时保留新旧架构断言。例如它仍要求保存 `openCodeSessionId` 双 ID，而当前 SDD 明确要求 Desktop 从创建起只使用真实 `ses_*`。
4. 媒体测试仍把 `grok-video-3`、`newapi/t8/seedance-2-0-pro` 当作可执行模型；当前注册表已明确标记渠道下线/`broken`。
5. `skillsManageStore.ts` 中 Discover/Obsidian 函数仍是死亡代码，但不再从 Store 返回；归档审计已经决定删除这批旧 Vault 能力，相应测试未同步。
6. 浏览器日志中的 `navigator.platform=MacIntel` 不是硬件证据。本机事实为 Apple Silicon（`uname -m=arm64`），因此 Intel baseline 仍未真机验收。
7. Rust 旧测试暴露了真实数据库缺口：`AgentSkillObservation.commands` 已存在于 Rust 结构体，但旧数据库没有列。现已增加新库建表、旧库 `ALTER TABLE` 迁移和 upsert 回归测试。
