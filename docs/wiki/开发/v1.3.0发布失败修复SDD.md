# v1.3.0 发布失败修复 SDD

> 日期：2026-07-20
> 状态：已完成；后续遗留测试已由 [[v1.3.0遗留测试对齐SDD]] 对齐
> 触发：`pnpm run build` 的 focused suite 失败；随后未带 `--branch=main` 的 Pages 部署只生成了 Preview。

## 1. 发布问题清单

| 优先级 | 现象 | 根因 | 处理 |
| --- | --- | --- | --- |
| P0 | `mediaTaskStore` 的 Desktop/Web 定向画布结果测试显示 `unwritten` 或超时 | 测试仍模拟旧画布读写接口；画布已改为通过 `ProjectFileService` 读取资源、带 revision 保存。测试双桩缺少 `dev_list_files`、revision 写入和 Web 资源列表能力，伪造了写入失败 | 升级测试双桩，覆盖真实共享文件服务合同；不改生产媒体流程 |

| P1 | Pages 命令返回部署成功但不是正式站 | 没有指定 `--branch=main`，Cloudflare Pages 创建 Preview | 发版命令明确要求 Production 分支；测试失败时不得部署 |

## 2. 根因与边界

画布的唯一写入链路是：

```text
MediaTask.canvasTarget（提交时冻结 owner）
-> 项目媒体落盘
-> writeCanvasTaskResult
-> ProjectFileService / revision 写入
-> canvas:task-result
```

本次 Desktop 测试只实现了旧的直接文件读写命令，Web 测试也只拦截了旧的 `webProjectFiles.read/write`。当前 `canvasPersistence` 已经统一经 `ProjectFileService` 先列举资源、再读取并按 revision 写回；测试双桩没有跟上，因此 `writeCanvasTaskResult` 抛错，再被 `writeCanvasResult` 正确标记为 `unwritten`。

因此不能通过放宽生产错误处理、回退到旧文件 API 或在没有 owner 时猜活动项目来“让测试通过”。这些都会破坏项目切换时任务仍写回提交时原画布的合同。

## 3. 目标与验收

1. Desktop：项目从 A 切到 B 后，定向任务仍将媒体和画布结果写入 A；恢复任务同样成立。
2. Web：定向任务仍将结果写入提交时的项目和画布；画布写入 gate 未释放前，任务保持 pending write，不被过早结算。
3. 画布事件合同只验证 `target/document/owner/release` 字段，不依赖格式化产生的换行。
4. 所有危险关闭操作通过 `confirmAction`，没有原生 `confirm`。
5. 网关模型测试承认已注册的 RH AI 应用为 text；商品图测试读取 `jc-product-image` 并继续限制为计划输出。
6. `pnpm run test:focused`、`pnpm exec vue-tsc -b`、`pnpm run build`、`pnpm run build:desktop` 成功后，才允许执行 Pages Production 和 App tag 发布。

## 4. 实施步骤

1. 先修改 `mediaTaskStore` 测试双桩及源码合同测试，使其模拟 `ProjectFileService` 的当前 Desktop/Web 合同；先运行媒体任务测试确认 Red。
2. 不动 `mediaTaskStore` 生产逻辑；让 Desktop、Web、gate 三条回归用例重新通过，确认这是测试基础设施修复而非功能回退。
3. 将 `EditorPanel` 的项目 Tab 关闭确认改为 `confirmAction`，保持保存和放弃两个独立判断；运行确认框守卫测试。
4. 更新 RH AI 应用和 `jc-product-image` 的测试预期；运行对应测试。
5. 运行完整 focused suite、类型检查和双端构建。构建失败时禁止部署。

## 5. 执行结果

- 画布生产链路未修改。Desktop/Web 双桩补齐当前 `ProjectFileService` 的列表、revision 读取与条件写入合同；Web V3 画布断言改为读取 `asset.resource.path`，不再使用已删除的 V2 `asset.path`。
- `EditorPanel` 关闭项目文件 Tab 改用统一 `confirmAction`，保留“先保存，否则再次确认放弃”的原有交互。
- 网关模型能力测试与商品图 Skill 路径测试更新为当前已发布的模型目录和 Skill 目录。
- 本文覆盖的局部 80/80、类型检查与双端构建当时通过；随后发现另一组与本文无关的 Workspace/创作模型旧测试，已由 [[v1.3.0遗留测试对齐SDD]] 对齐并重新通过完整门禁。
- 本次没有执行 Cloudflare Pages Production 部署或 Git tag 推送；必须使用下一节命令发布，避免误把 Preview 当正式站。

## 6. 正式发布门禁

Web Production：

```bash
pnpm run build
npx wrangler pages deploy dist --branch=main
```

桌面正式发包（GitHub tag 触发三平台 CI）：

```bash

git push origin v1.3.0
```

若 Git HTTPS 仍优先使用失效的 macOS Keychain 凭据，使用已登录 `gh` token 的 Basic 认证推 tag；具体命令在收尾交付中给出。
