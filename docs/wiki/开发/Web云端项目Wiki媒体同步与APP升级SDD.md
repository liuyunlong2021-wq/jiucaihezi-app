# Web 项目与媒体 APP 等价适配 SDD

> 日期：2026-07-14
> 状态：方案确认，待执行
> 分支：`0713-webchonggou`
> 产品标准：用户在 Web 看到的项目文件树、媒体落点、预览和文件操作与 Desktop APP 一致。

## 1. 前端能力

| 位置 | Web 与 APP 统一后的行为 |
|---|---|
| 项目顶部 | 新建文件、新建文件夹、上传文件、上传文件夹、切换项目、刷新、收起 |
| 项目空白处右键 | 新建文件、新建文件夹、上传文件、上传文件夹、切换项目、复制项目路径、导出项目 |
| 文件夹右键 | 新建文件、新建文件夹、上传到此处、重命名、删除、复制路径 |
| 文件右键 | 预览、加入画布、编辑区打开、重命名、删除、另存为、复制路径 |
| 文件拖入项目树 | 上传到拖入的文件夹；拖入空白处则上传到项目根目录 |
| 创作任务完成 | 自动写入当前项目的 `jc-media`，文件树立即出现，画布引用同一项目文件 |
| 点击媒体预览 | 图片、视频、音频都在站内预览；只有“另存为”触发下载 |

首次生成对应媒体时按需建立目录：

```text
jc-media/
├── images/
├── videos/
└── audios/
```

## 2. Web 项目存储

| 数据 | Web 真源 | 用途 |
|---|---|---|
| 项目、目录、文件路径、媒体元数据 | IndexedDB `documents` | 文件树、搜索、重命名、删除 |
| Markdown、JSON、剧本等文本 | IndexedDB `documents` | 编辑器和模型文件工具读取 |
| 图片、视频、音频及其他二进制文件 | 浏览器 OPFS | 保存真实字节，不占服务器存储 |
| 画布文件 | 当前项目中的 `.jccanvas` | Desktop 与 Web 使用相同项目表达 |

二进制文件在 OPFS 中按稳定文件 ID 保存；项目相对路径保存在 `documents`。重命名和移动只更新路径，不复制大文件。

浏览器首次建立项目存储时申请持久化存储权限，并在上传或生成落盘前检查剩余空间。大文件使用流式写入，不转成 Base64，不整体载入内存。

## 3. 上传

1. “上传文件”使用浏览器文件选择器，支持多选；
2. “上传文件夹”保留用户选择文件夹的相对目录结构；
3. 文本写入 `documents`，二进制流式写入 OPFS，再写入文件元数据；
4. 文件写入成功后刷新项目树；失败时不留下空文件记录；
5. 同名文件弹出覆盖、保留两份、取消选择；
6. 上传的图片、视频和音频可直接预览、加入画布、另存为，并可由模型读取对应项目路径。

## 4. 创作媒体

创作任务开始时固定当前项目 ID，任务完成后执行：

```text
模型返回媒体 URL
  -> 浏览器将媒体流写入当前项目 OPFS
  -> 在 documents 建立 jc-media/{类型}/{文件名} 记录
  -> 画布写入该项目相对路径
  -> 刷新第二列项目树
  -> 站内预览读取同一份项目媒体
```

图片进入 `jc-media/images/`，视频进入 `jc-media/videos/`，音频进入 `jc-media/audios/`。文件名继续使用 Desktop 的时间、提示词摘要和随机后缀规则。

浏览器优先直接从媒体 URL 流式写入 OPFS；上游不允许跨域读取时，由现有 API 网关只做流式转发，不持久化媒体。

没有选择项目时，生成按钮先要求用户新建或选择项目，避免产生没有项目归属的媒体。

## 5. 预览与导出

1. 图片使用站内大图预览；
2. 视频使用站内 `<video>` 播放器；
3. 音频使用站内 `<audio>` 播放器；
4. 文本继续在编辑区打开；
5. “预览”不调用外部浏览器地址，不触发下载；
6. “另存为”将 OPFS 文件流写到用户选择的位置；
7. “导出项目”按项目目录结构写到用户选择的本地文件夹；
8. “导入项目”通过文件夹选择器恢复目录、文本、画布和媒体。

## 6. 现有代码适配

| 现有模块 | 适配内容 |
|---|---|
| `ProjectFileTree.vue` | 补齐根目录右键、上传、拖放、站内预览和项目导入导出入口 |
| `webProjectFiles.ts` | 保持项目 CRUD 唯一入口，增加二进制文件引用和级联清理 |
| `mediaTaskStore.ts` | 创作任务完成后统一写当前 Web 项目，不再将 creation 任务排除在项目写入之外 |
| `creationMediaCache.ts` | 接入真实创作完成链，负责 Web 媒体流式落项目 |
| `CreationPanel.vue` | 预览改为站内预览，画布只引用项目文件路径 |
| `webProjectTools.ts` | `read/glob/grep/write/edit` 继续使用当前项目；媒体读取解析 OPFS 文件引用 |

## 7. 一致性规则

1. 当前项目是创作媒体、上传文件、画布和模型文件工具的共同边界；
2. 先写真实文件，再写元数据，避免文件树出现不存在的媒体；
3. 删除文件或文件夹时，同时删除对应 OPFS 文件和 `documents` 记录；
4. 同一项目写入继续使用 Web Locks 串行化；
5. 多标签页继续使用 `BroadcastChannel` 刷新项目树；
6. 媒体落盘失败时任务显示“保存到项目失败”和重试入口，不把远程 URL 冒充项目文件；
7. 浏览器空间不足时停止写入并提示用户导出或清理项目。

## 8. 验收

1. Web 项目空白处右键菜单具备新建、上传、导出能力；
2. 可上传单文件、多文件和完整文件夹，并保持目录结构；
3. 图片、视频、音频生成完成后自动出现在 `jc-media`；
4. 项目文件和画布引用同一媒体，重命名后仍可预览；
5. 点击“预览”不打开新标签页、不自动下载；
6. 文件“另存为”和项目导出得到可用的本地文件；
7. 刷新页面、关闭后重开，项目结构和媒体仍可用；
8. 两个标签页同时写入同一路径不会产生重复文件或丢失编辑；
9. 模型可按项目相对路径读取上传文本和媒体；
10. `pnpm run build`、Web 产物审计和相关回归测试通过。
