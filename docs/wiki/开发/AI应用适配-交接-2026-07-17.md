## 三、轮询链路（已修复）

```
前端提交 → NewAPI 包装 task_oV7f... → 前端用 rh_task_id（原始 RunningHub ID）
→ /rh/tasks/{id}?ai_app=true → Nginx → rh-adapter:8789 → RunningHub
```
之前走 NewAPI 轮询导致永远返回 "processing"，改为直连 rh-adapter。

## 四、关键技术细节

### instanceType: "plus"（48GB）

`rh_client.py` 的 `submit_ai_app` 默认 `instance_type="plus"` 解决"我是导演" OOM。

### Nginx /api/runninghub/ 路由

绑 `172.17.0.1:8789`（Docker bridge IP），`localhost:8789` 不可达。

### 通用 AI 应用 UI

选「AI 应用」→ 点「应用」下拉 → 5 个预置应用 → 点选自动发现节点。底部保留自定义 webappId 输入。
