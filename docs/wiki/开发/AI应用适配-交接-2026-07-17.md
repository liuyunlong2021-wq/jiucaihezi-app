## 三、不能用的

### "我是导演"（webappId: 2029950473750454274）视频迁移工作流

**症状**: RunningHub 返回 `torch.OutOfMemoryError at WanVideoSampler`

**用户验证**: 同样的图片+视频文件，在 RunningHub 网站上直接使用没问题，5分钟出结果。通过我们的 APP 提交每次都是 OOM。

**推测根因**: 该工作流需要 48GB 显存实例（`instanceType: "plus"`），但我们提交时可能默认用了 24GB 实例。

```python
async def submit_ai_app(
    client, api_key, webapp_id, node_list,
    instance_type: str = "plus",  # ← 新增，默认 48GB
) -> str:
    ...
    if instance_type and instance_type != "default":
        payload["instanceType"] = instance_type
    logger.info("submit_ai_app: webapp=%s instanceType=%s nodeCount=%d", ...)
```

同时 `video.py` 的 `_submit_via_app` 也会从 `request.extra_fields` 读 `instanceType`。

**该文件在本地存在，但服务器容器内未生效。** 尝试了 3 次部署均被 Docker build 缓存阻挡。最后用 `docker cp` 直接拷贝到容器并重启，但 logger.info 日志未出现，无法确认代码是否真正生效。
