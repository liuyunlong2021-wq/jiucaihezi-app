> **状态**: UI 设计稿 · 待评审

> **适用范围**: 韭菜盒子 Desktop（Tauri + Vue）全端

## 0. 一句话决断

产品核心价值已从"AI 对话"进化到"AI 创作"。现有布局把对话锁死在主位、把创作产出（画布/编辑器/流水线）塞在可选的右侧面板——这个优先级是反的。

改造目标：**Rail 定义创作模式 → FileTree 是数据真源 → 模式内容区（第三列）根据后端的 skill 管线展示结构化进度 → 基础功能区（第四列，创作画布 + 编辑器）默认展开且常驻。**

**核心分工**：Chat（col3）只做文字讨论和选项交互；所有媒体预览（图片/视频/音频）统一在 col4 创作面板画布中展示。Chat 不嵌播放器、不嵌图片预览。

对话 UI 可隐藏，对话 session 不切断。一键成X 模式本质是同一个 agent session + 一层折叠皮。

## 1. 目标与非目标

### 1.1 目标

1. Rail 语义从"切换右侧面板"升级为"切换创作模式"——对话是其中一种模式，不是默认绑定态
2. FileTree 保留隐藏能力（适配小屏幕），但默认展开
3. 新用户冷启动默认看到：Rail + FileTree + 一键漫剧（模式内容区）+ 创作面板（基础功能区）——立刻能开干
4. 创作面板（画布/极速流水线）和编辑器从"右侧面板的可选项"升级为"默认展开的基础功能区"
5. 旧的 Rail 条目（skills/tools/editor/review/files/creation）归并到不挡视线的地方

### 1.2 明确不做

- 不改 skill 层：JC-manju-skills / JC-xiaoshuo / JC-meitichuangzuo 等已存在的 skill 文件不变，本次只改前端"怎么调用它们"
- 不改 useChat.ts 核心消息流：pipeline 面板只是折叠原始消息，不是新建通信通道
- 不改 mediaTaskStore / creationModelRegistry：创作面板的模型能力不变
- 不引入新的状态库或事件总线
- 不重构 FileTreePanel 内部逻辑，只改其"可隐藏策略"的默认值

### 2.1 Rail 图标布局（重设计）

**上部 — 模式类（点击切换"模式内容区"，第一波平铺展示）**

Rail 不使用图标。每个模式是一个带文字的圆形按钮：两字名称单行展示；三个字及以上固定两行、每行两个字。例如"漫剧制作"显示为"漫剧"、"制作"。文字本身就是功能名，避免用户猜图标语义。

| 按钮文字 | mode | 第一波说明 |
|----------|------|------------|
| 对话 | `chat` | 传统聊天模式，col3 显示 ChatPanel |
| 漫剧／制作 | `pipeline:manju` | 一键漫剧：风格→角色/场景/道具→工程手册→分镜→音色→配音→合成 |
| 漫剧／剧本 | `pipeline:script` | 从零写漫剧剧本/临摹换皮 |
| 电商 | `pipeline:ecom` | 产品图/修图/视频/反推，4 项独立工具 |
| PPT | `pipeline:ppt` | PPT 专用 skill + GPT Image 2；有必要时再接入 Remotion |
| 自媒体 | `pipeline:zimeiti` | 14 种内容类型卡片 → 写文案/配图/配音/BGM/合成 5 步成片 |
| 短故／事 | `pipeline:short-story` | 短故事创作与成稿 |
| 小说 | `pipeline:novel` | 一键小说：灵感→市场→核心梗→角色→大纲→章纲→文案→正文→复盘 |

> 第一波固定平铺以上 8 个模式。后续只有在 Rail 的垂直空间确实不足时，才为新增模式引入折叠分组；不提前设计下拉菜单。

**底部 — 工具类（不变，保持可用但不挡路）**

| 图标 | 名称 | 说明 |
|------|------|------|
| ❓ | 帮助 | 使用指南 |
| 🌐 | 语言切换 | 中/英切换 |
| ⚙️ | 设置 | 用户中心 + 旧 Rail 条目的归入位置 |

### 2.2 旧 Rail 条目的去向

当前 `ActivityRail.vue` 的 `allTabs` 数组包含 6 个条目。模式化后它们的归属：

| 旧条目 | 去向 | 理由 |
|--------|------|------|
| `skills`（技能仓库） | **设置 → 技能管理子页** | 安装/管理第三方 skill 是偶尔操作，不需要常驻 Rail |
| `tools`（工具仓库） | **设置 → 工具管理子页** | 同上 |
| `editor`（编辑区） | **基础功能区（col4）**，默认展开 | 编辑器是日常写作/笔记工具，不该藏在 Rail 后面 |
| `review`（变更审查） | **设置 → 审查记录子页** | 审查是后处理操作，不是入口级功能 |
| `files`（文件树） | **独立为 col2 FileTree**，不再走 Rail 切换 | FileTree 是数据真源，且 rail 已有独立的 files 图标，逻辑矛盾 |
| `creation`（创作面板） | **基础功能区（col4）**，默认展开 | 创作面板是核心工作区，不该是 Rail 的一个选项 |

> **关于编辑区**：你的直觉是对的——编辑区应该保留且可见。它的位置从"Rail 点一下才出现"改为"col4 基础功能区里的一个 Tab（创作面板/编辑器两个 Tab 切换）"，用户随时可以切过去写作，不用先找入口。

## 3. 根本性决议：操作台 = 指令按钮的进化

> **日期**: 2026-07-14 确认

### 3.1 不是新建系统，是现有机制的进化

- 产品只有一个：**OpenCode 驱动的创作引擎**
- 现在输入框上方有 30+ 平铺指令按钮（`kbCommandPresets.ts`），本质是"粘贴 prompt 到输入框"
- 新手面对 30 个按钮无从下手，老手每次要回忆"上次角色设计的 prompt 怎么写来着"
- **操作台** = 把这些指令按钮进化成用户能看懂的步骤引导，帮小白知道"从哪开始、下一步干什么"

### 3.2 三层架构

| 层 | 是什么 | 用在哪 |
|---|--------|--------|
| OpenCode Agent | 核武器：代码、项目管理、复杂 workflow | Rail "对话" 模式 |
| 操作台（指令进化版） | 拼装 prompt → 填入输入框 → OpenCode 干活 | Rail "漫剧制作" 等创作模式 |
| NewAPI 媒体引擎 | 出图/出视频/出音频 | 创作面板 col4 |

操作台不是新系统。它是指令面板的进化——把"粘贴 prompt 到输入框"这件事包装成用户能看懂、能跟着走的步骤引导。

### 3.3 指标：操作台做了什么

| 按钮 | 预填的提示词 | 用户只需补充 |
|------|-------------|-------------|
| `[A]` / `[B]` / `[C]` | `我选 A` | `，色调再暗一点` |
| `[🗑️ 重做]` | `重新设计角色林昭，保持人设不变，换一种风格：` | `改成更凌厉的线条` |
| `[✏️ 修改人设]` | `修改角色林昭的人设：` | `外套改成藏青色` |
| `[⚡ 开始设计]` | `设计角色 Yuki：女·19岁·神秘接头人·银白短发·红围巾。风格：新海诚·冷调` | (可直接发送) |
| `[⚡ 全部开始设计]` | `开始设计所有角色：林昭、陈叔、Yuki。风格：新海诚《你的名字。》冷调` | (可直接发送) |

核心逻辑：按钮不直接调 API。按钮只做一件事——**往输入框里填一段上下文**。发送之后走现有对话引擎 → skill → 媒体的完整链路。

## 8. 自媒体 — 操作台 UI 设计

> 设计灵感: Pixelle-Video（AI 视频创作平台）
> 设计原则：卡片选内容类型 → 5 步串行进度条 → 步步产出可见、可 Chat 讨论修改 → 所有媒体预览在 col4 画布
> Chat 只做文字讨论，不嵌任何媒体播放器或图片预览。

### 8.1 自媒体 5 步流程

```
① 写文案 → AI 根据类型+主题写稿 → Chat 区展示文案 → 用户讨论修改
② 配图/视频 → AI 为每段生成配图或视频 → col4 画布展示 → 逐张确认
③ TTS 配音 → AI 合成语音 → col4 画布预览试听 → 调整音色/语速
④ 加 BGM → AI 推荐背景音乐 → col4 画布试听 → 替换/调音量
⑤ 合成 → 画面+配音+BGM 一键合成 → col4 画布预览 → 下载
```

### 8.2 14 种内容类型卡片（Pixelle-Video 模板体系）

```
┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
```

## 10. 已确认的产品决策

1. 产品只有一个：OpenCode 驱动的创作引擎。操作台是指令按钮的进化。
2. Rail 使用平铺文字圆钮。第一波：对话、漫剧制作、漫剧剧本、电商、PPT、自媒体、短故事、小说。
3. 每个模式有独立操作台 UI，共用 ConsolePanel 组件架构。
4. 操作台采用方案C——嵌在 Chat 输入框上方，始终显示步骤进度条，用户可随时切任意环节。
5. 漫剧制作 5 步：风格定调 → 角色/场景/道具 → 分镜 → 组装视频。
6. 电商 4 项（产品图/修图/视频/反推）为独立工具，无步骤依赖，随时切换。
7. 自媒体 5 步串行（写文案/配图/配音/BGM/合成）+ 14 种内容类型卡片选择，所有媒体预览在 col4 画布，Chat 只做文字讨论。
8. PPT 使用独立 skill + GPT Image 2；Remotion 是可选后续能力。
9. 基础功能区固定为创作面板 / 编辑器两个 Tab，默认创作面板。
10. 模型选择全局共用（设置里统一配置），操作台顶部显示当前模型（只读标签）。
