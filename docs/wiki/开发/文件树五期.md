# 文件树五期

> 日期：2026-07-18
> 分支：`0718-wenjianshu&bianjiqu`
> 总原则：每期只交付一个可验收的边界；所有文件变更继续经过 [[开发/文件树一期资源身份与文件安全SDD]] 定义的 `ProjectFileService` 和资源事件。

## 第一期：资源身份与文件安全（已完成）

已交付：统一 `ProjectResource`、Desktop/Web 文件服务和打开路由；文本安全读取；改名/删除同步编辑 Tab 与画布；音频画布；当前目录新建；Desktop/Web 菜单对齐；Desktop 上传/导入/导出与系统废纸篓。

关键经验：路径不能做 `trim()`。首尾空格是合法文件名的一部分，安全校验只拒绝绝对路径、`..` 和空字节。

详情见 [[开发/文件树一期资源身份与文件安全SDD]] 和 [[开发/编辑区与创作面板基础能力升级SDD]]。

## 第二期：批量文件操作（下一期）

只做用户直接需要的批量操作：

1. 多选。
2. 复制、剪切、粘贴。
3. 项目内拖放移动。
4. 重名处理、批量删除和批量另存为。
5. Desktop/Web 使用同一批量动作协议，完整发出资源映射事件。

完成标准：一次多文件操作后，文件树、已打开 Tab 和画布都落在同一结果，不存在旧路径复活或只更新部分后代。

## 第三期：Explorer 状态与性能

1. 用真正的增量文件事件替换 Desktop 5 秒全量轮询；轮询只保留为异常兜底。
2. 目录按需加载，解除渲染树的 1000 项静默上限。
3. 补排序、展开状态恢复、自动定位当前资源和筛选。

完成标准：大项目不丢节点，切换编辑区或画布后能可靠反向定位到文件树。

## 第四期：编辑区收尾

一期已落地每个 Tab 的资源、保存基线、脏状态、条件保存、改名/删除处理和外部修改冲突基础。剩余只做：

1. 全部保存的清晰交互。
2. 外部修改、冲突、关闭前确认的完整双端人工矩阵。
3. 临时草稿与项目文件的视觉区分和明确入口。

完成标准：编辑器行为接近 VS Code 的文件生命周期，不引入多编辑组或复杂合并编辑器。

## 第五期：可选 Explorer 增强

紧凑目录、文件嵌套、多根项目和 Git/任务状态装饰。

Desktop 废纸篓已在第一期完成，不再列为第五期目标。上述增强不得早于二、三期，否则只会放大当前状态边界的复杂度。
