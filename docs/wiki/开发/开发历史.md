### Task 3-5：直连对话、媒体联动与收尾

- Desktop 创模式新增 `creativeChat`，使用 `safeFetch`、`runDirectChatCompletion` 和项目工具执行器；取消通过 `AbortController` 停止后续工具轮次，不调用 OpenCode。
- 创模式文本会话持久化为 `creative_*`；模型必须在网关目录中声明 `tool_call/toolCall: true` 才可进入工具循环。

- Direct Engine 为缺失工具 id 的兼容模型补齐稳定 id；两端均缓冲工具轮次文本，只在工具循环结束后提交最终答复，避免中间文本闪现覆盖结果。
- 修复共享工具摘要的误报：不再从返回正文猜测失败，改读执行器写入的 `succeeded/failed/cancelled`；Skill 正文含“失败/error”不再显示红色失败，真实错误可在详情查看。
- 创模式工具过程只展示可验证的折叠调用与结果，不泄露原始思维链；工具循环完成后一次性提交最终答复，不使用逐帧揭示器。

### 输出与会话稳定性修复

- 根因一：Vue 响应式数组内的 assistant 占位对象曾在 push 后继续修改裸对象，导致最终文本和工具摘要不触发渲染。改为 push 后重新取得数组内 proxy，用户消息、工具状态和最终答复均更新同一响应式对象。
- 根因二：新创会话先激活、后首存盘时，异步历史水合可能以空记录替换消息数组，持久层最终只剩被 UI 隐藏的 `tool` 消息。现在先持久化用户与 assistant 可见对，再激活会话，并以 pinned 数组与运行 token 阻断旧请求清锁。
- `createPendingSession` 在创建时绑定项目；进入创模式或切项目同步清空共享 OpenCode 消息并进入水合锁，避免 `ses_*` 历史或项目 A 会话串入项目 B。
- 模型目录、Skill/command catalog 和共享 OpenCode 命令/发送路径在异步边界后再次检查创模式；切入创模式后不再建立或继续提交 OpenCode 请求，文、武模式原路径不变。

## 2026-07-14 · `0713-webchonggou` 画布定向任务项目隔离

- `CanvasTaskTarget` 持久化提交时的 owner，并在参考媒体异步解析前同时快照 `canvasId`、`canvasPath`。
- Desktop 正常完成和恢复轮询都用该 owner 本地化结果、写入 `.jccanvas`、派发 `canvas:task-result`；不再读取完成时的当前项目目录。
- 缺少 owner 的旧定向任务显式标记 `unwritten`，不会回退写入当前项目。
- `.jccanvas` 持久化收敛为单 `owner:path` 队列；任务结果在队列内读改写，当前画布用异步 `CanvasGate` 在任务写入和磁盘恢复之间冻结旧快照。
- 定向任务从提交起为 `pending`，因此清理、删除、重命名和删除画布都不能越过尚未落盘的目标结果；文件树在异步生命周期屏障后再次检查。
- 修复同路径重开、切到空项目和卸载三个竞态：等待中的旧请求不会打断新 gate，且不会把旧场景排到任务 RMW 之后。
- 任务结果恢复失败时不恢复旧场景：保留 restoring 阻断，只释放 task gate 给显式重开流程接管磁盘恢复。
- 回归覆盖 A 提交后切到 B、恢复轮询、旧任务、任务 RMW 串行化、文件生命周期门控和上述三条竞态；`pnpm run test:focused`（Rust `372 passed`）、`pnpm exec vue-tsc -b`、`git diff --check` 通过。
