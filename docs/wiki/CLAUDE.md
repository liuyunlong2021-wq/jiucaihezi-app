## 📖 Wiki 使用说明

Wiki 是项目的第二大脑。你扔内容进来，AI 整理成能搜索、能追踪的知识库。

### 五个 Skill

| Skill | 干啥 | 指令 |
|-------|------|------|
| Everything转Wiki | 把任何内容转成 Wiki 结构 | `Everything转Wiki：[内容]` |
| 记忆压缩 | 归档开发经验、填充 wiki | `整理记忆体` |
| Wiki查询 | 查状态、搜内容、画关系图 | `查询[关键词]` 或 `生成关系图` |
| 一致性巡检 | 检查断链、索引补缺、列体检单 | `巡检一下` |
| Wiki修正 | 把巡检单问题落地改回源文件；给 wiki 加新分类目录 | `执行修正` 或 `加个分类` |

巡检官只查不改，修正官接手把体检单上的问题落地——机械类（漏改/断链）直接改，语义类（设定矛盾）先问你哪个对。"巡检→修正→复巡" 是闭环，不然体检单会一直堆着没人处理。
修正官还管"架构扩展"：项目做大了想给 wiki 加新分类（比如小说中途想单独建"伏笔/"目录），跟它说一声就能建好骨架；巡检官发现某分类文件数明显失衡时也会主动建议拆分。新分类要不要吃掉已有内容（内容迁移），转交记忆压缩官处理。

### 怎么用

```
Everything转Wiki：[粘贴内容]     → 建壳
Wiki壳填充                        → 填内容
查看Wiki状态                      → 看进度
查询[关键词]                       → 搜东西
生成关系图                         → 画关系网（需 Obsidian Canvas Viewer 插件）
巡检一下                           → 体检，出体检单
执行修正                           → 把体检单上的问题落地改掉
加个分类                           → 给 wiki 建新的子目录/分类
```

## 🔧 知识库维护

| 什么时候 | 做什么 |
|----------|--------|
| 每次收尾 | 刷新 `hot.md`（更新日期、指向最新变更） |
| 踩了新类型坑 | 加一条到生存手册（#N + 速查表） |
| 分支合并/发版 | 更新 `开发历史.md` |
| 功能或排障完成 | 用 `JC-jiyiyasuo`（记忆压缩）归档经验到 wiki |
| 不确定知识库状态 | 用 `JC-Wiki-查询` 查状态、生成关系图 |
| 修了 bug/发现问题 | 更新 `OpenCode差异修复记录.md` |
| 架构变化 | 更新 `产品架构.md` + `对照表.md` |
| 运维操作变化 | 更新 `运维/*.md` |
