## 🔧 如何更新 Wiki（AI 必读）

> Wiki 是活文档。每次开发结束必须更新。以下是各文件的更新规则：

| 文件 | 什么时候更新 | 怎么更新 |
|------|-------------|---------|
| `hot.md` | **每次收尾** | 更新计数、指向最新变更、刷新日期 |
| `开发/OpenCode差异修复记录.md` | 修了 bug 或发现新问题 | 修复→加「已修复」表行；新问题→加「还没修」条目 |
| `学习/AI编程生存手册.md` | 踩了新类型的坑 | 加新概念（#N）+ 速查表加一行（症状→概念→告诉AI） |
| `开发/开发历史.md` | 合并分支/发版时 | 加一行：日期、分支、改了什么 |
| `架构/对照表.md` | 新增或移动文件时 | 更新文件映射 |
| `架构/产品架构.md` | 架构变化时 | 更新对应章节 |
| `运维/*.md` | 运维操作有变化时 | 更新命令/流程 |

### 收尾标准指令

开发者说这句，AI 自动执行更新：

```
收尾：把今天的改动更新到 Wiki，生存手册有新概念就加，刷新 hot.md
```
