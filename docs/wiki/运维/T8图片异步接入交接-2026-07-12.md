# T8 图片异步接入交接

> 状态：仅完成排查与服务器核验，尚未修改 NewAPI、前端或线上服务。
> 下一阶段目标：让 T8 `gpt-image-2` 的文生图、图生图不再因同步等待触发 Cloudflare 524。

## 已确认事实

- T8 协议见 `docs/wiki/归档/模型文档/T8 异步文档.md`：提交图片时在 `/v1/images/generations` 或 `/v1/images/edits` 加 `?async=true`，立即取得 `task_id`；查询为 `GET /v1/images/tasks/{task_id}`。
- 上游已出现“524 但任务实际 SUCCESS”的证据：同步 HTTP 连接超时，T8 后台继续生成。
- NewAPI 当前有图片提交路由，但没有 T8 所需的 `GET /v1/images/tasks/{task_id}` 路由。
- NewAPI 的通用任务数据结构已包含用户、渠道、上游任务 ID、上游 Key 和计费上下文，应复用它，不新增独立任务库。
- Nginx 已将 `^/v1/images` 转给 NewAPI，所以新增该 NewAPI 路由不需要改 Nginx。

## 正确边界

```text
创作面板 -> NewAPI -> T8
```

- 不放入 `rh-adapter`：它仅翻译 RunningHub 协议，保存的是 RunningHub Key。
- 不让前端直连 T8：前端不能持有 T8 上游 Key，也不能负责用户归属、渠道选择、计费或退款。
- 不靠管理员后台配置：后台不能新增 HTTP 查询路由。
