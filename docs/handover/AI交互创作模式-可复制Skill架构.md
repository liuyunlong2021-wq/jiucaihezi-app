# AI 交互创作模式 — 可复制 Skill 架构

> 基于 `JC-duanju-shijiemoxing` 提炼的通用创作 Skill 模式。任何 AI 工具拿到本文档，都应该能为其他领域复制出同等质量的创作 Skill。

---

## 一、这个模式解决了什么

### 传统 AI 创作的问题

| 问题 | 表现 |
|------|------|
| 记不住 | 写到第 20 章忘了第 3 章的角色设定 |
| 写不好 | 没有质量约束，写出来像聊天不像作品 |
| 不会引导 | 用户说"写个XX"，AI 直接胡写一通 |
| 无法协作 | 写完没法质检、没法整理、没法回溯 |

### 这个模式的回答

```
逐轮引导选择 → 建制完整 → 逐场按质量手册写 → 写完压缩记忆 → 定期巡检纠错
```

**核心公式**：`引导式建制 + 质量约束写作 + 外置记忆 + 巡检纠错`

## 二、Skill 文件架构（复制模板）

```
skill-name/
├── SKILL.md              ← 统帅：180行以内。架构/原则/流程概览/铁律/指令/ref索引
├── scripts/              ← 可执行脚本（可选）
│   └── validate.py       ← 质量控制脚本
└── references/           ← 按需加载的详细手册
    ├── 建制阶段.md        ← 逐轮交互收集模板（最重要）
    ├── 创作工作流.md      ← 核心创作循环
    ├── 核心机制.md        ← 领域特有的规则/公式
    ├── 写作质量手册.md    ← 质量控制规则（台词/节奏/冲突/钩子...）
    ├── 素材库.md          ← 参考素材（爽点/反派/母题...）
    └── 摄入规范.md        ← 外部资料如何归档
```

### SKILL.md 应该包含

| 章节 | 内容 | 行数 |
|------|------|------|
| 定位 | 一句话说清这个 skill 做什么 | 2-3 |
| 架构 | 核心循环/Agent角色 | 5-10 |
| 核心原则 | 5-9 条不可违反的规则 | 10-15 |
| 文件约定 | Wiki 目录结构 | 15-20 |
| 工作流程 | 概览，指向 references/ | 20-30 |
| 交互铁律 | 询问/选项/等待确认 | 5-10 |
| 交互时机表 | 所有交互点矩阵 | 10-15 |
| 自检清单 | 每场/每章结束后检查 | 10-15 |
| 质量守则 | ❌禁止/✅必须 | 3-5 |
| 快速参考 | Q&A | 5-10 |
| 指令 | commands | 5-10 |
| 参考文件 | refs 索引表 | 5-10 |

## 三、用户交互模式（最核心的可复制部分）

### 格式铁律

```
每个问题 = 3个具体选项（A/B/C）+ 用户自定义（D/E/F）
```

### 逐轮渐进式收集

```
第0轮：先问一句"你想做什么？"，判断完整性
  ├── 用户说清楚了 → 跳到关键设定轮
  └── 用户说了模糊方向 → 逐轮收集

第1轮：核心类型 + 基调     (A-F/A-E)
第2轮：主角/主体核心        (文本/A-H/A-F)
第3轮：初始状态 + 困境      (A-F/A-F/A-E)
第4轮：对立面 + 规则        (A-F/A-F)
第5轮：关键设定             (A-E/A-E/A-E)

建制完成 → 摘要确认 → 开始创作
```

### 每个选项的设计原则

1. **具体**：不说"选择世界观"，说"A.修仙世界（凡人流/宗门/秘境）"
2. **刺激联想**：选项本身就是创作方向的提示
3. **覆盖盲区**：帮用户想到他们可能遗漏的关键设定
4. **最后一个永远是自定义**：不绑架用户

## 四、Wiki 记忆系统（跨 Skill 共享）

```
项目根/
├── .raw/                  ← 原始素材（Obsidian 隐藏）
│   ├── 参考资料/           ← 书籍/文献（摄入模式用）
│   ├── 对话记录/           ← 创作对话（压缩模式用）
│   └── 灵感笔记/
├── wiki/                  ← 结构化知识
│   ├── 作品/               ← 创作产出正文
│   ├── 角色/               ← 角色档案（或领域对应的主体档案）
│   ├── 世界/               ← 设定/规则/关系
│   ├── 剧情/               ← 悬念/时间线/大纲（或领域对应的状态追踪）
│   ├── hot.md              ← 记忆小抄（~500字，创作前必读）
│   ├── index.md            ← 全库目录
│   └── 交付手册.md         ← 后置产出
└── CLAUDE.md               ← 记忆锚点
```

### 三级检索模式（照抄 claude-obsidian）

```
hot.md → index.md → drill into pages
  (快)      (中)         (慢但全)
```

## 五、三 Skill 管道（不失忆循环）

无论什么创作领域，都需要这条管道：

```
创作 Skill      写内容    → 产出正文 + 更新 wiki
压缩 Skill      养记忆    → 把零散信息沉淀进 wiki + 刷新 hot.md
巡检 Skill      查质量    → 全库检查矛盾/遗漏/断链
```

**衔接规则**：

| 时机 | 调用 |
|------|------|
| 每写完 3-5 个单元 | 压缩 Skill |
| 每写完 10-15 个单元 | 巡检 Skill |
| 每次续写前 | 读 hot.md |
| 巡检发现问题后 | 创作 Skill 修复 |

## 六、写作质量约束（领域特有）

这是每个 skill 差异最大的部分。但结构可以复制：

| 章节 | 通用可复制 | 领域特有 |
|------|-----------|---------|
| 核心公式 | ✅ 建立预期→打破预期 | ❌ 具体公式 |
| 节奏结构 | ✅ 起承转合 | ❌ 比例分配 |
| 质量法则 | ✅ 每句话有目的 | ❌ 具体标准 |
| 技法库 | ✅ 常见冲突/手法 | ❌ 具体技法 |
| 禁忌清单 | ✅ 格式 | ❌ 具体禁忌 |
| 自检清单 | ✅ 格式 | ❌ 具体条目 |

## 七、为其他领域复制 Skill 的检查清单

### 建制阶段（最重要）

- [ ] 第0轮：先问一句探路
- [ ] 第1-5轮：覆盖该领域所有关键设定
- [ ] 每轮 A/B/C 选项具体、能刺激联想
- [ ] 最后一个选项永远是自定义
- [ ] 建制完成后输出摘要确认

### 创作循环

- [ ] 每次创作前有"场景工作台"（内部）
- [ ] 每次创作后用户确认
- [ ] 有催化剂/选择机制（领域不一定叫"催化剂"，但要有"推动力选择"）

### 记忆系统

- [ ] Wiki 目录结构覆盖该领域所有信息类型
- [ ] 更新时机明确（每场/每章/每节）
- [ ] 有 hot.md + index.md 二级检索
- [ ] 压缩 Skill 能处理该领域的素材

### 质量控制

- [ ] 有可执行的质量规则（不只是"写好点"）
- [ ] 有自检清单
- [ ] 有 validate 脚本（可选但推荐）
- [ ] 巡检 Skill 能检查该领域的常见错误

### 交付

- [ ] 后置指令：提炼交付手册
- [ ] 交付手册包含该领域的标准产出

## 八、领域适配示例

### 小说

```
建制轮次：世界观→主角→配角→主线→章纲
创作循环：章纲→选催化剂→写一章→确认→更新
Wiki：角色/ 世界/ 剧情/ 章节/
质量：对话/描写/节奏/伏笔/人设一致性
```

### 论文

```
建制轮次：领域→论点→论据框架→参考文献→方法论
创作循环：节纲→选论证角度→写一节→确认→更新
Wiki：论点/ 文献/ 数据/ 章节/
质量：逻辑链/引用规范/论证强度/原创性
```

### 演讲稿

```
建制轮次：场合→受众→核心信息→情感曲线→时长
创作循环：段落→选修辞手法→写一段→确认→更新
Wiki：演讲稿/ 素材/ 受众分析/
质量：开场钩子/情感节奏/记忆点/行动号召
```

### 短故事

```
建制轮次：世界观→主角→困境→转折→结局走向
创作循环：场景→选推进事件→写一场→确认→更新
Wiki：故事/ 角色/ 设定/
质量：反转密度/情感冲击/结尾余韵
```

## 九、关键原则（跨领域通用）

1. **渐进式披露**：SKILL.md 是统帅，不超过 200 行；详细内容在 references/
2. **选项驱动**：每个决策点 A/B/C + 自定义，刺激用户联想，补全盲区
3. **外置记忆**：Wiki 结构 + hot.md，解决长文失忆
4. **质量约束**：不靠"写好点"，靠具体的可执行规则
5. **管道协作**：创作→压缩→巡检，三 Skill 各司其职
6. **后置交付**：创作时不写大纲手册，写完再提炼
