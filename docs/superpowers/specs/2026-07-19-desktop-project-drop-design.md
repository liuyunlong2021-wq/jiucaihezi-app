# Desktop 项目拖放统一设计

## 目标

Desktop 用户从 Finder 拖入文件时，画布、对话区、编辑区和项目树都先把文件复制到当前项目；随后各自引用同一 `ProjectResource`。项目内文件是唯一事实源，切换面板或重启不会依赖临时 Blob、Data URL 或外部绝对路径。

## 存储规则

| 落点 | 项目内目标 | 成功后的行为 |
|---|---|---|
| 画布 | `jc-media/` | 立即生成对应图片、音频或视频卡片 |
| 对话区 | `jc-imports/` | 将项目资源读成当前消息附件 |
| 编辑区 | `jc-imports/` | 文本资源直接打开；其他资源新建引用该资源的项目 Markdown 文档并打开 |
| 项目树 | 当前选中文件夹；无选中目录时项目根目录 | 刷新并展示导入后的资源 |

画布只接受图片、音频和视频。其他类型在画布落下时显示明确错误，不复制、不生成空卡片。文件夹拖入项目树保留现有文件夹导入语义；其他区域仅接受普通文件。

## 架构

Tauri 原生窗口拖放事件只由一个 Desktop 分发器接收。它使用落点坐标识别 `data-project-drop-target`，调用共享文件总管导入命令。Rust 负责从用户刚拖入的外部路径复制到项目目录，前端不再使用 `plugin-fs` 读取任意 Finder 路径，也不扩大文件系统 scope。

共享动作返回 `ProjectResource[]`，并由 `ProjectFileService` 统一发布 `created` 事件。各面板仅在导入完成后读取该资源、创建引用或打开文档。失败必须显示来源、目标或类型原因，不能静默吞掉。

## 文件树刷新

资源 `changed` 是内容保存，不改变树结构，文件树不得整棵重载。`created`、`deleted`、`renamed` 刷新已加载的受影响父目录；根目录或未加载父目录才重载根目录。这样切换画布时对旧画布的自动保存不会折叠 `jc-canvas`。

## 验收

1. Desktop 从 Finder 拖一张图片到画布：文件先出现在 `jc-media/`，随后画布有图片卡片。
2. 拖文本、图片或音视频到对话区：文件先出现在 `jc-imports/`，随后成为可发送附件。
3. 拖文本到编辑区：原项目文本打开；拖媒体或其他文件：项目里创建并打开引用它的 Markdown 文档。
4. 拖文件到项目树目录和空白处：分别进入该目录与项目根目录。
5. 点击一个画布切换到另一个画布后，原先展开的画布目录仍展开。
6. 导入失败、未选择项目、画布不支持的类型都有主题内错误提示；不产生临时资源或外部路径引用。
