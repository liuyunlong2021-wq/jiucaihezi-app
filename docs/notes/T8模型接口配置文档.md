# T8 供应商模型参数与接口对照表 (ai.t8star.cn)

本文档旨在统一记录 T8 供应商模型（基于 NewAPI / 自建扩展网关）的调用接口规范与相关支持参数。该结构参考了 `runninghub_Skills` 的管理方式，方便前端与代理层进行参数映射和功能开发。

---

## 1. 图像生成模型 (Image Generation)

所有文生图基础调用均遵循标准的 OpenAI `/v1/images/generations` 协议。

### 1.1 Gemini Flash 图像系列
* **模型名称**: 
  - `gemini-3.1-flash-image-preview-2k`
  - `gemini-3.1-flash-image-preview-4k`
* **文生图端点**: `https://ai.t8star.cn/v1/images/generations` (POST)
* **图片编辑端点**: `https://ai.t8star.cn/v1/chat/completions` (POST)  
  *(注：该模型图片编辑走多模态 Chat 协议，而非 Image Edits 协议)*
* **支持比例 (Aspect Ratios)**:
  `1:1`, `16:9`, `9:16`, `4:3`, `3:4`, `3:2`, `2:3`, `5:4`, `4:5`, `21:9`, `1:4`, `4:1`, `1:8`, `8:1`

### 1.2 Banana PRO 系列
* **模型名称**:
  - `nano-banana-pro-2k`
  - `nano-banana-pro-4k`
* **文生图端点**: `https://ai.t8star.cn/v1/images/generations` (POST)
* **图片编辑端点**: `https://ai.t8star.cn/v1/images/edits` (POST)
* **支持比例 (Aspect Ratios)**:
  `1:1`, `16:9`, `9:16`, `4:3`, `3:4`, `3:2`, `2:3`, `5:4`, `4:5`, `21:9`

### 1.3 GPT Image 系列
* **模型名称**: 
  - `gpt-image-2`
* **文生图端点**: `https://ai.t8star.cn/v1/images/generations` (POST)
* **图片编辑端点**: `https://ai.t8star.cn/v1/images/generations` (POST)  
  *(注：文生图与图生图均集成在 generations 端点)*
* **支持比例 (Aspect Ratios)**:
  `1:1`, `16:9`, `9:16`, `4:3`, `3:4`, `3:2`, `2:3`

## 2. 视频生成模型 (Video Generation)

视频模型采用了非常规的扩展接口版本 `/v2/videos/generations`。

### 2.1 Veo 3.1 系列
* **模型名称**: 
  - `veo3.1-fast`
  - `veo3.1-components`
* **文生视频端点**: `https://ai.t8star.cn/v2/videos/generations` (POST)
* **图生视频端点**: `https://ai.t8star.cn/v2/videos/generations` (POST)
* **参数建议**: 
  - 需要在 payload 中传递 `image_url` (图生视频) 或 `prompt` (文生视频)。
  - 需要根据工作台要求补充时长 `duration` 等字段。

---

## 3. 大语言与多模态模型 (LLM / Multimodal)

完全兼容标准 OpenAI 对话接口。

### 3.1 Gemini LLM 系列
* **模型名称**: 
  - `gemini-3.1-flash-lite-preview`
  - `gemini-3.1-pro-preview`
* **核心端点**: `https://ai.t8star.cn/v1/chat/completions` (POST)
* **功能映射**:
  1. **文生文 (Chat)**: 标准 `messages` 数组发送。
  2. **修改图片 (Image Editing via Prompt)**: 将目标图片转换为 base64，并通过多模态 `messages` 结构附加修改提示词发送至 completions 端点。
  3. **分析视频 (Video Analysis)**: 支持视频文件上传/解析，通常需携带视频流或提取的关键帧至 completions 端点。

## 4. 前端调用对接示例 (伪代码)

在韭菜盒子工作台 `runCreationPanel` 中组装这些参数的方法：

```javascript
// 针对 T8 图像生成模型的代理 Payload 示例
let payload = {
  model: "nano-banana-pro-4k", // 或其他模型名
  prompt: snap.prompt,
  size: snap.ar || "16:9", // 将前端的 16:9 直接透传给 size 参数
  n: 1
};

fetch("https://ai.t8star.cn/v1/images/generations", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": "Bearer T8_API_KEY"
  },
  body: JSON.stringify(payload)
});
```
