# 知识库内循环 — 文件协作方案

> **最后更新**: 2026-07-01
> **设计原则**: 一个项目 = 一个创作主力 + 巡检 + 压缩，三个 skill 协作同一套文件

---

## 架构

```
                      ┌──────────────────────────┐
                      │      JC-yizhixing        │
                      │   「全库巡检」            │
                      │   扫漏改/撞车/断链/矛盾   │
                      │   产出: wiki/巡检报告/    │
                      └──────────┬───────────────┘
                                 │ 读全库，写报告
                                 ▼
┌─────────────────┐    ┌─────────────────────────┐    ┌─────────────────┐
│  创作主力        │    │   项目文件系统            │    │  JC-jiyiyasuo  │
│  (四选一)        │───▶│                          │◀───│  「记忆压缩」   │
│                 │    │  .raw/    原始素材         │    │               │
│ 世界模型/临摹/   │    │  wiki/    Obsidian 主目录  │    │ .raw→wiki 归档 │
│ 小说/漫剧成稿    │    │  CLAUDE.md 记忆锚点        │    │ 补[[双链]]    │
│                 │    │                          │    │ 刷新 hot.md    │
│ 产出: wiki/作品/ │    └─────────────────────────┘    │ 更新 index.md  │
└─────────────────┘                                   └─────────────────┘
```

**一个项目只有一个创作主力**。四个可选：

| 创作主力 | 产出路径 | 配合 |
|---------|---------|------|
| JC-duanju-shijiemoxing | `wiki/作品/第X集.md` `wiki/角色/` `wiki/世界观/` `wiki/悬疑管理/` | 从零写短剧 |
| JC-linmoduanju | `wiki/改写稿/第X集.md` `wiki/映射表.md` | 换皮临摹（范本在 .raw/） |
| JC-linmoxiaoshuo | `wiki/改写稿/第X章.md` `wiki/映射表.md` | 小说换皮 |
| novel-writing | `wiki/作品/第X章.md` `wiki/角色/` | 从零写小说 |

```

│   │   └── 第X集.md

│   ├── 角色/                ← 角色档案（从零创作类 skill 维护）
│   │   └── 角色名.md
│   ├── 世界观/              ← 世界设定/关系网/势力（从零创作类 skill 维护）
│   │   ├── 世界设定.md
│   │   ├── 关系网.md
│   │   └── 势力分布.md
│   ├── 悬疑管理/            ← 悬念/伏笔账本（从零创作类 skill 维护）
│   │   ├── 悬念账本.md
│   │   └── 历史记录.md
│   ├── 映射表.md            ← 临摹类 skill 维护（范本名→新名对应表）
│   ├── 伏笔账本.md          ← 跨 skill 共享（巡检用）
│   │
│   ├── 巡检报告/            ← ★ JC-yizhixing 产出（每次巡检一份）
```

## 三大 skill 的分工

### 创作主力（四选一）

| 做什么 | 读什么 | 写什么 |
|-------|--------|--------|
| 从零创作 | CLAUDE.md 自己的区块 | `wiki/作品/` `wiki/角色/` `wiki/世界观/` `wiki/悬疑管理/` |
| 换皮临摹 | `.raw/范本/` + CLAUDE.md | `wiki/改写稿/` `wiki/映射表.md` |
| 每次产出后 | — | **更新 CLAUDE.md 自己的区块** |

### JC-yizhixing — 一致性巡检官

| 做什么 | 读什么 | 写什么 |
|-------|--------|--------|
| 全库体检 | 整个 `wiki/` + `wiki/映射表.md` | `wiki/巡检报告/日期.md` |
| 换皮专项检查 | `wiki/映射表.md` 存在时启用 | 报告漏改/撞车/漂移 |
| 通用检查 | 无映射表时 | 报告角色矛盾/断链/孤儿笔记/伏笔没收 |
| 巡检后 | — | **更新 CLAUDE.md 的 `## JC-yizhixing` 区块** |

### JC-jiyiyasuo — 记忆压缩官

| 做什么 | 读什么 | 写什么 |
|-------|--------|--------|
| 日常整理 | `.raw/` 新素材 | 提炼归档进 `wiki/` |
| 补双链 | 新写入的 wiki 文件 | 添加 `[[交叉引用]]` |
| 刷新热缓存 | `wiki/` 全部文件 | `wiki/hot.md`（最近活跃的条目摘要） |
| 更新索引 | `wiki/` 全部文件 | `wiki/index.md`（全库条目列表） |
| 压缩后 | — | **更新 CLAUDE.md 的 `## JC-jiyiyasuo` 区块** |

## CLAUDE.md 格式规范

三个 skill 各有自己的区块，互不覆盖：

```markdown
# 短剧：《重生之异能觉醒》

## [创作] JC-duanju-shijiemoxing
- 最新: [[wiki/作品/第5集]]（2026-07-01）
- 主角: [[wiki/角色/林风]]（压力值:65）
- 配角: [[wiki/角色/苏婉]] [[wiki/角色/陈烈]]
- 世界: [[wiki/世界观/世界设定]]
- 悬念: [[wiki/悬疑管理/悬念账本]]（已埋5, 待收3）

## JC-yizhixing
- 最近巡检: [[wiki/巡检报告/2026-07-01]]
- 结果: ❌0 🔻0 ⚠️1（第3集苏婉压力值未更新）
- 下次建议: 写完第6集后再巡

## JC-jiyiyasuo
- hot.md 刷新: 2026-07-01
- index.md 条目: 15
- .raw/ 待处理: 3 个素材
```

**另一种组合**（换皮临摹）：

```markdown
# 短剧临摹：《出马仙》（范本: 测字有术）

## [创作] JC-linmoduanju
- 范本: .raw/范本/测字有术（654章）
- 映射表: [[wiki/映射表]]（32条映射）
- 已完成: [[wiki/改写稿/第1集]] [[wiki/改写稿/第2集]] [[wiki/改写稿/第3集]]
- 下次: 改写第4集

## JC-yizhixing ...（同上）
## JC-jiyiyasuo ...（同上）
```

## 关于 .raw/ 和 wiki/

| 目录 | 用途 | 谁操作 | Obsidian 里能看到？ |
|------|------|--------|:--:|
| `.raw/` | 原始素材仓库 | 用户扔进去 / JC-jiyiyasuo 读取 | ❌（`.raw` 前缀在 Obsidian 中默认隐藏） |
| `wiki/` | 结构化知识库 | 创作 skill 写入 / yizhixing 巡检 / jiyiyasuo 归档 | ✅ Obsidian 打开的根目录 |

`.raw/` 的名字以 `.` 开头，Obsidian 默认不显示，用户不会在 Obsidian 里看到杂乱素材。JC-jiyiyasuo 负责从中提炼精华进 `wiki/`。

## 协作流程

```
1. 创作主力写一集 → 写入 wiki/作品/第X集.md → 更新 CLAUDE.md

2. 写完3-5集后 → JC-yizhixing 巡检 → 产出 wiki/巡检报告/ → 更新 CLAUDE.md
   ├── 有问题 → 创作主力修复 → 再巡
   └── 没问题 → 继续

3. 不定期 → JC-jiyiyasuo 整理 .raw/ 素材 → 归档进 wiki/ → 刷新 hot.md/index.md → 更新 CLAUDE.md
```

## 不需要一键建库

目录结构由**创作主力 skill 自动创建**。用户只需要：

1. 在对话框上方选一个空文件夹作为项目目录
2. 发送 `从0写短剧` / `小说换皮改写第1章` 等指令
3. skill 自动创建 `.raw/` `wiki/作品/` `wiki/角色/` 等子目录
4. Obsidian 打开同一个目录 → 直接看到 `wiki/` 下的所有内容
