# 音视频工坊主动工具 SDD

## 1. 目标

把现有音视频类本地工具收敛成一个用户可直接操作的主动工具工作台：**音视频工坊**。

用户只需要：

1. 打开 App。
2. 进入「工具仓库」。
3. 运行「音视频工坊」或任意音视频相关工具卡。
4. 添加音频/视频文件。
5. 选择一个直白动作：
   - 查看信息
   - 转文字
   - 压缩转格式
   - 视频上字幕
6. 点击开始。
7. 得到本地输出文件，并可打开、在 Finder 中显示、加入对话或进入下一步处理。

用户不需要理解 `ffmpeg`、`ffprobe`、`Whisper`、命令行参数、PATH、Homebrew、Python 或任何本地依赖安装。

## 2. 范围

本 SDD 只做 **音视频工坊**。

明确不做：

- 源码项目工作台。
- 浏览器工作台。
- 通用 Shell 面板。
- LLM 自动选择文件或自动决定处理动作。
- 开放式命令参数输入。
- 聊天区 Tools 开关改造。

## 3. 产品原则

- 这是工具仓库里的主动工具，不是对话 Agent。
- 用户必须显式选择或拖入本地音视频文件。
- 系统可以读取媒体信息做预检，但不能从聊天历史自动抓文件执行。
- 所有写文件动作必须由用户点击触发。
- 执行前必须展示目标文件、任务类型、关键选项和保存位置。
- 工具直接可用。正式 App 包必须内置所需媒体处理组件。
- UI 不显示「请安装 ffmpeg」「请安装 Whisper」「等待配置」「本地能力中心」。
- 如果内置组件缺失，只显示应用级错误：「媒体处理组件不可用，请重新安装应用后重试。」
- 底层执行统一走 Tauri/Rust command 或一手本地工具层，前端只传结构化参数。

## 4. 与现有优秀示例的一致性

参考两个已认可的主动工具：

| 示例 | 可复用模式 |
|------|------------|
| 格式转换 | 文件选择、任务队列、输出格式、进度、取消、结果动作 |
| 网页媒体采集 | 先解析再执行、业务语言 UI、不暴露底层采集组件、完成后打开/定位文件 |

音视频工坊继承这些规则：

- 工具仓库卡片点击「运行」进入独立子面板。
- 子面板有明确顶部、模式切换、输入区、选项区、任务区、结果区。
- 任务可取消，失败可重试。
- 输出文件可打开、Finder 显示、加入对话。
- 底层 runtime 不教育用户，不把内部依赖变成用户任务。

## 5. 工具仓库入口

### 5.1 卡片策略

以下工具卡都应显示「运行」，点击后进入同一个 `MediaWorkbenchPanel.vue`，但默认选中不同动作：

| 工具卡 | 默认动作 |
|--------|----------|
| 音视频识别 | 查看信息 |
| 语音转文字 | 转文字 |
| 音视频处理 | 压缩转格式 |
| 字幕烧录 | 视频上字幕 |

工具仓库也可以新增或重命名一张总入口卡：

```text
名称：音视频工坊
图标：movie_edit
分类：音视频
风险：可生成
描述：查看、转写、压缩转换和给视频添加字幕。
标签：视频 / 音频 / 字幕 / 转文字 / 压缩
```

### 5.2 命名规则

用户可见命名必须直白：

| 禁用或不推荐 | 推荐 |
|--------------|------|
| 识别 | 查看信息 |
| 转写 | 转文字 |
| 处理 | 压缩转格式 |
| 字幕烧录 | 视频上字幕 |

说明：

- 代码内部仍可保留 `media_burn_subtitles` 等历史 command 名。
- UI、工具卡、按钮、状态文案必须优先使用用户语言。

## 6. 用户流程

### 6.1 查看信息

1. 用户进入「音视频工坊」。
2. 用户添加 `demo.mp4`。
3. 系统自动读取基础信息。
4. 页面显示：
   - 文件名
   - 文件大小
   - 时长
   - 分辨率
   - 帧率
   - 音轨
   - 是否包含字幕
5. 用户可点击：
   - 加入对话
   - 复制信息
   - 转文字
   - 压缩转格式
   - 视频上字幕

### 6.2 转文字

1. 用户添加音频或视频文件。
2. 用户切到「转文字」。
3. 用户选择输出：
   - TXT
   - SRT
   - VTT
4. 用户选择语言：
   - 自动
   - 中文
   - 英文
5. 用户选择速度：
   - 快速
   - 高质量
6. 用户点击「开始转文字」。
7. 系统显示进度。
8. 完成后显示结果文件。
9. 用户可点击：
   - 打开文字
   - 在 Finder 中显示
   - 加入对话
   - 用于视频上字幕

### 6.3 压缩转格式

1. 用户添加音频或视频文件。
2. 用户切到「压缩转格式」。
3. 用户选择任务：
   - 压缩视频
   - 转成 MP4
   - 转成 MOV
   - 抽取音频
   - 截取片段
   - 静音视频
4. 用户选择质量：
   - 省空间
   - 平衡
   - 高质量
5. 如果是截取片段，用户填写开始/结束时间。
6. 用户点击「开始处理」。
7. 完成后显示输出文件。
8. 用户可点击：
   - 打开播放
   - 在 Finder 中显示
   - 加入对话

### 6.4 视频上字幕

1. 用户添加视频文件。
2. 用户切到「视频上字幕」。
3. 用户选择字幕来源：
   - 选择 SRT 文件
   - 粘贴 SRT 文本
   - 使用刚刚转文字产生的 SRT
4. 用户选择字幕样式：
   - 标准
   - 大字幕
   - 底部安全区
5. 用户点击「开始上字幕」。
6. 完成后显示带字幕的视频文件。
7. 用户可点击：
   - 打开播放
   - 在 Finder 中显示
   - 加入对话

## 7. 工作台 UI

组件：`MediaWorkbenchPanel.vue`

```text
       查看、转文字、压缩转格式和给视频上字幕
```

### 7.2 动作切换

使用分段控件，不使用技术名：

```text

```

规则：

- 点击工具卡进入时自动选中对应动作。
- 用户可在同一文件上切换动作。
- 文件不符合动作要求时，动作区显示明确提示，例如「视频上字幕需要选择视频文件」。

## 9. 后端与运行层

优先复用现有 Tauri commands：

| 用户动作 | 现有命令 | 说明 |
|----------|----------|------|
| 查看信息 | 可复用/补齐媒体信息读取 command | 现有 LLM tool 只读附件文本，主动工作台需要真实本地文件元信息 |
| 转文字 | `media_transcribe_file` | 输出 TXT/SRT/VTT |
| 压缩转格式 | `media_process_file` | 执行压缩、转码、截取、抽音频、静音 |
| 视频上字幕 | `media_burn_subtitles` | 输入视频 + SRT，输出带字幕视频 |

需要新增或补齐：

- `media_inspect_file`：读取本地媒体元信息，返回结构化 JSON。
- `cancel_media_workbench_task`：P0 如暂不支持真实取消，UI 仍需有取消状态；后端应逐步支持按 jobId 终止进程。
- `media_open_file` / `media_reveal_file` 可复用网页媒体采集已有安全打开模式，或抽成通用 generated output registry。

正式 App 要求：

- 媒体处理组件必须内置或由 App 托管。
- 不走 localCapabilities 中的用户安装引导。
- 不把 `ffmpeg` / `whisper` 缺失作为用户配置任务。

## 10. 安全边界

- 输入文件必须来自用户选择、拖入或 App 已缓存的本地附件。
- 后端必须 canonicalize 输入路径，防止路径遍历。
- 输出目录默认使用 App 管理目录或 `~/Movies/韭菜盒子/媒体工具/`。
- 打开/定位文件前，必须确认路径是本工具生成的输出。
- 不允许用户输入任意命令或任意 ffmpeg 参数。
- 前端枚举动作，后端按白名单生成参数。
- 错误信息必须业务化，不回显完整命令行和内部路径细节。

## 13. 验收标准

- 工具仓库中音视频相关工具可以进入同一个音视频工坊。
- 用户能看到四个直白动作：查看信息、转文字、压缩转格式、视频上字幕。
- 用户选择文件后能先看到媒体基础信息。
- 转文字能输出 TXT/SRT/VTT。
- 压缩转格式能完成至少 3 个 P0 动作：压缩视频、转成 MP4、抽取音频。
- 视频上字幕能选择 SRT 或使用转文字结果生成带字幕视频。
- 完成后能打开文件、在 Finder 中显示、加入对话。
- UI 不出现本地依赖安装/配置/检测引导。
- `pnpm run test:focused`、`pnpm exec vue-tsc -b`、`cargo check --manifest-path src-tauri/Cargo.toml` 通过。

## 14. 分阶段实现

### P0：可用主链路

- 新增 `MediaWorkbenchPanel.vue`。
- 工具仓库四张音视频卡进入同一工作台。
- 支持单文件。
- 支持查看信息。
- 支持转文字 TXT/SRT/VTT。
- 支持压缩视频、转 MP4、抽音频。
- 支持选择 SRT 做视频上字幕。
- 支持打开/定位输出文件。

### P1：体验补齐

- 支持 Finder 拖入。
- 支持转文字结果一键用于视频上字幕。
- 支持截取片段、静音视频、转 MOV。
- 支持任务取消。
- 支持自定义保存目录。

### P2：批量与联动

- 支持多文件队列。
- 支持批量转文字。
- 支持批量压缩。
- 支持结果加入知识库 raw。
- 与网页媒体采集结果联动：下载后可直接进入音视频工坊继续处理。
